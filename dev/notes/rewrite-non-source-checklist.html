ggpaintr Rewrite — Non-Source Checklist

dev/notes/rewrite-non-source-checklist.html · 2026-05-20

Purpose. If we rewrite ggpaintr, the R/paintr-*.R source is only one layer. This file enumerates every other artifact in this repo that must move in lockstep with a source rewrite, grouped by category, so nothing is silently left behind referring to a function, file, or invariant that no longer exists.

How to use. Treat it as a pre-merge gate: walk the list, mark each item Done / N/A / Deferred. Anything Deferred needs a follow-up issue under .scratch/.

1. Generated documentation (must regenerate)

  • roxygen2 doc blocks in R/paintr-*.R — every @param, @return, @examples, @inheritParams, @export must match the rewritten signatures and behavior. Use auditor-roxygen agent.
  • man/*.Rd — regenerate via devtools::document(); delete .Rd files for removed exports.
  • NAMESPACE — auto-generated; verify diff matches intended public API. Never hand-edit.
  • pkgdown site — _pkgdown.yml reference index sections must list only surviving exports; rebuild via source("dev/build-pkgdown.R"); build_pkgdown_clean(); the generated docs/ tree is gitignored output but the input config is not.
  • pkgdown favicon — pkgdown/favicon/ is fine to keep unless branding changes.

2. README & package metadata

  • README.Rmd — editable source; updates motivation, install snippet, minimum demo. Verify every ptr_* referenced still exists.
  • README.md + README.html — regenerated outputs; re-knit with devtools::load_all("."); rmarkdown::render("README.Rmd", envir = globalenv()).
  • DESCRIPTION — Version bump, Imports / Suggests churn (drop unused deps, add new ones), Title / Description prose if scope changes, Authors@R if contributors change.
  • NEWS.md — add a new top section documenting every breaking change, new export, removed export, behavior change. This is the user-facing changelog; the rewrite is invisible without it.
  • cran-comments.md + CRAN-SUBMISSION — refresh before any CRAN submission; reset to reflect this release's checks.
  • LICENSE / LICENSE.md — only if the license itself changes (rare).
  • .Rbuildignore — add any new dev-only paths (vignettes-pending-port, audit dirs, etc.) so they don't ship in the tarball.
  • .gitignore — add new generated paths.

3. Vignettes (long-form docs)

All under vignettes/. Each is both prose and runnable code; both sides must hold.

  • ggpaintr-use-cases.Rmd — main tour. Update every formula, every ptr_* call, every screenshot reference.
  • ggpaintr-customization.Rmd — L2/L3 examples. Verify against current ADRs (0005, 0006).
  • ggpaintr-gallery.Rmd — every gallery example must still run.
  • ggpaintr-llm.Rmd — LLM-facing API surface; re-check function names and arg names exactly.
  • ggpaintr-safety.Rmd — denylist + walker description; ensure security claims still match runtime code.
  • dev/vignettes-pending-port/ — staging area; either port forward or delete with a note in NEWS.
  • Run the auditor-vignette agent for prose↔code alignment and tester-vignette agent for runtime e2e of every chunk.

4. Tests & fixtures

  • tests/testthat/test-*.R — every test calling a renamed/removed function fails closed; fix or delete. Re-derive coverage map (see .claude/rules/testing.md).
  • tests/testthat/helper-*.R — shared helpers; refactor as signatures change.
  • tests/testthat/fixtures/ — including vignette-apps/<slug>/app.R files. These are diffable mirrors of vignette chunks (per vignette-fixture-drift-ledger); when a vignette chunk changes, the fixture's "EXCEPT" header is the authoritative drift signal — re-sync.
  • tests/testthat/_snaps/ — snapshot files; regenerate intentionally and review every diff, do not blanket-accept.
  • tests/manual/ — human-driven scripts (uploads, inline errors, custom integrations). Update to new API per .claude/rules/testing.md.
  • tests/browser/ — shinytest2 driver-level fixtures; check pkgload+app-dir pattern still holds (see project memory shinytest2-appdir-pkgload).
  • tests/testthat.R — usually static, verify.
  • Re-run the authoritative gate: NOT_CRAN=true Rscript -e 'suppressMessages(devtools::load_all(".")); testthat::test_dir("tests/testthat", reporter="progress", stop_on_failure=FALSE)' → expect FAIL 0 / ERROR 0 / SKIP 0 / PASS N.

5. Bundled assets (inst/)

  • inst/www/ — ggpaintr.css, ggpaintr-layer.css, ggpaintr-ui.js, ggpaintr-layer.js, ggpaintr-logo.png. CSS selectors and JS event hooks must match new UI DOM ids/classes; logo refs need addResourcePath("ggpaintr") (memory: htmldependency-resource-prefix).
  • inst/extdata/ — sample CSVs/RDS used by examples and tests; if column names change, regenerate.
  • inst/figures/ — README/vignette images; re-shoot if UI redesigned.
  • inst/llm/ — anything shipped for LLM consumers (prompts, schema dumps); regenerate from new API surface.

6. Domain & architectural docs

  • CONTEXT.md — single-source domain doc (L2/L3 model, UI-piece taxonomy, shared-coordinator vocab); locked by ADR 0005, edited as part of any rewrite that changes those concepts.
  • dev/adr/ — open a new ADR for any decision the rewrite reverses or supersedes; mark old ones Superseded-By with a link. Do not silently change behavior an ADR documents.
  • dev/developer-notes.md — human-maintained; user must ask before edits.
  • dev/high-level-picture.md — refresh if architecture diagram changes.
  • dev/feature_bank/ — canonical feature record (memory: project-feature-bank); rebuild via Rscript dev/feature_bank/build-feature-bank.R.
  • AGENTS.md — top-level agent instructions; only edit if process changes.
  • CLAUDE.md — project conventions; touch only if the authoritative gate, source pattern, or build commands change.

7. Agent / harness config

  • .claude/rules/coding.md, reviewing.md, testing.md, serena-tools.md — refresh if conventions / file layouts shift.
  • dev/agents/issue-tracker.md, triage-labels.md, domain.md — re-anchor any path/function refs.
  • .scratch/<feature>/ — open issues / PRDs; resolve or migrate.
  • dev/githooks/ — verify hooks still match build commands.
  • dev/scripts/ — utility scripts (e.g. audit-probe.js); re-check refs to ids/selectors.

8. CI & build infrastructure

  • .github/workflows/R-CMD-check.yaml — R version matrix, system deps, cache keys.
  • .github/workflows/pkgdown.yaml — publishing workflow; verify it still calls the right build entry.
  • .github/workflows/rhub.yaml — release verification.
  • Dockerfile + .dockerignore + docker-compose.codex.yml — container layer pins dep versions; bump alongside DESCRIPTION.
  • dev/build-pkgdown.R — local doc build script; verify it still excludes CLAUDE.md from rendered output.

9. Audit / scratch / archive

  • dev/audit/, dev/notes/, dev/plans/, dev/demo/, dev/assets/ — historical reports. Do not retroactively rewrite; instead, add a dated successor file noting which findings are now stale. Memory vignette-fixture-drift-ledger + project-feature-bank describe the canonical sources.
  • archive/ — frozen; leave alone.
  • dev/tasks/*.md — stored task files; do not edit unless user asks (per project rules).

10. User-facing memory & auto-memory

  • Project MEMORY.md entries that name a renamed function (e.g. ptr_module_server → ptr_server as in ADR 0006) must be updated or marked superseded. Verify with grep before claiming any memory still applies.
Definition of Done for a rewrite PR.
  1. devtools::document() → no diff outside intended changes.
  2. NOT_CRAN=true full test suite → FAIL 0 / ERROR 0 / SKIP 0.
  3. devtools::check(args = c("--as-cran","--no-manual")) → 0 errors, 0 warnings; only the by-design NOTEs.
  4. build_pkgdown_clean() → clean.
  5. README knit → no errors, screenshots current.
  6. Every vignette renders end-to-end (tester-vignette agent passes).
  7. NEWS.md has an entry for every external-visible change.
  8. No memory entry names a non-existent function (grep against current R/).
