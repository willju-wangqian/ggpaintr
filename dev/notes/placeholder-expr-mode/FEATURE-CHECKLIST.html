ggpaintr feature checklist — post ADR-0009

vignette-review HEAD at last refresh: 984b689 · 2026-05-23 (original draft: 2c504da, 2026-05-21; patched 2026-05-23 against audit audit-feature-checklist-postadr9-drift-2026-05-23.html) · companion to WORKFLOW.html · purpose: a feature inventory grouped by what is natural to exercise together in a single complex test app. The grouping is not orthogonal to the L1/L2/L3 axis — it is orthogonal to what can be observed without rebuilding state.

How to read this. Each section answers the question "what feature cluster could one super-app demonstrate in one screen?" A super-testcase will pick one item from each of several sections — every section here represents a natural axis along which super-tests vary. Items inside a section are the granular things to assert. Items marked (BROKEN) have a known regression on current HEAD; (UNTESTED) means no automated test asserts the user-visible behaviour even if internals are covered.

Section A — Placeholders & argument grammar

Group: the placeholder vocabulary itself. Any super-app will use at least 2-3 of these in one formula.
ItemKeyword / shapeNotes
A1. Bareword (no parens) formaes(x = ppVar, y = ppNum)Parser still recognises a placeholder used as a bareword inside aes() and other named-arg slots. Pre-ADR-0009 this was the only form; still works.
A2. Call form (parens, no args)geom_point(size = ppNum())Equivalent to bareword. Common in user code that wants to be explicit.
A3. ppVar — column pickerppVar(mpg)selectInput populated from upstream data cols. default = <sym> seeds the picker (PLAN-01 ptr_default_symbol_or_string).
A4. ppNum — numericInputppNum(0.7)Default validated via constant-fold registry: literals + arithmetic + named constants like pi (PLAN-01).
A5. ppText — textInputppText("Hello")Default must be a string literal.
A6. ppExpr — textInput → parsed user expressionppExpr(mpg * 2)User-typed expression is parsed at runtime; default is the AST. Returned as ptr_user_expr() so substitution preserves it as code, not a quoted value.
A7. ppUpload — file sourceppUpload() |> dplyr::filter(...)Either at formula head (data source) or mid-pipeline. Validates file extension against ptr_upload_accept_formats() = c(".csv", ".tsv", ".rds", ".xlsx", ".xls", ".json"). Optional name = "<sym>" companion (ADR-0010, registered default_arg) drives a companion textInput that round-trips through preserve-mode.
A8. Named arg: shared = "<key>"ppVar(shared = "a")Bind multiple occurrences to one widget. See Section E.
A9. Multiple same-type placeholdersaes(x = ppVar, y = ppVar)Each occurrence gets its own widget unless shared= links them. Widget ids are distinct.
A10. Empty input → silent absence—Unset ppText / ppNum / ppVar collapse to ptr_missing(); the surrounding call is dropped (eg. labs(title = ppText()) with no value emits no title).
A11. ppExpr empty → ptr_missing—Empty input string; user typed nothing yet.
A12. ppVar arity check via validate_input—Built-in ppVar rejects length-zero / length >1 (multi-select consumers can lift this).

Section B — Code panel & the two view modes

Group: every super-app has a code panel; both modes should be exercised. This is where the "show placeholder" bug lived (fixed 2c504da).
ItemNotes
B1. Final code mode (default)Shows the substituted ggplot expression — input values plugged in, missing placeholders elided. format_code_with_extras(state$runtime(), state$extras_exprs()).
B2. Code with placeholders modeRe-renders state$tree() with preserve_placeholders = TRUE; widget value appears as ppX(value) arg. PLAN-05.
B3. Toggle (radioGroupButtons)Reads input$<ns>-ptr_code_mode, values "final" / "preserve". PLAN-08 UI + commit 2c504da server wiring.
B4. Code panel style = "window"Floating mini-window (default in ptr_app()). Toggle lives inside.
B5. Code panel style = "panel"In-flow block (used by ptr_ui_code(id, style = "panel") — the exported helper; code_block_tag() is an unexported internal). Toggle still attaches.
B6. Code reactive on Update clickCode panel re-runs state$runtime() which fires on the Update button.
B7. Code panel survives hidden stateoutputOptions(suspendWhenHidden = FALSE) — output renders even when the window is initially collapsed.
B8. ptr_extract_code(state)Non-reactive snapshot of current final-mode code. (UNTESTED) for parity with preserve-mode (no ptr_extract_code(state, preserve = TRUE); consumer must call ptr_render themselves).
Pair B with E. The preserve-mode renderer is the only way to see whether shared= survived parsing — the substituted form erases the shared annotation. A super-test that flips B3 should also assert the rendered text contains shared = "key".

Section C — Overrides: ui_text, copy, defaults

Group: every override mechanism. Super-app demonstrates per-keyword vs per-aesthetic vs per-layer precedence.
ItemOverride slotNotes
C1. Per-placeholder label / hintui_text$defaults$ppNum$labelApplies to every ppNum widget unless overridden lower.
C2. Per-keyword empty-textui_text$defaults$ppVar$empty_textThe "Pick a column" hint shown before the user chooses.
C3. {param} interpolationcopy_defaults = list(label = "Range for {param}")Custom placeholders fill {param} from the surrounding aes/arg name.
C4. Per-layer / per-arg copyparams$linewidth$ppNum$labelLayer-name × arg-name × placeholder-kind precedence. Tested in test-copy-rules.R.
C5. Default seeding from formulappNum(0.7)PLAN-07 — node$default reaches the widget's value / selected arg. Persisted user input wins over default on re-render.
C6. Default seeding for shared bucket—First-occurrence-wins for the shared widget (PLAN-07 shared_widget_default). Conflicting defaults on later occurrences silently ignored.
C7. named_args threading—Validated named args from formula reach the build_ui hook via node$named_args (custom hooks only; built-ins swallow via ...).
C8. NULL default → widget initial state—No default arg → widget's own empty initial state is unchanged.

Section D — Custom placeholders (L2 extension)

Group: the registry surface. Super-app should always include at least one custom placeholder to flex this path.
ItemNotes
D1. ptr_define_placeholder_valueInline value placeholder (the num/text/expr shape). Required hooks: build_ui, resolve_expr.
D2. ptr_define_placeholder_consumerData consumer (the var shape). Required: build_ui, resolve_expr, optional validate_input.
D3. ptr_define_placeholder_sourceData source (the upload shape). Required: build_ui, resolve_data (or equivalent).
D4. Hook return value: callable (define-helpers)PLAN-03: define-helpers return the registered entry as a callable, so users can capture ppFoo <- ptr_define_placeholder_value(...) and reference it.
D5. default_arg validator on registry entryClosure factory ptr_default_*() declared on the registry entry; parser dispatches to it. Built-ins use this since PLAN-08.
D6. named_args schema on registry entryNamed-list of validator closures, one per declared named arg. Reject undeclared args at parse time.
D7. Keyword-shadow rejectionRegistering a keyword that exists in base R / utils / stats / dplyr / ggplot2 aborts with a shadow message. Grandfather list = c("range", "date") for vignette fixtures only.
D8. LHS-name vs keyword= drift warningBest-effort warning when ppFoo <- ptr_define_placeholder_value(keyword = "bar") would create a mismatch. Lexical-shadow path catches it; REPL primitive <- does not push a frame.
D9. Custom validate_input contractReturns TRUE / NULL (ok) or a character message (error displayed inline). Any other shape fails closed.
D10. Custom hook receives selected / named_args via ...PLAN-07 gates the injection on "..." %in% fmls || arg %in% fmls; older hooks with neither still work.

Section E — Shared widgets (shared=)

Group: cross-layer linkage. The only feature where two visually-distinct call sites collapse to one widget.
ItemNotes
E1. Same-keyword shared= collapseTwo ppVar(shared = "a") in different layers render one picker. Both layers re-render on its change.
E2. Shared section partitionShared widgets surface in a dedicated section (sidebar header / top bar / coordinator) rather than under their layer's tab.
E3. Shared label tiebreakFirst-occurrence label = wins; later differing labels silently ignored.
E4. Shared default tiebreak (PLAN-07)First-occurrence default = wins.
E5. Cross-role rejectionPLAN-06 ptr_validate_shared_roles() — a shared key spanning value + consumer roles aborts at translate time.
E6. Same-role acceptanceTwo ppVar with same shared key in two layers' aes mappings are accepted.
E7. Mid-pipeline shared= (data source)Shared= on ppVar used both in a dplyr::select stage and an aes mapping — same widget drives both.
E8. Preserve-mode renders shared = argThe dual-mode renderer should round-trip the shared = "key" annotation. (UNTESTED) for embed + grid hosts where the shared section's binder was historically inconsistent (see shared-section-binder memory).

How to build super-testcases from this

A "super" testcase = a single ggpaintr app that exercises one item from each of as many sections as possible without invalidating the formula. Suggested combinations:

  • Maximal default app: A1+A3+A4+A5+A8+E1+B1+B3+C5+I3 — single string formula, every placeholder kind, shared bucket, default seeding, both code modes via the toggle.
  • Custom + upload super: A7+D1+D5+D6+D9+H3+H4+H7+C3 — bare-expression formula, ppUpload mid-pipeline, custom placeholder with default_arg + named_args + validate_input, copy interpolation.
  • L3 + plotly super: F1+F4+F8+G2+B9+I4 — bare-expression formula, custom plotly output, both code-text shapes accessible from outside the default UI.
  • Multi-instance super: L1+L3+L4+F2+E1 — two ptr_app_components in a grid, both consuming a shared widget, both writing to bespoke outputs.
  • Theming + safety super: K1+K2+K3+J1+J3+G5 — bslib theme, user_css, asset references; on the safety axis, attempt to inject eval(parse(...)) through ppExpr and a paste() string-builder.

Every super-test should toggle B3 at least once and assert that both code-mode outputs are sensible — that single check alone protects against the regression that motivated this checklist.
