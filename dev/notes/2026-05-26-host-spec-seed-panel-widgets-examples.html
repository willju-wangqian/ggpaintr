Host-scope spec= for panel-shared widgets — working / non-working examples

Branch host-spec-seed-panel-widgets at HEAD with the FINDING #1 + #7 fix (pre-commit). Companion to placeholder-role-coverage2.html v7 — the v7 doc described the gaps; this file shows the runnable repros that proved them and the post-fix behaviour.

Every snippet below is real public-API ggpaintr code that should logically work. “Before” is the state on commit 1502c17 (the parent of the fix); “After” is the state with R/paintr-shared-ui.R's new apply_spec_at_boot_host() + ptr_setup_panel_values() + spec_seed= threading through ptr_bind_local_shared_consumers(), plus the spec= channel on ptr_shared_server() forwarded from ptr_app_grid().

Reading the verdicts

  • YES — works end-to-end (browser-truth verified via shinytest2 gate).
  • PARTIAL — verified at the structural seam (host spec_seed bag receives the value; deferred dispatch registered) but the full browser-truth assertion has not been pinned by a regression test yet.
  • REJECTED — rejected by design at parse time or by the dispatch policy; the rejection IS the contract.

1. Panel-shared CONSUMER + spec= (FINDING #1)

1a. AFTER: YES   BEFORE: silently dropped — cross-formula ppVar(shared='col') seeded via spec=list(shared_col="mpg") 
library(ggpaintr)

ptr_app_grid(
  plots = c(
    "ggplot(mtcars, aes(x = ppVar('cyl', shared = 'col'), y = mpg)) + geom_point()",
    "ggplot(mtcars, aes(x = ppVar('cyl', shared = 'col'), y = hp))  + geom_col()"
  ),
  spec = list(shared_col = "mpg")
)

Before fix — actual: picker boots showing "cyl" (the formula's default = 'cyl'; shared_widget_default() first-occurrence wins). The spec= entry is silently dropped because every per-instance ptr_server()'s apply_spec_at_boot() prefix-filters by its own module namespace (plot_1- etc.) at R/paintr-server.R:533, and ptr_shared_server() never received spec=.

After fix — actual: picker boots showing "mpg" (spec wins). ptr_app_grid() forwards spec into ptr_shared_server(); the new apply_spec_at_boot_host() writes host_spec_seed[["shared_col"]] <- "mpg"; ptr_bind_shared_consumer_uis()'s seed read now consults the spec_seed= argument before falling back to state$spec_seed (which is still NULL at host scope by ADR 0006).

Browser-truth pin: tests/testthat/test-spec-panel-widgets-boot.R test 1.

1b. UNCHANGED: YES — shared-section consumer + spec= (single-formula reference) was already working 
ptr_app(
  "ggplot(mtcars, aes(x = ppVar('cyl', shared = 'col'), y = mpg)) + geom_point()",
  spec = list(shared_col = "mpg")
)

Single formula referencing shared='col' ⇒ key is shared-section, not shared-panel. ptr_app() uses ptr_make_app_server() (no coordinator); the per-instance apply_spec_at_boot() runs at top level (ns is identity), so shared_col matches the prefix and lands in state$spec_seed. Picker shows "mpg" at boot. Test coverage: test-shared-consumer-spec-seed.R.

2. Panel-shared VALUE + spec= (FINDING #7)

2a. AFTER: YES   BEFORE: widget literally absent from the DOM — cross-formula ppNum(shared='lw') seeded via spec=list(shared_lw=2) 
library(ggpaintr)

ptr_app_grid(
  plots = c(
    "ggplot(mtcars, aes(x = wt, y = mpg)) + geom_line(linewidth = ppNum(1, shared = 'lw'))",
    "ggplot(mtcars, aes(x = wt, y = hp))  + geom_line(linewidth = ppNum(1, shared = 'lw'))"
  ),
  spec = list(shared_lw = 2)
)

Before fix — actual: the <div id="shared_lw_ui" class="shiny-html-output"></div> placeholder is emitted by shared_panel_body_tag() at the host's un-namespaced root, but no server-side renderUI registers output$shared_lw_ui. The per-instance shared-value loop in ptr_setup_value_uis() registers at the per-instance namespaced id (output[["plot_1-shared_lw_ui"]]), which never reaches the host's UI div. Net: the numericInput never materializes; app$get_value(input = "shared_lw") returns NULL.

After fix — actual: ptr_setup_panel_values() (new, at host scope, parallel to ptr_setup_panel_sources()) registers output$shared_lw_ui <- renderUI(...) reading from host_spec_seed. Widget materializes with value="2". app$get_value(input = "shared_lw") returns 2.

Browser-truth pin: tests/testthat/test-spec-panel-widgets-boot.R test 2.

2b. UNCHANGED: YES — panel-shared value without spec= still gets the formula's first-occurrence default 
ptr_app_grid(
  plots = c(
    "ggplot(mtcars, aes(x = wt, y = mpg)) + geom_line(linewidth = ppNum(3, shared = 'lw'))",
    "ggplot(mtcars, aes(x = wt, y = hp))  + geom_line(linewidth = ppNum(1, shared = 'lw'))"
  )
)

shared_widget_default() still picks the first occurrence (3) via node$default; has_rendered = FALSE on first fire of the new host-scope renderUI means selected_arg = seed (here NULL), so invoke_build_ui() falls back to node$default just like the per-instance path. Widget shows 3 at boot.

3. Panel-shared SOURCE COMPANION + spec=

3a. AFTER: PARTIAL (structural)   BEFORE: silently dropped — panel-shared ppUpload(shared='ds')'s companion textInput seeded via spec=list(shared_ds_name="df_main") 
library(ggpaintr)

df_main <- mtcars
df_aux  <- iris

ptr_app_grid(
  plots = c(
    "ppUpload(shared = 'ds') |> ggplot(aes(x = ppVar('cyl', shared = 'col'), y = mpg)) + geom_point()",
    "ppUpload(shared = 'ds') |> ggplot(aes(x = ppVar('cyl', shared = 'col'), y = hp))  + geom_col()"
  ),
  spec = list(shared_ds_name = "df_main")
)

Before fix — actual: the companion textInput at the host's un-namespaced root id shared_ds_name first-renders blank. The per-instance apply_spec_at_boot() drops the un-namespaced spec id; no host-scope handler exists; the FINDING #8 onFlushed updateTextInput dispatch never fires for host-owned source companions.

After fix — actual at the seam: apply_spec_at_boot_host() matches shared_ds_name against the shared_<k>_name companion pattern, normalizes the value as text, writes host_spec_seed[["shared_ds_name"]] <- "df_main", and registers a deferred session$onFlushed(once = TRUE, updateTextInput(session, "shared_ds_name", "df_main")) — the same shape as the per-instance source_companion branch in apply_spec_at_boot() (R/paintr-server.R:680–700), which already has browser-pinned coverage at test-finding8-source-companion-spec.R. Verified at the structural seam via a manual probe (matching logic finds the companion id; onFlushed callback registered).

Caveat (the PARTIAL): not yet pinned by a shinytest2 regression test on this branch. The pre-existing per-instance companion path (FINDING #8) uses the same dispatch mechanism end-to-end and IS browser-pinned, so the host variant should follow the same end-to-end success — but the gate is not yet there. Follow-up: add tests/testthat/fixtures/vignette-apps/spec-panel-source-companion/app.R + a third assertion to test-spec-panel-widgets-boot.R.

3b. REJECTED (unchanged) — targeting the fileInput itself via spec=list(shared_ds=…) is silently skipped 
ptr_app_grid(
  plots = c(
    "ppUpload(shared = 'ds') |> ggplot(aes(x = ppVar('cyl', shared = 'col'), y = mpg)) + geom_point()",
    "ppUpload(shared = 'ds') |> ggplot(aes(x = ppVar('cyl', shared = 'col'), y = hp))  + geom_col()"
  ),

The fileInput cannot accept a programmatic value (recorded in source at R/paintr-server.R:582–584). Both per-instance apply_spec_at_boot() and the new apply_spec_at_boot_host() silently skip the row when the matched node is_ptr_ph_data_source(node). Spec the bare-name companion (shared_ds_name, example 3a) instead.

4. Composite: panel source + panel consumer + spec= for BOTH

4a. AFTER: YES (consumer) + PARTIAL (companion) — ADR 0023 textbook case with both halves seeded 
df_main <- mtcars

ptr_app_grid(
  plots = c(
    "ppUpload(shared = 'ds') |> ggplot(aes(x = ppVar(shared = 'col'), y = mpg)) + geom_point()",
    "ppUpload(shared = 'ds') |> ggplot(aes(x = ppVar(shared = 'col'), y = hp))  + geom_col()"
  ),
  spec = list(
    shared_ds_name = "df_main",   # panel source companion (PARTIAL — see 3a)
    shared_col     = "mpg"         # panel consumer (YES — see 1a)
  )
)

Before fix: both halves silently dropped; the picker boots blank (no default= in the consumer formula) and the companion textInput boots blank. The user has to type the dataset name AND pick a column manually before any plot draws.

After fix: consumer half is pinned by the gate; companion half is verified at the structural seam (see 3a). End-to-end: the page boots with the dataset name pre-filled, the panel resolver mirrors df_main through state$resolved_sources + state$bound_names, the consumer picker takes its dep on panel_sources[["shared_ds"]]() + reads spec_seed[["shared_col"]], and both plots draw x = mpg from df_main at first paint.

5. Rejected-by-design (unchanged, still rejected)

5a. REJECTED — panel-shared consumer + non-shared / shared-section source (FINDING #4) 
# Panel-owned consumer ppVar(shared='col') appears in both formulas,
# but each formula has its OWN bare-data source (mtcars vs iris).
ptr_shared(c(
  "ggplot(mtcars, aes(x = ppVar(shared = 'col'))) + geom_point()",
  "ggplot(iris,   aes(x = ppVar(shared = 'col'))) + geom_point()"
))
# Aborts at parse time with condition class
# `ptr_panel_consumer_source_mismatch` — ADR 0023 §6, "one widget, one
# source-of-truth". Fix the formula, not the package: either promote the
# source to panel-shared too, or demote the consumer to shared-section.

Adding a spec= entry doesn't change this: the abort fires before any server / spec-apply code runs.

5b. REJECTED — spec=list(<unknown-id>=...) at host scope is dropped with an i message

If the spec key doesn't match any panel-shared id pattern (shared_<k> for a panel key, or shared_<k>_name for a panel-owned source companion), the host applier silently skips the row. Per-instance apply_spec_at_boot() still emits its cli::cli_inform() “Dropped N unknown ids” if the same id doesn't match any per-instance widget either. (The host applier is intentionally quieter: unknown un-namespaced ids could legitimately belong to another coordinator on the same page; a noisy warning would false-positive in that pattern.)
