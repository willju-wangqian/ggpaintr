PLAN-01 — Default-validator helpers

Owning ADR: dev/adr/0009-pp-placeholders-and-expression-input.html · Branch: adr9/01-default-arg-helpers · Parallelism: PARALLEL with PLAN-02, 03, 04, 05 · Depends on: nothing · Date: 2026-05-20

What this plan does (self-contained)

Adds five small, pure-function helpers that ggpaintr will use to validate the unevaluated AST of a placeholder's positional default and named arguments. These helpers do not change any existing behaviour: they introduce new exported helpers that no other code calls yet. PLAN-03 (registry) and PLAN-06 (parser) will consume them later.

Each helper accepts an unevaluated R expression (the AST captured by the parser) and either returns a canonical value or aborts with a clear, keyword-specific message. The helpers never call eval() — they pattern-match the AST. Symbol policy is per-helper (see table below); the parser remains keyword-agnostic.

Context from ADR 0009 (only the bits this plan touches)

ADR 0009 specifies a registry-declared argument schema where the placeholder definition picks which validator runs on its positional argument. The validators ship as named helpers so authors do not have to write their own for common shapes. The symbol policy lives in the helper, not in the parser, so the same AST (ppX(mpg)) can mean "column-name string" in one keyword's definition and "rejected" in another.

Helper contract (closure shape)

Every helper is a factory:

ptr_default_numeric <- function() {
  function(arg_expr) {
    # arg_expr is unevaluated language (number, symbol, call, string, ...)
    if (is.numeric(arg_expr) && length(arg_expr) == 1L && !is.na(arg_expr)) {
      return(arg_expr)
    }
    rlang::abort(
      "Default for this placeholder must be a single numeric literal."
    )
  }
}

The closure is what gets stored in entry$default_arg. PLAN-06's parser calls it with the placeholder's positional arg AST and stores the returned canonical value on node$default.

Why factories rather than bare functions: some helpers (like ptr_default_numeric_vector(length = 2L)) need parameters. Making them all factories keeps the call shape uniform — every default_arg = ptr_default_X() reads the same way — and keeps the door open to per-helper configuration without a future API churn.

AST primer (for the agent implementing this)

Helpers receive the result of arg_expr from a placeholder call's AST. In R:

  • ppX(mpg) → arg_expr is mpg (a symbol); test with is.symbol() and use as.character() for its name.
  • ppX("mpg") → arg_expr is "mpg" (a character scalar).
  • ppX(5) → arg_expr is 5 (a numeric scalar of length 1).
  • ppX(c(1, 2)) → arg_expr is a call (is.call(x) is TRUE); x[[1]] is the symbol c, x[[2]], x[[3]], ... are the literal arguments.
  • ppX(x + 1) → arg_expr is a call (+); language object — for ptr_default_expression() only.

Do not call eval() on arg_expr. Pattern-match the AST shape only. This is the safety invariant that lets the whole-formula walker stay the single eval-time gate.

Success Criteria

  1. R/paintr-default-args.R exists and exports five factory functions: ptr_default_symbol_or_string, ptr_default_string, ptr_default_numeric, ptr_default_numeric_vector, ptr_default_expression.
  2. Each factory returns a closure of shape function(arg_expr) -> canonical_value | abort().
  3. Symbol policy follows the accept/reject matrix above. ptr_default_symbol_or_string() returns the symbol's name as a character string when given a symbol; returns the string unchanged when given a string literal; aborts otherwise.
  4. ptr_default_expression() returns its input unchanged (it is purely a tag, not a transformer); helpful so the parser still stamps "this placeholder accepts a default" regardless of shape.
  5. ptr_default_numeric_vector(length = 2L) accepts c(0, 1) and rejects c(0) (length mismatch); without length argument, length is unchecked.
  6. Every abort message is a single sentence naming the constraint (e.g. "Default for ppNum must be a single numeric literal.") and is produced via rlang::abort(), not stop().
  7. Helpers do not call eval(), parse(), or any deparse → reparse cycle on the input.
  8. Unit tests in tests/testthat/test-default-args.R cover every cell of the matrix plus length-check behaviour. New tests are the only test additions; no existing test changes.
  9. The authoritative gate (NOT_CRAN=true devtools::test()) reports 0/0/0 with at least the previously-passing tests still passing.
  10. devtools::document() regenerates NAMESPACE cleanly with five new export() lines.

Constraints

  • No call sites yet. The five helpers are exported and tested, but nothing in R/paintr-*.R outside the new file calls them. PLAN-03 and PLAN-06 wire them up.
  • No eval at translate-time. The closures must operate on AST only.
  • No registry changes. R/paintr-registry.R is not touched in this plan.
  • No parser changes. R/paintr-translate.R is not touched.
  • No breaking changes. Existing public surface unchanged. The five helpers are pure additions.
  • roxygen docs required. Each helper gets a roxygen block with @param, @return, @export, and at least one @examples block. Document the symbol policy explicitly per helper.
  • No emojis in source or docs.
  • NOT_CRAN gate only. Do not declare done off a raw Rscript -e 'testthat::test_dir(...)' run (proxy trap: silently skips browser e2e).
