PLAN-05 — Dual-mode renderer (erased vs preserved placeholders)

Owning ADR: dev/adr/0009-pp-placeholders-and-expression-input.html · Branch: adr9/05-render-dual-mode · Parallelism: PARALLEL with PLAN-01, 02, 03, 04 · Depends on: nothing · Date: 2026-05-20

What this plan does (self-contained)

Adds a single new parameter preserve_placeholders (default FALSE) to ptr_render() and threads it through the render_walk.* S3 dispatchers. When FALSE, the renderer behaves exactly as today (the placeholder node is rendered as its current substituted value — "final code"). When TRUE, the renderer emits the placeholder call shape ppX(current_pick) instead — the "formula with placeholders" view that round-trips as valid plain R (because ppX is exported as identity; see PLAN-08).

This plan is purely renderer-side: it adds the capability to ptr_render() but does not wire any UI toggle. The toggle widget that switches between the two views in the running app lives in PLAN-08 (it must land with the rename to use the right keyword surface).

Mechanism note (verify before relying on this for the design): the existing render_walk.ptr_ph_value at line 209 (and its _data_consumer / _data_source siblings at lines 212 / 215) emit rlang::expr_text(node$expr) — i.e. they print the placeholder's source expression. The "final code" view shown in today's app is therefore produced by a substitution-before-render pass (the substitute_walk.ptr_ph_* methods in R/paintr-substitute.R at lines 80 / 102 / 138) that rewrites placeholder nodes into literal / resolved nodes before ptr_render walks the tree. Implementer: read those substitute_walk.* bodies (lines 80–175 of R/paintr-substitute.R) and confirm the swap shape before adopting the plan's "preserve-mode walks pre-substitution tree" framing. The two possible designs are:

  1. Two trees — preserve-mode renders the pre-substitution AST; erase-mode renders the post-substitution AST. The substitute pass is called once and the result kept alongside the original. Cleanest if the substitute pass is referentially transparent on the root.
  2. One tree with a flag — placeholder nodes carry the current pick (proposed field: node$current_pick); the renderer chooses between emitting the literal pick (erase) or ppX(pick) (preserve). Requires PLAN-07/08 to set current_pick at substitution time.

This plan assumes design (2) for the renderer-side change; PLAN-07 / PLAN-08 wire the current_pick setter. Tests in this plan stub current_pick by hand on synthetic placeholder nodes.

Threading strategy

Two acceptable patterns; choose whichever keeps the diff smaller while staying internally consistent:

  1. Pass-through second arg. Every render_walk.* method gains a preserve_placeholders parameter (default FALSE). The generic and all dispatchers thread it. Easy to grep; mechanically verifiable.
  2. Options list. A single opts list contains preserve_placeholders and the existing indent. Cleaner for future per-render options; bigger diff today.

Default to pattern (1) unless the diff becomes unwieldy. Pattern (1) keeps each S3 method's signature small and matches the existing indent argument shape.

Placeholder-mode rendering shape

For a placeholder node with node$keyword = "ppVar" and node$current_pick = "mpg":

ModeOutput
preserve_placeholders = FALSEmpg (today's behaviour: substitution upstream has already replaced the placeholder node with a ptr_literal node holding the symbol mpg)
preserve_placeholders = TRUEppVar(mpg) (placeholder preserved, current pick as the positional argument; uses the bareword form for symbol-or-string defaults, the quoted form for string defaults, the literal numeric for numeric defaults, etc.)

Where the renderer encounters a placeholder node with no current_pick recorded (e.g. before the user has selected anything), it should emit ppVar() — the call with empty arg list — when preserve_placeholders = TRUE. This is a valid R call (the identity function accepts no args via the ... sink). The "final code" mode would normally never see such a node (substitution would have pruned the layer), but if it does, fall back to today's render_walk.ptr_ph_value behaviour (rlang::expr_text(node$expr)).

Keyword-agnostic: the renderer reads node$keyword directly. After PLAN-08 renames the built-in keywords, the renderer emits the new names automatically with no further change here.

Symbol vs string formatting for the inner positional (G1.d from ADR 0009 edge-case resolutions):

  • For ppVar (ptr_default_symbol_or_string): emit a bareword when the column name is a syntactic R identifier (use base R's make.names(x) == x as the check); otherwise emit a backticked bareword — ppVar(mpg) or ppVar(`miles per gallon`). Never emit ppVar("mpg") as a string — inside aes() that would resolve to a constant aesthetic, not a column reference. The .data[[ppVar("col")]] form is a power-user alternative mentioned in ?ptr_app; this renderer does not emit it.
  • For ppText (ptr_default_string): emit quoted — ppText("hello").
  • For ppNum (ptr_default_numeric): emit numeric literal — ppNum(5).
  • For ppExpr (ptr_default_expression): emit deparsed expression via rlang::expr_text() — ppExpr(x + 1).
The renderer decides by inspecting typeof(node$current_pick) and the registry's default_arg (looked up via ptr_registry_lookup(node$keyword)). Implement a small helper format_placeholder_value(value, keyword) that does the dispatch; tests cover each shape including non-syntactic column names (backtick path).

G2.a — shared placeholders render at every occurrence. When a placeholder has a non-NULL node$shared, the renderer emits the full ppX(current_pick, shared = "k") form at every occurrence in the rendered formula, not only the first. Rationale: the rendered formula round-trips correctly (re-translation applies first-occurrence-wins and arrives at the same value); the renderer does not need to track occurrence index, keeping the code path simple. Authoring-time intent (where the user wrote the default) is preserved at every site.

G3.a — current pick wins over author default in preserve mode (already implied by the renderer-uses-current_pick design above): when the user's runtime pick differs from the formula-time default, preserve mode emits the current pick — e.g. user authored ppNum(5), picked 7.5, renderer emits ppNum(7.5). The preserve view is "formula + current state", consistent across var/num/text/expr.

Success Criteria

  1. ptr_render(root) with no second argument produces output identical to today's ptr_render(root) on every baseline tree.
  2. ptr_render(root, preserve_placeholders = FALSE) is equivalent to the no-arg form.
  3. ptr_render(root, preserve_placeholders = TRUE) emits the placeholder call form for every placeholder node in the tree, using its current_pick as the positional argument.
  4. The two modes agree on all non-placeholder structure: identical whitespace, identical indentation, identical + joining, identical pipe chains.
  5. Symbol-vs-string formatting follows the default_arg-implied shape (per the table above); a small helper handles the dispatch.
  6. Placeholder nodes with no current_pick render as ppX() in preserve mode and as today (expr_text(node$expr)) in erase mode.
  7. The authoritative gate reports 0/0/0 PASS N ≥ baseline + new tests in test-render-dual-mode.R.

Constraints

  • No substitution-pass changes. R/paintr-substitute.R is not touched by this plan. PLAN-07/08 will set node$current_pick on placeholder nodes; until then, tests in this plan can stub a placeholder node with a hand-set current_pick.
  • No UI changes. The toggle widget that picks between the two modes in the running app lives in PLAN-08.
  • No keyword changes. The renderer reads node$keyword — works with both today's vocabulary (var/num/...) and tomorrow's (ppVar/ppNum/...).
  • Backward-compat default. Calls that do not pass preserve_placeholders behave identically to today. Existing callers in the codebase (e.g. server-side code-gen) need not be modified by this plan.
  • S3 method exports. Each render_walk.* method must keep its @export roxygen line; preserving S3 dispatch across packages is documented in the existing code.
