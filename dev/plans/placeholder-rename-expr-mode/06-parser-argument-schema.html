PLAN-06 — Parser argument schema (extract_placeholder_args)

Owning ADR: dev/adr/0009-pp-placeholders-and-expression-input.html · Branch: adr9/06-parser-schema · Parallelism: SEQUENTIAL (after PLAN-01, 02, 03) · Date: 2026-05-20

What this plan does (self-contained)

Replaces the parser-internal, keyword-agnostic helper extract_shared(expr, keyword) with a registry-driven extract_placeholder_args(expr, entry). The new helper reads entry$default_arg and entry$named_args from the registry entry and applies the per-keyword schema:

  • If entry$default_arg is NULL, reject any positional argument with today's error text.
  • Otherwise call entry$default_arg(positional_arg_expr); store the canonical return on node$default.
  • For each named arg other than shared, look it up in entry$named_args; reject unknown names; call the validator; store on node$named_args.
  • Continue to extract shared as today; store on node$shared.

Built-in registrations are not changed in this plan — they still register with default_arg = NULL and named_args = list(), so the parser's externally observable behaviour is identical to today (positional + unknown-named both rejected). PLAN-08 wires the actual default_arg helpers into the built-in registrations as part of the rename.

Context from ADR 0009 (only the bits this plan touches)

Per ADR 0009 §6 ("Parser change"):

  • extract_shared(expr, keyword) at R/paintr-translate.R:381 is renamed to extract_placeholder_args(expr, entry).
  • Signature changes from receiving the keyword string to receiving the full registry entry — so the validator closures are reachable.
  • Existing error texts ("positional args are not allowed", "got unknown argument(s)") survive as the no-schema-declared fallbacks.
  • Validation failures coming back from a validator closure are translated to translate-time aborts; the validator's message becomes the user-facing error.
  • Validators receive the unevaluated AST. The parser never eval()s a placeholder's positional or named-arg AST.

Storage on the placeholder node:

node$default     # validated positional, or NULL  (PLAN-02 added the slot)
node$named_args  # named list of validated named args, or list()  (PLAN-02)
node$shared      # existing slot, behaviour unchanged

Target helper shape

extract_placeholder_args <- function(expr, entry) {
  if (is.symbol(expr)) {
    return(list(shared = NULL, default = NULL, named_args = list()))
  }
  if (length(expr) == 1L) {
    # bare call without parens shouldn't happen but be defensive
    return(list(shared = NULL, default = NULL, named_args = list()))
  }
  args      <- as.list(expr[-1L])
  arg_names <- names(args) %||% rep_len("", length(args))
  keyword   <- entry$keyword

  # Split named vs positional
  named_idx <- nzchar(arg_names)
  named     <- args[named_idx]
  positional <- args[!named_idx]

  # Positional handling
  default <- NULL
  if (length(positional) >= 1L) {
    if (is.null(entry$default_arg)) {
      rlang::abort(paste0(
        "Placeholder `", keyword, "(...)` accepts only named arguments; ",
        "positional args are not allowed."
      ))
    }
    if (length(positional) > 1L) {
      rlang::abort(paste0(
        "Placeholder `", keyword, "(...)` accepts at most one positional ",
        "argument (the default); got ", length(positional), "."
      ))
    }
    default <- entry$default_arg(positional[[1L]])
  }

  # Named handling
  recognized <- c("shared", names(entry$named_args %||% list()))
  unknown    <- setdiff(names(named), recognized)
  if (length(unknown) > 0L) {
    rlang::abort(paste0(
      "Placeholder `", keyword, "(...)` got unknown argument(s): ",
      paste(unknown, collapse = ", "), ". ",
      "Allowed: ", paste(recognized, collapse = ", "), "."
    ))
  }

  # Validate each declared named arg
  named_args_out <- list()
  for (nm in names(entry$named_args %||% list())) {
    if (nm %in% names(named)) {
      validator <- entry$named_args[[nm]]
      named_args_out[[nm]] <- validator(named[[nm]])
    }
  }

  # Extract shared (existing semantics, lifted out)
  shared <- extract_shared_value(named[["shared"]], keyword)

  list(shared = shared, default = default, named_args = named_args_out)
}

extract_shared_value <- function(shared_ast, keyword) {
  if (is.null(shared_ast)) return(NULL)
  if (!is.character(shared_ast) || length(shared_ast) != 1L ||
      is.na(shared_ast) || !nzchar(shared_ast)) {
    rlang::abort(paste0(
      "`", keyword, "(shared = ...)` requires a single non-empty string."
    ))
  }
  shared_ast
}

Success Criteria

  1. extract_shared is replaced by extract_placeholder_args(expr, entry) across all call sites.
  2. For an entry with default_arg = NULL and named_args = list(), the helper behaves identically to today's extract_shared: positional → abort; unknown-named → abort; shared = "k" → returned in the shared field; non-string shared → abort.
  3. For an entry with default_arg = ptr_default_numeric(), a call like foo(5) stores 5 on node$default.
  4. For an entry with named_args = list(step = ptr_default_numeric()), a call like foo(step = 0.5) stores list(step = 0.5) on node$named_args.
  5. For an entry with both, foo(5, step = 0.5, shared = "k") populates all three: node$default = 5, node$named_args$step = 0.5, node$shared = "k".
  6. Validator aborts propagate cleanly with the validator's message as the user-facing error.
  7. The parser does not call eval() on any placeholder argument AST. (Inspectable by reading the source — no eval, parse, get calls inside the helper.)
  8. All existing tests pass without modification; the new tests for the schema behaviour pass.
  9. The authoritative gate reports 0/0/0 PASS N ≥ baseline + new tests.

Shared-key role-uniqueness check (B1.a from ADR 0009 §"Edge-case resolutions")

After the tree is annotated and ptr_shared_bind has bucketed the occurrences by shared key, run a sibling validator that aborts when two occurrences with the same shared key come from different placeholder roles (e.g. one ppVar consumer + one ppNum value, both with shared = "k"). This catches a footgun where the same shared id would back two incompatible widget types.

Where it lands (R/paintr-translate.R, inside the annotate block around lines 60-66): add a new line ptr_validate_shared_roles(root) right after the existing root <- ptr_shared_bind(root) call.

Helper sketch:

ptr_validate_shared_roles <- function(root) {
  occs <- collect_shared_occurrences(root)   # walk tree; returns list keyed by shared
  for (key in names(occs)) {
    bucket <- occs[[key]]
    roles <- vapply(bucket, function(n) {
      if (is_ptr_ph_data_consumer(n)) "consumer"
      else if (is_ptr_ph_data_source(n)) "source"
      else "value"
    }, character(1))
    if (length(unique(roles)) > 1L) {
      keywords <- vapply(bucket, function(n) n$keyword, character(1))
      rlang::abort(paste0(
        "Shared key `\"", key, "\"` is used by placeholders with incompatible roles: ",
        paste0(keywords, "=", roles, collapse = ", "), ". ",
        "A shared widget can only back one role at a time."
      ))
    }
  }
  invisible(root)
}

The collect_shared_occurrences helper may already exist as collect_shared_consumer_occurrences in R/paintr-shared.R:144 — generalize or write a sibling that includes all placeholder roles, not only consumers.

BDD scenarios

Feature: parser dispatches placeholder argument validation through the registry

  Scenario: positional rejected when default_arg is NULL
    Given a registered placeholder with default_arg = NULL
    When the formula contains foo(5)
    Then ptr_translate aborts with the message "accepts only named arguments"

  Scenario: positional accepted when default_arg is a numeric validator
    Given a registered placeholder with default_arg = ptr_default_numeric()
    When the formula contains foo(5)
    Then ptr_translate succeeds
    And the resulting node$default equals 5

  Scenario: named arg rejected when not declared
    Given a registered placeholder with named_args = list()
    When the formula contains foo(label = "X")
    Then ptr_translate aborts with the message "unknown argument(s): label"

  Scenario: named arg accepted when declared
    Given a registered placeholder with named_args = list(step = ptr_default_numeric())
    When the formula contains foo(step = 0.5)
    Then ptr_translate succeeds
    And node$named_args$step equals 0.5

  Scenario: shared continues to work
    Given a registered placeholder
    When the formula contains foo(shared = "k")
    Then node$shared equals "k"
    And node$default is NULL
    And node$named_args is list()

  Scenario: default + named + shared combine
    Given a placeholder registered with default_arg, named_args, and shared support
    When the formula contains foo(5, step = 0.5, shared = "k")
    Then node$default = 5, node$named_args$step = 0.5, node$shared = "k"

  Scenario: validator failure surfaces as translate-time abort
    Given a placeholder registered with default_arg = ptr_default_numeric()
    When the formula contains foo("not a number")
    Then ptr_translate aborts with the validator's message
    And the error is an rlang_error

  Scenario: no eval on placeholder ASTs
    Given a placeholder registered with default_arg = ptr_default_expression()
    When the formula contains foo(system("rm -rf /"))
    Then ptr_translate succeeds (the expression is stored as language)
    And no system command is executed
