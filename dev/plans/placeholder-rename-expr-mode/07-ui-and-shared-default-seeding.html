PLAN-07 — UI hooks + shared coordinator default seeding

Owning ADR: dev/adr/0009-pp-placeholders-and-expression-input.html · Branch: adr9/07-ui-shared-defaults · Parallelism: SEQUENTIAL (after PLAN-02 + PLAN-06) · Date: 2026-05-20

What this plan does (self-contained)

Wires node$default (set by PLAN-06's parser) into two consumer paths:

  1. Per-placeholder widget initialization (R/paintr-builtins.R + R/paintr-build-ui.R): when a placeholder's widget is built, the widget's initial value is set from node$default (when no persisted prior input exists). This requires extending the built-in build_ui hooks for num/text/expr to accept and use a default argument (today only var's hook accepts selected); also requires the build-ui orchestrator at the call-site of each hook to pass node$default through.
  2. Shared coordinator default seeding (R/paintr-shared.R): the label-resolution helper (shared_widget_label at line 177) currently picks occurrences[[1L]] for the first-occurrence-wins rule. This plan adds a sibling helper (or extends shared_widget_label) to surface the first occurrence's default so the shared widget's initial value can use it.

The third touch (R/paintr-substitute.R) is small and defensive: the substitution walk that converts placeholder nodes to their resolved values continues to read input$<id> as the source of truth; the default participates only at boot, through the UI initialization above. No new behaviour in substitute.R itself, but a comment is added to record the invariant for future readers.

Premise correction (2026-05-20 audit). An earlier draft of this plan claimed "the orchestrator computes a selected from persisted input state today" — that is incorrect. grep -n "selected\b" R/paintr-build-ui.R returns zero matches; the orchestrator does not currently compute a selected argument. The only built-in hook that has a selected formal is ptr_builtin_var_build_ui (paintr-builtins.R:132); the four others (text/num/expr/upload) take node, label, copy, ... and would silently swallow any extra arg via .... This plan therefore introduces the orchestrator-level computation and extends the value-placeholder hooks to consume it, rather than just adjusting an existing path.

Context from ADR 0009 (only the bits this plan touches)

ADR 0009 §8 ("Default + shared, combinable"): ppVar(mpg, shared = "axis") is legal; first-occurrence-in-formula-order wins for the shared widget's default. Matches today's tiebreak rule for label and widget-config conflicts among multiple shared = "k" occurrences (see R/paintr-shared.R:177 — shared_widget_label picks occurrences[[1L]] for its config).

ADR 0009 §5 ("Default-validator helpers"): the canonical default value stored on node$default matches the helper's return type — string for ptr_default_symbol_or_string and ptr_default_string; numeric scalar for ptr_default_numeric; numeric vector for ptr_default_numeric_vector; language for ptr_default_expression.

Build-ui change (sketch)

Today, the orchestrator in R/paintr-build-ui.R does not compute any initial-value argument. Locate the call site that does do.call(entry$build_ui, ...) (a likely candidate is the invoke_build_ui helper around line 669, and build_ui_for.ptr_ph_data_source around line 68 — grep do.call.*build_ui in the file to enumerate). Each call site assembles the named args passed to the hook.

After this plan, the orchestrator computes an initial value from persisted input + node$default and passes it to the hook:

persisted <- isolate(input[[node$id]])
initial   <- if (length(persisted) && nzchar(as.character(persisted)[[1L]])) {
  persisted
} else if (!is.null(node$default)) {
  node$default
} else {
  character(0)
}
build_ui_args <- list(
  rendered_node, label = copy$label,
  selected = initial,                          # consumed by var hook today; extended below for value hooks
  named_args = node$named_args %||% list()
)
do.call(entry$build_ui, c(build_ui_args, list(...)))

Shared coordinator change (sketch)

Mechanism correction (2026-05-20 audit). The actual resolve_shared_consumer at line 192 returns list(kind, value, error) — an upstream/source arbitration record — not a label/widget-config record. The first-occurrence-wins rule for the shared widget's label lives in a separate helper, shared_widget_label at line 177, which picks occurrences[[1L]] for its config. The default-seeding hook therefore lands alongside or inside shared_widget_label (or a new sibling helper called from the same orchestrator that already calls shared_widget_label) — not inside resolve_shared_consumer.

Recommended shape — add a tiny new helper that mirrors shared_widget_label's first-occurrence-wins rule:

shared_widget_default <- function(occurrences) {
  if (length(occurrences) == 0L) return(NULL)
  occurrences[[1L]]$default   # NULL if first has no default
}

And wire it at the existing call site that builds the shared widget (find it via grep -n "shared_widget_label\|canonical_shared_id" R/; the orchestrator passes the resolved label through to the per-keyword shared-widget builder). The new helper's return value is passed alongside the label to the builder, which threads it into the widget's initial value the same way the per-layer orchestrator threads node$default into per-layer widgets.

Do not touch resolve_shared_consumer. Its list(kind, value, error) shape is unrelated to widget configuration and would not be improved by carrying a default. Its callers parse $kind against "upstream"/"source"/"error" and would break if extra fields appeared in unexpected positions.

If a second occurrence has a different default, it is silently ignored. This matches ADR 0009 §8 ("If two occurrences disagree on the default, the first wins silently; no translate-time abort.") and mirrors today's silent-tiebreak behaviour in shared_widget_label.

Success Criteria

  1. For a placeholder with node$default = "mpg" and no persisted input, the built picker has "mpg" as its initial selected value.
  2. For a placeholder with persisted input "disp" and node$default = "mpg", the picker's initial value is "disp" (persisted wins).
  3. For a placeholder with no persisted input and node$default = NULL, the picker's initial value matches today's behaviour (empty / character(0)).
  4. For a shared bucket with two occurrences, both with shared = "k" and default = "mpg" and default = "disp" respectively, the resolved coordinator carries default = "mpg" (first-occurrence-wins). No warning is emitted.
  5. For a shared bucket where the first occurrence has default = NULL and the second has default = "disp", the resolved coordinator carries default = NULL.
  6. build_ui hooks receive named_args as an additional argument (named list, possibly empty). Existing hooks that don't consume it continue to work via the ... sink.
  7. The substitution walk in R/paintr-substitute.R is unchanged in behaviour; the new comment documents the invariant.
  8. The authoritative gate reports 0/0/0 PASS N ≥ baseline + new tests.
