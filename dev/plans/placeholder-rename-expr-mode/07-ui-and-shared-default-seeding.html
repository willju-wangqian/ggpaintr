PLAN-07 — UI hooks + shared coordinator default seeding

Owning ADR: dev/adr/0009-pp-placeholders-and-expression-input.html · Branch: adr9/07-ui-shared-defaults · Parallelism: SEQUENTIAL (after PLAN-02 + PLAN-06) · Date: 2026-05-20

What this plan does (self-contained)

Wires node$default (set by PLAN-06's parser) into two consumer paths:

  1. Per-layer widget initialization (R/paintr-build-ui.R): when a placeholder's widget is built and there is no persisted prior input value, the widget's initial selected / value uses node$default.
  2. Shared coordinator seeding (R/paintr-shared.R): when multiple ppX(..., shared = "k") occurrences exist with potentially different defaults, the first occurrence's default wins; the shared widget uses that value as its initial selection.

The third touch (R/paintr-substitute.R) is small and defensive: the substitution walk that converts placeholder nodes to their resolved values continues to read input$<id> as the source of truth; the default participates only at boot, through the UI initialization above. No new behaviour in substitute.R itself, but a comment is added to record the invariant for future readers.

Context from ADR 0009 (only the bits this plan touches)

ADR 0009 §8 ("Default + shared, combinable"): ppVar(mpg, shared = "axis") is legal; first-occurrence-in-formula-order wins for the shared widget's default. Matches today's tiebreak rule for label and widget-config conflicts among multiple shared = "k" occurrences (see R/paintr-shared.R:177 — shared_widget_label picks occurrences[[1L]] for its config).

ADR 0009 §5 ("Default-validator helpers"): the canonical default value stored on node$default matches the helper's return type — string for ptr_default_symbol_or_string and ptr_default_string; numeric scalar for ptr_default_numeric; numeric vector for ptr_default_numeric_vector; language for ptr_default_expression.

Build-ui change (sketch)

Today, the orchestrator passes a computed selected to the placeholder's build_ui hook. The computation currently looks roughly like:

selected <- isolate(input[[node$id]]) %||% character(0)

After this plan:

persisted <- isolate(input[[node$id]])
selected  <- if (length(persisted) && nzchar(persisted[[1L]])) {
  persisted
} else if (!is.null(node$default)) {
  node$default
} else {
  character(0)
}
build_ui_args <- list(
  node = node, cols = cols, data = data, label = label,
  copy = copy, selected = selected,
  named_args = node$named_args %||% list()
)
do.call(entry$build_ui, build_ui_args)

Built-in hooks already accept the ... sink, so passing named_args through as an extra named argument is backward-compatible: hooks that do not consume it ignore it via the sink.

Shared coordinator change (sketch)

Today's resolve_shared_consumer picks occurrences[[1L]] for label and widget config. Extend it to also extract the default:

resolve_shared_consumer <- function(occurrences) {
  if (length(occurrences) == 0L) return(NULL)
  first <- occurrences[[1L]]
  default <- first$default  # first-occurrence-wins; NULL if first has no default
  # ... existing label / widget-config resolution ...
  list(
    id = canonical_shared_id(first$shared),
    label = shared_widget_label(occurrences),
    keyword = first$keyword,
    default = default,
    occurrences = occurrences
  )
}

The downstream coordinator that builds the shared widget reads resolved$default as its initial value. Today's path (label / widget config) is unchanged; default is a new field on the resolved record.

If a second occurrence has a different default, it is silently ignored. This matches ADR 0009 §8: "If two occurrences disagree on the default, the first wins silently; no translate-time abort." No warning emitted — consistent with the existing label-conflict behaviour, which also wins silently.

Success Criteria

  1. For a placeholder with node$default = "mpg" and no persisted input, the built picker has "mpg" as its initial selected value.
  2. For a placeholder with persisted input "disp" and node$default = "mpg", the picker's initial value is "disp" (persisted wins).
  3. For a placeholder with no persisted input and node$default = NULL, the picker's initial value matches today's behaviour (empty / character(0)).
  4. For a shared bucket with two occurrences, both with shared = "k" and default = "mpg" and default = "disp" respectively, the resolved coordinator carries default = "mpg" (first-occurrence-wins). No warning is emitted.
  5. For a shared bucket where the first occurrence has default = NULL and the second has default = "disp", the resolved coordinator carries default = NULL.
  6. build_ui hooks receive named_args as an additional argument (named list, possibly empty). Existing hooks that don't consume it continue to work via the ... sink.
  7. The substitution walk in R/paintr-substitute.R is unchanged in behaviour; the new comment documents the invariant.
  8. The authoritative gate reports 0/0/0 PASS N ≥ baseline + new tests.

Constraints

  • No changes to built-in hooks. The five built-in placeholders' hooks in R/paintr-builtins.R are not modified by this plan. They already accept a selected argument and a ... sink.
  • No keyword changes. The rename to ppX is PLAN-08.
  • No parser changes. PLAN-06 set node$default; this plan consumes it.
  • First-occurrence-wins silently. No warning on default conflicts among shared occurrences — consistent with today's label-conflict policy.
  • NOT_CRAN gate only. Browser-driven UI assertions require NOT_CRAN=true.
  • No emojis.
