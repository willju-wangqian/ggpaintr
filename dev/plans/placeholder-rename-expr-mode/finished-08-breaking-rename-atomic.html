PLAN-08 — Breaking rename (built-ins + vignettes + tests + UI toggle)

Owning ADR: dev/adr/0009-pp-placeholders-and-expression-input.html · Branch: adr9/08-rename-atomic · Parallelism: ATOMIC (single merge; everything flips together) · Date: 2026-05-20

This plan must land as a single merge. It flips every formula in the codebase from var/num/text/expr/upload to ppVar/ppNum/ppText/ppExpr/ppUpload in lock-step. Splitting the changes across PRs would leave the suite red between merges (the authoritative gate, NOT_CRAN=true devtools::test(), requires 0/0/0 — and any single uncoordinated rename leaves stale fixtures using the wrong vocabulary).

What this plan does (self-contained)

The final, breaking step of the ADR-0009 work:

  1. Re-register the five built-in placeholders under their new keyword names (ppVar, ppNum, ppText, ppExpr, ppUpload) in R/paintr-builtins.R. Each declares its default_arg helper (consumed by the parser as wired in PLAN-06).
  2. Export plain-R callables for the five names: identity functions for ppVar/ppNum/ppText/ppExpr, a guard function for ppUpload. These are returned by the PLAN-03 define helpers; assign them to the keyword names at the top level of R/paintr-builtins.R with @export roxygen.
  3. Remove the old keyword registrations. Old names cease to be placeholders. A formula that still says var(mpg) fails at translate-time with the standard "unknown keyword" error.
  4. Rename every formula in vignettes/, tests/testthat/, tests/testthat/fixtures/, tests/manual/, README.Rmd, and any in-source roxygen example from old keywords to new ones.
  5. Wire the two-version code-gen UI toggle. Server-side code-output assembly is in ptr_register_code() at R/paintr-server.R:1391–1402 (the shiny::renderText({...}) call is at line 1394). UI shell is code_window_tag() at R/paintr-app.R:353, with shiny::verbatimTextOutput(ns("ptr_code")) rendered at R/paintr-app.R:376 and :386. R/paintr-shared-ui.R has no code-panel UI. Toggle: default tab "final code"; toggled view calls ptr_render(root, preserve_placeholders = TRUE) (PLAN-05). The toggle widget itself (radio / segmented control) goes alongside verbatimTextOutput(ns("ptr_code")) in R/paintr-app.R; the server reads its reactive value and switches between the two render calls inside ptr_register_code().
  6. Update NEWS.md with the breaking-change announcement and DESCRIPTION if any user-visible bump is appropriate.
  7. Update CONTEXT.md only if it cites old keyword names (the placeholder taxonomy itself is unchanged; only the surface vocabulary moves).

Order of edits within this plan

Sequence designed to minimize the time the suite is red within the worktree (the merge is atomic, but during local development the working tree must converge to green):

  1. Step 1: Add new built-in registrations under ppX names in R/paintr-builtins.R, keeping the old var/num/... registrations in place. Both vocabularies are registered simultaneously. ptr_builtin_keywords() returns both old and new. Tests still pass.
  2. Step 2: Rename inside formulas across vignettes, tests, fixtures. Many places use old names; they keep passing because old names are still registered.
  3. Step 3: Run the suite (NOT_CRAN gate) — must be 0/0/0. If anything still uses the old names that should not, it surfaces here.
  4. Step 4: Remove the old registrations. ptr_builtin_keywords() returns only ppX names. Run the suite again — any stray use of an old keyword now hard-errors.
  5. Step 5: Update internal string lookups in R source that were keyed on old names (default-copy maps, safety-walker lists, etc.). Run the suite.
  6. Step 6: Wire the UI toggle.
  7. Step 7: Update NEWS, README, DESCRIPTION. Re-knit README. Run devtools::document(). Run devtools::check().
  8. Step 8: Final authoritative-gate run (NOT_CRAN). Confirm 0/0/0.

This is local-only sequencing — the merge itself is one PR. The user merges once and the entire flip happens.

Success Criteria

  1. Five new exports — ppVar, ppNum, ppText, ppExpr, ppUpload — appear in NAMESPACE.
  2. Old keywords (var, num, text, expr, upload) are no longer registered in the placeholder registry. ptr_registry_keywords() returns only the new names plus any user-registered custom keywords.
  3. ptr_translate("ppVar(mpg)") succeeds and produces a placeholder node with node$keyword == "ppVar" and node$default == "mpg".
  4. ptr_translate("var(mpg)") aborts with a "keyword not registered" error.
  5. ppVar(mpg) evaluated as plain R inside ggplot's tidy-eval (e.g. ggplot_build()) returns the column value — identity behaviour. The same plain-R histogram baseline and ppVar(mpg)-wrapped version produce identical plots.
  6. ppUpload() called outside ptr_app() aborts with the documented guard message.
  7. Every vignette knits without error against the new vocabulary.
  8. Every fixture app under tests/testthat/fixtures/vignette-apps/ boots and exercises the new vocabulary.
  9. The code panel in ptr_app() shows a toggle between "final code" and "formula with placeholders". Default view = final code. Switching the toggle shows the preserved form using the new keyword vocabulary.
  10. The authoritative gate reports 0/0/0 PASS N where N is at least the baseline plus the new test-rename-coverage.R tests.
  11. devtools::check(--as-cran) → 0 errors / 0 warnings; by-design NOTEs allowed.
  12. README re-knits cleanly with the new vocabulary; README.md diff is consistent with README.Rmd.
  13. NEWS.md carries a clear "Breaking changes" entry.

Constraints

  • No old-name aliases. ADR 0009 mandates a hard rename, no deprecation cycle. Old names cease to be placeholders entirely.
  • Plain-R exports must not shadow base R. None of ppVar, ppNum, ppText, ppExpr, ppUpload exists in base R or in any common tidyverse package — verify with ?ppVar etc. in a fresh R session, expecting "no documentation".
  • The five exports must work as identity (or guard for ppUpload) when called outside ptr_app(). Tests cover this via ggplot_build()-flavored use.
  • UI toggle defaults to "final code". Existing users see the same default they see today; the toggle is an opt-in.
  • roxygen docs. Each exported plain-R function gets a one-paragraph roxygen with @param, @return, @export, and at least one @examples block showing both the in-formula use and the plain-R identity use.
  • Use devtools::document() to regenerate NAMESPACE. Do not hand-edit.
  • NOT_CRAN gate only. See proxy-trap list below.
  • Surface-not-absorb. If during implementation you discover work outside this plan's "Files touched" scope — a related bug, a refactor opportunity, an adjacent test gap, a doc inconsistency — file it as a follow-up (add a new entry under dev/plans/ or a one-line NEWS note) and STOP. Do NOT silently expand scope. Do NOT paper over by editing the out-of-scope file "while you're there." Returning a smaller correct change beats absorbing scope creep. (Within the in-scope rename: yes, every fixture/vignette/test using an old keyword IS in-scope by definition — that's the plan's mandate. Out-of-scope means adjacent unrelated changes.)
  • No emojis.
  • Re-knit README. Per CLAUDE.md, README.Rmd is the editable source.

Proxy traps — DO NOT be fooled:

  • Raw Rscript -e 'testthat::test_dir(...)' without NOT_CRAN: skip_on_cran() fires and every shinytest2 browser test silently SKIPs. A "green" run off this is NOT the gate. Only NOT_CRAN=true / devtools::test() exercises the browser e2e.
  • devtools::check --as-cran: does not run the e2e tests (devtools sets NOT_CRAN=true in the check subprocess, but the source-root guard in boot_vignette_app() causes the e2e file to cleanly SKIP). An ERROR on the e2e file there is a harness defect (source-root guard regressed), not a product failure.
  • Browser stack absent locally: shinytest2 + chromote + Chrome are required. If absent, the e2e file SKIPs cleanly under the NOT_CRAN gate too — but that is NOT acceptable as the final gate. CI must run them.
  • Sub-agent self-report: any "tests pass / scope clean" from a sub-agent or worktree-internal verification is a claim to verify, not proof. Run the gate yourself before declaring done.
  • vignette↔fixture drift (project memory): fixture app.R headers carry "EXCEPT" notes that are authoritative drift signals. After renaming, re-audit fixture EXCEPT-headers and update them to reflect the new vocabulary.

Independence (within this plan)

This plan is internally cohesive — it cannot be split. All seven prior plans must have landed for this plan's edits to make sense (without PLAN-01's helpers, the new built-in registrations cannot declare a default_arg; without PLAN-06's parser, the schema is not consulted; without PLAN-05's renderer, the toggle has nothing to switch between). PLAN-08 is the single, atomic step that turns the prior groundwork into externally observable change.

Verification independence. Any implementer self-report ("tests pass", "DoD met", "scope clean") is a claim to be verified, never proof. The integrator MUST re-run the authoritative gate command from the Definition of Done before merging this plan, and MUST NOT treat the implementer's report (or any status block in this plan) as a substitute for that re-run. This is especially load-bearing for PLAN-08 because the rename is atomic — a partial flip leaves the suite red and the merge unsafe.
