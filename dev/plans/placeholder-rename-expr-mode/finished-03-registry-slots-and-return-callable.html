PLAN-03 — Registry slots + define-helper return value

Owning ADR: dev/adr/0009-pp-placeholders-and-expression-input.html · Branch: adr9/03-registry-slots · Parallelism: PARALLEL with PLAN-01, 02, 04, 05 · Depends on: nothing · Date: 2026-05-20

What this plan does (self-contained)

Extends the three public define helpers (ptr_define_placeholder_value, ptr_define_placeholder_consumer, ptr_define_placeholder_source) with three new parameters: default_arg, named_args, and an optional runtime override. Validates the new parameters at registration time. Changes the return value from "invisibly, the registry entry" to "the runtime callable" so authors can write ppRange <- ptr_define_placeholder_value(keyword = "ppRange", ...) in one statement. Adds a best-effort sys.calls() drift warning when the enclosing <- LHS differs from the keyword string.

This plan is additive: any existing call to ptr_define_placeholder_*() that omits the new parameters continues to register identical entries. Callers that ignore the return value (today's pattern in R/paintr-builtins.R) continue to work; the runtime callable is just returned and unused. PLAN-06 (parser) and PLAN-08 (built-ins) consume the new behavior later.

Context from ADR 0009 (only the bits this plan touches)

Per ADR 0009 §4 ("Registry-declared argument schema") and §7 ("Define-helper surface returns the runtime callable"), each registry entry gains:

SlotTypeDefaultMeaning
default_argNULL or validator function (closure)NULLSchema for the (single) positional argument. NULL ⇒ positional rejected at translate-time. Closure ⇒ called by parser with unevaluated AST; returns canonical value or aborts.
named_argsnamed list of validator closureslist()Schema for additional named arguments beyond shared. Each entry's closure receives the unevaluated AST and returns canonical value or aborts.

And the define helpers now return a runtime callable:

Define helperReturned runtime function (default)
ptr_define_placeholder_value()function(x, ...) x (identity)
ptr_define_placeholder_consumer()function(x, ...) x (identity; matches ppVar's aes() NSE shape)
ptr_define_placeholder_source()function(...) rlang::abort(sprintf("%s() is only meaningful inside ptr_app()", keyword))

Authors who want a custom plain-R body pass runtime = function(...) .... Reserved name shared may not appear in named_args (hard error at registration).

Why keyword = stays explicit: R does not give a function access to the LHS of the <- it is being assigned to (the RHS is fully evaluated before <- binds). Walking sys.calls() works in simple REPL contexts but breaks inside lapply, local({...}), assign("X", ...), package-source files, and pipe contexts. R6::R6Class and setClass took the explicit-string path for the same reason. As a cheap safety net, the define helpers do a best-effort sys.calls() walk and warn (not error) when the enclosing <- LHS differs from keyword =; silent in non-assignment contexts.

Target signature (one example; others mirror it)

ptr_define_placeholder_value <- function(keyword, build_ui, resolve_expr,
                                          default_arg = NULL,
                                          named_args = list(),
                                          runtime = NULL,
                                          copy_defaults = list(
                                            label = "Enter a value for {param}"
                                          )) {
  validate_keyword(keyword)
  validate_hook(build_ui, "build_ui", c("node"))
  validate_hook(resolve_expr, "resolve_expr", c("value", "node"))
  validate_default_arg(default_arg, keyword)
  validate_named_args(named_args, keyword)
  validate_copy_defaults(copy_defaults)
  ptr_check_keyword_lhs_drift(keyword)

  runtime_fn <- runtime %||% function(x, ...) x
  validate_hook(runtime_fn, "runtime", character())

  entry <- list(
    keyword = keyword, role = "value", data_aware = FALSE,
    build_ui = build_ui, resolve_expr = resolve_expr,
    default_arg = default_arg, named_args = named_args,
    runtime = runtime_fn,
    copy_defaults = copy_defaults
  )
  ptr_registry_register(entry)
  runtime_fn
}

For _source, the default runtime is the guard:

runtime_fn <- runtime %||% function(...) rlang::abort(
  paste0("`", keyword, "()` is only meaningful inside `ptr_app()`.")
)

%||% is rlang's null-coalesce; it is already in use elsewhere in the package.

RuleHow to enforceError class

Keyword does not shadow base R / ggplot2 (per ADR 0009 §"Edge-case resolutions" — F1.a)For each package in c("base", "stats", "utils", "methods", "graphics", "grDevices", "ggplot2"), check exists(keyword, envir = asNamespace(pkg), inherits = FALSE) AND is.function(get(keyword, envir = asNamespace(pkg))). If any match, abort.rlang_error with message naming the conflicting package, e.g.: "Placeholder keyword 'sqrt' shadows base::sqrt. Pick a name that does not collide with base R or ggplot2 functions (the pp-prefix convention sidesteps this)."

Why F1.a: formulas inside ptr_app() see registered keywords as placeholder calls. A custom registration of "sqrt" would make sqrt(4) inside a formula get parsed as a placeholder call, breaking every legitimate use of sqrt. The pp-prefix convention used by the built-ins (ppVar/ppNum/etc.) avoids this automatically; the check is a tripwire for users who deviate.

F2.a: do not enforce AST-only contract on validators. The roxygen for each define-helper notes "validators are expected to operate on AST only and not call eval(); ggpaintr trusts the author. Authors who eval in a validator are opting into the risk of running user code at translate-time." No runtime enforcement.

Drift-warning helper

ptr_check_keyword_lhs_drift <- function(keyword) {
  calls <- sys.calls()
  for (i in seq.int(length(calls), 1L)) {
    call <- calls[[i]]
    if (is.call(call) && length(call) == 3L &&
        identical(call[[1L]], quote(`<-`))) {
      lhs <- call[[2L]]
      if (is.symbol(lhs) && !identical(as.character(lhs), keyword)) {
        cli::cli_warn(
          "Placeholder definition assigned to `{deparse(lhs)}` but registered under keyword \"{keyword}\". The LHS name and the keyword should match for the plain-R callable to be in scope under the same identifier."
        )
      }
      return(invisible(NULL))
    }
  }
  # No enclosing <- found (e.g. inside lapply); silent.
  invisible(NULL)
}

Success Criteria

  1. Each of ptr_define_placeholder_value, _consumer, _source accepts default_arg, named_args, runtime with documented defaults.
  2. The registry entry list stored under ptr_registry_register carries the three new fields.
  3. ptr_define_placeholder_value(keyword = "k", build_ui = ..., resolve_expr = ...) returns a function (the runtime callable), not the registry entry. The return value passes is.function().
  4. The default runtime for _value and _consumer is function(x, ...) x; called with any single argument it returns that argument unchanged.
  5. The default runtime for _source aborts with the documented message.
  6. A user-supplied runtime overrides the default exactly.
  7. Validation rejects named_args = list(shared = ...) with an rlang_error whose message names "shared is reserved".
  8. Validation rejects default_arg = "not a function" with an rlang_error.
  9. The drift-warning helper warns when an assignment ppX <- ptr_define_placeholder_value(keyword = "ppY", ...) happens at the top of a script or in a package source file. It stays silent inside lapply and other no-enclosing-<- contexts.
  10. Existing calls to the three define helpers in R/paintr-builtins.R continue to compile and register entries that pass the no-default_arg path. (The current built-ins do not need to be modified in this plan.)
  11. The authoritative gate reports 0/0/0 PASS N ≥ baseline + the new tests.
  12. devtools::document() regenerates roxygen cleanly with the new parameters documented.

Constraints

  • Backward-compatible default behavior. A define call with no default_arg, no named_args, no runtime produces an entry that behaves identically to today (the parser, once PLAN-06 lands, sees default_arg = NULL and applies today's "only shared allowed" rule).
  • No parser changes. R/paintr-translate.R is not touched.
  • No node changes. R/paintr-nodes.R is not touched (PLAN-02 handles slots).
  • No built-in re-registration. R/paintr-builtins.R is not touched in this plan. Built-ins migrate in PLAN-08.
  • Return-value change is non-breaking only if today's callers ignore the return. Verify by grepping for assignments off these helpers in the existing source (today's pattern: bare-statement call inside ptr_register_builtins, no LHS).
  • Drift warning, not error. Mismatched LHS does not break the call. The warning is informational; cli::cli_warn per project convention (not warning()).
  • roxygen. Each define helper's roxygen gets a @param entry per new parameter, a @return line updated to "the runtime callable", and the existing examples re-validated.
  • NOT_CRAN gate only.
  • Surface-not-absorb. If during implementation you discover work outside this plan's "Files touched" scope — a related bug, a refactor opportunity, an adjacent test gap, a doc inconsistency — file it as a follow-up (add a new entry under dev/plans/ or a one-line NEWS note) and STOP.
