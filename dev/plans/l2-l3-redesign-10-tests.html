Step 10 — Test suite sweep (pass #TEST)

Phase 7 (final gate). Large, separate. Authority: ADR 0005 · .claude/rules/testing.md.

Goal

Bring the whole tests/testthat/ suite onto the new API and add the net-new coverage the redesign demands. Steps 01–06 ship their own focused tests; this step is the cross-cutting consolidation + the explicit new regressions.

Affected / new test files

  • test-shared-partition.R — new (Step 01 lands it; consolidate edge cases here): 1-formula, 2-formula, key in all formulas, consumer vs value keys, empty.
  • test-ui-pieces.R — bare-piece arity (no ptr_ui_plot flags), combinator markup snapshots, removed composites/split absent, ptr_ui_page still works.
  • test-rewrite-app.R — ptr_module_ui DOM/id parity vs pre-redesign snapshot.
  • test-asset-bundle.R, test-user-css.R — drop ptr_controls_ui/ptr_outputs_ui refs; assert one .ptr-app + deduped assets via the new composition.
  • test-copy-rules.R — if ui_text now flows through ptr_shared(), cover that channel.
  • New 2-formula lockstep integration test (Step 02's scenario) at app level.

Success criteria

  • devtools::test() fully green; no skipped redesign-relevant tests.
  • Every removed symbol has a test asserting its absence from the public API; every new symbol has behavior coverage.
  • The f1=A+A+B / f2=C+C+B scenario is an automated test (A,C inline; B in one panel; B drives both in lockstep).
  • Combinator markup is snapshot-locked and equals the old ptr_ui_code_toggle DOM.
  • devtools::check(document=FALSE, manual=FALSE, args=c("--as-cran","--no-manual")) = 0 errors / 0 warnings.
  • pkgdown builds clean: source("dev/build-pkgdown.R"); build_pkgdown_clean(); _pkgdown.yml retiered L1/L2/L3, no dangling refs.

Constraints

  • testthat edition 3; withr::local_* for temp state; group by behavior not source file (project testing rules).
  • Do not assert via bare ptr_server testServer where the behavior depends on ptr_app/module code-gen (project memory: not a faithful proxy) — use an app-level harness for those.
  • Run only after Steps 01–08 land; this is the gate, not a parallel task.
  • _pkgdown.yml retiering: mirror the L1/L2/L3 tiers from CONTEXT.md; exclude CLAUDE.md from the rendered site (existing doc-build contract).
