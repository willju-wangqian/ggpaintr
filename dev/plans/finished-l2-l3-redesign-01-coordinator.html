Goal

Introduce the partition (per shared key: # of formulas referencing it; ==1 ⇒ formula-local, ≥2 ⇒ panel) as a pure function, and split ptr_shared_ui (R/paintr-shared-ui.R:173–298) into three exports: ptr_shared() (coordinator), ptr_shared_panel() (L2 self-contained UI), ptr_ui_shared_panel() (L3 bare UI). ptr_shared_server keeps its name; signature changes formulas → obj (server wiring is Step 02).

Why net-new

collect_shared_placeholders() (R/paintr-build-ui.R:330) is single-tree. No code today counts cross-formula occurrence. This step builds that algorithm.

Signatures (the contract)

ptr_shared(formulas, shared_ui = list(), ui_text = NULL,
           expr_check = TRUE, draw_all_label = "Draw all")  # -> obj, S3 class "ptr_shared_spec"
ptr_shared_panel(obj, css = NULL)     # self-wraps .ptr-app + ptr_assets()
ptr_ui_shared_panel(obj)              # bare; no shell, no assets
# obj must expose (read-only, pure): obj$panel_keys (>=2 formulas),
#   obj$local_keys_by_formula (named list), obj$widgets/config from shared_ui/ui_text.

Success criteria

  • Pure function shared_partition(trees) returns, for keys across all formulas: the set with occurrence-count ≥2 (panel_keys) and, per formula, the count-1 keys (local_keys). No reactivity, no Shiny.
  • ptr_shared() returns an S3 ptr_shared_spec carrying the partition + resolved widget builders/config. Idempotent & deterministic for a given formulas.
  • ptr_shared_panel(obj) renders only obj$panel_keys, self-wrapped in one div.ptr-app + ptr_assets(); ptr_ui_shared_panel(obj) renders the identical inner markup with no wrapper/assets.
  • ptr_shared_ui removed from NAMESPACE; the three new names exported; devtools::document() clean.
  • New tests/testthat/test-shared-partition.R green.

Constraints

  • Single-formula input ⇒ panel_keys is empty (every key is formula-local). ptr_shared() is multi-instance API; do not special-case a single-formula "promote" path (ADR 0005 rejected it).
  • Reuse existing helpers: shared_translate_formulas, shared_first_nodes, shared_consumer_representatives, canonical_shared_id. Do not change global id scheme (shared_<key>, ptr_shared_draw_all, ptr_shared_errors) — Step 02/05 depend on it.
  • obj is pure/non-reactive — no reactive(), no session. Server reactives are Step 02 only.
  • No deprecation shim for ptr_shared_ui.
  • Do not touch ptr_module_*, ptr_ui_controls, or the vignette here (later steps).
