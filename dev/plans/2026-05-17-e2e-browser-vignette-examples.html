E2E browser test of vignette examples (pass #E2E)

Created 2026-05-17. Depends on: the vignette-prose-sweep plan (examples must already be on the shipped API). Authority: ADR 0005 · .claude/rules/testing.md · project memory (testServer != ptr_app; htmlDependency resource prefix; benign shinytest2 globals warning).

Hard ordering: run only after the vignette-prose-sweep lands. An e2e test that boots an example still calling a removed symbol fails on construction and tells you nothing new. This plan assumes every runnable example already composes the shipped L1/L2/L3 API.

Goal

Knitting proves a vignette's chunks evaluate; it does not prove the generated Shiny app works in a browser (inputs render, the plot/code/error wire up, an interaction re-renders). Deliver an automated shinytest2 suite that boots every runnable vignette example in a real (headless Chromium) browser and asserts it functions. shinytest2 drives a real browser from R, so the artifact is repeatable and CI-gateable; claude-in-chrome is an authoring aid only (discover selectors / eyeball the plotly/ggiraph/bslib flows) and is never the committed test.

Scope — example inventory (enumerate precisely during impl; re-grep)

  • ggpaintr-use-cases.Rmd — canonical L1 (ptr_app), L1-grid (ptr_app_grid), L2 (ptr_module_ui+ptr_module_server), L2 coordinator (ptr_shared/ptr_shared_panel/ptr_shared_server), L3 (bare pieces + combinators), L3 custom render (moduleServer(id) + state$runtime()).
  • ggpaintr-gallery.Rmd — the worked plotly + ggiraph custom-render examples (the realistic L3 patterns users copy).
  • ggpaintr-customization.Rmd — the bslib page_sidebar wrapper (post-sweep, on ptr_module_ui), the ui_text example, and the placeholder-author examples (value/consumer/source) that build runnable apps.
  • Excluded (documented): non-app prose chunks and ggpaintr-safety.Rmd (no interactive app); any chunk that is intrinsically display-only and not an app — list each with the reason in the test file header.

Deliverables

  • tests/testthat/helper-vignette-apps.R (or tests/testthat/fixtures/vignette-apps.R) — one factory per example returning a shiny.appobj. For eval=FALSE display chunks, the app-constructing code is lifted verbatim-equivalent into a factory so the test exercises the same code the vignette shows.
  • tests/testthat/test-e2e-vignette-examples-shinytest2.R — one test_that per example: boot via shinytest2::AppDriver$new(), wait for idle, assert the documented input ids exist, perform one minimal interaction (set an input / click Update plot or the draw-all), assert the plot output rendered (an <img> / plotly / girafe node) and the generated-code output is non-empty, and assert no inline error is shown on the happy path.
  • A short header comment block enumerating every vignette example and its disposition (covered / excluded + why) — the audit trail.

Success criteria

  • Every runnable example in the inventory has a passing shinytest2 test that (a) boots without error, (b) renders the documented inputs, (c) survives one real interaction with the plot + code outputs updating, (d) shows no inline error on the happy path.
  • With shinytest2 + chromote installed the suite is fully green; without them every test skips cleanly (skip_on_cran() + skip_if_not_installed("shinytest2")) — no spurious failure.
  • The factories construct the same app code the vignette presents (incl. lifted eval=FALSE chunks) — a reviewer can diff factory ⇄ vignette and see equivalence.
  • devtools::test() remains green overall; the new file adds the only new skips (browser-gated), no new failures.
  • Excluded chunks are enumerated with a stated reason; nothing runnable is silently dropped.

Constraints

  • testthat edition 3; group by example/behavior; withr::local_* for temp state. Every test starts skip_on_cran(); skip_if_not_installed("shinytest2").
  • Wrap AppDriver$new() in suppressWarnings() (scoped) — the "Failed to locate globals" / htmlDependency-prefix warnings are benign and already documented in project memory; do not let them fail the test, but do not blanket-suppress assertion output.
  • Do not assert via a bare ptr_server testServer as a faithful proxy (project memory) — boot the real shiny.appobj the vignette builds.
  • Bound runtime: explicit AppDriver load/idle timeouts; keep the suite to a few minutes; prefer DOM/value assertions over screenshot snapshots (brittle across machines) unless a screenshot is the only signal.
  • Do not modify the vignettes (the prose-sweep plan owns them); if an example cannot be made runnable, exclude + document — do not silently rewrite the vignette here.
  • shinytest2 is already in DESCRIPTION Suggests (added Step 02); confirm chromote resolves in the dev env, otherwise the suite is all-skip and that is acceptable but must be reported, not hidden.
  • claude-in-chrome may be used while authoring to find selectors / verify the gallery/bslib flows; nothing from it is committed — the committed artifact is pure shinytest2.
