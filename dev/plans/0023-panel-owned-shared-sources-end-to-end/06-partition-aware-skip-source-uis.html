PLAN-06 — Partition-aware skip restored in ptr_setup_source_uis()

Source: ADR 0023 · Depends on: 01, 04 · Parallel group: G5 · Status: draft

Context

ADR 0023 §7 (Decision): "Cut 1 dropped the partition-blind skip entirely. With panel_sources now telling each ptr_server which ids the host owns, the skip is reintroduced in partition-aware form: if (s$id %in% names(state$panel_sources %||% list())) next. The panel-side renderUI from point 2 owns those containers; the per-instance renderUI would otherwise write to orphan output slots (harmless today but a latent collision with future namespacing changes)."

ptr_setup_source_uis() lives in R/paintr-server.R at line 1693 (per the survey). The current implementation contains a long Cut-1 comment block (lines ~1697–1709) noting the prior blanket skip was partition-blind and that "Cut 2 will reintroduce a partition-aware skip here once shared_state$panel_sources makes panel-owned ids addressable." Plan 06 IS that Cut-2 step.

Accessor note. The ADR §7 pseudocode reads state$panel_sources, but dots is a local variable inside ptr_server() (R/paintr-server.R:756) — it is not in ptr_setup_source_uis()'s scope. The reachable accessor is state$panel_sources, populated by Plan 01's ptr_init_state() extension. This plan reads state$panel_sources.

Governing principle

One widget, one owner. The shared panel emits the source-widget container; the host server is its sole binder (Plan 04). The per-instance binder must not also write to that container, because two writers to the same output id is exactly the orphan-output / multi-writer collision the partition rule exists to eliminate. The Cut-1 fix removed a blanket skip because the partition information wasn't reachable at the per-instance binder; now that Plan 01 puts panel_sources on the bundle, the skip is restored in partition-aware form: skip exactly the ids the host owns, render every other shared source on the per-instance path.

Success Criteria

  • ptr_setup_source_uis() in R/paintr-server.R contains the line if (s$id %in% names(state$panel_sources %||% list())) next at the head of the for (s in sources) loop body (after the existing if (is.null(s$id)) next guard). The line replaces the Cut-1 comment block that flagged this work as deferred.
  • For a config where state$panel_sources includes id "shared_ds" and the per-instance tree references ppUpload(shared='ds') (a panel-owned source from the instance's view): the per-instance ptr_setup_source_uis() does NOT assign output[[ns(source_output_id("shared_ds"))]]. The host server (Plan 04) owns that output.
  • For a config where the per-instance tree references a formula-local ppUpload(shared='ds_local') NOT in names(state$panel_sources): the per-instance ptr_setup_source_uis() DOES assign output[[ns(source_output_id("shared_ds_local"))]] — the Cut-1 fix for formula-local shared sources is preserved.
  • For a single-instance config (shared_state = NULL): state$panel_sources is absent / NULL; the %||% list() fallback yields an empty character vector from names(), the skip never fires, and the existing single-instance path (Cut 1) runs unchanged. Verified by every existing test in test-shared-source-rendering.R continuing to PASS.
  • All criteria verified by NOT_CRAN=true Rscript -e 'suppressMessages(devtools::load_all(".")); testthat::test_dir("tests/testthat", reporter="progress", stop_on_failure=FALSE)' → FAIL 0 / ERROR 0 / SKIP 0 / PASS rises by ≥ 2 (one positive, one negative scenario below).

Constraints

  • Files this plan owns: R/paintr-server.R (specifically ptr_setup_source_uis() — only the loop-head guard and the surrounding Cut-1 comment cleanup), tests/testthat/test-setup-source-uis-partition-aware-skip.R.
  • Files this plan must NOT touch: R/paintr-shared-ui.R (Plans 01/04), R/paintr-shared-coordinator.R (Plan 03), ptr_setup_pipelines() (Plan 05), ptr_bind_shared_consumer_uis() / ptr_bind_local_shared_consumers() (Plan 07).
  • The accessor is state$panel_sources — Plan 01 places panel_sources on state via the ptr_init_state() extension; this plan reads it directly. Do NOT invent a new field-access path; do NOT reach for dots (out of scope).
  • The %||% list() fallback is mandatory — without it, single-instance configs (where Plan 01's default panel_sources = list() may still produce an empty list rather than NULL, but defensive code is cheaper than a regression) would risk erroring on names(NULL).
  • The Cut-1 multi-line comment block is shortened to a one-line comment pointing at ADR 0023 §7 (the long explanation is now stale — the work it deferred is done).
  • SCOPE NARROWING FORBIDDEN — see the implementer/auditor prompts in /exec-plan.
