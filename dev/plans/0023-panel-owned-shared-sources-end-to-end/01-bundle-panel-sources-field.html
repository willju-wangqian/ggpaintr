PLAN-01 — Bundle gains panel_sources field

Source: ADR 0023 · Depends on: none · Parallel group: G1 · Status: draft

Context

ADR 0023 §1 (Decision): "ptr_shared_state (returned by ptr_shared_server()) carries one new field: panel_sources, a named list of reactive data.frame values keyed by canonical shared id (shared_<key>), populated for exactly the panel-owned source keys (intersect of obj$panel_keys with source-role keys). The bundle stays "signals + payloads"; shared_resolutions is already a payload, so this is the same shape."

This plan adds the new field to the value-type constructor, validator, and printer — AND surfaces it onto the per-instance state object via ptr_init_state(), because that is the channel through which ptr_setup_pipelines() / ptr_setup_source_uis() read bundle fields (the dots variable is local to ptr_server() at R/paintr-server.R:756 and flows into ptr_init_state() via do.call(ptr_init_state, c(list(formula, envir), dots)) at R/paintr-server.R:781). Downstream Plans 05/06/07 read state$panel_sources, not dots$panel_sources — the ADR's dots$ pseudocode is shorthand for "the bundle field via the existing per-instance channel". Nothing else changes in this plan.

Governing principle

The bundle channel between ptr_shared_server() and per-instance ptr_server() is the single channel for cross-server payloads (signals + resolved data). ADR-0006 forbids a separate top-level state object; ADR 0023 enforces this by growing the existing bundle rather than inventing a new lookup chain. panel_sources sits beside shared and shared_resolutions as one more payload of the same shape: a named list of reactives keyed by canonical id. Any future cross-scope payload follows the same rule.

Success Criteria

  • new_ptr_shared_state(shared = list(), draw_trigger = NULL, shared_resolutions = list(), shared_stage_enabled = list(), panel_sources = list()) constructs a ptr_shared_state whose $panel_sources slot is the supplied named list of reactives. Default value is list() (empty). Verified by testthat::expect_equal(names(x$panel_sources), c("shared_ds")) on a constructed instance.
  • validate_ptr_shared_state(x) rejects panel_sources values that are not Shiny reactives, with rlang::abort() error message "\`shared_state$panel_sources\` values must be Shiny reactives.". Verified by testthat::expect_error(validate_ptr_shared_state(bad), "must be Shiny reactives").
  • validate_ptr_shared_state(x) rejects a non-list panel_sources with "\`shared_state$panel_sources\` must be a (possibly empty) named list of reactives.".
  • validate_ptr_shared_state(x) rejects panel_sources with duplicate / empty names with "\`shared_state$panel_sources\` must have unique non-empty names.".
  • print.ptr_shared_state(x) emits one new line: " panel source keys :" followed by the canonical ids (or <none> when empty).
  • ptr_init_state() in R/paintr-server.R accepts a new keyword arg panel_sources = list(), positioned after existing args so byte-stable callers need no edits, and stores it on the returned state list as state$panel_sources. Verified by state <- ptr_init_state(formula, envir, panel_sources = list(shared_ds = r1)); expect_identical(state$panel_sources, list(shared_ds = r1)).
  • The existing do.call(ptr_init_state, c(list(formula, envir), dots)) call inside ptr_server() (at R/paintr-server.R:781) propagates the bundle's panel_sources field into state$panel_sources without code change at the call site — the new arg flows through the existing dots list. Verified by testServer: passing shared_state = list(panel_sources = list(shared_ds = r1)) to ptr_server() results in state$panel_sources$shared_ds being identical to r1.
  • An embedder constructing shared_state manually (the escape-hatch path noted in ADR 0023 worked-example R4) can pass list(panel_sources = list(shared_ds = my_df)) and have it validate cleanly when my_df is a shiny::reactiveVal().
  • All criteria verified by running NOT_CRAN=true Rscript -e 'suppressMessages(devtools::load_all(".")); testthat::test_dir("tests/testthat", reporter="progress", stop_on_failure=FALSE)' and observing FAIL 0 / ERROR 0 / SKIP 0 / PASS rises by ≥ 8 (six bundle-shape scenarios + two ptr_init_state propagation scenarios).

Constraints

  • Files this plan owns: R/paintr-shared-ui.R (constructor + validator + print method), R/paintr-server.R (only ptr_init_state()'s signature + the state-list slot for panel_sources; nothing else in that file), tests/testthat/test-ptr-shared-state-panel-sources.R (new test file).
  • Files this plan must NOT touch: R/paintr-server.R beyond the single ptr_init_state() signature + state-list edit — specifically NOT ptr_setup_pipelines() (Plan 05), NOT ptr_setup_source_uis() (Plan 06), NOT ptr_bind_shared_consumer_uis() (Plan 07); R/paintr-shared-coordinator.R (parser invariant — Plan 03); any vignette or fixture.
  • The new field is added as a keyword-only argument to new_ptr_shared_state(), defaulting to list(), positioned AFTER existing args so byte-stable callers (the existing ptr_shared_server() call) need no edits.
  • No call to ptr_shared_server() changes in this plan — it still passes no panel_sources, so the default list() applies and the bundle's observable shape is unchanged for the existing test suite.
  • SCOPE NARROWING FORBIDDEN — see the implementer/auditor prompts in /exec-plan.
  • Tidyverse style, snake_case, 2-space indent, rlang::abort() for errors (per project CLAUDE.md / .claude/rules/coding.md).

BDD scenarios

Feature: ptr_shared_state carries a panel_sources payload

Scenario: Constructor stores supplied panel_sources reactives under canonical ids
  Given a reactive r1 = shiny::reactiveVal(mtcars)
  When state <- new_ptr_shared_state(shared = list(), draw_trigger = NULL,
                                     shared_resolutions = list(),
                                     panel_sources = list(shared_ds = r1))
  Then state$panel_sources has names == c("shared_ds")
  And  state$panel_sources$shared_ds is identical to r1
  And  inherits(state, "ptr_shared_state") is TRUE

Scenario: Default constructor argument yields an empty panel_sources slot
  Given no panel_sources argument supplied
  When state <- new_ptr_shared_state(shared = list(), draw_trigger = NULL,
                                     shared_resolutions = list())
  Then state$panel_sources is identical to list()

Scenario: Validator rejects non-reactive values in panel_sources
  Given state with panel_sources = list(shared_ds = mtcars)
  When validate_ptr_shared_state(state) is called
  Then it aborts with a message containing "must be Shiny reactives"

Scenario: Validator rejects duplicate / empty names in panel_sources
  Given state with panel_sources = list(shared_ds = r1, shared_ds = r2)
  When validate_ptr_shared_state(state) is called
  Then it aborts with a message containing "unique non-empty names"

Scenario: Print method names the panel source keys
  Given state with panel_sources = list(shared_ds = r1)
  When capture.output(print(state)) is run
  Then the captured text contains "panel source keys" and "shared_ds"

Scenario: ADR worked example R4 (escape-hatch) — embedder-supplied reactive validates
  Given my_df <- shiny::reactiveVal(mtcars)
  And  state <- list(shared = list(), draw_trigger = NULL,
                     shared_resolutions = list(),
                     shared_stage_enabled = list(),
                     panel_sources = list(shared_ds = my_df))
  And  class(state) <- c("ptr_shared_state", "list")
  When validate_ptr_shared_state(state) is called
  Then it returns invisibly without error

Scenario: ptr_init_state() accepts panel_sources arg and stores it on state
  Given a parsed formula tree and an envir
  And  r1 <- shiny::reactiveVal(mtcars)
  When state <- ptr_init_state(formula, envir, panel_sources = list(shared_ds = r1))
  Then state$panel_sources is identical to list(shared_ds = r1)
  And  no other observable field of state has changed shape

Scenario: bundle panel_sources propagates to state via existing do.call channel
  Given shared_state = list(panel_sources = list(shared_ds = shiny::reactiveVal(mtcars)))
  And  shared_state is validated and class()ed as ptr_shared_state
  When testServer runs ptr_server(formula, "p1", shared_state = shared_state)
  Then state$panel_sources$shared_ds is identical to shared_state$panel_sources$shared_ds
  And  no edit was required at the do.call(ptr_init_state, c(list(formula, envir), dots)) call site
