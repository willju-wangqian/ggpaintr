PLAN-03 — Parse-time invariant: panel-owned consumer requires panel-owned source

Source: ADR 0023 · Depends on: none · Parallel group: G2 · Status: draft

Context

ADR 0023 §6 (Decision): "Inside ptr_shared() (the coordinator), after the partition is computed: for each panel-owned consumer key, every source it depends on (via the consumer's upstream pipeline) must also be panel-owned. If any source is formula-local, abort with a clear message naming the consumer key, the dependent source key, and the formulas where the asymmetry surfaces."

The ADR's worked example R1 demonstrates this: two formulas with aes(x = ppVar(shared='col')) but different bare-data ggplot(mtcars, ...) / ggplot(iris, ...) sources must abort at ptr_shared() time with a message identifying the consumer, the conflicting datasets, and the two formulas.

Governing principle

The partition is at hand wherever it needs to be read — including the parser's own scope, where it can fail fast. "One widget, one source-of-truth" is the locked invariant from CONTEXT.md: every shared id is driven by one widget in lockstep across the formulas that reference it. A single panel-scope picker cannot list columns from two different formula-local datasets without inventing arbitrary semantics (union? intersection? per-instance widgets?). The ADR rejects all such semantics in favour of an early, named, actionable abort at coordinator construction — where the embedder can act on it — rather than a silent runtime degradation where the picker quietly populates from one dataset.

Success Criteria

  • ptr_shared(c("ggplot(mtcars, aes(x = ppVar(shared='col'))) + geom_point()", "ggplot(iris, aes(x = ppVar(shared='col'))) + geom_point()")) aborts with an error whose message contains all four substrings: "col" (the consumer key), "mtcars" (formula 1 source), "iris" (formula 2 source), and a phrase identifying that one is panel-owned and the upstreams are formula-local. Verified by testthat::expect_error(ptr_shared(...), regexp = "col.*mtcars.*iris") plus a content assertion on the message.
  • The valid configurations remain valid: panel-owned consumer with panel-owned source (shared='ds' on the source), panel-owned consumer with no source (e.g. ggplot(mtcars, aes(x = ppVar(shared='col'))) alone — formula-local source is fine when the consumer is also formula-local), and mixed-scope (panel-owned source + formula-local consumer, the ADR Worked example 3) all construct without error.
  • The check fires AFTER the existing shared_partition() call inside ptr_shared() and uses obj$panel_keys / obj$consumer_reps as inputs. No new partition computation; the predicate reads what is already in scope.
  • The error is raised with rlang::abort() with class "ptr_panel_consumer_source_mismatch" so embedders can tryCatch() it specifically.
  • The error message names every offending formula by 1-indexed position ("formula 1", "formula 2") and quotes the bare-data source expression (mtcars, iris) verbatim from the AST.
  • All criteria verified by NOT_CRAN=true Rscript -e 'suppressMessages(devtools::load_all(".")); testthat::test_dir("tests/testthat", reporter="progress", stop_on_failure=FALSE)' → FAIL 0 / ERROR 0 / SKIP 0 / PASS rises by ≥ 6.

BDD scenarios

Feature: ptr_shared rejects panel-owned consumer with formula-local sources

Scenario: ADR worked example R1 — different bare-data sources behind one shared consumer
  Given formulas =
    c("ggplot(mtcars, aes(x = ppVar(shared='col'))) + geom_point()",
      "ggplot(iris,   aes(x = ppVar(shared='col'))) + geom_point()")
  When ptr_shared(formulas) is called
  Then it aborts with class "ptr_panel_consumer_source_mismatch"
  And  the message contains the substring "col" (the consumer key)
  And  the message contains the substring "mtcars" (formula 1's source)
  And  the message contains the substring "iris"   (formula 2's source)
  And  the message contains "formula 1" and "formula 2"

Scenario: Panel-owned consumer with panel-owned source passes
  Given formulas =
    c("ggplot(ppUpload(shared='ds'), aes(x = ppVar(shared='col'))) + geom_point()",
      "ggplot(ppUpload(shared='ds'), aes(x = ppVar(shared='col'))) + geom_col()")
  When ptr_shared(formulas) is called
  Then it returns a ptr_shared_spec object
  And  obj$panel_keys contains both "ds" and "col"

Scenario: Mixed scope (panel-owned source, formula-local consumer) passes
  Given formulas =
    c("ggplot(ppUpload(shared='ds'), aes(x = ppVar(shared='colA'), y = mpg)) + geom_point()",
      "ggplot(ppUpload(shared='ds'), aes(x = ppVar(shared='colB'), y = hp)) + geom_col()")
  When ptr_shared(formulas) is called
  Then it returns a ptr_shared_spec object
  And  obj$panel_keys contains "ds" only (colA, colB are formula-local)

Scenario: Single-formula spec is unaffected by the new check
  Given formulas = c("ggplot(mtcars, aes(x = ppVar(shared='col'))) + geom_point()")
  When ptr_shared(formulas) is called
  Then it aborts (existing single-formula check) — NOT with class
       "ptr_panel_consumer_source_mismatch"

Scenario: Panel-owned consumer with no source upstream (literal data, no ppUpload) passes
  when both formulas use the same bare-data symbol
  Given formulas =
    c("ggplot(mtcars, aes(x = ppVar(shared='col'))) + geom_point()",
      "ggplot(mtcars, aes(x = ppVar(shared='col'))) + geom_col()")
  When ptr_shared(formulas) is called
  Then it returns a ptr_shared_spec object
  And  no abort fires (no ppUpload source present anywhere)

Scenario: ADR worked example R2 — two bare ppUpload() with shared consumer falls to R1
  Given formulas =
    c("ggplot(ppUpload(), aes(x = ppVar(shared='col'))) + geom_point()",
      "ggplot(ppUpload(), aes(x = ppVar(shared='col'))) + geom_col()")
  When ptr_shared(formulas) is called
  Then it aborts with class "ptr_panel_consumer_source_mismatch"
  And  the message contains the substring "col"
  And  the message indicates the upstream sources are formula-local
       (each bare ppUpload widget is independent; they do not collapse into a shared widget without `shared = "..."`)
