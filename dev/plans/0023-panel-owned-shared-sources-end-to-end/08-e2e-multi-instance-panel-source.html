PLAN-08 — End-to-end multi-instance panel-shared source fixture

Source: ADR 0023 · Depends on: 01–07 · Parallel group: G7 · Status: draft

Context

Plans 01–07 build the mechanism (bundle field, resolver helper, parser invariant, host resolver, per-instance substitution, partition-aware skip, consumer-picker dep). This plan ships the regression test that proves the composition works in a real browser: the original Bug 1 reproducer from ADR 0023's Context section — two plots sharing one uploaded dataset via a panel-owned ppUpload(shared='ds') and a panel-owned ppVar(shared='col').

Per .claude/rules/testing.md and project memory shinytest2-appdir-pkgload, the fixture is an app directory whose app.R starts with pkgload::load_all(Sys.getenv("GGP_PKG"), quiet = TRUE, helpers = FALSE, attach_testthat = FALSE); the test sets withr::local_envvar(GGP_PKG = normalizePath(test_path("..", ".."))) and drives via AppDriver. The CONTEXT.md correction (line 125 promised in ADR 0023 Consequences) is folded into this plan because it documents the now-true invariant the e2e proves.

Governing principle

Mechanism that PASSes per-plan but never composes into the promised user observable IS a shipped bug. The textbook multi-instance use case — "upload one dataset, drive two plots from it" — is the entire reason ADR 0023 exists. Plans 01–07 each PASS their own gates, but none of them assert the composition in a real browser. This plan IS that assertion: render the panel UI, upload a CSV, observe both plots populate. Without it, every per-plan gate could be green and the ADR's promise could still ship broken — the canonical failure mode the worked-example coverage audit in /decision-to-plan exists to catch (ggpaintr ADR 0021 / ppVerbSwitch, 2026-05-25). The fixture is also the documentation artifact: future maintainers read it to see what "Cut 2 working" looks like.

Success Criteria

  • A new test file tests/testthat/test-shared-source-panel-multi-instance.R opens the new fixture under tests/testthat/fixtures/vignette-apps/shared-source-panel-multi/ via AppDriver. The fixture's app.R mirrors the ADR §1 worked-example body verbatim (the textbook reproducer): two formulas referencing ppUpload(shared='ds') + ppVar(shared='col'), wired with ptr_shared_panel(panel) in the UI and s <- ptr_shared_server(panel); ptr_server(panel$formulas[[i]], paste0("p", i), shared_state = s) in the server.
  • The test boots the fixture, uploads tests/testthat/fixtures/datasets/penguins.csv into the panel-owned #shared_ds fileInput, opens the layer subtabs for both plot instances, and asserts that the panel-shared #shared_col picker's choices include every column of penguins.csv (uses app$get_html("#shared_col") and parses option values, or app$get_value(input="shared_col")).
  • The test asserts both plot outputs (#p1-plot and #p2-plot) render non-empty SVG after the user picks a column and clicks Draw — verified by nchar(app$get_html("#p1-plot")) > 0 AND containing a <svg tag, same for #p2-plot.
  • A second test scenario in the same file asserts the rejected configuration from ADR Worked-example R1: constructing ptr_shared(c("ggplot(mtcars, aes(x = ppVar(shared='col'))) + geom_point()", "ggplot(iris, aes(x = ppVar(shared='col'))) + geom_point()")) aborts at ptr_shared() time with class "ptr_panel_consumer_source_mismatch" (this is the Plan 03 invariant exercised in a non-fixture context — included here as an end-to-end smoke).
  • A third scenario asserts the ADR Worked-example 4 single-instance shape (ptr_app(ppUpload(shared='ds') |> ggplot(...))) still works post-Cut-2: this is the Cut-1 regression preserved by Plan 06's partition-aware (rather than blanket) skip. Re-uses the existing single-instance fixture from test-shared-source-rendering.R if present, otherwise adds a minimal app-dir fixture.
  • CONTEXT.md line 125 (the "the owning instance binds every formula-local key" clause) is updated to the role-explicit wording the ADR Consequences promised: "the owning instance binds every formula-local key — value, consumer, and source alike." Exactly one line changes; the surrounding paragraph is otherwise untouched.
  • All criteria verified by NOT_CRAN=true Rscript -e 'suppressMessages(devtools::load_all(".")); testthat::test_dir("tests/testthat", reporter="progress", stop_on_failure=FALSE)' → FAIL 0 / ERROR 0 / SKIP 0 / PASS rises by ≥ 5.

Definition of Done

Authoritative gate: see CLAUDE.md §"Authoritative gate" — copy verbatim:

NOT_CRAN=true Rscript -e 'suppressMessages(devtools::load_all(".")); testthat::test_dir("tests/testthat", reporter="progress", stop_on_failure=FALSE)'

Expected: FAIL 0 / ERROR 0 / SKIP 0 / PASS N, with PASS rising by ≥ 5.

Plan-specific gate risks: (a) The shinytest2 child process resolves library(ggpaintr) against the stale system install unless the fixture's app.R first calls pkgload::load_all(Sys.getenv("GGP_PKG")) — without that, every assertion is testing dead code while looking green (memory shinytest2-appdir-pkgload). The auditor MUST inspect the fixture's first line. (b) devtools::check() would clean-skip this file via the source-root guard, so checking against check() is not the gate (memory shinytest2-appdir-pkgload's devtools::check sets NOT_CRAN=true so e2e RUNs+ERRORs in .Rcheck unless source-root skip guard). (c) Raw Rscript without NOT_CRAN=true SKIPs every shinytest2 test silently — the e2e file would show SKIP 3, FAIL 0, looking green. Run the authoritative gate.
