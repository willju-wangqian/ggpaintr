PLAN-07 — Consumer-picker dep reads panel-owned sources (preserves has_rendered)

Source: ADR 0023 · Depends on: 04 · Parallel group: G6 · Status: draft

Context

ADR 0023 §5 (Decision, amended): "ptr_bind_shared_consumer_uis() (and ptr_bind_local_shared_consumers() when invoked at host scope by ptr_shared_server()) gains a panel_sources arg. When the consumer's resolved upstream source id sits in names(panel_sources), the renderUI takes a dep on panel_sources[[sid]]() instead of (or in addition to) state$resolved_sources / state$resolved_data. The existing if (!is.null(state)) gate (which today excludes the host path) is replaced with a panel_sources-aware path. One resolution code path, partition-routed. Cut 2 must preserve the has_rendered closure-flag pattern introduced by commit 401804c — first-render-omits-selected and subsequent-render-passes-current %||% character(0)-verbatim are independent invariants that Cut 2's panel_sources dep wiring must not regress; the panel_sources-aware path threads through the existing has_rendered closure, not around it."

The current code is in R/paintr-server.R: ptr_bind_shared_consumer_uis at line 1987 and ptr_bind_local_shared_consumers at line 2186. The has_rendered closure-flag lives at line 2027 (declared FALSE), is consulted at line 2135 (if (has_rendered) seed %||% current %||% character(0)), and flipped to TRUE at line 2150 (after the picker is built). All three sites must survive this plan unchanged.

Governing principle

The consumer-picker reactive must wake when its upstream source resolves — regardless of which scope owns the source. Today it watches per-instance state$resolved_sources / state$resolved_data; when the source is panel-owned, that per-instance slot only mirrors the panel reactive (Plan 05), so depending on only the per-instance slot still works but routes the signal through one extra hop. Depending directly on panel_sources[[sid]] shortens the chain and makes the dep explicit at the host scope (where there is no per-instance state to read). Equally important: the has_rendered closure-flag pattern from commit 401804c is an orthogonal invariant — it preserves a user's emptied widget across renderUI re-fires by omitting selected on the first render and passing current %||% character(0) verbatim thereafter. Adding a new dep changes when the renderUI re-fires; it must not change the closure-flag logic. The ADR explicitly amends the decision to spell this out because the natural rewriting temptation is to consolidate the gate region wholesale.

Success Criteria

  • ptr_bind_shared_consumer_uis() in R/paintr-server.R gains a new keyword argument panel_sources = list(), positioned after state = NULL in the signature so existing call sites are byte-stable (they pass the default).
  • ptr_bind_local_shared_consumers() gains the same panel_sources = list() argument and threads it into its single call to ptr_bind_shared_consumer_uis().
  • ptr_shared_server() (Plan 04 already populates panel_sources) passes the populated list to ptr_bind_local_shared_consumers() when it binds consumer pickers at host scope. Existing per-instance call sites pass panel_sources = list() (the default).
  • Inside the renderUI body, for each consumer's resolved upstream source id sid: if sid %in% names(panel_sources), the renderUI calls panel_sources[[sid]]() to take a reactive dep (in addition to the existing state$resolved_sources / state$resolved_data deps, which remain because formula-local consumers under panel-owned sources still route through the per-instance mirror). For a host-scope invocation where state = NULL, the panel_sources dep is the sole signal.
  • The renderUI body's snap snapshot (for upstream_source_companion_ids and upstream_source_self_ids) reads top-level (un-namespaced) input ids for panel-owned source ids — i.e., input[[paste0("shared_", key, "_name")]], NOT input[[ns(...)]] — matching the panel's global id convention.
  • has_rendered invariant is preserved verbatim: the closure-flag declaration (today line 2027), the selected_arg <- if (has_rendered) seed %||% current %||% character(0) else seed ternary (today line 2135), and the post-picker flip to TRUE (today line 2150) all survive unchanged. The auditor checks this by diffing the three regions against the pre-Cut-2 HEAD; any change other than re-indentation MUST be flagged.
  • All criteria verified by NOT_CRAN=true Rscript -e 'suppressMessages(devtools::load_all(".")); testthat::test_dir("tests/testthat", reporter="progress", stop_on_failure=FALSE)' → FAIL 0 / ERROR 0 / SKIP 0 / PASS rises by ≥ 3. Specifically: the regression test in test-shared-source-rendering.R for the 401804c user-emptied-widget invariant (introduced post-add-expr merge) MUST continue to PASS — this is the auditor's primary signal that has_rendered survived.

Constraints

  • Files this plan owns: R/paintr-server.R (specifically ptr_bind_shared_consumer_uis() and ptr_bind_local_shared_consumers()), R/paintr-shared-ui.R (only the call site inside ptr_shared_server() where the new arg is passed), tests/testthat/test-consumer-picker-panel-sources-dep.R.
  • Files this plan must NOT touch: R/paintr-shared-coordinator.R (Plan 03), ptr_setup_pipelines() (Plan 05), ptr_setup_source_uis() (Plan 06).
  • has_rendered closure-flag region is read-only: do not refactor it, do not consolidate it with the new dep, do not move it. The new dep is added as a separate line near the top of the renderUI body (with the other for (rv in state$resolved_sources) rv() deps), not inside the selected_arg ternary.
  • The new arg is added with a non-NULL default (list()) so existing direct callers (single-instance host, per-instance modules, every test that calls these helpers directly) don't need updating.
  • SCOPE NARROWING FORBIDDEN — see the implementer/auditor prompts in /exec-plan.
  • Tidyverse style, snake_case, 2-space indent.

  Given a fresh renderUI invocation (has_rendered starts FALSE)
  And  panel_sources = list(shared_ds = reactiveVal(mtcars))
  And  no spec_seed for this consumer id
  When the renderUI body runs once
  Then the resulting picker is built WITHOUT a `selected` extra
       (selected_arg resolves to NULL on this branch)
  And  has_rendered is now TRUE

Scenario: has_rendered invariant — subsequent render preserves a user-emptied widget
  Given has_rendered = TRUE (one prior render fired)
  And  the user has cleared the picker (input slot is character(0))
  And  panel_sources reactive fires again
  When the renderUI body re-runs
  Then selected_arg = character(0) (the user-emptied value, NOT the seed default)
  And  the rendered picker has no selected items

Scenario: Host-scope invocation (state = NULL) with populated panel_sources still wires the dep
  Given state = NULL and panel_sources = list(shared_ds = rv)
  When ptr_bind_shared_consumer_uis(state = NULL, panel_sources = panel_sources, ...)
  And  rv() updates
  Then output[[consumer_output_id]] re-fires
       (the new path replaces the legacy `if (!is.null(state))` exclusion)
