PLAN-05 — Per-instance pipeline substitutes panel-owned source ids

Source: ADR 0023 · Depends on: 01, 04 · Parallel group: G4 · Status: draft

Context

ADR 0023 §4 (Decision): "In ptr_setup_pipelines() (R/paintr-server.R near line 1001), add a branch at the head of the per-source observer loop: panel_owned <- sid %in% names(state$panel_sources %||% list()); if (panel_owned) { shiny::observe({ df <- state$panel_sources[[sid]](); if (is.null(df)) { slot(NULL); return(invisible()) }; bind_source_value(state, sid, <binding_name>, df, slot) }); return(invisible()) } # else: existing per-instance path".

The branch sits at the head of the pipeline-head source loop (today at R/paintr-server.R:1004 — the for (src in find_nodes(tree, is_ptr_ph_data_source)) block). It also applies to the bare-data-source-layer loop at line ~917 in the same function. Both loops gain the same head-branch.

Accessor note. The ADR §4 pseudocode reads state$panel_sources[[sid]](), but dots is a local variable inside ptr_server() (R/paintr-server.R:756) — it is not in ptr_setup_pipelines()'s scope. The actual reachable accessor is state$panel_sources, populated by Plan 01's ptr_init_state() extension which adds panel_sources = list() to the state list (the bundle's panel_sources field flows through the existing do.call(ptr_init_state, c(list(formula, envir), dots)) at R/paintr-server.R:781). This plan reads state$panel_sources.

Governing principle

Per-instance pipelines read the partition through the bundle, not through their own input slots. "One widget, one owner" is the invariant; the per-instance side has no input slot for a panel-owned source (the canonical id shared_<key> is at top-level, not in the per-instance namespace), so attempting to watch it would yield a permanent NULL. The substitution branch redirects the per-instance binder to read from state$panel_sources[[sid]]() instead, mirroring the resolved frame into state$resolved_sources / bound_names so every downstream consumer sees one unified resolved view keyed identically — there is no second code path for downstream consumers. Substitution is local to the source-observer loop; the rest of the per-instance machinery is unaware of panel ownership.

Success Criteria

  • ptr_setup_pipelines() in R/paintr-server.R reads state$panel_sources (Plan 01 surfaces this slot via ptr_init_state()'s new panel_sources = list() arg; the bundle's panel_sources flows in unchanged via the existing do.call(ptr_init_state, c(list(formula, envir), dots)) call). For each source id sid in names(state$panel_sources %||% list()), the per-instance source-observer loop skips the input-watching observer and instead installs a shiny::observe({ df <- state$panel_sources[[sid]](); ... }) that mirrors df into state$resolved_sources[[sid]] and state$bound_names[[sid]] via bind_source_value().
  • The substitution branch applies to BOTH the bare-data-source-layer loop (today line ~917) and the pipeline-head source loop (today line ~1004): a panel-owned source can appear in either AST position. For each, the branch is at the head of the loop body (after any guard next and after the local({...}) binding, but before the shiny::observe that watches input slots).
  • The binding name passed to bind_source_value() on the panel-owned branch is derived using the same logic as the input-watching path — companion-driven sources read the companion text-input value at top-level (input[[paste0("shared_", key, "_name")]]); companion-less sources derive via entry$resolve_expr. Same name-derivation, different input-slot origin.
  • When state$panel_sources[[sid]]() returns NULL, the branch calls slot(NULL) and returns invisibly — symmetric with the existing input-NULL path.
  • For a config with no panel-owned sources (the common case today), the new branch is a no-op (names(state$panel_sources %||% list()) is empty), and every existing test that exercises ptr_setup_pipelines continues to PASS byte-for-byte.
  • For the textbook multi-instance config (two formulas, panel-shared ppUpload(shared='ds')), each per-instance state$resolved_sources$shared_ds is populated from the host's panel_sources$shared_ds reactive — verified by a testServer test that constructs the panel + two embedded servers and reads each instance's state$resolved_sources.
  • All criteria verified by NOT_CRAN=true Rscript -e 'suppressMessages(devtools::load_all(".")); testthat::test_dir("tests/testthat", reporter="progress", stop_on_failure=FALSE)' → FAIL 0 / ERROR 0 / SKIP 0 / PASS rises by ≥ 5.

Constraints

  • Files this plan owns: R/paintr-server.R (specifically ptr_setup_pipelines() — both per-source loops), tests/testthat/test-pipeline-panel-source-substitution.R.
  • Files this plan must NOT touch: R/paintr-shared-ui.R (Plans 01/04), R/paintr-shared-coordinator.R (Plan 03), ptr_setup_source_uis() (Plan 06), ptr_bind_shared_consumer_uis() / ptr_bind_local_shared_consumers() (Plan 07).
  • The branch must NOT also wire the input-watching observer for panel-owned ids — the comment in ADR 0023 §4 reads "do not also wire the input-watching observer". A panel-owned source has no per-instance input slot; wiring it would watch a permanently-NULL slot and cause spurious invalidations.
  • The branch reuses Plan 02's resolve_upload_source() only if the per-instance binding-name derivation needs the same logic — otherwise it calls bind_source_value() directly. The ADR's pseudocode shows the second shape, which is acceptable.
  • SCOPE NARROWING FORBIDDEN — see the implementer/auditor prompts in /exec-plan.
  • Tidyverse style, snake_case, 2-space indent.
