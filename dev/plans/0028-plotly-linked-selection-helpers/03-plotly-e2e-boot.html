PLAN-03 — e2e boot fixture: headline two-instance composition

Source: ADR 0028 (Decisions 4, 8 tier 2; worked example "becomes possible #1/#2" setup) · Depends on: 01, 02 · Parallel group: G3 · Status: draft

Context

ADR 0028 Decision 8 tier 2: "One e2e boot fixture (two instances + table; boots, widget renders, empty state, all-FALSE flags)." The fixture encodes the ADR's headline composition (Decision 4 / worked example "becomes possible #1"): ptr_options(gate_draw = FALSE) live mode; instance 1 ("p1") rendered through plotly::renderPlotly(ptr_ggplotly(state1)); sel <- ptr_plotly_selection(state1, mode = "flag"); instance 2 ("p2") a ggpaintr formula whose pipeline head is sel() — sel() |> ggplot(aes(x = ppVar(hp), y = ppVar(qsec), color = .ptr_selected)) + geom_point() — plus a renderTable off mode = "rows". The reactive-pipeline-head boot behavior this composition rides on is already pinned (tests/testthat/test-runtime-gate.R::"reactive pipeline head boots and seeds pickers"; live-mode redraw at ::"gate_draw=FALSE redraws on reactive pipeline-head change", mechanism R/paintr-server.R:1957-1971); this plan pins that the selection-fed composition boots end-to-end. The fixture follows the established app-dir pattern (tests/testthat/fixtures/vignette-apps/*/app.R with the pkgload::load_all(Sys.getenv("GGP_PKG"), ...) first line; cf. adr28-gate-live-reactive-head/app.R) and the runnable demo dev/scripts/plotly_linked_select.R.

Governing principle

The headline promise is compositional, with zero core changes: the second plot is an ordinary ggpaintr formula whose pipeline head happens to be the selection reactive, and it redraws per brush purely because live mode (gate_draw = FALSE) runs the render body un-isolated, making the formula's sel() read an ordinary reactive dependency. No trigger machinery, no adapter, no special-cased selection plumbing in core. The fixture must mirror that composition exactly — a fixture that hand-wires the second plot (the demo's pre-ADR shape) or adds a trigger would test something other than the ADR's promise. The pre-draw window (instance 2 shows its inline pre-draw state until instance 1's first draw, one flush under live mode) is accepted and documented, not papered over.

Success Criteria

  • A new fixture directory tests/testthat/fixtures/vignette-apps/adr28-plotly-linked/ whose app.R boots both instances under ptr_options(gate_draw = FALSE), renders instance 1 via plotly::renderPlotly(ptr_ggplotly(state1, source = "ptr_e2e")) (explicit source so PLAN-04's injected event ids are deterministic), wires ptr_plotly_selection in both modes (flag → instance 2's formula head; rows → a tableOutput), and stays diffable against the ADR's worked-example code.
  • New e2e tests in tests/testthat/test-plotly-e2e.R (gated per .claude/rules/testing.md: skip_on_cran(), skip_if_not_installed("shinytest2") + "chromote" + "plotly"; booted via boot_vignette_app("adr28-plotly-linked"), tests/testthat/helper-vignette-apps.R:20) asserting, on boot with no interaction:
    • widget renders — the instance-1 output container's HTML contains a rendered plotly htmlwidget (a js-plotly-plot/plotly class marker), via app$get_html() — never app$get_values() (forbidden, .claude/rules/testing.md);
    • no inline errors — instance 2's host settles as a drawn ggplot with an empty error output (expect_host_settled(), helper-vignette-apps.R:228), i.e. the all-FALSE flag data drew;
    • empty state — the rows-mode table output renders unconditionally with zero data rows (header-only / no data rows in its HTML): the zero-row-same-columns contract means no req() dance and no error placeholder;
    • seeding — instance 2's ppVar pickers carry the formula defaults (hp / qsec) via expect_input_eventually() (helper-vignette-apps.R:114), pinning that a selection-fed head seeds pickers like any reactive head.
  • Verified by Rscript -e 'devtools::test(filter="plotly-e2e", reporter="progress", stop_on_failure=FALSE)' → FAIL 0 / ERROR 0, tests RUN (not skipped) on the dev machine.
