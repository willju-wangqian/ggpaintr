PLAN-02 — ptr_plotly_selection(): projections + reset semantics

Source: ADR 0028 (Decisions 2, 3, 5, 6, 7; testing tier 1) · Depends on: 01 · Parallel group: G2 · Status: draft

Context

ADR 0028 Decision 2 defines the second helper: ptr_plotly_selection(state, mode = c("rows", "flag"), source = NULL) — returns a reactive. Decision 3 (CONTEXT.md, LOCKED) fixes its semantics verbatim: "A selection refers to the drawn data — state$runtime()$plot$data of the draw that produced the widget — never to 'the original object' […] Keys are minted per draw and are meaningless across draws, so the selection resets to empty on every draw. The helper returns rows, not bare indices […] Two projections of the one selection: mode = "rows" (the selected slice; zero rows, same columns, when empty) and mode = "flag" (the full drawn data + logical .ptr_selected)." Decision 5 makes .ptr_selected a documented reserved name, silently overwritten on collision (overwrite keeps chained selection-fed instances correct), and keeps .ptr_row invisible in both projections. This plan consumes PLAN-01's store (source string + per-draw snapshot on session$userData) and maintains the selection state in it across plotly_selected / plotly_deselect events and redraws, reading events the way the demo does (plotly::event_data(..., source = <source>), dev/scripts/plotly_linked_select.R).

Governing principle

A selection names rows of the drawn data of the draw that produced the widget — nothing older, nothing upstream. After a filter()/mutate() pipeline head or an upload swap there is no stable mapping back to "the original object", so keys are per-draw and the selection resets to empty on every draw; carrying it over would silently mis-attribute rows. Rows, not indices, because indices invite subsetting an object that no longer lines up. The flag column is silently overwritten on collision because chained selection-fed instances ride instance N's stale flag into instance N+1's drawn data — a warning there would spam every live draw. Per-session bookkeeping lives in session$userData, never a package-global (that would leak state across concurrent users). Do not persist selections, do not return indices, do not add a third projection.

Success Criteria

  • ptr_plotly_selection(state, mode = c("rows", "flag"), source = NULL) is a new exported function in R/paintr-plotly.R returning a Shiny reactive; roxygen with full-loop @examples (ADR Decision 7), including one sentence each on the pre-draw window and the live-mode picker-change key reset (ADR Consequences). Verified by devtools::test(filter = "plotly-selection").
  • Projection cores are pure (ADR Decision 8 locked factoring) and unit-tested without a browser in tests/testthat/test-plotly-selection.R (guarded by skip_if_not_installed("plotly") where plotly is touched): given a drawn-data snapshot and a set of selected keys, mode = "rows" returns exactly the selected slice and mode = "flag" returns the full snapshot plus logical .ptr_selected.
  • Empty state: with no selection, mode = "rows" yields a zero-row data frame with the same columns as the drawn data (so a renderTable() consumer needs no req() dance), and mode = "flag" yields the full drawn data with .ptr_selected all FALSE.
  • Neither projection exposes .ptr_row (it lives on the widget copy only, ADR Decision 5): asserted on both projections' column names.
  • Collision overwrite: a snapshot already containing a .ptr_selected column (stale flag from a chained instance) has it silently replaced — no warning — and the new flag reflects only the current selection.
  • Reset on draw: when a new per-draw snapshot lands (keys re-minted), the selection state resets to empty — the rows projection returns the zero-row slice of the NEW snapshot. A plotly_deselect event likewise clears the selection.
  • Keyless events stay empty (ADR rejected-by-design #1): an event payload carrying no key field — what brushing a non-identity-stat trace (e.g. geom_smooth()) produces — leaves the selection at its empty state (zero-row rows projection, all-FALSE flag projection).
  • Bare-reactive pipeline head is pinned as an error (ADR rejected-by-design #4): evaluating ggplot2::ggplot(sel, ...) where sel is a function/reactive (the formula head sel |> ggplot(...) — missing the parentheses) raises an error whose message contains "`data` cannot be a function" (ggplot2's literal, probe-verified 2026-06-10).
  • mode is validated via match.arg()-style behavior: an unknown mode aborts.

Interface Contract

ptr_plotly_selection(state, mode = c("rows", "flag"), source = NULL) (exported)

  • Signature: ptr_plotly_selection(state, mode = c("rows", "flag"), source = NULL) — state: a ptr_state handle (required); mode: "rows" or "flag" (match.arg semantics, default "rows"); source: character scalar or NULL (default — the same derived id as ptr_ggplotly(), via plotly_source_id() from PLAN-01). Returns a Shiny reactive expression.
  • Input domain: inside — a state whose widget is rendered by ptr_ggplotly() (same session, same derived/explicit source). Outside — mode = "indices" or any other string: abort (the bare-indices projection is rejected by design); a non-ptr_state: abort.
  • Output spec (the reactive's value): mode = "rows" → the selected slice of the current draw's snapshot; empty state = zero-row data frame with the same columns as the drawn data. mode = "flag" → the FULL drawn data plus logical .ptr_selected, TRUE exactly at selected rows (all FALSE when empty). Neither projection ever contains .ptr_row. Pre-draw (no snapshot yet): the documented pre-draw window — the reactive req()s the first draw (silent condition), so a selection-fed instance shows its inline pre-draw state for one flush under live mode.
  • Error modes: unknown mode → abort; everything event-shaped is non-erroring by contract (keyless events, deselect, redraw all land on the empty state, never an error).
  • Side effects: registers observers for the instance's plotly_selected/plotly_deselect events (via plotly::event_data(..., source)) and maintains the selection state in the PLAN-01 store on session$userData; resets the selection to empty whenever a new per-draw snapshot lands.
  • Worked example: snapshot mtcars[1:10, ], selected keys c(2, 5): mode = "rows" → the 2-row frame mtcars[c(2, 5), ] (snapshot columns, no .ptr_row); mode = "flag" → 10 rows, .ptr_selected == c(FALSE, TRUE, FALSE, FALSE, TRUE, FALSE, FALSE, FALSE, FALSE, FALSE). No selection → rows: 0-row same-columns frame; flag: all FALSE.

plotly_project_rows(snapshot, keys) / plotly_project_flag(snapshot, keys) (internal pure cores — the tier-1 unit-test seam)

  • Signature: both take snapshot (data frame — the per-draw drawn data) and keys (integer vector of selected .ptr_row keys, possibly integer(0)); each returns a data frame.
  • Input domain: inside — any data frame snapshot (including one already carrying a stale .ptr_selected column) and keys within seq_len(nrow(snapshot)); keys outside that range are dropped (a key can only be stale after a redraw, and stale keys must not select wrong rows). Outside — a non-data-frame snapshot: abort.
  • Output spec: plotly_project_rows → snapshot[keys, ] shape: nrow = length(valid keys), columns identical to snapshot's (minus any .ptr_row, which never appears); keys integer(0) → zero-row same-columns frame. plotly_project_flag → all snapshot rows + logical .ptr_selected (silently replacing a pre-existing .ptr_selected), TRUE exactly at valid keys.
  • Error modes: none for event-shaped inputs (NULL/empty/stale keys → empty-state outputs, by contract).
  • Side effects: none (pure).
  • Worked example: plotly_project_flag(transform(mtcars[1:10, ], .ptr_selected = TRUE), keys = 1L) → no warning, .ptr_selected is TRUE only at row 1 (chained-instance overwrite, ADR Decision 5).
