PLAN-02 — snapshot + UI readers honor default_active / default_stage_enabled

Source: ADR 0020 (Decision §4-§5, Verified facts 2, 3, 4) · Depends on: 01 · Parallel group: G2 · Status: draft

Context

Plan 01 stamps default_active on ptr_layer from ppLayerOff(..., hide=) and default_stage_enabled on ptr_call from ppVerbOff(..., hide=). Until reader sites consume those fields, the formula-side metadata has no effect — the snapshot still hardcodes TRUE for stage checkboxes (R/paintr-headless.R:110) and consults checkbox_defaults for layer checkboxes (R/paintr-headless.R:98, 102-106); the UI still routes through resolve_layer_default() for the layer checkbox (R/paintr-build-ui.R:144-145) and hardcodes TRUE for both stage-checkbox call sites (R/paintr-build-ui.R:230-232 per-layer; R/paintr-build-ui.R:486-488 shared); and ptr_init_state() in R/paintr-server.R:136-142 resolves resolved_cd from checkbox_defaults, stuffs it into state$checkbox_defaults (line 236), and ptr_spec_defaults_from_state() at R/paintr-server.R:329-348 reads state$checkbox_defaults to seed the diff baseline used by spec emission. (ADR 0020 enumerated only the two snapshot/UI sites; this server-side reader is a third site found during plan carving — same logical role, must learn the new field for the same correctness reason.)

This plan teaches all reader sites to honor the new node-level fields. checkbox_defaults consultation is dropped from the snapshot, from the layer-checkbox UI at the read site, and from ptr_init_state() + ptr_spec_defaults_from_state(). state$checkbox_defaults as a state field is removed (no remaining consumers). The checkbox_defaults = formal across the public API surface is not removed here (Plan 04). After this plan merges, checkbox_defaults = still appears in signatures and is silently ignored; this is the deprecation half-step.

Drift risk this plan must close: all four sites must read the same node-level field. If only some learn, snapshot ↔ UI disagree at boot — and that disagreement is exactly what ADR 0020's Risk §1 names as the highest-priority regression class.

Governing principle

One field, all readers. A node-level default field on a translated tree node is a single source of truth — it MUST be read identically by every consumer that decides the boot value of the corresponding widget. The snapshot (which seeds the initial reactive state and decides spec-emission diffs) and the UI emitter (which decides what the user first sees in the rendered checkbox) read the same field, in the same way, with the same fallback (%||% TRUE). Anything that lets a layer's snapshot value disagree with its checkbox's rendered value violates the ADR's Risk §1 invariant and produces a boot-time UI/state mismatch that no other layer in the stack can repair.

Corollary: this plan does not "fix bugs" in checkbox_defaults consultation — it removes that consultation from these reader sites entirely. checkbox_defaults's formal argument lingers (Plan 04 removes it from the public surface) but no read site this plan owns reads from it after merge. Mixing the old and new mechanisms produces split state and re-introduces exactly the side-channel inconsistency ADR 0020 was written to eliminate.

Success Criteria

  • SC1 — Snapshot honors default_active for layer_checkbox. Given a translated tree where layer "geom_point" has default_active = FALSE, ptr_default_snapshot(spec, tree)'s entry for "geom_point_checkbox" equals FALSE. Layers without a default_active field (or with NULL) default to TRUE. Verified by testthat::test_file("tests/testthat/test-pp-off-readers.R"); expected new PASS >= 2 for these.
  • SC2 — Snapshot honors default_stage_enabled for stage_enabled. Given a translated tree where the stage "ggplot_<path>_stage_enabled" carries a ptr_call with default_stage_enabled = FALSE, ptr_default_snapshot(spec, tree)'s entry for that stage_id equals FALSE. Stage ptr_calls without the field (or with NULL) default to TRUE. Verified by testthat::test_file("tests/testthat/test-pp-off-readers.R"); expected new PASS >= 2.
  • SC3 — Snapshot no longer consults checkbox_defaults for layer_checkbox. ptr_default_snapshot(spec, tree, checkbox_defaults = list(geom_point = FALSE)) returns the same value for "geom_point_checkbox" as ptr_default_snapshot(spec, tree) — i.e. the checkbox_defaults argument has no effect at the snapshot site. Verified by expect_equal() over both calls in tests/testthat/test-pp-off-readers.R.
  • SC4 — Layer-checkbox UI honors node$default_active. The UI emitted by ptr_app(ggplot() + ppLayerOff(geom_point(), TRUE)) renders the "geom_point_checkbox" input with value = FALSE. The check site is R/paintr-build-ui.R:144-145; after this plan, default_on <- isTRUE(node$default_active %||% TRUE) (or equivalent) — no checkbox_defaults consultation here. Verified by tests/testthat/test-pp-off-readers.R using build_layer_ui() directly (or any internal helper that returns the constructed shiny::checkboxInput tag list, asserting tag$attribs$value == "FALSE" on the relevant input).
  • SC5 — Per-layer Data-panel stage-checkbox UI honors default_stage_enabled. Given a formula ggplot(mtcars |> ppVerbOff(mutate(x = 1), TRUE), aes(x = mpg, y = wt)) + geom_point(), the per-layer Data-panel stage block for the ggplot layer renders its controllable_region(..., default_on = ...) with default_on = FALSE. Verified by inspecting the rendered UI structure (or by calling the entry-builder find_layer_placeholders_with_stage() directly and asserting the entry shape now carries default_stage_enabled = FALSE) in tests/testthat/test-pp-off-readers.R.
  • SC6 — Shared-stage UI honors default_stage_enabled. Given a formula ggplot(mtcars |> ppVerbOff(filter(ppVar(shared = "v") > 0), TRUE), aes(x = ppVar(shared = "v"))) + geom_point() (i.e. a shared-only orphan stage), the shared-section controllable_region() for that stage is rendered with default_on = FALSE. Verified by inspecting collect_orphan_shared_stages(tree)'s emitted record shape (now carries default_stage_enabled) and the consumer's default_on wiring in tests/testthat/test-pp-off-readers.R.
  • SC7 — End-to-end: snapshot agrees with UI value. For each of ppLayerOff(..., TRUE), ppLayerOff(..., FALSE), ppVerbOff(..., TRUE), ppVerbOff(..., FALSE), the snapshot entry, the UI's checkboxInput(..., value=) attribute, and (after a shinytest2 boot) app$get_value(input = <id>) all agree. Asserted in tests/testthat/test-pp-off-readers.R for the snapshot+UI half and (gated under skip_on_cran() + skip_if_not_installed("shinytest2")) in tests/testthat/test-pp-off-e2e.R for the post-flush half — see testing.md for the app-dir + pkgload::load_all() fixture pattern this project uses; mirror an existing browser test (e.g. tests/testthat/test-shared-stage-checkbox.R).
  • SC8 — spec = at boot still overrides the new default. Given ppLayerOff(geom_point(), TRUE) (formula says off), a spec = list(geom_point_checkbox = TRUE) entry at boot flips the layer back on (the existing apply_spec_at_boot() dispatch in R/paintr-server.R is unchanged). Verified by tests/testthat/test-pp-off-readers.R.
  • SC9 — Authoritative gate. NOT_CRAN=true Rscript -e 'suppressMessages(devtools::load_all(".")); testthat::test_dir("tests/testthat", reporter="progress", stop_on_failure=FALSE)' → FAIL 0 / ERROR 0 / SKIP 0; PASS rises by ≥ 9 over baseline (the new test-pp-off-readers.R + test-pp-off-e2e.R files; the existing test-checkbox-defaults.R may need adjustment if it relied on snapshot consulting checkbox_defaults — see Constraints).

Definition of Done

Authoritative gate (copy verbatim from CLAUDE.md's "Authoritative gate" block):

NOT_CRAN=true Rscript -e 'suppressMessages(devtools::load_all(".")); testthat::test_dir("tests/testthat", reporter="progress", stop_on_failure=FALSE)'

Expected output: FAIL 0 / ERROR 0 / SKIP 0 / PASS N (N = baseline + this plan's new tests; the SKIP count may rise by the number of test-checkbox-defaults.R / test-options.R tests this plan skips with the "superseded by ADR 0020" message — record the delta in the audit report so the post-merge integration step can verify Plan 04 removes the skipped tests rather than letting them rot).

Proxy traps — do not be fooled:

  • A raw Rscript -e 'testthat::test_dir()/test_file()' with no NOT_CRAN → skip_on_cran() fires → every shinytest2 browser test silently SKIPs. "green, SKIP n" off that is NOT the gate.
  • devtools::check(... --as-cran) does not make skip_on_cran() fire (devtools runs the check subprocess with NOT_CRAN=true). The browser fixtures instead cleanly SKIP via boot_vignette_app()'s source-root guard (no DESCRIPTION in the .Rcheck sandbox). If a check run shows the e2e file ERRORing, that guard is missing/broken — a harness defect, not a product failure, and never the gate.
  • Only NOT_CRAN=true / devtools::test() runs the browser tests (source tree present → 0 SKIP for fixture-backed tests). That alone is the gate.

This plan adds at least 9 new tests across test-pp-off-readers.R + test-pp-off-e2e.R. The gate must show PASS count rise by ≥ 9 with FAIL/ERROR still 0. SKIP may rise by the count of checkbox_defaults-related tests this plan defers to Plan 04 (record the exact delta in the audit report).
