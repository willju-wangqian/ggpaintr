PLAN-03 — structural-equality comparator excludes new fields + adr12 test switches to real comparator

Source: ADR 0020 (Structural-equality-comparator §, Risk §4, Verified facts 7-8) · Depends on: 01 · Parallel group: G2 · Status: draft

Context

ptr_tree_structural_equal() at R/paintr-nodes.R:123-150 answers "do these two trees compute the same plot?" — it deliberately excludes op (the pipe operator surface notation) and expr (the cached original R expression) from its name-set walk because those fields encode how the user wrote it, not what the plot computes. The new default_active / default_stage_enabled fields belong in the same exclusion category: both encode a request about the UI widget's boot value, not anything the plot evaluation produces.

Separately, tests/testthat/test-adr12-consumer-uniformity.R:68 defines its own inline ptr_tree_structural_equal helper. The inline helper is class-dispatched — it compares hand-picked fields per node class (ptr_layer: name / data_arg / children; etc.) — and therefore already ignores default_active / default_stage_enabled by construction. But it's also a weaker comparator than the real one (hand-picked fields vs full slot set). Switching the file to ggpaintr:::ptr_tree_structural_equal gains tighter coverage and clears a few ::: uses (aligns with ADR-0019).

This plan does exactly two things: extend the real comparator's exclusion list to c("op", "expr", "default_active", "default_stage_enabled"), and replace the inline helper in test-adr12-consumer-uniformity.R with a delegation to the real one. Nothing else.

Governing principle

ADR 0012 commits the project to a semantic tree model: surface-syntax variants and UI-state metadata are separate from the tree's plot computation. ptr_tree_structural_equal() is the canonical implementation of "do these two trees compute the same plot?", and its exclusion list is the project's authoritative catalog of slots that don't participate in that question. Every UI-state-only field added in the future (this ADR adds two) MUST be added to this exclusion list, or the comparator's contract silently weakens — equal plots will start comparing unequal whenever UI-state metadata differs.

Corollary for the test switch: a test file that re-implements the comparator inline is a fork of the project's source of truth. If the real comparator's contract evolves (as it does in this very plan), the fork drifts silently. Switching to the real comparator is not just hygiene — it pins the test against the canonical definition, so the next ADR that touches comparator contract automatically updates the test's behavior. (The drift-audit-D6 invariant for this plan: the test file MUST call the real comparator, not paraphrase its logic.)

Success Criteria

  • SC1 — Comparator exclusion list extended. The body of ptr_tree_structural_equal() at R/paintr-nodes.R:123-150 computes names_a <- setdiff(names(a), c("op", "expr", "default_active", "default_stage_enabled")) (and the symmetric names_b) at the existing setdiff lines (currently 130-131 in the pre-plan file). Verified by reading the file or by an inline regex check in tests/testthat/test-pp-off-comparator.R.
  • SC2 — Trees differing only on default_active compare equal. Construct two ptr_layer nodes identical except default_active = TRUE vs FALSE; wrap each in matching ptr_root. ggpaintr:::ptr_tree_structural_equal(a, b) returns TRUE. Verified in tests/testthat/test-pp-off-comparator.R; expected PASS >= 1.
  • SC3 — Trees differing only on default_stage_enabled compare equal. Construct two ptr_call nodes identical except default_stage_enabled = TRUE vs FALSE; wrap each in matching ptr_pipeline. ggpaintr:::ptr_tree_structural_equal(a, b) returns TRUE. Verified in tests/testthat/test-pp-off-comparator.R.
  • SC4 — Trees differing on a non-excluded field still compare unequal. Regression check: two ptr_layer nodes identical except name = "geom_point" vs "geom_line" compare unequal; two ptr_call nodes identical except fun = quote(mutate) vs quote(filter) compare unequal. Verified in tests/testthat/test-pp-off-comparator.R.
  • SC5 — End-to-end: ppLayerOff(..., FALSE) ≡ bare layer. ggpaintr:::ptr_tree_structural_equal(ptr_translate(quote(ggplot() + ppLayerOff(geom_point(), FALSE))), ptr_translate(quote(ggplot() + geom_point()))) returns TRUE. This duplicates Plan 01's SC4 and is intentional — it asserts the comparator + translate continue to agree after this plan's change.
  • SC6 — End-to-end: ppLayerOff(..., TRUE) ≡ bare layer (comparator level). ggpaintr:::ptr_tree_structural_equal(ptr_translate(quote(ggplot() + ppLayerOff(geom_point(), TRUE))), ptr_translate(quote(ggplot() + geom_point()))) returns TRUE (the trees differ on default_active, which is now excluded). This is the new capability — before this plan, the comparator would have returned FALSE because default_active was a compared slot.
  • SC7 — REMOVED (2026-05-24). Originally: "End-to-end: ppVerbOff(..., TRUE) ≡ bare stage (comparator level)". Dropped during execution because Plan 01's stamp_default_stage_enabled_ids() stamps a runtime-semantic stage_id slot on every ppVerbOff(hide=TRUE)-wrapped stage that the bare stage does not carry; stage_id is read by disable_walk at runtime so cannot be added to the comparator's exclusion list under Plan 03's own Constraints. The verb-stage structural-equality invariant will be addressed by a future ppControl placeholder ADR (see ADR 0020 "Out of scope"). SC6 already proves the principle at the layer level. SC numbering preserved (no shift on later SCs).
  • SC8 — test-adr12-consumer-uniformity.R uses the real comparator. The file no longer contains an inline ptr_tree_structural_equal <- function(a, b) {...} definition. Instead, the top of the file (or each test_that() block as appropriate) calls ggpaintr:::ptr_tree_structural_equal() directly. Verified by grep -c "^ptr_tree_structural_equal <-" tests/testthat/test-adr12-consumer-uniformity.R returning 0 and by grep -c "ggpaintr:::ptr_tree_structural_equal" tests/testthat/test-adr12-consumer-uniformity.R returning >= 1. Asserted programmatically in tests/testthat/test-pp-off-comparator.R via a readLines() + regex check.
  • SC9 — test-adr12-consumer-uniformity.R still passes against the real comparator. Every test_that() block in the file passes after the helper switch. If a test was relying on the weaker hand-picked-fields behavior (i.e. a test was passing only because the inline helper ignored some slot the real comparator compares), the test must either be updated to match the real comparator's tighter semantics or — if updating loses real coverage — flagged in the post-impl audit for a follow-up plan. Do not weaken the real comparator to keep these tests green.
  • SC10 — Authoritative gate. NOT_CRAN=true Rscript -e 'suppressMessages(devtools::load_all(".")); testthat::test_dir("tests/testthat", reporter="progress", stop_on_failure=FALSE)' → FAIL 0 / ERROR 0 / SKIP 0; PASS rises by ≥ 6 over baseline (the new test-pp-off-comparator.R); SKIP unchanged.

  • SCOPE NARROWING FORBIDDEN. If SC9 ("test-adr12 still passes against the real comparator") fails because the real comparator catches a slot the inline helper ignored: STOP and escalate. Do not "fix" by re-introducing the inline helper. Do not "fix" by adding the slot to the exclusion list unless the slot is genuinely UI-state-only and not part of the tree's plot-computation semantics (this is exactly the judgment call the ADR's "decouple comparator from runtime" principle exists to make — and a slot like active, active_input_id, stage_id, source_pipe_op would NOT qualify; only fields whose value the plot evaluation never reads do). The expected pattern is: real-comparator-tighter ⇒ the test had a bug ⇒ fix the test. If the test was correct and the real comparator's tighter check is wrong, escalate — the ADR didn't anticipate the case and a follow-up ADR is needed.
  • Conventions: snake_case for internals; 2-space indent; no new exported surface (the comparator stays internal, accessed as ggpaintr:::ptr_tree_structural_equal per ADR-0019's :::-clearing plan when it lands).

Definition of Done

Authoritative gate (copy verbatim from CLAUDE.md's "Authoritative gate" block):

NOT_CRAN=true Rscript -e 'suppressMessages(devtools::load_all(".")); testthat::test_dir("tests/testthat", reporter="progress", stop_on_failure=FALSE)'

Expected output: FAIL 0 / ERROR 0 / SKIP 0 / PASS N (N = baseline + this plan's new tests, ≥ +6).

Proxy traps — do not be fooled:

  • A raw Rscript -e 'testthat::test_dir()/test_file()' with no NOT_CRAN → skip_on_cran() fires → every shinytest2 browser test silently SKIPs. "green, SKIP n" off that is NOT the gate.
  • devtools::check(... --as-cran) does not make skip_on_cran() fire. Browser fixtures cleanly SKIP via the source-root guard. If a check run shows the e2e file ERRORing, that's a harness defect, not the gate.
  • Only NOT_CRAN=true / devtools::test() runs the browser tests. That alone is the gate.

This plan adds 1 new test file (tests/testthat/test-pp-off-comparator.R) with ≥ 6 tests. The gate must show PASS count rise by ≥ 6 with FAIL/ERROR still 0 and SKIP unchanged.
