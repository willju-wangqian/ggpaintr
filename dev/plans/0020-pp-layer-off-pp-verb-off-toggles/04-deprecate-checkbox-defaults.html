PLAN-04 — remove checkbox_defaults = formal + checkbox_default_all_other_layer option + R/paintr-checkbox-defaults.R

Source: ADR 0020 (Decision §Precedence + §Deprecation, Verified facts 5) · Depends on: 02 · Parallel group: G3 · Status: draft

Context

Plan 02 stopped every read site from consulting checkbox_defaults or the global option checkbox_default_all_other_layer — the snapshot, the layer-checkbox UI, the stage-checkbox UI (per-layer + shared), and the server-side spec-defaults reader all now read node$default_active / node$default_stage_enabled directly. The checkbox_defaults = argument is therefore vestigial: present on the signatures of the entire public API surface (ptr_app, ptr_app_components ×2, ptr_app_grid, ptr_app_grid_components, ptr_app_bslib, ptr_init_state, ptr_run_formula, ptr_default_snapshot), forwarded via ... from ptr_server, and a formal on the internal build_ui_for.ptr_layer method — but no longer read anywhere.

This plan rips out the deprecated machinery in one change, per ADR 0020's "straight remove" cadence ("ggpaintr has no external users to deprecate-warn for — there is no production deployment surface to protect"). It also deletes the now-orphan source file R/paintr-checkbox-defaults.R (the resolver, two validators, plus ptr_strip_dedup_suffix — verified to have only one caller, also inside that file, so it dies with the file), the dedicated test file tests/testthat/test-checkbox-defaults.R (287 lines, all about the deleted machinery), the checkbox_default_all_other_layer half of R/paintr-options.R, the corresponding half of tests/testthat/test-options.R, and the now-dead resolve_layer_default() helper at R/paintr-build-ui.R:531-540. Tests that Plan 02 skipped with the "superseded by ADR 0020" marker are deleted or rewritten here against the formula-side ppLayerOff equivalent. Vignettes and manual fixtures that document the deprecated path are migrated to ppLayerOff.

Governing principle

One mechanism for boot state, no side-channels. ADR 0020's deprecation rests on the conviction that four entry points for one bit of state (spec =, per-call checkbox_defaults =, global checkbox_default_all_other_layer, implicit TRUE) is too many — three of them are side-channels disconnected from the formula. ppLayerOff + ppVerbOff live inside the formula and strictly subsume checkbox_defaults = + the global option (every shape they expressed has a mechanical per-layer ppLayerOff equivalent). Leaving checkbox_defaults = as a no-op formal indefinitely re-introduces the same configuration-vs-source-of-truth confusion the ADR was written to eliminate: a future reader sees the argument in ?ptr_app, assumes it does something, and silently passes values that are ignored. The argument must come out of every signature this plan can reach.

Corollary: the removal cadence is "all at once", not "deprecate-warn, remove later". ggpaintr has no external users. There is no production surface to protect. Keeping the formal around with a .Deprecated() shim trades one form of confusion (silently ignored value) for another (a warning the user can't escape because their formula is correct). The ADR locked this decision explicitly; this plan executes it.

Definition of Done

Authoritative gate (copy verbatim from CLAUDE.md's "Authoritative gate" block):

NOT_CRAN=true Rscript -e 'suppressMessages(devtools::load_all(".")); testthat::test_dir("tests/testthat", reporter="progress", stop_on_failure=FALSE)'

Expected output: FAIL 0 / ERROR 0 / SKIP 0 / PASS N. PASS count drops by the count of deleted tests minus any rewritten replacement tests; SKIP count drops back to its pre-Plan-02 baseline (the "superseded by ADR 0020" skips disappear because the deferred-test files are deleted or rewritten). Record the exact deltas in the audit report; the post-merge integration step verifies the drop matches the expected count and is not hiding a regression.

Proxy traps — do not be fooled:

  • A raw Rscript -e 'testthat::test_dir()/test_file()' with no NOT_CRAN → skip_on_cran() fires → every shinytest2 browser test silently SKIPs. "green, SKIP n" off that is NOT the gate.
  • devtools::check(... --as-cran) does not make skip_on_cran() fire. Browser fixtures cleanly SKIP via the source-root guard. If a check run shows the e2e file ERRORing, that's a harness defect, not the gate.
  • Only NOT_CRAN=true / devtools::test() runs the browser tests. That alone is the gate.

This plan removes ~287 lines of tests (test-checkbox-defaults.R) + ~30 lines (the four removed test-options.R blocks) + several rewritten tests under test-rewrite-*. The net PASS-count delta is expected to be negative; the audit must record the exact number and confirm every "test deleted" maps to either (a) "tested only the deleted machinery" or (b) "rewritten in this plan against ppLayerOff".
