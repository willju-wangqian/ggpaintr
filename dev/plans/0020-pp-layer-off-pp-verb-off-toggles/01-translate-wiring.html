PLAN-01 — translate-side wiring for ppLayerOff / ppVerbOff

Source: ADR 0020 (Decision §1-§3, Verified facts 1, 6) · Depends on: none · Parallel group: G1 · Status: draft

Context

Today, layer-checkbox boot state can only be set via checkbox_defaults = (schema-side, by layer name) or by hand-writing a spec = entry against a namespaced input id. Stage-checkbox boot state has no schema or formula entry point at all — every stage starts TRUE. The formula is therefore not a faithful expression of the app's boot state.

ADR 0020 introduces two new pp* keywords — ppLayerOff(layer_expr, hide = TRUE) and ppVerbOff(.data, verb_expr, hide = TRUE) — that are structural unwrappers, not value placeholders. They become normal layers / stages after translate-time unwrapping, with the hide argument stamped as a node-level default field. Outside ptr_app, the function bodies behave per their R semantics so naked-ggplot scripts still render.

This plan implements the translate-side: exported function shells with naked-R semantics, placeholder_keyword() recognition, special unwrap rules in the translator, and node-field stamping. Reader sites (snapshot, UI) are out of scope — Plan 02 owns them.

Governing principle

The formula is the single source of truth for the app's boot state. ADR 0020 commits to deprecating side-channel layer-checkbox state (the schema-side checkbox_defaults = and the global checkbox_default_all_other_layer option) so that what the user sees at boot is visible in the same expression that defines the plot. Inside ggpaintr, every first-class control's boot value must be derivable from the formula alone (or from an explicit spec = override). The translator's job in this plan is to faithfully convert each hide = literal into a node-level field that downstream readers (Plan 02) consume directly — no fallback through global state, no inference from other arguments. If the implementation lets a ppLayerOff(..., hide = TRUE) in the formula produce a node whose default_active is TRUE (or unset), the formula has silently lost authority over its own boot state and the rest of the ADR's machinery cannot recover.

A second invariant from ADR 0012 applies: trees are semantic, not syntactic. ppLayerOff(layer_expr, FALSE) must produce a tree structurally indistinguishable from the bare layer_expr (modulo the default_active slot, which is the requested signal). Same for ppVerbOff(.data, verb_expr, FALSE) vs .data |> verb_expr. Anything that preserves the wrapper-call shape in the tree (e.g. translating ppLayerOff(geom_point(), FALSE) as a generic ptr_call(fun = ppLayerOff, ...)) violates the ADR.

Success Criteria

  • SC1 — Exported function shells (naked-R semantics). ppLayerOff(geom_point(), TRUE) returns NULL; ppLayerOff(geom_point(), FALSE) returns the unevaluated geom_point() expression evaluated in the caller's frame (i.e. a ggplot2 layer object). ppVerbOff(mtcars, mutate(mpg = mpg + 100), TRUE) returns mtcars unchanged; ppVerbOff(mtcars, mutate(mpg = mpg + 100), FALSE) returns mutate(mtcars, mpg = mpg + 100). Both functions default hide = TRUE. Verified by running testthat::test_file("tests/testthat/test-pp-off-naked.R"); expected FAIL 0 / ERROR 0 / SKIP 0 / PASS >= 4.
  • SC2 — Registry recognition. "ppLayerOff" %in% ptr_registry_keywords() and "ppVerbOff" %in% ptr_registry_keywords() both return TRUE. placeholder_keyword(quote(ppLayerOff(geom_point(), FALSE))) returns "ppLayerOff"; placeholder_keyword(quote(ppVerbOff(.data, mutate(x = 1), FALSE))) returns "ppVerbOff". Verified by running testthat::test_file("tests/testthat/test-pp-off-translate.R"); expected PASS >= 2 for these assertions.
  • SC3 — Layer-position unwrap with hide=TRUE. ptr_translate(quote(ggplot() + ppLayerOff(geom_point(), TRUE))) produces a ptr_root whose second layer (index 2) is a ptr_layer with name == "geom_point", active_input_id == "geom_point_checkbox", and default_active == FALSE. Verified by running testthat::test_file("tests/testthat/test-pp-off-translate.R").
  • SC4 — Layer-position unwrap with hide=FALSE produces structurally-equal tree. ggpaintr:::ptr_tree_structural_equal(ptr_translate(quote(ggplot() + ppLayerOff(geom_point(), FALSE))), ptr_translate(quote(ggplot() + geom_point()))) returns TRUE. (Plan 03 will extend the comparator to ignore default_active; in the interim — i.e. when this plan merges first — the trees ARE identical on every slot because the bare-geom_point layer also has default_active = TRUE.) Verified by running testthat::test_file("tests/testthat/test-pp-off-translate.R").
  • SC5 — Stage-position unwrap with hide=TRUE. ptr_translate(quote(ggplot(mtcars |> ppVerbOff(mutate(mpg = mpg + 100), TRUE)))) produces a tree whose ggplot layer has a data_arg that is a ptr_pipeline containing a stage ptr_call with fun == quote(mutate), stage_id set to the usual "ggplot_<path>_stage_enabled" form, and default_stage_enabled == FALSE. Verified by running testthat::test_file("tests/testthat/test-pp-off-translate.R").
  • SC6 — Stage-position unwrap with hide=FALSE produces structurally-equal tree. ggpaintr:::ptr_tree_structural_equal(ptr_translate(quote(ggplot(mtcars |> ppVerbOff(mutate(x = 1), FALSE)))), ptr_translate(quote(ggplot(mtcars |> mutate(x = 1))))) returns TRUE. Verified by running testthat::test_file("tests/testthat/test-pp-off-translate.R").
  • SC7 — Translate-time validation: non-literal hide. ptr_translate(quote(ggplot() + ppLayerOff(geom_point(), some_var))) aborts with class "ptr_translate_error" (introduced by this plan; see Constraints). Error message contains the keyword name (ppLayerOff) and the discriminator token literal. Same for non-literal hide in ppVerbOff (error message contains ppVerbOff and literal). Verified by expect_error(..., class = "ptr_translate_error", regexp = "...") in tests/testthat/test-pp-off-translate.R.
  • SC8 — Translate-time validation: wrong-position keyword. ptr_translate(quote(ggplot(aes(x = ppLayerOff(geom_point(), FALSE))))) aborts with class "ptr_translate_error"; message contains ppLayerOff and position. ptr_translate(quote(ggplot() + ppVerbOff(mtcars, mutate(x = 1), FALSE))) aborts with class "ptr_translate_error"; message contains ppVerbOff and position. Verified by expect_error(..., class = "ptr_translate_error", regexp = "...") in tests/testthat/test-pp-off-translate.R.
  • SC9 — default_stage_enabled slot exists on every ptr_call. A ptr_call constructed without a default_stage_enabled argument has the slot present and equal to TRUE (the on-by-default value). Verified by inspecting names(ptr_call(...)) in tests/testthat/test-pp-off-translate.R.
  • SC10 — Authoritative gate. NOT_CRAN=true Rscript -e 'suppressMessages(devtools::load_all(".")); testthat::test_dir("tests/testthat", reporter="progress", stop_on_failure=FALSE)' shows FAIL 0 / ERROR 0 / SKIP 0; PASS count rises by ≥ 8 over the pre-plan baseline (the new test-pp-off-naked.R + test-pp-off-translate.R files).

Constraints

  • Files this plan owns (read+write):
    • R/paintr-translate.R — extend placeholder_keyword() recognition; add special-unwrap branches in the layer-translate path and the pipeline-stage-translate path; add the translate-time validators (literal-only hide; wrong-position errors).
    • R/paintr-nodes.R — extend the ptr_call() constructor (formerly R/paintr-nodes.R at the ptr_call definition near ptr_pipeline) to add a default_stage_enabled = TRUE formal and slot. Do not touch ptr_tree_structural_equal() — that is Plan 03's territory.
    • R/paintr-registry.R — register ppLayerOff / ppVerbOff in ptr_registry_keywords() via the existing built-in registration path (look at how ppVar / ppText / ppExpr are registered; mirror that). The registry hook fields these keywords need are minimal — they never become ptr_ph_* nodes — so a registry entry sufficient to make placeholder_keyword() return their names is enough; extract_placeholder_args() never runs on them because the special-unwrap branches in translate intercept first.
    • R/paintr-builtins.R — define and @export ppLayerOff and ppVerbOff as user-callable functions with the naked-R semantics from SC1. Roxygen on both; for ppVerbOff, the roxygen MUST include a "Data-argument position" note stating the data must be in position 1 of verb_expr (matches ADR 0020 Risks §3).
    • tests/testthat/test-pp-off-naked.R — NEW. Tests SC1 (naked-R semantics, both keywords, both hide values).
    • tests/testthat/test-pp-off-translate.R — NEW. Tests SC2–SC9 (registry recognition, unwrap with both hide values for both keywords, structural-equality for the FALSE case, translate-time validation for non-literal and wrong-position).
    • NAMESPACE, man/ppLayerOff.Rd, man/ppVerbOff.Rd — regenerate via devtools::document() after editing roxygen. Generated; the integration step (not this plan) verifies cleanliness; this plan must run document() before claiming gate-green so the new exports land in NAMESPACE.
  • Files this plan must NOT touch:
    • R/paintr-headless.R (Plan 02 — snapshot readers).
    • R/paintr-build-ui.R (Plan 02 — UI plumbing + Plan 04 — resolve_layer_default removal).
    • R/paintr-nodes.R ptr_tree_structural_equal() function body (Plan 03).
    • R/paintr-checkbox-defaults.R (Plan 04 — deletion).
    • R/paintr-options.R (Plan 04 — option removal).
    • R/paintr-app.R, R/paintr-app-bslib.R, R/paintr-server.R (Plan 04 — formal removal).
    • tests/testthat/test-adr12-consumer-uniformity.R (Plan 03).
    • vignettes/ (Plan 04 + Plan 05).
  • No scope creep, no incidental refactors, no shims for "future plans". Do not pre-empt Plan 02's snapshot/UI changes even if reading them feels natural while wiring translate.
  • SCOPE NARROWING FORBIDDEN. If implementing SC4/SC6 (the structural-equality-after-FALSE-unwrap criteria) breaks because ptr_tree_structural_equal() happens to compare a field this plan adds (default_stage_enabled) and the bare side does not set it: STOP and escalate. The intended fix lives in Plan 03 (extend the comparator's exclusion list); narrowing SC4/SC6 to "the trees differ only on the new slot, that's fine" is exactly the failure mode this plan is structured to prevent. Note that the SC9 contract — every ptr_call has default_stage_enabled = TRUE by default — closes the most likely failure here, but if a non-default code path somewhere produces a ptr_call without the slot, the answer is to fix the constructor, not weaken the criterion.
  • Translate-time errors: rlang::abort() with a new class string "ptr_translate_error". There is no existing class convention in R/paintr-translate.R (its sole pre-existing rlang::abort() at line 122 carries no class= arg); this plan introduces the class for these two validators and any future translate-time validation should reuse it. Each error message MUST include the keyword name (ppLayerOff / ppVerbOff) verbatim; non-literal-hide errors MUST also include the token literal; wrong-position errors MUST also include the token position. Tests assert via expect_error(..., class = "ptr_translate_error", regexp = "...").
  • Conventions: snake_case for internals; ptr_ prefix for newly-exported helpers (the function names ppLayerOff / ppVerbOff are camelCase by design — they match the pp* family already in the registry); 2-space indent.

BDD scenarios

Feature: ppLayerOff / ppVerbOff translate as structural unwrappers with default-state metadata

Scenario: ppLayerOff at layer position with hide = TRUE produces an off-by-default layer
  Given the ADR-0020 keyword "ppLayerOff" is registered
  When  ptr_translate(quote(ggplot() + ppLayerOff(geom_point(), TRUE))) runs
  Then  the resulting ptr_root has 2 layers
  And   layer 2 is a ptr_layer with name "geom_point"
  And   layer 2 has active_input_id "geom_point_checkbox"
  And   layer 2 has default_active equal to FALSE
  And   the wrapper call ppLayerOff(...) does NOT appear anywhere in the tree (no ptr_call with fun == quote(ppLayerOff))

Scenario: ppLayerOff at layer position with hide = FALSE is structurally equal to the bare layer
  Given a translated tree T_off  = ptr_translate(quote(ggplot() + ppLayerOff(geom_point(), FALSE)))
  And   a translated tree T_bare = ptr_translate(quote(ggplot() + geom_point()))
  When  ggpaintr:::ptr_tree_structural_equal(T_off, T_bare) is called
  Then  it returns TRUE
  And   T_off's layer 2 has default_active equal to TRUE

Scenario: ppVerbOff at pipeline-stage position with hide = TRUE produces an off-by-default stage
  Given the ADR-0020 keyword "ppVerbOff" is registered
  When  ptr_translate(quote(ggplot(mtcars |> ppVerbOff(mutate(mpg = mpg + 100), TRUE)))) runs
  Then  the resulting ggplot layer has a data_arg that is a ptr_pipeline
  And   that pipeline's first downstream stage is a ptr_call with fun equal to quote(mutate)
  And   that stage's stage_id matches the regex "^ggplot_.*_stage_enabled$"
  And   that stage's default_stage_enabled equals FALSE
  And   no ptr_call with fun == quote(ppVerbOff) appears anywhere in the tree

Scenario: ppVerbOff at pipeline-stage position with hide = FALSE is structurally equal to the bare stage
  Given a translated tree T_off  = ptr_translate(quote(ggplot(mtcars |> ppVerbOff(mutate(x = 1), FALSE))))
  And   a translated tree T_bare = ptr_translate(quote(ggplot(mtcars |> mutate(x = 1))))
  When  ggpaintr:::ptr_tree_structural_equal(T_off, T_bare) is called
  Then  it returns TRUE

Scenario: ppLayerOff with non-literal hide aborts at translate time
  Given a translate input quote(ggplot() + ppLayerOff(geom_point(), some_var))
  When  ptr_translate(input) runs
  Then  it aborts with class "ptr_translate_error" (introduced by this plan)
  And   the error message regex matches "ppLayerOff" AND "literal"
  And   expect_error(..., class = "ptr_translate_error", regexp = "ppLayerOff.*literal|literal.*ppLayerOff") passes

Scenario: ppLayerOff outside layer position aborts at translate time
  Given a translate input quote(ggplot(aes(x = ppLayerOff(geom_point(), FALSE))))
  When  ptr_translate(input) runs
  Then  it aborts with class "ptr_translate_error"
  And   the error message regex matches "ppLayerOff" AND "position"
  And   expect_error(..., class = "ptr_translate_error", regexp = "ppLayerOff.*position|position.*ppLayerOff") passes

Scenario: ppVerbOff outside pipeline-stage position aborts at translate time
  Given a translate input quote(ggplot() + ppVerbOff(mtcars, mutate(x = 1), FALSE))
  When  ptr_translate(input) runs
  Then  it aborts with class "ptr_translate_error"
  And   the error message regex matches "ppVerbOff" AND "position"
  And   expect_error(..., class = "ptr_translate_error", regexp = "ppVerbOff.*position|position.*ppVerbOff") passes

Scenario: naked-R ppLayerOff with hide = TRUE returns NULL outside ptr_app
  Given no ggpaintr evaluator on the stack
  When  ppLayerOff(geom_point(), TRUE) is called
  Then  the return value is NULL

Scenario: naked-R ppLayerOff with hide = FALSE returns the layer
  Given no ggpaintr evaluator on the stack
  When  result <- ppLayerOff(geom_point(), FALSE)
  Then  the return value is a ggplot2 layer (inherits "Layer" or "LayerInstance")
  And   ggplot(mtcars, aes(x = mpg, y = wt)) + result renders without error

Scenario: naked-R ppVerbOff with hide = TRUE returns .data unchanged
  Given no ggpaintr evaluator on the stack
  When  result <- ppVerbOff(mtcars, mutate(mpg = mpg + 100), TRUE)
  Then  identical(result, mtcars) is TRUE

Scenario: naked-R ppVerbOff with hide = FALSE routes .data through the verb call
  Given no ggpaintr evaluator on the stack
  When  result <- ppVerbOff(mtcars, mutate(mpg = mpg + 100), FALSE)
  Then  result$mpg[1] equals 121
  And   nrow(result) equals nrow(mtcars)

Scenario: ptr_call has default_stage_enabled = TRUE by default
  Given no explicit default_stage_enabled argument
  When  n <- ptr_call(fun = quote(mutate), args = list(quote(x), quote(1)))
  Then  "default_stage_enabled" %in% names(n) is TRUE
  And   n$default_stage_enabled equals TRUE

Proxy traps — do not be fooled:

  • A raw Rscript -e 'testthat::test_dir()/test_file()' with no NOT_CRAN → skip_on_cran() fires → every shinytest2 browser test silently SKIPs. "green, SKIP n" off that is NOT the gate.
  • devtools::check(... --as-cran) does not make skip_on_cran() fire (devtools runs the check subprocess with NOT_CRAN=true). The browser fixtures instead cleanly SKIP via boot_vignette_app()'s source-root guard (no DESCRIPTION in the .Rcheck sandbox). If a check run shows the e2e file ERRORing (R CMD check found ERRORs, pkgload "DESCRIPTION not found"), that guard is missing/broken — a harness defect, not a product failure, and never the gate.
  • Only NOT_CRAN=true / devtools::test() runs the browser tests (source tree present → 0 SKIP). That alone is the gate. Never claim the suite green, merge a branch, mark a step done, or clear context off the skipping/erroring harness.
