PLAN — ptr_shared() accepts quoted-expression formulas

Source: grill session 2026-05-30 · ADR: none (additive, trivially reversible) · Status: draft · Branch: improve-code-base-workflow-run

Context — the feature gap

ptr_app() / ptr_server() / ptr_ui() accept a formula as either a string or an unquoted ggplot expression. They do it at the single-argument public boundary: rlang::enexpr(formula) captures the argument unevaluated, then ptr_capture_formula() (R/paintr-app.R:196) normalizes it to a single string by deparse (rlang::expr_text). Everything downstream of the boundary — ptr_translate() and the whole engine — is string-only; expression mode is purely a front-door deparse convenience. ADR 0009 (dev/adr/0009-pp-placeholders-and-expression-input.html §28) scoped expression mode to the single-formula entries only.

ptr_shared(formulas, …) (R/paintr-shared-coordinator.R:230) is the multi-formula coordinator. It is string-only: shared_translate_formulas() (R/paintr-shared-ui.R:120) asserts all(vapply(formulas, rlang::is_string, logical(1))). Passing a quoted expression element today fails with the cryptic "Elements 2 of vapply(formulas, rlang::is_string, logical(1)) are not true" (verified). That is the gap.

Design decision (grill, 2026-05-30): formulas stays a normally-evaluated argument (signature unchanged — ... / !!! rejected to avoid breaking ~150 existing call sites). Because expr()/quote() evaluate to a language object, by the time ptr_shared() receives the list each element is already either a character string or a language object — no NSE, no enexpr, no envir needed. The change is per-element normalization: deparse language elements to strings; reject anything that is neither a string nor a quoted expression.

Why not transfer the enexpr mechanism? formulas is a collection. rlang::enexpr(formulas) would capture the whole list(...)/c(...) call as one expression, and letting R evaluate list(ggplot(...) + geom()) eagerly builds ggplot objects (source text lost — deparse yields <object>, verified). The "list of strings or expr()-quoted language" contract sidesteps both: expr() defers evaluation to a recoverable language object; R's ordinary argument evaluation of the surrounding list() is harmless because each element is already a string or a language object.

Why no ADR

The skill's three-criteria test (hard to reverse · surprising without context · real trade-off) requires all three. After reverting the ... proposal, the change is additive and trivially reversible (re-tighten one assertion to undo it) — criterion 1 fails. Documented instead via ptr_shared roxygen + NEWS.md + a one-line CONTEXT.md status-header pointer.

Interface Contract (frozen)

  • Signature unchanged: ptr_shared(formulas, id = NULL, ui_text = NULL, expr_check = TRUE, draw_all_label = "Draw all"). No new arguments. No envir.
  • formulas accepts: a non-empty list or character vector. Each element is one of:
    • a single non-empty string (existing behavior, byte-identical), or
    • a language object — a call or symbol, as produced by rlang::expr(...) / quote(...).
  • Normalization: string → kept verbatim; language (is.call(x) || rlang::is_symbol(x)) → rlang::expr_text(x, width = 500L) (same width as ptr_capture_formula).
  • Rejected with a clear, index-naming error (replacing the cryptic vapply message): any element that is neither a string nor a language object — notably a built ggplot/gg object (is.call/is.language both FALSE; deparse → <object>), a numeric, a list, etc. The message must name the 1-based index and the offending class, and tell the user to quote the expression: expr(ggplot(...) + ...).
  • Return: a ptr_shared_spec identical in every field to the all-string call with the deparsed equivalents, including obj$formulas which holds the normalized character strings (not raw language objects), so shared_assert_panel_consumer_sources() and the code panel get clean source text.
  • Mixed lists allowed: list("ggplot(...)", expr(ggplot(...))) is valid.
  • Native-pipe caveat (inherited from ADR 0009 §29): a |> inside a quoted expression is desugared by R's parser before expr() captures it, so the deparsed string loses the |> surface (use %>% or string mode to preserve it). Documented, not fixed.

Success Criteria

  1. ptr_shared(formulas = list(expr(ggplot(aes(x = ppVar(mpg), colour = ppVar(g, shared = "k"))) + geom_point()), expr(ggplot(aes(x = ppVar(wt), colour = ppVar(g, shared = "k"))) + geom_line()))) returns a ptr_shared_spec whose panel_keys, local_keys_by_formula, formula_keys, and formula_count are equal to the spec built from the byte-equivalent all-string formulas.
  2. obj$formulas is a list of character strings (each rlang::is_string() TRUE) after an all-expression or mixed call.
  3. A mixed list(string, expr) call succeeds and is equal to the corresponding all-string spec.
  4. A built ggplot object element (ggplot(...) + geom_point(), not quoted) is rejected with an error that (a) names the 1-based index, (b) names the class, (c) mentions expr(.
  5. A numeric / logical / nested-list element is rejected with the same index-naming error shape.
  6. Every existing all-string call site continues to pass unchanged (no regression): full suite at FAIL 0 / ERROR 0 / SKIP 0.
  7. devtools::document() regenerates man/ptr_shared.Rd with the widened @param formulas; NAMESPACE unchanged.

Constraints

  • No signature change. Do not switch to ...; do not add envir. The whole point of the revert is zero call-site churn.
  • No new translate-time eval surface. Normalization must only deparse language objects — never eval() an element. (Symbol resolution / string-builder forcing from ptr_capture_formula is deliberately out of scope: elements arrive pre-evaluated, so there is nothing to resolve, and adding eval would open a surface ADR 0009 §56 closed.)
  • String path byte-identical. A string element must flow through unchanged; the deparse branch must never touch it.
  • Reuse existing helpers. Deparse via rlang::expr_text(x, width = 500L) (matches ptr_capture_formula); error via rlang::abort(); do not introduce a new dependency.
  • Do not edit ptr_app_grid. Out of scope (unmaintained, per user); its identical guard at R/paintr-app.R:1318-1322 stays string-only. The new helper is local to the shared coordinator, not a shared utility.
  • Do not edit fixtures/vignettes/tests to use the new form except the new dedicated tests. Existing call sites must keep working as-is (they prove non-regression). Vignettes are flagged outdated (project rule) — not gating.
  • Follow project idioms: rlang::abort() not stop(); snake_case; internal helper needs no ptr_ prefix; run devtools::document() after roxygen edits.

Implementation

1. R/paintr-shared-ui.R — add normalize_shared_formulas(), route shared_translate_formulas() through it

Add an internal helper (no ptr_ prefix) and call it at the top of shared_translate_formulas(), replacing the string-only assertion:

normalize_shared_formulas <- function(formulas) {
  if (is.character(formulas)) formulas <- as.list(formulas)
  if (!is.list(formulas) || length(formulas) < 1L) {
    rlang::abort(paste0(
      "`formulas` must be a non-empty list or character vector of formula ",
      "strings and/or quoted expressions (`expr(...)` / `quote(...)`)."
    ))
  }
  lapply(seq_along(formulas), function(i) {
    x <- formulas[[i]]
    if (rlang::is_string(x)) return(x)
    if (is.call(x) || rlang::is_symbol(x)) {
      return(rlang::expr_text(x, width = 500L))
    }
    rlang::abort(paste0(
      "`formulas[[", i, "]]` must be a single formula string or a quoted ",
      "ggplot expression (`expr(...)` / `quote(...)`); got ", class(x)[1L], ". ",
      "A built ggplot object cannot be used here -- quote the expression ",
      "instead: `expr(ggplot(...) + ...)`."
    ))
  })
}

shared_translate_formulas <- function(formulas, expr_check) {
  formulas <- normalize_shared_formulas(formulas)   # now always a list of strings
  lapply(formulas, ptr_translate, expr_check = expr_check)
}

Note the old assertthat::assert_that(...) block is gone; normalization both validates and converts.

2. R/paintr-shared-coordinator.R — store normalized strings in the spec

ptr_shared() currently calls shared_translate_formulas(formulas, …) at L244 but stores the raw formulas in the spec (L305) and passes raw formulas to shared_assert_panel_consumer_sources(formulas = formulas) (L299). Normalize once at the top so both the spec field and the assert get strings:

# near L244, before shared_translate_formulas:
formulas <- normalize_shared_formulas(formulas)
trees    <- shared_translate_formulas(formulas, expr_check = expr_check)

With formulas already normalized here, shared_translate_formulas() calling normalize_shared_formulas() again is idempotent (strings pass through) — acceptable; or pass a pre-normalized flag. Prefer the idempotent double-call for simplicity unless profiling says otherwise. formulas = formulas at L305 and the assert at L299 now see strings with no further change.

3. Roxygen — widen @param formulas on ptr_shared

At R/paintr-shared-coordinator.R:184 change to document both forms and the native-pipe caveat:

#' @param formulas A non-empty list or character vector, one element per
#'   embedded [`ptr_ui()`] instance. Each element is either a formula
#'   **string** or a **quoted ggplot expression** (`rlang::expr(...)` /
#'   `quote(...)`); quoted expressions are deparsed to their source string.
#'   A built ggplot object (e.g. `ggplot(...) + geom_point()` assigned to a
#'   variable) is **not** accepted -- quote it with `expr()`. Note: a native
#'   pipe `|>` inside a quoted expression is desugared by R before capture,
#'   so it does not survive into the generated code (use `%>%` or a string).

Add one @examples line showing the quoted form. Run devtools::document().

4. Tests — new file tests/testthat/test-shared-expression-input.R

Encode the BDD scenarios. Build FS1/FS2 strings and FE1/FE2 = rlang::expr(...) of the byte-equivalent code. Assert spec-field equality (not object identity — environments/attrs may differ), obj$formulas string-ness, mixed-list equality, and the two reject paths via expect_error(..., regexp =) on index + class + "expr(". Teeth: the all-expression acceptance test must fail on the pre-change code (it does — current guard aborts with the vapply message); the reject tests must fail if the guard were dropped entirely. Expectations are derived from the contract (deparse round-trip), not from running the new code.

5. NEWS.md + CONTEXT.md pointer

NEWS: one bullet — "ptr_shared() now accepts quoted ggplot expressions (expr()/quote()) as formulas elements, alongside strings; mixed lists allowed." CONTEXT.md: one line in the status header noting expression input extended to ptr_shared() (2026-05-30, additive, no ADR).

6. (Optional) one e2e fixture parity smoke

Lowest-priority. A construction-time unit equality test (Scenario 1) already proves parity because the deparsed string is byte-equivalent to the string form and the entire engine is downstream of the string. An e2e boot adds little. Include only if the merge-boundary reviewer wants belt-and-suspenders; otherwise the unit equality + full-suite regression is sufficient.

Blast radius

Change is at the construction/parse boundary only (no runtime/reactive/codegen change), and the string path is byte-identical, so the seeding ↔ source-binding ↔ boot-default runtime cluster is not touched. Inner-loop coverage: all non-e2e tests + the shared-coordinator unit cluster (test-shared-coordinator, test-shared-partition, test-shared-ui, the new test-shared-expression-input). The full e2e suite runs only at the merge boundary per .claude/rules/testing.md.

Baseline / drift reference (item 7)

The sole authoritative ledger is devtools::test() at FAIL 0 / ERROR 0 / SKIP 0 / PASS N (Gate 2). Before implementing, capture the known-good baseline: on the unchanged tree run Gate 2 and record N (expected ≈ 3388 pre-change). After implementing, N must equal baseline + the new file's assertion count, with FAIL/ERROR/SKIP still 0. For R CMD check (Gate 3), the baseline is whatever check already reports on the stashed tree — diff against it; pre-existing failures (per memory rcmd-check-preexisting-failures) are not a regression. Distinguish pre-existing from new by the A/B stash described in Gate 3.

Independence clause (item 6)

Any "done / tests pass / scope clean" reported by an implementer, sub-agent, worktree session, or a prior turn is a claim to verify, never proof — it is the reporter's belief from the same fallible process. The integrator does not trust a status block: they independently re-run the Gate-2 authoritative command themselves and see FAIL 0 / ERROR 0 / SKIP 0 with their own eyes before this change is considered green, merged, or context cleared.

Out-of-scope handling (item 8)

This change owns exactly: normalize_shared_formulas() + its wiring in shared_translate_formulas()/ptr_shared(), the ptr_shared roxygen, the NEWS.md bullet, the one-line CONTEXT.md pointer, and the new test-shared-expression-input.R. Any unrelated bug, drift, or smell discovered while implementing is surfaced, not absorbed: record it as a .scratch/<feature>/ follow-up issue and leave it untouched. Do not fold an adjacent fix into this commit, do not paper over a pre-existing failure, do not widen scope to ptr_app_grid.

Revertable integration (item 9)

The whole change lands as one commit (helper + guard + roxygen + regenerated man/ptr_shared.Rd + NEWS.md + CONTEXT.md line + new test). No ordering dependencies on other work. git revert <sha> undoes it cleanly as a unit. The additive, low-blast-radius nature does not warrant a sibling worktree / isolated --no-ff merge.
