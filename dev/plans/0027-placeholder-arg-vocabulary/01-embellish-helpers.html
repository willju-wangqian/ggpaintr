PLAN-01 — embellish_eval built-in helpers

Source: ADR 0027 (D4, D1) · Depends on: none · Parallel group: G1 · Status: draft

Context

ADR 0027 D4 introduces two built-in helpers for the plain-R evaluation slot: embellish_identity() (the identity function(x, ...) x, the slot's default) and embellish_symbol_to_string() (NSE-captures the call argument and returns column names as a character vector, so a column-selecting consumer works inside a tidyselect verb when the expression is run as plain R with no app). This plan ships those two functions additively — usable immediately via the current runtime = constructor argument. The slot rename (runtime → embellish_eval) and making embellish_identity() the wired default are PLAN-03's job.

Governing principle

A placeholder-embellished ggplot expression must remain valid plain R that renders the original plot with no app, and each placeholder's plain-R meaning is author-controlled, never derived — only the author knows the intended live-R semantics. embellish_symbol_to_string() exists because a function wrapper around a tidyselect selection cannot be transparent: tidyselect evaluates the unknown wrapper call in non-masked scope, so bare column symbols throw "object not found." The helper sidesteps this by returning column names as strings, which tidyselect accepts. Do NOT "simplify" it to forward bare symbols or to eagerly evaluate its argument — that reintroduces the exact failure this helper removes.

Success Criteria

  • embellish_identity() returns a function equivalent to function(x, ...) x; embellish_identity()(5L) returns 5L. Verified by devtools::test(filter="embellish-eval").
  • embellish_symbol_to_string() returns a function that, used as a consumer's runtime, makes the naked (no-app) expression mtcars |> dplyr::select(colvars(c(mpg, hp))) return a data frame whose names are exactly c("mpg", "hp") — no "object 'mpg' not found", no tidyselect type error. Verified by the same test file.
  • With the identity runtime (the current default), the same naked dplyr::select(colvars_bad(c(mpg, hp))) raises an error whose message contains "mpg" and "not found" — pinning the anti-pattern ADR 0027 negative example #2 demonstrates.
  • Both helpers are exported (appear in NAMESPACE after devtools::document()) with roxygen.

Interface Contract

embellish_identity()

  • Signature: embellish_identity() — no parameters; returns a function of signature function(x, ...).
  • Input domain: the factory takes nothing. The returned function accepts any single value x plus ignored ....
  • Output spec: the returned function returns its first argument x unchanged (identity). embellish_identity() is referentially the slot's default behavior, equivalent to function(x, ...) x.
  • Error modes: none (does not validate or coerce).
  • Side effects: none (pure).
  • Worked example: f <- embellish_identity(); f(5L) → 5L; f(c("a","b")) → c("a","b").

embellish_symbol_to_string()

  • Signature: embellish_symbol_to_string() — no parameters; returns a function of signature function(x, ...) that captures x unevaluated (NSE via rlang::enexpr(x)).
  • Input domain (inside the circle): the returned function is called with one argument expression that is (a) a call to c(...) whose elements are bare symbols and/or string literals, (b) a single bare symbol, or (c) a single string literal. Outside the circle: any other expression (e.g. starts_with("a"), an arithmetic call) falls through to an eval(expr, parent.frame()) fallback — explicitly best-effort, not contract-guaranteed.
  • Output spec: for inputs (a)–(c) it returns a character vector of the column names — bare symbols converted via rlang::as_string, string literals passed through. Order preserved. c(mpg, hp) → c("mpg", "hp"); mpg → "mpg"; c("mpg","hp") → c("mpg","hp").
  • Error modes: the NSE capture never forces the symbols, so it does not itself raise "object not found"; a malformed element inside c(...) that is neither symbol nor string reaches the eval fallback, whose error is the caller's (out of contract).
  • Side effects: none (pure; reads only the captured expression).
  • Worked example (the contract that matters): with a consumer registered runtime = embellish_symbol_to_string(), evaluating mtcars |> dplyr::select(colvars(c(mpg, hp))) as plain R returns a data frame with names c("mpg","hp") — because the returned function yields the character vector tidyselect selects by name, instead of forcing c(mpg, hp) in non-masked scope.
