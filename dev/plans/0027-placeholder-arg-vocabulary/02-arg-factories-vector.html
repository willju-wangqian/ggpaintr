PLAN-02 — ptr_arg_* vector= collapse + ptr_arg_symbol

Source: ADR 0027 (D3) · Depends on: none · Parallel group: G1 · Status: draft

Context

ADR 0027 D3: collapse the scalar/vector validator-factory pair into a single vector= flag and add a missing factory. Concretely: ptr_arg_numeric(vector = FALSE, length = NULL) absorbs the behavior of ptr_arg_numeric_vector (under vector = TRUE; length applies only then); ptr_arg_string(vector = FALSE) and ptr_arg_symbol_or_string(vector = FALSE) gain the flag; and a new ptr_arg_symbol(vector = FALSE) (bare-name → string) is added so a multi-column consumer's positional default colvars(c(mpg, hp)) has a vector-capable parser. This plan is additive: ptr_arg_numeric_vector is retained as a thin alias delegating to ptr_arg_numeric(vector = TRUE) so the gate stays green; PLAN-03 removes the alias atomically with the call-site sweep.

Governing principle

The validator factories are the single declared schema for a placeholder's positional/named default arguments; their return value is not a mere pass/fail but the parsed seed value the widget boots with. A vector= flag must therefore preserve that parse-and-return contract — vector = TRUE returns the whole vector, vector = FALSE stays scalar — and must keep folding only constant ASTs (the placeholder-argument trust boundary). Do not let vector= widen what counts as an acceptable literal beyond "a single c(...) / vector of the declared element type."

Success Criteria

  • ptr_arg_numeric()(quote(5)) returns 5 (scalar default unchanged); ptr_arg_numeric(vector = TRUE)(quote(c(1, 2))) returns c(1, 2); ptr_arg_numeric(vector = TRUE, length = 2L)(quote(c(1,2,3))) errors on length. Verified by devtools::test(filter="default-args").
  • ptr_arg_symbol()(quote(mpg)) returns "mpg"; ptr_arg_symbol(vector = TRUE)(quote(c(mpg, hp))) returns c("mpg", "hp"); a non-symbol literal is rejected.
  • ptr_arg_string(vector = TRUE)(quote(c("a","b"))) returns c("a","b"); ptr_arg_symbol_or_string(vector = TRUE)(quote(c(mpg, "hp"))) returns c("mpg","hp").
  • ptr_arg_numeric_vector() still exists and behaves identically to ptr_arg_numeric(vector = TRUE) (alias retained this round); all are exported after devtools::document().

Interface Contract

Every factory is a zero-or-few-arg constructor returning a validator of signature function(arg_expr) where arg_expr is an unevaluated language object (the formula's positional/named default). The validator returns the parsed seed value on success and rlang::abort()s on a rejected literal. All numeric/vector parsing goes through the existing constant-only ptr_constant_fold (no symbol evaluation beyond the constant-fold env).

ptr_arg_numeric(vector = FALSE, length = NULL)

  • Signature: vector logical scalar (default FALSE); length non-negative integer or NULL, honored only when vector = TRUE. Returns function(arg_expr).
  • Input domain: inside — vector=FALSE: a literal folding to a single finite non-NA numeric; vector=TRUE: a literal folding to a numeric vector of non-NA values (length == length if given). Outside — non-numeric, NA, wrong length, non-constant AST.
  • Output spec: the folded numeric scalar (scalar mode) or numeric vector (vector mode), returned verbatim.
  • Error modes: abort "Default must be a single numeric literal." (scalar); "Default must be a numeric vector of non-NA values." / "…of length N." (vector); length itself non-scalar-integer → abort at construction.
  • Side effects: none.
  • Worked example: ptr_arg_numeric()(quote(5)) → 5; ptr_arg_numeric(vector = TRUE)(quote(c(1, 2))) → c(1, 2); ptr_arg_numeric(vector = TRUE, length = 2L)(quote(c(1,2,3))) → abort (length 2 required).

ptr_arg_string(vector = FALSE) / ptr_arg_symbol_or_string(vector = FALSE) / ptr_arg_symbol(vector = FALSE)

  • Signature: vector logical scalar (default FALSE). Returns function(arg_expr).
  • Input domain (scalar mode): ptr_arg_string — a single string literal; ptr_arg_symbol — a single bare symbol; ptr_arg_symbol_or_string — either. Vector mode (vector=TRUE): a c(...) literal whose elements each satisfy the scalar element rule. Outside — any element violating the element rule (e.g. a number for ptr_arg_symbol), or a non-c() compound.
  • Output spec: scalar mode → a length-1 character; vector mode → a character vector, order preserved. Bare symbols are converted via rlang::as_string; string literals pass through.
  • Error modes: abort with a message naming the required element kind (e.g. "Default must be a bareword column name…", "Default must be a single string literal.").
  • Side effects: none.
  • Worked example: ptr_arg_symbol()(quote(mpg)) → "mpg"; ptr_arg_symbol(vector = TRUE)(quote(c(mpg, hp))) → c("mpg","hp"); ptr_arg_symbol()(quote(1 + 1)) → abort; ptr_arg_symbol_or_string(vector = TRUE)(quote(c(mpg, "hp"))) → c("mpg","hp").

ptr_arg_numeric_vector(length = NULL) (retained alias this round)

  • Signature/behavior: identical to ptr_arg_numeric(vector = TRUE, length = length); exists only so existing callers keep working until PLAN-03 migrates them and removes it.
  • Worked example: ptr_arg_numeric_vector()(quote(c(1, 2))) → c(1, 2), identical to the vector = TRUE form.
