Plan 06 revisited — Shared placeholders & UI cohesion

Working tree: core-rewrite-impl · Source-of-truth verified against R/paintr-shared.R, paintr-substitute.R, paintr-build-ui.R, paintr-app.R, paintr-server.R, paintr-utils.R, paintr-resolve.R.

Heads-up: Plan 06 was written against the pre-rewrite layout (R/paintr-placeholders.R, R/paintr-ui.R, ptr_setup_data_pipeline_observers, per-layer ptr_update_data_<layer> buttons). Those files and that button-driven model no longer exist. About half of plan 06's decisions are already realised by the rewrite; the other half — auto-render + dedup + validation — remain real bugs and are the scope of this implementation.

1. Current architecture (verified)

1.1 How shared = "..." works today

  1. Translate stage — ptr_translate() calls ptr_shared_bind() (R/paintr-shared.R:6). For every placeholder with a non-NULL shared key, its id is rewritten to the canonical form shared_<key>. So var(shared = "axis") in layer A and layer B end up sharing the same Shiny inputId.
  2. UI build — build_ui_for.ptr_layer (R/paintr-build-ui.R:96) collects every placeholder in the layer via find_layer_placeholders() with no filter on the shared field. Each call emits a widget whose inputId is the canonical id — so the same widget can be rendered N times across N tabs.
  3. Server resolve — read_placeholder_value() (R/paintr-substitute.R:155) consults ctx$shared[[node$shared]] first; if NULL it returns NULL, which flows through is_missing_value_input() and becomes ptr_missing(). The local widget's input value (ctx$snapshot[[node$id]]) is never read for shared nodes.
  4. Validation — ptr_validate_shared_bindings() (R/paintr-utils.R:54) checks the embedder-supplied list is reactive-valued with unique names. It does not cross-check that every name matches a formula annotation, nor that every formula annotation has a binding.
  5. Grid host — ptr_app_grid_components() (R/paintr-app.R:214) takes shared_ui = list(key = function(id) ...), renders the widgets in a top-level wellPanel, wires each key's input[[key]] into a reactive, and passes the reactives to ptr_module_server(..., shared = ...). Each module's resolver then finds them in ctx$shared.
  6. Single-plot host — ptr_build_app_ui() (R/paintr-app.R:74) renders only the layer picker, hidden tabset, and a single ptr_update_plot button. No shared section. ptr_app() never threads a shared = ... argument down.

1.2 The three concrete defects this implies

#DefectSource evidence
B1 Single-plot shared placeholders are inert. Widget renders, but resolver only reads ctx$shared, which is always empty when invoked through ptr_app() → ptr_missing() every time. paintr-substitute.R:155-164; paintr-app.R:25-72 (no shared= threaded).
B2 Duplicate DOM input ids across layer tabs. A shared = "x" referenced from layer A's aesthetic and layer B's pipeline yields two widgets with the same inputId = "shared_x". Shiny tolerates it but it's an HTML spec violation and a footgun for any future updateXxxInput() call. paintr-build-ui.R:201-220 (find_layer_placeholders has no $shared filter).
B3 Silent mis-binding on the grid path. Typos in either direction (formula uses shared = "n" but caller passes shared_ui = list(rows = ...), or vice versa) produce no warning. Widget renders for nothing, or formula reference silently resolves to missing. paintr-utils.R:54-74 never sees the formula tree.

5.4 Keying contract (don't get this wrong)

Three names look interchangeable but aren't. Pin them explicitly:

SymbolValue for shared = "axis"Set byRead by
node$shared"axis" (raw key) ptr_translate via extract_shared() (paintr-translate.R:370) read_placeholder_value() (paintr-substitute.R:156): looks up ctx$shared[["axis"]]
node$id"shared_axis" (canonical) ptr_shared_bind() (paintr-shared.R:32-40) build_ui_for sets inputId = ns_fn(node$id); producer-input setup binds input[[ns(node$id)]]
shared_reactives list name"axis" (raw key) New server block in ptr_app_components() ctx$shared[[node$shared]]

Concretely, the auto-bind block is:

shared_entries <- collect_shared_placeholders(tree)   # one per unique key
shared_keys    <- vapply(shared_entries, `[[`, character(1), "key")
shared_reactives <- stats::setNames(
  lapply(shared_keys, function(k) {
    canonical <- paste0("shared_", k)             # matches ptr_shared_bind
    shiny::reactive(input[[ns(canonical)]])
  }),
  shared_keys                                       # named by raw key
)
ptr_server(input, output, session, formula, ...,
              shared = shared_reactives, ns = ns)

This mirrors ptr_app_grid_components() (paintr-app.R:281-288) except the grid host uses bare-key ids (shared_ui[[nm]](nm) → inputId = nm), so its input[[nm]] read is correct for that host. Single-plot uses build_ui_for() on the node, which respects the canonical id — hence the paste0("shared_", k) step. Don't copy-paste the grid block without this adjustment.

5.5 Namespace threading

ptr_app() calls ptr_app_components(ns = shiny::NS(NULL)) (paintr-app.R:31). That makes ns the identity function; the rendered inputId is the bare string "shared_axis", and input[[ns("shared_axis")]] == input[["shared_axis"]]. No session$ns dance needed — that's only for ptr_module_server(), which already wires shared reactives via the grid host's pre-existing path. Net: the auto-bind block above is correct as-written for ptr_app(); do not introduce a separate server_ns on top of it.

5.6 Host gating for shared var (data-consumer) placeholders

ptr_ph_data_consumer nodes need ctx$upstream_cols[[node$id]] populated by ptr_setup_consumer_uis() per-layer. A shared widget rendered outside any layer panel has no consumer-ui setup, so its column choices have no source. Until plan 05 lands, gate this at the host:

  • The gate lives in ptr_server_state() (paintr-server.R:43), not in ptr_translate() — translate is host-agnostic and is shared with the grid path, which can legitimately handle a shared var if the embedder supplies a column-aware widget.
  • Condition: any(vapply(collect_shared_placeholders(tree), function(e) inherits(e$node, "ptr_ph_data_consumer"), logical(1))) AND length(shared) == 0L (no external bindings) AND the single-plot host (detect via identical(ns, shiny::NS(NULL)) OR pass an explicit auto_bind_shared = TRUE flag from ptr_app_components()).
  • Error message: "Shared var(shared = '…') placeholders are not yet supported in single-plot ptr_app(). Use ptr_app_grid() with shared_ui for now."

Confirmed in scope for C2: shared num, text, expr, upload — none of these read ctx$upstream_cols in substitute_walk.ptr_ph_value (paintr-substitute.R:74-88), so they work end-to-end with the auto-bind block above.

6. Success criteria

  • ptr_app() with one or more shared = "..." annotations renders a single widget per key in the shared section, no duplicates in layer tabs, and the widget's value drives the plot.
  • ptr_app_grid() behaviour unchanged on the happy path; typos in either direction now raise informative errors at server-state construction time.
  • Automated coverage: new test-rewrite-shared.R green; no regressions in existing rewrite tests.
  • R CMD check: 0 errors, 0 warnings.
  • Manual checklist updated with the new single-plot shared-section scenario.
