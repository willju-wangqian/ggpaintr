PLAN-02 — Super-app 1: kitchen-sink default (built-ins only)

Source: ADR 0013 §"App 1 — Kitchen sink default" + D1/D3/D4/D5/D6 (G2 row) + D7 implicitly-not-triggered · Depends on: 01 · Parallel group: G2 · Status: draft

Context

ADR 0013 §App-1 locks the slot picks: ptr_app() · literal mtcars head · G2 bare-expression capture · window code panel · default theme. The roster (per ADR): ppVar×4 → 2 widgets after shared "grp" collapse · ppNum×4 → 3 widgets after shared "lw" collapse · ppText×3 · ppExpr×2 · ppRange×1 (one custom value placeholder, registered locally in the app process). Total widgets ≈ 10.

ADR §App-1 names the unique pressure axes: (i) ppExpr-derived column flows into a downstream ppVar picker via dplyr::mutate(adj = ppExpr(mpg / wt)); (ii) shared "grp" key spans aes() and facet_wrap(vars(...)) — two different call types under one key; (iii) C6 shared-default tiebreak with ppNum(1, shared = "lw") seeded then ppNum(shared = "lw") default-less; (iv) C3+C4+C5 override channels via ui_text$defaults with {param} interpolation and per-arg ui_text$params$linewidth; (v) B3 toggle in the canonical-host default window code panel.

This plan ships the App-1 fixture (one app.R + one reference.R) and inserts one test_that() block between PLAN-01's # >>> super-1 begin and # <<< super-1 end markers. No R/paintr-*.R source change.

Governing principle

ADR 0013 commits the suite to propagation assertions: every placeholder in this app gets a literally-unique sentinel that round-trips through final-mode substitution (the sentinel appears at its expected role-specific position) and preserve-mode round-trip (the sentinel appears wrapped as ppX(<sentinel>) for value placeholders, or as a bare symbol for var placeholders, consistent with how render_placeholder_preserved (R/paintr-render.R:535) dispatches per keyword).

App 1 is the canonical-host baseline: the simplest host (ptr_app()), the simplest data head (mtcars literal), the simplest capture (G2 bare-expression). If a propagation assertion fails here, every other super-app's same-shape assertion is also at risk. App 1 therefore carries the B3 toggle test in its canonical-host form — the original regression (2c504da) lived in exactly this host shape; the propagation assertion is what would have caught it on first run. Soft "the code text is non-empty" wording is forbidden under this plan; if the orchestrator-driven implementation reveals that a particular sentinel cannot be propagated as written, STOP and escalate — narrowing the Then clause to assert something weaker is an ADR-0013 violation.

Success Criteria

  • Fixture directory exists with the two required files. tests/testthat/fixtures/vignette-apps/super-1-kitchen-sink/app.R exists. tests/testthat/fixtures/vignette-apps/super-1-kitchen-sink/reference.R exists. Both are tracked. No .csv sibling needed (no upload).
  • app.R shape. First line is the pkgload scaffolding: pkgload::load_all(Sys.getenv("GGP_PKG"), quiet = TRUE, helpers = FALSE, attach_testthat = FALSE) (verbatim from the existing fixture convention, project memory shinytest2-appdir-pkgload). The body declares the custom ppRange value placeholder via ptr_define_placeholder_value(keyword = "ppRange", ..., default_arg = ptr_default_numeric_vector(length = 2)), then calls ptr_app(<G2 bare-expression formula>, ui_text = list(defaults = list(ppNum = list(label = "{param}"), ppText = list(label = "{param} text")), params = list(linewidth = list(ppNum = list(label = "Line thickness"))))). The formula contains the roster ADR 0013 §App-1 specifies (ppVar×4 with shared "grp" collapse to 2 widgets; ppNum×4 with shared "lw" collapse to 3 widgets; ppText×3; ppExpr×2; ppRange×1), verbatim-equivalent to the ADR's code block in §App-1.
  • reference.R shape. Doc-only ggplot2 expression that is the all-defaults equivalent of the App-1 formula with no placeholders. NOT parsed by any test, NOT asserted against the substituted output. Filename is exactly reference.R; first line is a leading comment # Doc-only reference (ADR 0013 D3). Not parsed by tests..
  • One test_that() block lands between PLAN-01's # >>> super-1 begin and # <<< super-1 end markers in tests/testthat/test-super-pressure.R. No code is inserted outside that pair. grep -c "^test_that" tests/testthat/test-super-pressure.R rises from 0 to 1 after this plan.
  • Sentinel propagation — ppNum. The test sets the size = ppNum(2) input id (the one rendered for the size argument of geom_point) to the literally-unique numeric sentinel 0.7314159 via set_sentinel, clicks Update via draw_and_wait, then in final mode calls expect_sentinel_in_code(app, <code_output_id>, "0.7314159", "geom_point\\([^)]*size\\s*=\\s*([^,)]*)", "final"). In preserve mode (after toggle_code_mode(app, "preserve")), it calls expect_sentinel_in_code(app, <code_output_id>, "0.7314159", "ppNum\\(([^)]*)\\)", "preserve"). Both succeed.
  • Sentinel propagation — ppVar (root). The test sets the x = ppVar(mpg) input id to the literally-unique column choice "disp" (an mtcars column distinct from the seeded default "mpg"), draws, and asserts in final mode expect_sentinel_in_code(app, code_output_id, "disp", "aes\\([^)]*x\\s*=\\s*([^,)]*)", "final") AND in preserve mode the same regex still matches with the captured group containing "disp" as a bare symbol (ADR D4: "ppVar sentinel \"cyl\" appears as the bare symbol cyl").
  • Sentinel propagation — ppText. The test sets the title = ppText("Title") input id to the literally-unique string sentinel "S_T_2718", draws, asserts final-mode expect_sentinel_in_code(app, code_output_id, "\"S_T_2718\"", "labs\\([^)]*title\\s*=\\s*([^,)]*)", "final") AND preserve-mode expect_sentinel_in_code(app, code_output_id, "\"S_T_2718\"", "ppText\\(([^)]*)\\)", "preserve").
  • Sentinel propagation — ppExpr feeding a downstream ppVar picker. The test sets the second ppExpr (mutate(adj = ppExpr(mpg / wt))) to the expression sentinel hp + 1 (a syntactically valid AST whose result is a new numeric column equal to hp + 1), draws, and asserts in final mode that expect_sentinel_in_code(app, code_output_id, "hp + 1", "dplyr::mutate\\(adj\\s*=\\s*([^)]*)", "final") matches AND the downstream y = ppVar(adj) picker's option set still includes "adj" as a choice (asserted via expect_picker_populated(app, <y_input_id>, "adj"), reusing the helper from helper-vignette-apps.R via the package's testthat helper-load conventions).
  • Shared collapse — "grp" reaches both aes() and facet_wrap. The test sets the shared "grp" widget (input id "shared_grp", the project's canonical-shared-id form per R/paintr-app.R:1322) to the unique choice "gear" (an mtcars column not seeded as the default), draws, and asserts in final mode that "gear" appears as a bare symbol in BOTH aes\\([^)]*color\\s*=\\s*([^,)]*) AND facet_wrap\\(vars\\(([^)]*)\\) captures via two independent expect_sentinel_in_code calls. This is the call-type-agnostic check that ADR §App-1 names as pressure axis (ii).
  • C6 shared-default tiebreak. The test reads the initial value of input id "shared_lw" at app boot (before any user interaction). Asserts app$get_value(input = "shared_lw") == 1 (first-occurrence default, from ppNum(1, shared = "lw") in geom_smooth). The defaultless ppNum(shared = "lw") in geom_line is silently ignored — no error, no warning.
  • B3 toggle differential. After the propagation assertions, the test toggles code mode preserve↔final once, asserts that final-mode code does NOT contain the literal substring "ppVar(" (expect_sentinel_nowhere(app, code_output_id, "ppVar(")), and preserve-mode code DOES contain it (expect_sentinel_in_code(app, code_output_id, "ppVar(", "(ppVar)\\(", "preserve")). This is the canonical-host instance of the B3 regression net.
  • No plot error class on happy path. expect_no_plot_error(app) succeeds in both final and preserve modes.
  • Authoritative gate returns FAIL 0 / ERROR 0 / SKIP 0 / PASS (baseline + 1) after this plan lands, when the browser stack (shinytest2 + chromote + Chrome) is available. When the stack is absent, the new test_that() SKIPs cleanly (one new SKIP) — that is acceptable on a developer machine without the stack, but the orchestrator-side authoritative gate must show PASS+1, not SKIP+1.

Definition of Done

Authoritative gate (copy verbatim from CLAUDE.md):

NOT_CRAN=true Rscript -e 'suppressMessages(devtools::load_all(".")); testthat::test_dir("tests/testthat", reporter="progress", stop_on_failure=FALSE)'

Expected output: FAIL 0 / ERROR 0 / SKIP 0 / PASS (baseline + 1).

Proxy traps — do not be fooled (verbatim from CLAUDE.md):

  • A raw Rscript -e 'testthat::test_dir()/test_file()' with no NOT_CRAN → skip_on_cran() fires → every shinytest2 browser test silently SKIPs. "green, SKIP n" off that is NOT the gate.
  • devtools::check(... --as-cran) does NOT make skip_on_cran() fire (devtools runs the check subprocess with NOT_CRAN=true). The browser fixtures instead cleanly SKIP via boot_vignette_app()'s source-root guard (no DESCRIPTION in the .Rcheck sandbox). If a check run shows the e2e file ERRORing (R CMD check found ERRORs, pkgload "DESCRIPTION not found"), that guard is missing/broken — a harness defect, not a product failure, and never the gate.
  • Only NOT_CRAN=true / devtools::test() runs the browser tests (source tree present → 0 SKIP). That alone is the gate. Never claim the suite green, merge a branch, mark a step done, or clear context off the skipping/erroring harness.

This plan adds 1 new test_that() call at tests/testthat/test-super-pressure.R (in the super-1 anchor region). The gate must show PASS count rise by ≥ 1 with FAIL/ERROR still 0 and SKIP unchanged from the pre-plan baseline. When the browser stack is unavailable, SKIP may rise by 1 — that is acceptable on a developer machine but is NOT the gate; the orchestrator must verify on a stack-present machine.
