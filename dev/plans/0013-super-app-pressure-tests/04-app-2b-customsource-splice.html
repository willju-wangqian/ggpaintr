PLAN-04 — Super-app 2b: custom source + splice + layer-upload

Source: ADR 0013 §"App 2b — Custom source + splice + layer-upload" + D1/D2/D3/D4/D5/D6 (G3 row)/D7/D9/D10 · Depends on: 01 · Parallel group: G2 · Status: draft

Context

ADR 0013 §App-2b locks the slot picks: ptr_app() · custom ppSample() source (D3) at formula head · G3 !! splice with placeholders inside the spliced expression · window code panel · default theme. Three custom placeholders register locally: value ppCoef (slider 0–1), consumer ppFactor (single-select picker → bare symbol, supports shared=), and source ppSample (selectInput from a list of base R datasets, resolve_data reads the chosen dataset out of asNamespace("datasets")).

The roster: ppSample×1 (D3) · ppUpload×1 (layer-data via geom_rug(data = ppUpload(name = "df_rug"), aes(...)), NOT at formula head) · ppVar×4 · ppNum×2 · ppText×2 · ppCoef×1 (inside the splice) · ppFactor×2 → 1 after shared "fac" collapse. Total widgets ≈ 10. One sample CSV ships sibling: sample_rug.csv (mtcars subset with mpg + wt columns).

Unique pressure axes: (i) custom D3 source at head + ppUpload as layer-data — two distinct data-source mechanisms in one formula; (ii) placeholders inside the !!-spliced expression (the AST walker must descend into the spliced segment to discover them — FEATURE-CHECKLIST G3's canonical example splices a placeholder-free layer; this one stresses the harder shape); (iii) shared= on a custom consumer placeholder (ppFactor(Species, shared = "fac") in aes(color=) + facet_wrap collapse to one widget); (iv) G6 forwarded-symbol resolution (smooth_template <- rlang::expr(...) is a local binding the splice resolves at capture time).

Governing principle

ADR 0013 commits the suite to propagation assertions across the densest placeholder × host × capture interactions. App 2b's distinguishing load: the AST walker that powers placeholder discovery MUST descend into the body of a spliced expression. If !!smooth_template resolves to geom_smooth(method = ppText("lm"), linewidth = ppNum(1), alpha = ppCoef(0.3)) at capture time, the discovered placeholder set MUST include those three. A presence-style assertion that only checks "code panel non-empty" would survive a regression where the splice resolves but the walker fails to traverse it — exactly the integration shape this ADR was written to catch. The plan's BDD Then clauses name the spliced placeholders' sentinels at exact positions; weakening to "spliced layer is somewhere in the output" is forbidden.

Likewise, the two-source pressure (custom D3 source at head + ppUpload at layer-data) is the failure shape this app uniquely covers. The two data sources have disjoint column sets (iris has Species; sample_rug.csv does not); a ppVar picker bound to the layer's aes() MUST populate from the layer's data, not the root's. STOP-and-escalate if the picker shows the wrong columns; do not narrow the assertion.

Success Criteria

  • Fixture directory exists with three files. tests/testthat/fixtures/vignette-apps/super-2b-customsource-splice/app.R, reference.R, sample_rug.csv. CSV has columns mpg, wt (≥ 6 rows).
  • app.R shape. First line is the pkgload scaffolding. Body: three ptr_define_placeholder_* registrations (ppCoef value, ppFactor consumer, ppSample source). ppSample uses default_arg = ptr_default_string() and its resolve_data reads from asNamespace("datasets") with a hard whitelist of four dataset names: "iris", "mtcars", "ChickWeight", "ToothGrowth". A local binding smooth_template <- rlang::expr(geom_smooth(method = ppText("lm"), linewidth = ppNum(1), alpha = ppCoef(0.3))). Then ptr_app(<G2 bare-expression formula with !!smooth_template splice>, ui_text = list(defaults = list(ppCoef = list(label = "Coef: {param}")))). The formula matches the ADR §App-2b code block verbatim.
  • reference.R shape. Doc-only ggplot2 expression, all-defaults equivalent. First line: # Doc-only reference (ADR 0013 D3). Not parsed by tests..
  • One test_that() block lands between PLAN-01's # >>> super-2b begin and # <<< super-2b end markers.
  • ppSample (D3 custom source) drives the formula-head data. The test sets the ppSample selectInput to "mtcars" (distinct from the default "iris"), draws, and asserts: (a) the host output has no plot-error class; (b) the root ppVar pickers (for x = ppVar(Sepal.Length) and y = ppVar(Sepal.Width)) populate from mtcars's column set — verified via expect_picker_populated(app, <x picker id>, "mpg"); (c) preserve-mode code contains the literal substring ppSample(mtcars) (the bare-symbol/string-passthrough shape ppSample's resolve_data accepts).
  • Layer-data ppUpload populates its own aes pickers from sample_rug.csv's columns. The test uploads sample_rug.csv to the df_rug file input, draws, asserts: expect_picker_populated(app, <geom_rug's x picker id>, "mpg") succeeds AND expect_picker_populated(app, <same id>, "Sepal.Length") would FAIL (asserted via HTML-substring inverse: the picker's HTML does NOT contain "Sepal.Length"). The two data sources stay scoped — root pickers offer iris/mtcars columns, layer pickers offer sample_rug.csv columns.
  • Placeholders inside the splice are discovered. The test sets the ppText sentinel inside the spliced geom_smooth (the method = ppText("lm") input id) to the literally-unique value "loess", draws, asserts final-mode expect_sentinel_in_code(app, code_output_id, "\"loess\"", "geom_smooth\\([^)]*method\\s*=\\s*([^,)]*)", "final"). In preserve mode, expect_sentinel_in_code(app, code_output_id, "\"loess\"", "ppText\\(([^)]*)\\)", "preserve"). Mirroring assertions for the spliced ppNum (linewidth) and the spliced ppCoef (alpha) — each with its own literally-unique sentinel. If the AST walker fails to descend into the spliced segment, the corresponding input id never appears in the page and the set_sentinel call fails with "Unable to find input binding" — that failure mode is the auditor's evidence the regression existed.
  • Shared "fac" key collapses ppFactor in aes(color=) and facet_wrap(vars(...)). The test sets the shared widget "shared_fac" to "Species" (an iris column), draws, asserts in final mode that "Species" appears as a bare symbol in BOTH aes(... color = Species ...) AND facet_wrap(vars(Species)) captures. This is the custom-consumer-shared variant of App-1's call-type-agnostic collapse — the test is independently warranted because ADR §App-2b notes FEATURE-CHECKLIST E1/E6 cover shared= on built-ins only.
  • B3 toggle differential. Toggle preserve↔final. In final, "ppSample(", "ppFactor(", "ppText(", "ppCoef(", "ppNum(", "ppVar(", "ppUpload(" all ABSENT (expect_sentinel_nowhere per literal). In preserve, all PRESENT.
  • No plot-error class on happy path in either mode.
  • Authoritative gate returns FAIL 0 / ERROR 0 / SKIP 0 / PASS (baseline + 1).

Definition of Done

Authoritative gate (copy verbatim from CLAUDE.md):

NOT_CRAN=true Rscript -e 'suppressMessages(devtools::load_all(".")); testthat::test_dir("tests/testthat", reporter="progress", stop_on_failure=FALSE)'

Expected output: FAIL 0 / ERROR 0 / SKIP 0 / PASS (baseline + 1).

Proxy traps — do not be fooled (verbatim from CLAUDE.md):

  • A raw Rscript -e 'testthat::test_dir()/test_file()' with no NOT_CRAN → skip_on_cran() fires → every shinytest2 browser test silently SKIPs. "green, SKIP n" off that is NOT the gate.
  • devtools::check(... --as-cran) does NOT make skip_on_cran() fire (devtools runs the check subprocess with NOT_CRAN=true). The browser fixtures instead cleanly SKIP via boot_vignette_app()'s source-root guard (no DESCRIPTION in the .Rcheck sandbox). If a check run shows the e2e file ERRORing (R CMD check found ERRORs, pkgload "DESCRIPTION not found"), that guard is missing/broken — a harness defect, not a product failure, and never the gate.
  • Only NOT_CRAN=true / devtools::test() runs the browser tests (source tree present → 0 SKIP). That alone is the gate. Never claim the suite green, merge a branch, mark a step done, or clear context off the skipping/erroring harness.

This plan adds 1 new test_that() call at the super-2b anchor region. Gate must show PASS+1 with FAIL/ERROR still 0 and SKIP unchanged from pre-plan baseline.
