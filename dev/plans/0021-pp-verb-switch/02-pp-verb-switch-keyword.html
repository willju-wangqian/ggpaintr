PLAN-02 — ppVerbSwitch naked-R + keyword registration

Source: ADR 0021 · Depends on: 01 · Parallel group: G2 · Status: draft

Context

ADR 0021 introduces a new exported function ppVerbSwitch(.data, verb_expr, switch_on = TRUE, label = NULL) in R/paintr-builtins.R, registered as a built-in keyword and a structural keyword (so the placeholder-registry will not let any consumer drop it, and the translator can recognize the wrapper via is_structural_keyword_call()).

From the ADR's Decision section:

Naked-R semantics. Identical shape to ppVerbOff: .data first, verb_expr captured via base-R substitute(), first-arg injection via as.call() + eval() in parent.frame(). switch_on = FALSE ⟹ return(.data) (skip the verb); switch_on = TRUE ⟹ apply the verb to .data. label is ignored outside ggpaintr (pure UI metadata).

This plan owns only the new function + the two single-line registry edits. Translation-time logic (validators, unwrap, wiring) is plan 03. UI emission of the label is plan 05. Hard-removal of ppVerbOff is plan 06 — do not touch ppVerbOff in this plan, the two keywords coexist on this branch until plan 06 merges.

Governing principle

Formula-as-source-of-truth: every UI affordance that affects pipeline structure or visibility must be expressible inside the formula text the user wrote against ggplot — there is no parallel side-channel (no checkbox_defaults =-style argument) for boot state, no ui_text-side override for the new label. ppVerbSwitch closes the last gap in the verb-stage UI surface: today the only way to force a UI checkbox onto a bare verb is to put a placeholder inside it, and there is no way to give that checkbox a meaningful label. The new keyword exposes both knobs (boot state + label) inside the formula, and its naked-R semantics MUST be identical to ppVerbOff for the switch_on = FALSE case so the structural-equality side of ADR 0021 holds: ppVerbSwitch(verb, switch_on = FALSE) at runtime equals ppVerbOff(verb, hide = TRUE) equals the script's "skip the verb" path. An implementer who deviates from ppVerbOff's argument-splicing recipe (as.call(c(list(call_expr[[1L]]), list(data_sym), as.list(call_expr[-1L]))) + eval(new_call, envir = parent.frame()) at R/paintr-builtins.R:486-496) breaks the "naked-R script runs both bare and inside ptr_app" contract that ADR 0020 and ADR 0009 rest on. Mirror, do not improvise.

Success Criteria

  • SC-1. ppVerbSwitch(mtcars, dplyr::filter(mpg > 20), switch_on = TRUE) returns a data frame with 14 rows (mtcars rows with mpg > 20), identical to dplyr::filter(mtcars, mpg > 20). Verified by an expect_equal(nrow(...), 14L) assertion in tests/testthat/test-pp-verb-switch-naked.R.
  • SC-2. ppVerbSwitch(mtcars, dplyr::filter(mpg > 20), switch_on = FALSE) returns mtcars unchanged. Verified by expect_identical(ppVerbSwitch(mtcars, dplyr::filter(mpg > 20), FALSE), mtcars) in the same file.
  • SC-3. ppVerbSwitch(mtcars, dplyr::mutate(mpg = mpg + 100), TRUE)$mpg[[1]] equals mtcars$mpg[[1]] + 100. Verified by an arithmetic equality assertion in the same file.
  • SC-4. switch_on argument validation: ppVerbSwitch(mtcars, dplyr::filter(mpg > 20), switch_on = NA) errors with the message `switch_on` must be a length-1 non-NA logical. (verbatim, including backticks); the same error fires for switch_on = c(TRUE, FALSE) and for a non-logical value like switch_on = "yes". Verified by three expect_error assertions in the same file.
  • SC-5. verb_expr-not-a-call validation: ppVerbSwitch(mtcars, mtcars, FALSE) errors with the message `ppVerbSwitch(verb_expr = )` must be a verb call, e.g. `mutate(x = 1)`. (the message text mirrors the existing ppVerbOff error verbatim except for the keyword name). Verified by an expect_error in the same file.
  • SC-6. label is accepted as NULL (default) or a length-1 character (e.g. "Transform x") without affecting runtime behaviour. ppVerbSwitch(mtcars, dplyr::filter(mpg > 20), TRUE, label = "X") returns the same data frame as ppVerbSwitch(mtcars, dplyr::filter(mpg > 20), TRUE). Verified by an expect_identical(...) in the same file. (Argument validation for label outside ptr_app is intentionally permissive — naked-R is forgiving; translate-time validation is plan 03's territory.)
  • SC-7. "ppVerbSwitch" %in% ptr_builtin_keywords() is TRUE. Verified by an assertion in tests/testthat/test-pp-verb-switch-keyword.R.
  • SC-8. "ppVerbSwitch" %in% ptr_registry_keywords() is TRUE after a fresh devtools::load_all() (the boot path runs ptr_register_structural_keyword("ppVerbSwitch")). Verified by an assertion in the same file.
  • SC-9. ppLayerOff and ppVerbOff remain present and functional during this branch (this plan does NOT remove them). Verified by re-running the existing tests/testthat/test-pp-off-naked.R and observing zero regressions.
  • SC-10. The DoD command returns FAIL 0 / ERROR 0 / SKIP 0 / PASS N with N rising by exactly the number of new test_that blocks (SC-1..SC-8 = 8 blocks; SC-9 is regression coverage, not new) from the baseline.

Definition of Done

Authoritative gate (copy verbatim from project CLAUDE.md "Authoritative gate" block):

NOT_CRAN=true Rscript -e 'suppressMessages(devtools::load_all(".")); testthat::test_dir("tests/testthat", reporter="progress", stop_on_failure=FALSE)'

Expected output: FAIL 0 / ERROR 0 / SKIP 0 / PASS N. Baseline N = plan-01's post-merge N; expected N after = baseline + 8.

Proxy traps — do not be fooled:

  • A raw Rscript -e 'testthat::test_dir()/test_file()' with no NOT_CRAN → skip_on_cran() fires → every shinytest2 browser test silently SKIPs. "green, SKIP n" off that is NOT the gate.
  • devtools::check(... --as-cran) does NOT make skip_on_cran() fire (devtools runs the check subprocess with NOT_CRAN=true). The browser fixtures instead cleanly SKIP via boot_vignette_app()'s source-root guard. If a check run shows the e2e file ERRORing, that guard is missing/broken — a harness defect, not a product failure, and never the gate.
  • Only NOT_CRAN=true / devtools::test() runs the browser tests. That alone is the gate.

This plan adds 8 new test_that blocks (one per SC-1..SC-8) at tests/testthat/test-pp-verb-switch-naked.R (SC-1..SC-6) and tests/testthat/test-pp-verb-switch-keyword.R (SC-7, SC-8). The gate must show PASS count rise by exactly 8 with FAIL/ERROR still 0 and SKIP unchanged from the pre-plan baseline.
