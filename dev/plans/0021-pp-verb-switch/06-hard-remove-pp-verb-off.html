PLAN-06 — hard-remove ppVerbOff across code, tests, vignettes, fixtures, man

Source: ADR 0021 · Depends on: 02, 03, 04, 05 · Parallel group: G5 · Status: draft

Context

From the ADR's Decision section:

Hard-remove ppVerbOff. Per user direction, no deprecation window. Removed in the same change: the ppVerbOff exported function in R/paintr-builtins.R:477-497 (and its roxygen @rdname pp_off_toggles entry); the ppVerbOff entry in ptr_builtin_keywords() at line 261; the ptr_register_structural_keyword("ppVerbOff") call at line 361; the unwrap_pp_verb_off_stage helper and its caller in build_pipeline_from_lift (replaced by the ppVerbSwitch equivalents); the ppVerbOff-specific branches in ptr_assert_no_surviving_structural_wrappers and the validate_pp_off_hide helper (replaced by the ppVerbSwitch-specific validators); the man page combined-Rd pp_off_toggles splits into one Rd for ppLayerOff alone (unchanged) and one new Rd for ppVerbSwitch; the use-cases vignette section "Layer and stage toggles — ppLayerOff and ppVerbOff" retitled to "Layer and stage toggles — ppLayerOff and ppVerbSwitch" with the ppVerbOff prose rewritten for ppVerbSwitch; the six test-pp-off-*.R files (comparator, e2e, naked, readers, translate, vignette) renamed to test-pp-toggles-*.R with the ppVerbOff halves rewritten as ppVerbSwitch coverage (the ppLayerOff halves stay verbatim).

The largest plan, and necessarily atomic: removing ppVerbOff from R/ while leaving tests/ or vignettes/ calling it would break the gate. Every reference is migrated to ppVerbSwitch in one change.

Governing principle

One mental model, no aliases. The user direction explicit in the ADR is "no multiple surfaces doing the same thing": once ppVerbSwitch exists and subsumes ppVerbOff (ppVerbSwitch(verb, switch_on = FALSE) ≡ ppVerbOff(verb, hide = TRUE), and ppVerbOff(verb, hide = FALSE) ≡ bare verb), ppVerbOff must not survive as a synonym, deprecation shim, or compatibility alias. This is justified by two facts the ADR records: (a) ppVerbOff shipped only two days prior (ADR-0020, 2026-05-22) and has no production users beyond the project's own vignettes/tests; (b) leaving two surfaces forces every reader, contributor, and audit run to ask "which one and why" indefinitely. The migration map is mechanical: ppVerbOff(.data, verb, TRUE) → ppVerbSwitch(.data, verb, switch_on = FALSE) (or positionally, ppVerbSwitch(.data, verb, FALSE)); ppVerbOff(.data, verb, FALSE) → bare .data |> verb (the no-op wrapper has no replacement — its semantics is just "run the verb", which is what bare verbs do; per ADR §"Verified facts": "ppVerbOff(verb, hide=FALSE) is a no-op wrapper whose migration target is bare verb"). An implementer who keeps a ppVerbOff <- function(.data, verb_expr, hide = TRUE) ppVerbSwitch(.data, verb_expr, switch_on = !hide) shim "for safety" is re-introducing exactly the surface-multiplicity the ADR exists to remove; do not do this, and FAIL audit any such hedge on the PR. The naked-R / translate / disable / UI paths have already been put behind ppVerbSwitch by plans 02-05; this plan's job is to take the scaffolding away.

Definition of Done

Authoritative gate (copy verbatim from project CLAUDE.md "Authoritative gate" block):

NOT_CRAN=true Rscript -e 'suppressMessages(devtools::load_all(".")); testthat::test_dir("tests/testthat", reporter="progress", stop_on_failure=FALSE)'

Expected output: FAIL 0 / ERROR 0 / SKIP 0 / PASS N. Baseline N = the orchestrator HEAD after plan-04 AND plan-05 have both merged; expected N after ≥ baseline (parity-or-better — the test rewrite preserves coverage but may consolidate blocks).

Proxy traps — do not be fooled:

  • A raw Rscript -e 'testthat::test_dir()/test_file()' with no NOT_CRAN → skip_on_cran() fires → every shinytest2 browser test silently SKIPs. "green, SKIP n" off that is NOT the gate. This trap is especially dangerous for THIS plan because the renamed e2e file is what verifies the migrated ppVerbSwitch(filter(mpg > ppNum), switch_on = FALSE) fixture; running without NOT_CRAN would silently skip the exact regression coverage the plan exists to provide.
  • devtools::check(... --as-cran) does NOT make skip_on_cran() fire. The browser fixtures instead cleanly SKIP via the source-root guard. If a check run shows the e2e file ERRORing, that guard is a harness defect, not a product failure.
  • Only NOT_CRAN=true / devtools::test() runs the browser tests. That alone is the gate.
  • Project-specific proxy: devtools::document() emits a roxygen2 churn on every run for unrelated files if the workspace is cold; commit only the diff that mentions ppVerbOff / ppLayerOff / ppVerbSwitch / pp_off_toggles — re-run devtools::document() twice to confirm the second run is a no-op before committing.
  • Vignette knit: devtools::build_vignettes() is NOT the gate, but the auditor MUST run it once to confirm both vignettes render to HTML without error (ppVerbOff calls in chunks would still parse at load_all time but would FAIL the chunk at knit time — the gate catches this only if a test_that block knits the chunk, which is what test-pp-toggles-vignette.R is for).

This plan does not add new test_that blocks beyond the rewrites; net PASS-count delta is expected to be near zero with the rename. The hard requirement is FAIL 0 / ERROR 0 / SKIP 0 AND the coverage-parity check in SC-11.
