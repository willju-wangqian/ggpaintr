Goal

The formula-local shared section is part of the controls piece (decision 2026-05-16). Factor the existing inline-section block out of ptr_controls_panel (R/paintr-app.R:256–282) into a helper, and have ptr_ui_controls (R/paintr-app.R:651–662) render it, excluding obj's panel-owned keys.

Signature

ptr_ui_controls(id = NULL, formula, ui_text = NULL,
                checkbox_defaults = NULL, expr_check = TRUE,
                shared = NULL)        # `shared` is an obj from ptr_shared(); NULL = single-instance
# internal helper:
shared_section_tags(tree, ui_text, ns, exclude_keys = character())

Success criteria

  • The shared-section construction at app.R:256–282 is extracted verbatim-equivalent into shared_section_tags(), parameterised by exclude_keys.
  • ptr_ui_controls(id, formula) with no shared renders layer controls + a shared section containing all the formula's shared keys (single-instance behavior).
  • ptr_ui_controls(id, formula, shared = obj) renders the section with keys in obj$panel_keys excluded (those belong to ptr_shared_panel/ptr_ui_shared_panel).
  • The old internal render_shared_section boolean flag is gone from the public piece; any internal default-layout caller uses the new helper.
  • Section widgets remain namespaced via the controls' ns (NOT identity) — unchanged from today.

Constraints

  • No new export — this is folded into ptr_ui_controls (ADR 0005 rejected a separate ptr_ui_shared_section piece).
  • shared = obj is read-only/pure; do not derive a partition here — consume obj$panel_keys (Step 01 owns the algorithm).
  • Do not regress the existing DOM/ids for the non-shared parts (layer picker ptr_layer_select, hidden tabset, ptr_update_plot) — guarded by test-ui-pieces.R.
  • ptr_module_ui rewiring is Step 05, not here.
  • Generated files are deferred. If you run devtools::document() it is a local verify check only — do not commit the regenerated NAMESPACE/man/*.Rd on this feature branch (git restore NAMESPACE man/ before committing).
