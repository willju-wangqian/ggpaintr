1 · Part A — the traversal cursor

1.1 · Why: two passes hand-thread the same state

Placeholder ids and stage ids are location-derived. Today that location is recomputed by two separate recursions over the same tree.

R/paintr-ids.R — assign_ids_walk() (current)

# threads (layer_name, path, param) down by hand
assign_ids_walk <- function(node, layer_name, path, param) {
  if (is.null(node)) return(NULL)
  if (is_ptr_placeholder(node))
    return(assign_id_to_placeholder(node, layer_name, path, param))
  if (is_ptr_call(node)) {
    arg_names <- names(node$args) %||% rep_len("", length(node$args))
    for (i in seq_along(node$args)) {
      child_param <- if (nzchar(arg_names[i])) arg_names[i] else NA_character_
      node$args[[i]] <- assign_ids_walk(node$args[[i]], layer_name,
                                       path = c(path, i), param = child_param)
    }
    return(node)
  }
  if (is_ptr_pipeline(node)) {
    for (i in seq_along(node$stages))
      node$stages[[i]] <- assign_ids_walk(node$stages[[i]], layer_name,
                                         path = c(path, i), param = param)  # param passes through
    return(node)
  }
  node
}

R/paintr-disable.R — assign_stage_ids_chain() (current)

# threads (layer_name, path) down by hand, but ONLY along the data-arg chain
assign_stage_ids_chain <- function(node, layer_name, path) {
  if (is.null(node)) return(NULL)
  if (is_ptr_pipeline(node)) {
    for (i in seq_along(node$stages)) {
      stage <- node$stages[[i]]; sub_path <- c(path, i)
      if (is_data_chain_call(stage)) {                # ptr_call && walk_has_placeholder()
        stage$stage_id <- stage_id_from_path(layer_name, sub_path)
        if (i == 1L) {                                # only stage 1 is in data-arg position
          pos <- data_arg_position(stage)
          if (!is.null(pos)) stage$args[[pos]] <- assign_stage_ids_chain(
            stage$args[[pos]], layer_name, path = c(sub_path, pos))
        }
        node$stages[[i]] <- stage
      }
    }
    return(node)
  }
  if (is_data_chain_call(node)) {
    node$stage_id <- stage_id_from_path(layer_name, path)
    pos <- data_arg_position(node)
    if (!is.null(pos)) node$args[[pos]] <- assign_stage_ids_chain(
      node$args[[pos]], layer_name, path = c(path, pos))
    return(node)
  }
  node
}

Both recursions reconstruct exactly the same thing: "where am I in the tree, which layer am I under, and is this an argument-in-data-position?" A cursor makes that the traversal's job, computed once.

1.2 · The cursor object

A plain list, fields populated as known at each location (sparse is fine):

# R/paintr-walk.R  (proposed)
ptr_cursor <- function(layer_name      = NA_character_,
                       path            = integer(),     # index path relative to the layer
                       slot            = NA_character_, # "layers" | "data_arg" | "children" | "args" | "stages"
                       param           = NA_character_, # nearest enclosing argument NAME, or NA
                       in_data_position = FALSE,        # TRUE iff this node sits where pipeline data flows
                       pipeline_index  = NA_integer_) { # this node's stage index, if a pipeline stage
  list(layer_name = layer_name, path = path, slot = slot, param = param,
       in_data_position = in_data_position, pipeline_index = pipeline_index)
}

It is empty for a node sitting directly in data_arg; the literal string "0" only appears at id-formatting time. A var placeholder at children[1] → aes(...) → args[1] has path = c(1, 1) ⇒ id ends …_1_1_var_NA (this exact example is in the comment block of R/paintr-ids.R). 1.3 · The traversal primitive that threads it

A small top-down rewrite — visit each ptr_node with fn(node, cursor) before its children; fn returns a (possibly modified) node; the cursor for each child is derived per the table above. This is the minimal primitive the cursor needs; it is not the general ptr_rewrite(pre, post, descend) from the old handoff (no bottom-up phase, no descend callback). Hardcode the five container shapes here — do not introduce a ptr_children generic until a new node class actually exists.

# R/paintr-walk.R  (proposed)  — top-down rewrite, cursor-threaded.
# `fn(node, cur)` is called on every ptr_node before its children and must
# return a ptr_node (usually the same one, possibly with fields stamped).
# The `upstream` slot is never descended (same rule as ptr_walk()).
ptr_rewrite_pre <- function(node, fn, cur = ptr_cursor()) {
  if (!is_ptr_node(node)) {
    if (is.list(node)) return(lapply(node, ptr_rewrite_pre, fn = fn, cur = cur))
    return(node)
  }
  node <- fn(node, cur)
  if (!is_ptr_node(node)) return(node)   # fn may have replaced it with a leaf

  desc <- function(child, slot, idx, nm)
    ptr_rewrite_pre(child, fn, ptr_cursor_descend(cur, node, slot, idx, nm))

  if (is_ptr_root(node)) {
    for (i in seq_along(node$layers)) node$layers[[i]] <- desc(node$layers[[i]], "layers", i, NA_character_)
  } else if (is_ptr_layer(node)) {
    if (!is.null(node$data_arg)) node$data_arg <- desc(node$data_arg, "data_arg", NA_integer_, NA_character_)
    nms <- names(node$children) %||% rep_len("", length(node$children))
    for (i in seq_along(node$children)) node$children[[i]] <- desc(node$children[[i]], "children", i, nms[i])
  } else if (is_ptr_call(node)) {
    nms <- names(node$args) %||% rep_len("", length(node$args))
    for (i in seq_along(node$args)) node$args[[i]] <- desc(node$args[[i]], "args", i, nms[i])
  } else if (is_ptr_pipeline(node)) {
    for (i in seq_along(node$stages)) node$stages[[i]] <- desc(node$stages[[i]], "stages", i, NA_character_)
  }
  # placeholders / literal / missing / user_expr: no structural children — nothing to descend
  node
}

# Derive the child cursor from the parent cursor + (slot, 1-based index, arg name).
# Encodes the table in §1.2 — this is the ONE place those rules live.
ptr_cursor_descend <- function(cur, parent, slot, idx, nm) {
  nm_or_na <- if (!is.na(nm) && nzchar(nm)) nm else NA_character_
  if (slot == "layers")
    return(ptr_cursor(layer_name = parent$layers[[idx]]$name, path = integer(), slot = slot))
  if (slot == "data_arg")
    return(ptr_cursor(layer_name = cur$layer_name, path = integer(), slot = slot,
                       param = "data", in_data_position = TRUE))
  if (slot == "children")
    return(ptr_cursor(layer_name = cur$layer_name, path = idx, slot = slot, param = nm_or_na))
  if (slot == "args") {
    pos <- data_arg_position(parent)                       # NULL or the data-arg index
    return(ptr_cursor(layer_name = cur$layer_name, path = c(cur$path, idx), slot = slot,
                       param = nm_or_na,
                       in_data_position = isTRUE(cur$in_data_position) && identical(idx, pos)))
  }
  if (slot == "stages")
    return(ptr_cursor(layer_name = cur$layer_name, path = c(cur$path, idx), slot = slot,
                       param = cur$param,
                       in_data_position = isTRUE(cur$in_data_position) && idx == 1L,
                       pipeline_index = idx))
  ptr_cursor(layer_name = cur$layer_name)                  # defensive
}

1.4 · How ptr_assign_ids becomes one cursor-threaded pass

The id format is unchanged — copy it verbatim from assign_id_to_placeholder() / stage_id_from_path():

# placeholder id  (from R/paintr-ids.R, must stay byte-identical):
#   path_str    = if (length(path) == 0) "0" else paste(path, collapse = "_")
#   shared_part = node$shared %||% "NA"
#   id          = paste0(layer_name, "_", path_str, "_", node$keyword, "_", shared_part)
# stage id  (from R/paintr-disable.R::stage_id_from_path):
#   id = paste0(layer_name, "_", path_str, "_stage_enabled")

# R/paintr-ids.R  (proposed rewrite — replaces ptr_assign_ids + assign_ids_walk +
# assign_ids_in_layer; folds in assign_stage_ids).  Keep assign_id_to_placeholder
# and stage_id_from_path exactly as they are; only the recursion changes.
ptr_assign_ids <- function(node, ns_fn = shiny::NS(NULL)) {
  if (!is_ptr_root(node)) rlang::abort("ptr_assign_ids expects a ptr_root.")
  validate_ns_fn(ns_fn)
  node$ns_fn <- ns_fn
  ptr_rewrite_pre(node, function(n, cur) {
    if (is_ptr_placeholder(n)) {
      # NB: a placeholder that IS a layer (rare: a bare `upload` as a layer) keeps
      # the special-cased layer_name = n$keyword from the old code path — handle that
      # by setting cur$layer_name = n$keyword in ptr_cursor_descend's "layers" branch
      # when parent$layers[[idx]] is itself a placeholder. (See current ptr_assign_ids.)
      return(assign_id_to_placeholder(n, cur$layer_name, cur$path, cur$param))
    }
    if (is_ptr_call(n) && isTRUE(cur$in_data_position) && walk_has_placeholder(n)) {
      n$stage_id <- stage_id_from_path(cur$layer_name, cur$path)
    }
    n
  })
}
Do not drift — stage-id placement. Today, assign_stage_ids_chain only stamps a stage_id on a call that is (i) a ptr_call, (ii) walk_has_placeholder(call), and (iii) structurally on the data-arg chain (it only recurses into data_arg_position()). The cursor's in_data_position flag is what makes condition (iii) explicit. Two things that must still hold: (a) for a pipeline, every stage that is a data-chain call gets a stage_id from its c(path, i) — including stage 1 (the source) and stages k≥2 — but only stage 1's args are recursed into for nested chains (stages k≥2 take data implicitly); the cursor table encodes "stage k≥2's in_data_position is FALSE", so a ptr_call argument inside stage k≥2 will not be treated as a chain link — match that. (b) walk_has_placeholder stays as the gate; it's a narrow recursion (only into $args/$stages) and intentionally not the same as scanning all node fields — keep it. After the rewrite, collect_stage_ids(tree) (already on ptr_walk) must return the same set in the same order.

Lower-risk alternative if the merge feels too hot: keep ptr_assign_ids and assign_stage_ids as two logical passes, but implement each via ptr_rewrite_pre + the cursor instead of two bespoke recursions. You still delete the duplicated (layer_name, path, param) threading; you just don't share the single traversal. Do this first if you want the smallest possible diff.

1.5 · Acceptance criteria — Part A

  • Every placeholder's id, param, layer_name, and (for sources) companion_id are byte-identical to before. (companion_id is still computed by the registry's companion_id_fn inside assign_id_to_placeholder — don't move that.)
  • Every stage_id is byte-identical; collect_stage_ids(tree) returns the same vector in the same order; disable_walk behaviour is unchanged.
  • ptr_runtime_input_spec() output (rows + order + ids) is unchanged. (This is the strongest single regression test for this part.)
  • New test file tests/testthat/test-rewrite-walk.R: build a small typed tree by hand, run ptr_rewrite_pre collecting (node-class, cursor) at every node, and assert the cursor fields (path, layer_name, param, in_data_position, pipeline_index) match a hand-written expectation — including that upstream is not descended.
  • devtools::test(filter = "rewrite") green after the change.

2 · Part B — tree validators

2.1 · Why

The transform pipeline is order-sensitive: classify_walk must run before substitute (it writes upstream, read by substitute_walk.ptr_ph_data_consumer); ptr_substitute must run before ptr_prune before ptr_render/ptr_eval. When that's violated — or when a future change leaves a placeholder or a ptr_missing in the tree handed to ptr_eval — the failure today is a confusing error from deep inside eval() or ggplot2. A handful of cheap assertthat-style checks at the pass boundaries turn those into local, named AST errors. Because ptr_assemble_plot_safe() already wraps ptr_eval in tryCatch, an abort from a validator surfaces as a clean inline error in the app instead of a crash — that's the point.

2.2 · Proposed validators internal, in R/paintr-walk.R

All built on the existing ptr_collect / ptr_walk. Each is O(number of nodes).

# R/paintr-walk.R  (proposed)

# After ptr_assign_ids: every placeholder has a non-NA id.
ptr_assert_ids_assigned <- function(tree) {
  bad <- ptr_collect(tree, function(n) is_ptr_placeholder(n) && (is.null(n$id) || is.na(n$id)))
  if (length(bad))
    rlang::abort(paste0("Internal: ", length(bad),
      " placeholder(s) without an id reached a pass that needs one (keyword(s): ",
      paste(unique(vapply(bad, `[[`, character(1), "keyword")), collapse = ", "), ")."))
}

# After ptr_classify_data: consumers carry an `upstream`; sources have none.
ptr_assert_classified <- function(tree) {
  miss <- ptr_collect(tree, function(n) is_ptr_ph_data_consumer(n) && is.null(n$upstream))
  if (length(miss))
    rlang::abort("Internal: a `var` consumer has no `upstream` — ptr_classify_data() did not run or did not reach it.")
  src  <- ptr_collect(tree, function(n) is_ptr_ph_data_source(n) && !is.null(n$upstream))
  if (length(src))
    rlang::abort("Internal: a data-source placeholder has a non-NULL `upstream` (sources produce data).")
}

# Before ptr_eval / ptr_render: substitution must have removed every placeholder.
ptr_assert_no_placeholders <- function(tree) {
  bad <- ptr_collect(tree, is_ptr_placeholder)
  if (length(bad))
    rlang::abort(paste0("Internal: ", length(bad),
      " unresolved placeholder(s) reached evaluation — ptr_substitute() did not run, or pass order is wrong (keyword(s): ",
      paste(unique(vapply(bad, `[[`, character(1), "keyword")), collapse = ", "), ")."))
}

# After ptr_prune, before ptr_eval: no ptr_missing should remain.
ptr_assert_no_missing <- function(tree) {
  bad <- ptr_collect(tree, is_ptr_missing)
  if (length(bad))
    rlang::abort(paste0("Internal: ", length(bad), " `ptr_missing` node(s) survived pruning."))
}

# Test-only / debug-flag-only (NOT on the hot path): no node-id appears twice when
# descending WITH `upstream`. Guards a future bug where classify_walk points an
# `upstream` at a descendant of the consumer, creating a cycle. O(nodes) but the
# with-upstream walk can blow up on pathological hand-built trees, so don't ship it
# in the reactive path.
ptr_assert_acyclic <- function(tree) {
  seen <- new.env(parent = emptyenv())
  ptr_walk(tree, descend_upstream = TRUE, visit = function(n) {
    id <- n$id
    if (!is.null(id) && !is.na(id)) {
      if (!is.null(seen[[id]])) rlang::abort(paste0("Internal: node id `", id, "` visited twice — cyclic `upstream`?"))
      seen[[id]] <- TRUE
    }
  })
}

2.4 · Acceptance criteria — Part B

  • No behaviour change on any valid input — every existing test stays green, including the rewrite-error tests that check inline-error surfacing.
  • New tests (e.g. in tests/testthat/test-rewrite-walk.R): build deliberately-broken trees — a tree with an un-id'd placeholder, a consumer with upstream = NULL, a tree handed to ptr_eval with a stray ptr_ph_value, a pruned tree with a leftover ptr_missing, and (for ptr_assert_acyclic) a tree whose upstream points back into itself — and assert each validator aborts with a message mentioning the cause.
  • The validators are internal (no @export); they appear only in NAMESPACE's implicit set, not the public surface.

3 · Scope & non-goals

In scope (this patch): the ptr_cursor object + ptr_rewrite_pre + ptr_cursor_descend in R/paintr-walk.R; rewriting ptr_assign_ids (and folding in assign_stage_ids, or keeping it a separate cursor-threaded pass — implementer's call) onto it; the ptr_assert_* validators + their wiring at two pass boundaries; new tests in test-rewrite-walk.R. Internal helpers only — no public API change.
Out of scope (declined, not deferred-with-intent):
  • A general ptr_rewrite(pre, post, descend, include_upstream) bottom-up rewrite primitive, and porting substitute_walk / prune_walk / disable_walk / classify_walk onto it. Why: their container methods are not boilerplate — classify_walk threads a non-trivial data context (ggplot-layer-first ordering; building a pipeline-prefix as the context for stage k); prune_walk does operator-call escalation, positional-missing escalation, arg-name preservation, drop-when-empty, pipeline collapse, and the layer-drop policy; disable_walk has the stage-1-vs-stage-k≥2 distinction. The shared recursion in each is ~4 lines. A fat callback signature would hide that logic — the "too-clever walker" risk the original handoff itself lists. Revisit only if a genuinely new structural node class is added (then a ptr_children()/ptr_replace_children() generic earns its keep).
  • A ptr_children() generic. Hardcode the five container shapes inside ptr_rewrite_pre; the indirection isn't worth it for one consumer.
  • ptr_render / ptr_eval refactors — they are conversions (tree → text / tree → language), not tree rewrites; leave them.
  • ptr_translate — owns the raw-R-to-typed-AST special cases (top-level +, pipe desugaring, placeholder parsing, layer-call detection, call-form shared); stays custom.
  • Replacing the typed AST with an external R AST library (lobstr, treesitter.r, xmlparsedata, astgrepr, …). ggpaintr's hard problem is semantic traversal — ggplot layer inheritance, pipe upstream pointers, placeholder roles, Shiny ids, shared bindings, data-source nodes, pruning policy — not R syntax traversal. Those tools are useful for source-location/lint/refactor work, not for ggpaintr's runtime model.
  • Creating or modifying anything under .claude/specs/ on the strength of this plan alone. This is design input, not an accepted architectural decision.

4 · Implementation order

  1. Add the kernel. ptr_cursor(), ptr_cursor_descend(), ptr_rewrite_pre() in R/paintr-walk.R. Add test-rewrite-walk.R asserting cursor field values on a hand-built tree and that upstream is not descended. devtools::test(filter = "rewrite").
  2. Port ids. Rewrite ptr_assign_ids onto ptr_rewrite_pre (folding in assign_stage_ids, or keeping it a second cursor-threaded pass). Keep assign_id_to_placeholder and stage_id_from_path verbatim. Verify ids / stage ids / ptr_runtime_input_spec() unchanged. devtools::test() (full — this touches a foundational pass).
  3. Add validators. ptr_assert_* in R/paintr-walk.R + their tests (broken-tree fixtures). devtools::test(filter = "rewrite").
  4. Wire validators. Calls in ptr_translate() and ptr_eval() per §2.3. devtools::test() full. Confirm the rewrite-error tests still surface inline errors (the validators are additional guards, not replacements).

Each step is independently revertible. Do not start step n+1 until step n's suite is green.

5 · Decision log (carried forward + updated)

  • Keep ggpaintr's typed AST rather than adopting an external AST framework — ggpaintr needs domain semantics, not R syntax traversal. unchanged
  • Skip upstream by default in every traversal — prevents accidental cycles and duplicate work. now enforced in ptr_walk()
  • Port read-side collectors before mutation passes — lower risk, validates the API. done: ptr_collect + 5 collectors
  • Add a traversal cursor before any further pass rewrites — the cursor is the missing piece that makes id assignment and data-position queries declarative. this patch, Part A
  • Add tree validators at pass boundaries — converts pass-order / pass-omission bugs into local AST errors that the safe wrappers surface cleanly. this patch, Part B
  • Do NOT build a general ptr_rewrite(pre, post, descend) / port substitute·prune·disable — their container logic isn't boilerplate; the win is ~4 lines/pass against a real "hidden behaviour" risk. new decision, 2026-05-11
  • Defer any .claude/specs/ decision — this remains design input. unchanged

Confidence the cursor/path rules above reproduce current ids: ~75/100 — the rules were derived by reading assign_ids_in_layer / assign_ids_walk / assign_id_to_placeholder / assign_stage_ids_chain, but the in_data_position ⇄ data_arg_position() interaction for nested data-chain calls is the subtle bit; the implementer must verify against test-rewrite-ids.R and ptr_runtime_input_spec() before trusting it.
