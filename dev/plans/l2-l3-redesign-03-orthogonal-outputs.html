Step 03 — Orthogonal outputs + combinators (pass #O)

Phase 3 (independent of shared; parallelisable with Steps 01–02). Authority: ADR 0005 §4 · CONTEXT.md §"Combinator".

Goal

Make output pieces truly bare and move behavior into nestable combinators. Remove error=/code_toggle= from ptr_ui_plot (R/paintr-app.R:412–413). Delete ptr_ui_code_toggle (R/paintr-app.R:480–487). Add ptr_ui_inline_error(plot, error) and ptr_ui_toggle_code(plotish, code).

Verified mechanism (no server coupling)

inst/www/ggpaintr-ui.js drives the toggle DOM-locally: click .ptr-code-toggle → nearest .ptr-output ancestor → toggle .ptr-code-window's .ptr-open. Combinators only need to emit that markup. Reuse plot_card_tag (:315), code_block_tag (:339), the .ptr-output wrapper (ptr_outputs_panel, ~:376).

Signatures

ptr_ui_plot(id = NULL)                 # bare: just the plot slot. NO error=/code_toggle=
ptr_ui_error(id = NULL)                # bare (unchanged)
ptr_ui_code(id = NULL, style = c("panel","window"))  # bare (unchanged)
ptr_ui_inline_error(plot, error)       # -> .ptr-output card with error inline
ptr_ui_toggle_code(plotish, code)      # -> wraps plotish + code; code behind  toggle
# nest: ptr_ui_toggle_code(ptr_ui_inline_error(ptr_ui_plot(id), ptr_ui_error(id)), ptr_ui_code(id))

Success criteria

  • ptr_ui_plot arity is function(id = NULL) — no behavior flags. Grep shows no remaining error=/code_toggle= callers except the new combinators / Step 05.
  • ptr_ui_code_toggle removed from NAMESPACE and source; its only internal caller (ptr_outputs_ui) is itself removed in Step 05's prerequisite or rewired there.
  • Combinators are nestable and return a single tag; the innermost plotish may be a bare ptr_ui_plot() or the output of ptr_ui_inline_error().
  • Snapshot test: ptr_ui_toggle_code(ptr_ui_inline_error(ptr_ui_plot("p"), ptr_ui_error("p")), ptr_ui_code("p")) produces DOM with one .ptr-output, a .ptr-code-toggle and a .ptr-code-window sharing it, and ids p-ptr_plot/p-ptr_error/p-ptr_code — i.e. functionally equivalent to the deleted ptr_ui_code_toggle("p").

Constraints

  • Pure DOM helpers — no session, no reactivity, no change to the server-registered ids (ptr_plot/ptr_error/ptr_code).
  • Must not require the bundled JS to change. If the existing JS scoping (closest('.ptr-output')) cannot resolve a nested composition, fix the markup, not the JS — and document why.
  • No deprecation alias for ptr_ui_code_toggle.
  • Do not touch ptr_module_ui internals here (Step 05 consumes these combinators).
