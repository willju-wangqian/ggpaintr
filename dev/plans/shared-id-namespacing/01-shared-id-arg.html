Shared-coordinator id argument — multi-panel namespacing

Drafted 2026-05-25 on branch shared-id-upgrade · status: draft, awaiting /implementable stamp

Adds an id argument to ptr_shared() so two or more independent shared panels can coexist in one Shiny session without input-id collision. The id lives on the spec object; ptr_shared_panel(), ptr_ui_shared_panel(), and ptr_shared_server() consume it implicitly — their signatures do not change. id = NULL (the default) preserves today's flat ids verbatim — full backward compatibility.

Baseline (verified 2026-05-25)

Success criteria

  • A user can write panelA <- ptr_shared(formulasA, id = "panelA") and panelB <- ptr_shared(formulasB, id = "panelB") in one app; both panels are functional simultaneously even when formulasA and formulasB both reference shared = "x" (or any overlapping key set).
  • The spec object exposes obj$id (character scalar or NULL); ptr_shared_panel(), ptr_ui_shared_panel(), and ptr_shared_server() derive their namespace from obj$id — no new args on those three functions.
  • When id = "panelA", every id in the table above gains a panelA- prefix (Shiny-NS()-style separator -). When id = NULL, every id matches today exactly — byte-for-byte regression guard.
  • ptr_server(formula, …, shared_state = sA) links to coordinator A by reactive-bundle handoff (unchanged API). The bundle returned by ptr_shared_server(panelA) carries the namespacing internally, so ptr_server needs no shared_id arg.
  • ptr_shared(id = …) validates: NULL or a single non-empty string matching ^[A-Za-z][A-Za-z0-9_]*$. Anything else aborts via rlang::abort() with a message naming the bad value.
  • The full test suite passes the project's authoritative gate (see Definition of Done) with zero edits to any pre-existing test file. Existing tests assert flat ids like ptr_shared_draw_all / shared_B — they keep passing because the default is id = NULL.
  • One new file tests/testthat/test-shared-id-namespacing.R covers the BDD scenarios below.
  • devtools::check(document = FALSE, manual = FALSE, args = c("--as-cran","--no-manual")) — 0 errors, 0 warnings; only the by-design NOTEs from project CLAUDE.md.

Constraints

  • Spec carries the id. Only ptr_shared() grows a new arg. ptr_shared_panel(), ptr_ui_shared_panel(), ptr_shared_server() retain their current signatures and read obj$id internally. Rationale: the id is a property of the coordinator's identity; duplicating it on three consumers invites drift.
  • Backward compatibility is byte-level. ptr_shared(formulas) with no id must produce identical HTML and bind to identical Shiny inputs as today. No conditional behavior change beyond id prefixing.
  • Namespacing scheme matches Shiny NS(). The separator is "-"; the prefix function is function(x) if (is.null(id) || !nzchar(id)) x else paste(id, x, sep = "-"). Build a single shared_ns(obj) helper that returns this function; reuse it on both UI and server sides.
  • Reuse the ns_fn seam. shared_panel_body_tag(), build_ui_for(), controllable_region(), and ptr_bind_local_shared_consumers() already accept ns_fn and pass identity from the shared path today (R/paintr-shared-coordinator.R:193,203,230; R/paintr-shared-ui.R:299). Replace these identity values with shared_ns(obj). Do not introduce a second namespacing mechanism.
  • Out of scope (call out, do not modify):
    • ptr_server's auto_bind_shared self-bind path (R/paintr-app.R:1156) — fires only with no coordinator.
    • ptr_app(), ptr_app_components(), ptr_app_bslib() — single-instance, never call ptr_shared().
    • Detection of duplicate ids across coordinators in one session — no session-wide registry exists; user's responsibility.
    • CSS / JS literal id refs — grep confirms zero hits in inst/ for #shared_ or ptr_shared_draw_all.
  • Docs nudge (in scope, minimal): vignettes/ggpaintr-use-cases.Rmd:419 currently lists ptr_shared_draw_all / ptr_shared_errors as top-level reserved ids. Append one sentence: when ptr_shared(id = "<id>") is supplied, those ids carry an <id>- prefix.
  • Coding rules (from .claude/rules/coding.md): tidyverse style, snake_case, 2-space indent; rlang::abort() not stop(); assertthat::assert_that() for arg validation; regenerate NAMESPACE/man/ via devtools::document() — do not hand-edit.
  • Serena edit-tool rule (from .claude/rules/serena-tools.md): sub-agents in nested worktrees must use Edit/Write only; never mcp__serena__replace_symbol_body / insert_after_symbol / rename_symbol.

BDD scenarios

Feature: ptr_shared(id =) namespacing

  Scenario: default id keeps today's flat ids
    Given a formulas vector that uses shared = "k" in two formulas
    When  I call ptr_shared(formulas)
    Then  obj$id is NULL
    # ptr_shared_panel() emits every shared widget through a server-rendered
    # `renderUI` container (the `_ui` slot); the picker/widget itself is
    # inserted into that slot at server runtime. The static HTML contains
    # only the container id. The bare input id ("shared_k") is verified as
    # an addressable input slot via shiny::testServer in the "server reads"
    # scenario below — that is the behavioral pin.
    And   ptr_shared_panel(obj) HTML contains exactly one element with id "shared_k_ui"
    And   ptr_shared_panel(obj) HTML contains exactly one element with id "ptr_shared_draw_all"
    And   ptr_shared_panel(obj) HTML contains exactly one element with id "ptr_shared_errors"

  Scenario: non-null id prefixes every panel-owned id
    Given a formulas vector that uses shared = "k" in two formulas
    When  I call ptr_shared(formulas, id = "panelA")
    Then  obj$id equals "panelA"
    And   ptr_shared_panel(obj) HTML contains an element with id "panelA-shared_k_ui"
    And   ptr_shared_panel(obj) HTML contains no element with id "shared_k_ui"
    And   ptr_shared_panel(obj) HTML contains an element with id "panelA-ptr_shared_draw_all"
    And   ptr_shared_panel(obj) HTML contains an element with id "panelA-ptr_shared_errors"

    Given panelA <- ptr_shared(formulasA, id = "panelA") with one shared key "k"
    When  I run ptr_shared_server(panelA) inside shiny::testServer
    And   I set session$setInputs(`panelA-shared_k` = "wt", `panelA-ptr_shared_draw_all` = 1L)
    Then  shared_state$shared$k() returns "wt"
    And   shared_state$draw_trigger() advances to 1L

  Scenario: two coordinators in one session, overlapping keys, no collision
    Given panelA <- ptr_shared(formulasA, id = "panelA") referencing shared = "x"
    And   panelB <- ptr_shared(formulasB, id = "panelB") referencing shared = "x"
    When  both ptr_shared_server() calls run inside one Shiny session
    And   session$setInputs(`panelA-shared_x` = "wt", `panelB-shared_x` = "drat")
    Then  panelA's bundle$shared$x() returns "wt"
    And   panelB's bundle$shared$x() returns "drat"

  Scenario: shared-stage checkbox id is namespaced too
    Given a formulas vector whose shared key "k" hosts an orphan pipeline stage
    When  I call ptr_shared(formulas, id = "panelA")
    Then  ptr_shared_panel(obj) HTML contains an element with id "panelA-shared_k_stage_enabled"

    When  I call ptr_shared(formulas, id = c("a","b"))
    Then  rlang::abort fires with a message containing "id"

Implementation outline

  1. R/paintr-shared.R — add shared_ns(obj) helper: returns function(x) if (is.null(obj$id) || !nzchar(obj$id)) x else paste(obj$id, x, sep = "-"). Keep canonical_shared_id() and shared_stage_input_id() as the unprefixed id builders — the ns_fn wrapper does the prefixing at use-site.
  2. R/paintr-shared-coordinator.R / ptr_shared() —
    • Add id = NULL parameter (after formulas).
    • Validate: is.null(id) || (is.character(id) && length(id) == 1L && nzchar(id) && grepl("^[A-Za-z][A-Za-z0-9_]*$", id)); abort with the bad value otherwise.
    • Add id = id to the structure(list(...), class = "ptr_shared_spec") payload.
  3. R/paintr-shared-coordinator.R / shared_panel_body_tag() — replace the four ns_fn = identity sites (widget build, controllable_region, button, errors uiOutput) with the result of shared_ns(obj) wrapped around literal ids:
    • line 188 shared_ui[[k]](canonical) → pass shared_ns(obj)(canonical).
    • lines 191-205 build_ui_for(..., ns_fn = identity, ...) → ns_fn = shared_ns(obj).
    • line 230 controllable_region(shared_stage_input_id(k), …, ns_fn = identity) → ns_fn = shared_ns(obj).
    • line 242 shiny::actionButton("ptr_shared_draw_all", …) → shiny::actionButton(shared_ns(obj)("ptr_shared_draw_all"), …).
    • line 244 shiny::uiOutput("ptr_shared_errors") → shiny::uiOutput(shared_ns(obj)("ptr_shared_errors")).
  4. R/paintr-shared-ui.R / ptr_shared_server() —
    • Build local ns <- shared_ns(obj) at top of function.
    • line 266 cid <- canonical_shared_id(k) → cid <- ns(canonical_shared_id(k)).
    • line 283 input$ptr_shared_draw_all → input[[ns("ptr_shared_draw_all")]].
    • line 299 ptr_bind_local_shared_consumers(..., ns = identity, ...) → ns = ns.
    • lines 305, 315 output$ptr_shared_errors → output[[ns("ptr_shared_errors")]].
    • lines 329, 344 sid_input <- shared_stage_input_id(k) → sid_input <- ns(shared_stage_input_id(k)) (so input[[sid_input]] and block_dom_id at line 345 both reach the right slot — block_dom_id is already derived from sid_input so flows through).
  5. R/paintr-shared-ui.R / ptr_shared_state — no change. The state already carries reactives, not ids. ptr_server consumes via reactive call.
  6. Docs:
    • Roxygen on ptr_shared(): add @param id paragraph naming the validation rule and back-compat default.
    • Roxygen on ptr_shared_panel(), ptr_ui_shared_panel(), ptr_shared_server(): one sentence each — "Namespacing is inherited from obj$id; supply it to ptr_shared()."
    • vignettes/ggpaintr-use-cases.Rmd:419: append the prefix sentence noted in Constraints.
    • Regenerate with devtools::document().
  7. Tests — create tests/testthat/test-shared-id-namespacing.R with one test_that() per BDD scenario above. Use shiny::testServer() for the two server scenarios; HTML-render scenarios use htmltools::renderTags() on the panel and assert on the rendered string. Do not modify any pre-existing test file — back-compat is the regression guard.

Proxy traps — do NOT be fooled (from project CLAUDE.md)

  • A raw Rscript -e 'testthat::test_dir()' without NOT_CRAN=true silently SKIPs every shinytest2 browser test. "Green, SKIP n" off that command is NOT the gate.
  • devtools::check(... --as-cran) does NOT trip skip_on_cran() (devtools sets NOT_CRAN=true in the subprocess). The browser fixtures cleanly SKIP via the source-root guard instead. If the e2e file ERRORs under check, that's a harness defect — not a product failure, and never the gate.
  • A self-report from any sub-agent ("done / green / tests pass") is a claim to verify by running the authoritative command yourself — never proof.

Scope guard

  • This plan is id-side only. No behavioral change to the partition logic, the draw-all wiring semantics, the consumer-binding logic, or the auto_bind_shared self-bind path.
  • No new public function. No new file under R/. shared_ns() is an unexported helper added to R/paintr-shared.R.
  • No change to ptr_server(), ptr_app(), ptr_app_grid(), ptr_ui(), or the placeholder registry.
