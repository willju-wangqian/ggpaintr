Goal

Make ptr_shared_server(obj) derive the same partition Step 01 computed, and make ptr_module_server's shared_state path (R/paintr-app.R:801–874) mirror only the formula-local keys for that formula. Cross-formula keys are owned solely by the top-level ptr_shared_server. The returned ptr_shared_state bundle and the ptr_module_server(shared_state=) downstream contract are otherwise unchanged.

Success criteria

  • ptr_shared_server(obj) (was ptr_shared_server(formulas, ...), R/paintr-shared-ui.R:338+) reads the partition from obj — it must not re-derive it independently.
  • For a key in obj$panel_keys: exactly one reactive source exists (top-level ptr_shared_server); no per-module mirror is created for it.
  • For a formula-local key: it is driven by that formula's own server (the inline section), never by the panel.
  • Bundle shape ({shared, draw_trigger, shared_resolutions, shared_stage_enabled}) and validate_ptr_shared_state() unchanged.
  • 2-formula integration test green (below).

Constraints

  • Do not duplicate the partition algorithm — consume obj (single source of truth, ADR 0005). If ptr_shared_server and ptr_shared() disagree, that is a bug, not a tunable.
  • Preserve global ids (shared_<key>, ptr_shared_draw_all, ptr_shared_errors) and that ptr_shared_server runs at the top level (never inside moduleServer).
  • No change to ptr_server's public signature.
  • No UI work here (Steps 01/04/05 own UI).
