LLM materials audit — corpus + enum + README/external (pass #LLMA)

Created 2026-05-17. Follows the L2/L3 redesign + Step 08 (LLM topics rewrite). Authority: ADR 0005 · CONTEXT.md (LOCKED). Motivated by the Step-10 discovery that ggpaintr-llm.Rmd still hard-coded the Step-08-removed level2_embed topic — proof Step 08 swept the corpus but the wider LLM surface was not verified end-to-end.

Goal

Step 08 rewrote inst/llm/topics/* + primer.md. This pass is the verification + gap-fill over the entire LLM surface: the corpus, the ptr_llm_topics() enumeration, the ptr_llm_* exported behavior, README/ellmer discoverability copy, and a consistency audit (read-only) of ggpaintr-llm.Rmd against the post-Step-08 corpus. It guarantees an LLM consuming ggpaintr's bundle is told only the shipped API and the same examples the human docs use.

Scope — surface & ownership

  • Edited by this plan: inst/llm/topics/*.md, inst/llm/primer.md, the topic-id registry/enumeration behind ptr_llm_topics(), and LLM-discoverability copy in README.Rmd (the "use with an LLM / ellmer" section) + any inst/llm registration guidance.
  • Audited (read-only) by this plan, edited by the vignette-prose-sweep plan: vignettes/ggpaintr-llm.Rmd. This plan files concrete findings (line + required change); it must not edit the vignette.
  • Reference truth: CONTEXT.md (L1/L2/L3, partition, combinators) and the already-shipped vignettes/ggpaintr-use-cases.Rmd (canonical examples) — the corpus must not diverge from these.
  • ptr_llm_* roxygen is audited; edit only if it names a removed symbol (no signature changes).

Success criteria

  • No file under inst/llm/, nor README.Rmd's LLM section, mentions a removed symbol (ptr_controls_ui, ptr_outputs_ui, ptr_ui_code_toggle, ptr_shared_ui), the ptr_ui_plot flags, dead topic ids (level2_embed, level2_namespacing), or state$server_ns_fn as a user tool — except as an explicit "removed/superseded" note.
  • ptr_llm_topics() enumerates exactly the files present; invisible(lapply(ptr_llm_topics(), ptr_llm_topic)) runs with no error; no dead id is reachable and no present topic is unlisted.
  • The corpus states, per CONTEXT.md: the ptr_<x>=L2 / ptr_ui_<x>=L3 convention, the partition rule, single-vs-multi-instance, the combinator recipe, and custom-render-is-L3 via the moduleServer(id) pattern; no headless/testServer-as-feature framing.
  • Corpus examples are token-consistent with ggpaintr-use-cases.Rmd (same formulas, ids, combinator recipes) — a diff shows zero divergence.
  • README.Rmd LLM/ellmer section reflects the current topic set and API; README.md regenerated from it.
  • An llm-audit-findings note (this plan's deliverable for the vignette) lists every required ggpaintr-llm.Rmd change with line + rationale, handed to the vignette-prose-sweep plan.

Constraints

  • Content-only — no change to ptr_llm_* code or signatures, no R/ logic, tests/, NAMESPACE, or man/ (roxygen text edits only if a removed symbol is named, then devtools::document() as verify, generated files committed by whoever owns the doc tree, not casually).
  • Markdown: no hard-wrap — one paragraph = one line.
  • README.Rmd is the editable source; never hand-edit README.md — re-knit it.
  • Do not edit vignettes/ggpaintr-llm.Rmd (the vignette-prose-sweep plan owns its edits) or ggpaintr-use-cases.Rmd (Step 07). This plan's vignette output is a findings list, nothing more.
  • Stay consistent with the human docs — if the corpus and ggpaintr-use-cases.Rmd disagree, the vignette + CONTEXT.md win; flag, do not silently re-invent examples.
