PLAN-02 — Canonical pipeline lift via desugar + round-trip + grounding

Source: ADR 0012 (§1, §2 "Lift rule", §3.1 "The lift transform", §3.2 "Terminal-grounding check", §3.4 "+ fence", §4 S2, §5 OQ2) · Depends on: 01 · Parallel group: G2 (lands atomically with PLAN-04) · Status: draft (revision 3 — supersedes the merged-then-reverted commit cd9b74a and supersedes revision 2's narrow ADR §3.1 engine after a grill session that surfaced the engine was incomplete for canonical equivalence with the existing `%>%` translation path)

Governing principle (overrides scope tactics)

The tree is semantic, not syntactic. ADR 0012 §1 / §2 row 1: pipe operators are pure first-arg-insertion sugar that R desugars away (`|>` at parse time; `%>%` survives only because magrittr is a function). The canonical translated tree must be the same shape regardless of which surface form the user wrote (`|>`, `%>%`, nested-call, or any mix). The implementer's job is to make the lift produce one canonical tree for all four surface forms via a structural transformation, not to preserve any one surface form's idiosyncrasies. The auditor's job is to mechanically reject any implementation that leaves a surface-form divergence in the tree.

This plan introduces the canonical-pipeline-construction path. It supersedes the existing `%>%`-only `translate_pipeline` / `collect_pipe_stages` / `translate_piped_layer` path: all four surface forms now route through one uniform desugar → split → round-trip-check → ground → lift pipeline. After this plan and PLAN-04 land atomically, downstream consumers (prune, render, eval, runtime, safety, warn) treat the three surface forms uniformly because there is exactly one canonical shape they all produce.

Context

ADR 0012 §1.1–§1.2 documents the root divergence: a formula written with `|>` (expression mode) or as nested calls produces a `data_arg` of class `ptr_call`; the semantically-identical `%>%` form produces a `ptr_pipeline`; the string-input `|>` form (via `preprocess_native_pipe`) also produces a `ptr_pipeline` but via a separate translation route. Three different code paths for one semantic concept. The role-based redesign collapses them.

The ADR §3.1 engine sketch is incomplete. The sketch's loop terminates with `cur` still a call (whose first arg is a non-call), then returns `c(list(cur), stages)` — putting the entire last call (e.g., `filter(penguins, x > 1)`) into the source slot. But the existing `%>% path (`collect_pipe_stages`) splits `penguins %>% filter(x > 1)` into `[penguins, filter(x > 1)]` — source extracted as its own non-call element. The two shapes are not `identical()`. This plan fills the gap with a post-loop split + a round-trip verification step + a placeholder-aware grounding check (see Engine below).

History note (do not repeat). The prior revision of this plan (merged as cd9b74a, reverted at f2954e5) shipped a narrowed predicate that lifted only when the terminal was a `role='source'` placeholder symbol. That narrowing preserved the very divergence the ADR was supposed to eliminate. The auditor accepted it on the strength of an inline rationale about downstream consumers. That acceptance was wrong — see Constraints §SCOPE NARROWING FORBIDDEN.

Engine

The lift is a 4-step pipeline. Pseudo-code (final form may vary slightly for R idiom but must be semantically equivalent):

# Step 1 — desugar all pipe operators (`%>%` and the native-pipe sentinel)
# to fully-nested call form. Pipes are pure first-arg-insertion sugar.
desugar_pipes_to_nested(expr) {
  if (!is.call(expr)) return(expr)
  if (length(expr) >= 3 && is.symbol(expr[[1]])) {
    head_str <- as.character(expr[[1]])
    # `.ptr_pipe_native_sentinel` is the existing constant defined in
    # `R/paintr-translate.R` (used by `preprocess_native_pipe` to subst
    # `|>` for `%ptrPipeNative%` before parse). Reused here unchanged.
    if (head_str == "%>%" || head_str == .ptr_pipe_native_sentinel) {
      lhs <- desugar_pipes_to_nested(expr[[2]])
      rhs <- desugar_pipes_to_nested(expr[[3]])
      if (is.call(rhs))   return(as.call(c(list(rhs[[1]]), list(lhs), as.list(rhs[-1]))))
      if (is.symbol(rhs)) return(as.call(list(rhs, lhs)))
      # other RHS shapes — leave expr untouched (rare; pathological)
    }
  }
  # not a pipe — recurse into args, preserving names
  out <- as.call(c(list(desugar_pipes_to_nested(expr[[1]])),
                   lapply(as.list(expr[-1]), desugar_pipes_to_nested)))
  names(out) <- names(expr)
  out
}

# Step 2 — split fully-nested form into source + ordered stages. Each stage
# carries `first_arg_name` so the original positional/named-arg shape can be
# reconstructed losslessly. Result: list(source = node, stages = list(stage))
# where each stage = list(call = call_with_first_arg_stripped,
#                         first_arg_name = name_or_"")
resugar_pipeline_stages(expr) {
  stages <- list()
  cur <- expr
  # Outer loop: descend through any call whose first positional arg is itself
  # a call. Always-aggressive descent per ADR §1's "tree is semantic, not
  # syntactic" — pipe operators carry no boundary info, so neither do nested
  # calls. The lift always descends as far as it can.
  while (is.call(cur) && length(cur) >= 2 && is.call(cur[[2]])) {
    head <- cur[[1]]; rest_args <- as.list(cur[-c(1, 2)])
    fan  <- (names(cur) %||% rep_len("", length(cur)))[2]
    stages <- c(list(list(call = as.call(c(list(head), rest_args)),
                          first_arg_name = fan)), stages)
    cur <- cur[[2]]
  }
  # Post-loop split: if cur is a call whose first arg is a non-call,
  # extract that non-call as the source and bank cur (stripped of its
  # first arg) as the source-side stage.
  if (is.call(cur) && length(cur) >= 2 && !is.call(cur[[2]])) {
    head <- cur[[1]]; rest_args <- as.list(cur[-c(1, 2)])
    fan  <- (names(cur) %||% rep_len("", length(cur)))[2]
    source_arg <- cur[[2]]
    stages <- c(list(list(call = as.call(c(list(head), rest_args)),
                          first_arg_name = fan)), stages)
    cur <- source_arg
  }
  list(source = cur, stages = stages)
}

# Step 3 — rebuild the fully-nested form from (source, stages). Inverse of
# Step 2; round-trip identity is the structural-correctness gate.
rebuild_nested_from_stages(parts) {
  if (length(parts$stages) == 0) return(parts$source)
  acc <- parts$source
  for (st in parts$stages) {
    rest_args <- as.list(st$call[-1])
    new_args  <- c(list(acc), rest_args)
    nms <- names(new_args) %||% rep_len("", length(new_args))
    nms[1] <- st$first_arg_name  # restore the captured first-arg name
    names(new_args) <- nms
    acc <- as.call(c(list(st$call[[1]]), new_args))
  }
  acc
}

# Step 4 — round-trip + grounding gates. Lift fires iff both pass.
try_lift_to_pipeline(expr) {
  canonical <- desugar_pipes_to_nested(expr)
  parts <- resugar_pipeline_stages(canonical)
  if (length(parts$stages) < 2) {
    # Single-stage list = no chain to lift (degenerate single-call or bare value).
    return(list(success = FALSE, reason = "single-stage"))
  }
  # GATE 1 — round-trip identity. If the rebuild does not equal the
  # canonical-nested form, something was bisected (e.g. a named arg lost a
  # name in the split). Refuse to lift.
  rebuilt <- rebuild_nested_from_stages(parts)
  if (!identical(rebuilt, canonical)) {
    return(list(success = FALSE, reason = "round-trip-mismatch"))
  }
  # GATE 2 — source grounding. The extracted source must be a SYMBOL.
  # A CALL is an "opaque-call terminal" per ADR §3.2 — quiet failure.
  # A literal (numeric, string, logical) is not a data source.
  # NOTE: at lift time, raw AST is what we hold — a placeholder like
  # `ppUpload` is a plain symbol at the AST level; it does not become a
  # ptr_ph_data_source NODE until translate_node runs over it. So the
  # check here is purely `is.symbol`, not `is_ptr_ph_data_source`.
  if (is.call(parts$source)) {
    return(list(success = FALSE, reason = "opaque-call-source"))
  }
  if (!is.symbol(parts$source)) {
    return(list(success = FALSE, reason = "non-data-source"))  # e.g. a literal
  }
  list(success = TRUE, parts = parts)
}

Three independent gates decide whether to lift. GATE 0 (single-stage) rejects degenerate chains where `length(parts$stages) < 2` — there is no chain to lift. GATE 1 (round-trip) catches structurally-non-reversible bisects. With `first_arg_name` preservation in Step 2/3, this gate accepts more inputs than a naive engine would (a named-first-arg call like `ggplot(data = penguins, ...)` round-trips losslessly via the captured `"data"` name). It still catches genuinely non-reversible structures should any arise (e.g., from pathological AST shapes that the desugar step can't normalize). GATE 2 (grounding) catches sources that are not structurally valid data sources — opaque-call sources (e.g., `make_data()` at the deepest position) and literals (e.g., a bare `1` that fell out of bisecting `data.frame(x = 1)`). The three gates together accept every case where (a) there is a real multi-stage chain, (b) the lift produces a structurally-reversible canonical pipeline, and (c) the source slot is a symbol that could plausibly resolve to a data frame.

Pass-order requirement. Two ordering constraints justify the reorder:

  1. `ptr_classify_calls` must run BEFORE `ptr_assign_ids` so that ids are assigned on the canonical post-lift shape; otherwise nested-call and `%>%` users get different ids for the same widget. This also fixes a latent bug: `assign_stage_ids` (called inside `ptr_assign_ids`) only finds stages to id-stamp when the tree is in pipeline form; under the old order, nested-call layers had no stage-disable ids assigned because no pipeline existed yet at id-assignment time.
  2. `ptr_classify_data` runs before `ptr_classify_calls` for downstream consistency: classify_data tags placeholders' source/consumer role and stamps `ptr_ph_data_source` class on data-source placeholders. The lift itself does not consult these tags (GATE 2 uses only `is.symbol`, not `is_ptr_ph_data_source`, because the lift operates on raw AST where a placeholder is still a plain symbol), but `ptr_shared_bind` and downstream walkers rely on the tags, and keeping classify_data first preserves today's invariant that classification precedes structural rewrites.

The annotate-block sequence in `ptr_translate` (`R/paintr-translate.R`) changes from

root <- ptr_assign_ids(root, ns_fn = ns_fn)
root <- ptr_classify_data(root)
root <- ptr_classify_calls(root)
root <- ptr_shared_bind(root)

to

root <- ptr_classify_data(root)
root <- ptr_classify_calls(root)   # the lift; runs on un-id-ed canonical tree
root <- ptr_assign_ids(root, ns_fn = ns_fn)  # assigns ids on the post-lift tree
root <- ptr_shared_bind(root)

Verified: `ptr_classify_data` (`R/paintr-classify.R:5-10`) is id-independent; `ptr_classify_calls`'s lift is id-independent; `ptr_assign_ids`'s id encoding depends on `path_str` (structural position) — so id stability across surface forms requires assignment on the canonical shape; `ptr_shared_bind` (`R/paintr-shared.R:6-11`, line 18 reads `x$id`) must run after `ptr_assign_ids` — both old and new order satisfy this.

Stage node shape change. Each pipeline stage now carries `first_arg_name` (a chr scalar, `""` when the user wrote the first arg positionally). Eval (`pipeline_to_eval_expr` in `R/paintr-eval.R`) and render (preserve-mode in `R/paintr-render.R`, modified by PLAN-04) consult this field to reconstruct named-arg shape losslessly. Walkers that iterate stages without reading `first_arg_name` are unaffected (the field is inert).

Existing `%>%` path is retired. The current `translate_pipeline` / `collect_pipe_stages` / `translate_piped_layer` in `R/paintr-translate.R` collectively implement a `%>%`-only pipeline-construction path. After PLAN-02, `%>%` source AST is handled by `desugar_pipes_to_nested` (which collapses `%>%` to nested form in step 1), then by the uniform lift. The three retired functions are deleted in this plan; any caller is rerouted through the new path.

Success Criteria

  • `desugar_pipes_to_nested(expr)` (internal, in `R/paintr-translate.R`) implements Step 1: recursively rewrites every `%>%` and `.ptr_pipe_native_sentinel` infix call as a nested-call by inserting LHS as the first positional arg of RHS. Recurses into all args of non-pipe calls, preserving arg names. Returns the input unchanged when there are no pipe operators.
  • `resugar_pipeline_stages(expr)` (internal) implements Step 2: descends through every call whose first positional arg is itself a call (always-aggressive per ADR §1); on loop termination, executes the post-loop split when `cur` is a call whose first arg is a non-call. Returns `list(source = node, stages = list(list(call = ..., first_arg_name = ...)))`. Preserves the first-arg name of each call it descends through, so the round-trip rebuild is lossless.
  • `rebuild_nested_from_stages(parts)` (internal) is the inverse of Step 2: given `list(source, stages)`, folds left to reconstruct the fully-nested call, restoring the captured `first_arg_name` on each step.
  • `try_lift_to_pipeline(expr)` (internal) implements three gates: GATE 0 is `length(parts$stages) >= 2` (single-stage rejected with `reason = "single-stage"`); GATE 1 is `identical(rebuild_nested_from_stages(parts), desugar_pipes_to_nested(expr))` (`reason = "round-trip-mismatch"` on failure); GATE 2 is `!is.call(parts$source) && is.symbol(parts$source)` (call source → `reason = "opaque-call-source"`; non-call non-symbol, i.e. literal → `reason = "non-data-source"`). Returns `list(success = TRUE, parts = parts)` only when all three pass; otherwise `list(success = FALSE, reason = <one of the four reasons above>)`.
  • `ptr_classify_calls(root)` (body filled in this plan; slot reserved by plan 01) iterates the layers of `root`. For each layer it processes `layer$data_arg` only (NOT the layer's `$children` — the `+` fence holds, ADR §3.4). When `layer$data_arg` is a `ptr_call`, it extracts the raw R AST from `data_arg$expr` (the field every `ptr_call` already carries — confirmed at `R/paintr-translate.R:336` and `R/paintr-nodes.R:29`), runs `try_lift_to_pipeline(raw_ast)` ONCE per layer (no recursive walk into nested ptr_calls — post-desugar there are no nested liftable structures to find; one canonical lift per data_arg covers the chain in full), and on success rebuilds `data_arg` as a fresh `ptr_pipeline`: the source (a raw symbol) is fed through `translate_node` to become a typed source node; each stage's raw call is fed through `translate_node` to become a typed stage node; the stage's `first_arg_name` (a chr scalar captured at split time) is then attached as `stage_node$first_arg_name`; the resulting list of typed nodes is passed to `ptr_pipeline(stages = ..., op = "|>", expr = canonical_nested_expr)`. On `try_lift_to_pipeline` failure, `data_arg` is left as `ptr_call` unchanged.
  • Canonical-tree identity invariant. Define `ptr_tree_structural_equal(a, b)` in `R/paintr-nodes.R` as recursive structural identity that ignores `$op` and `$expr` (both are surface-form metadata, not load-bearing tree shape). For any layer formula whose data-arg chain has at least one placeholder, the three surface forms `|>` (string-input or expression-input — they desugar identically post-step-1), `%>%` (magrittr), and nested-call form produce trees where `ptr_tree_structural_equal(t1$layers[[1]]$data_arg, t2$layers[[1]]$data_arg)` returns TRUE pairwise.
  • Pass-order reorder in `ptr_translate` annotate block: `classify_data → classify_calls → assign_ids → shared_bind`. Verified id-independence of `ptr_classify_data` and `ptr_classify_calls` (the reorder is safe). Side benefit: `assign_stage_ids` (called inside `ptr_assign_ids`) now finds stages to id-stamp for all surface forms, closing the latent gap where nested-call forms had no stage-disable ids.
  • Existing `translate_pipeline`, `collect_pipe_stages`, and `translate_piped_layer` in `R/paintr-translate.R` are deleted; their callers (the layer dispatcher inside `translate_node`, the `%>%`-handler) are rerouted through `try_lift_to_pipeline`.
  • Every lifted `ptr_pipeline` carries `$op = "|>"` (the ADR §5 OQ2 default). Because all four surface forms route through `desugar_pipes_to_nested` in Step 1 — which erases the source surface-pipe info by construction — the lift cannot (and must not try to) preserve which pipe operator the user originally typed. Preservation of original pipe-operator choice is ADR §5 OQ2 deferred work, out of scope here. `$op` is non-structural metadata, excluded from `ptr_tree_structural_equal`.
  • Authoritative gate (see Definition of Done) shows FAIL 0 / WARN 0 / SKIP 0 / PASS rising by ≥ N_new tests with all existing tests still passing.

Constraints

  • SCOPE NARROWING FORBIDDEN. If the always-aggressive descent in `resugar_pipeline_stages` appears to break a downstream consumer (e.g., a render-side walker, a `pipe_layer_warnings_for_layer` scanner, a `prune_walk.ptr_call` branch), the resolution is NEVER to narrow the descent (add an early-stop heuristic, a verb whitelist, a "stop at non-chain-shape head" check, or any other ad-hoc surface-syntax preservation). The acceptable resolutions are: (a) coordinate with PLAN-04 to update the downstream consumer in the same atomic merge group; (b) if a real ADR-vs-consumer conflict surfaces, STOP and surface the conflict in the audit report — do not paper over it with a narrowing. Any in-source comment of the form "narrowed for backward compat with downstream X", "stop descent at verb Y", or any test that asserts "form Z does NOT lift" (other than the three plan-documented rejection cases — single-stage, round-trip-mismatch, opaque-call/non-data-source — see BDD) is a SCOPE VIOLATION that the auditor MUST FAIL.
  • BDD is contract. A test that asserts behavior contradicting any `Then` clause in this plan's BDD section is a SCOPE VIOLATION, regardless of inline rationale. The auditor MUST FAIL such a test mechanically. Inline rationales explaining why the contradiction is "actually fine" do not save it.

    • `R/paintr-nodes.R` — add `ptr_tree_structural_equal(a, b)` (internal, no `@export`). Cases where this descent produces "weird-looking" stages (e.g., `my_data_dict[["x"]]` source decomposes into `[my_data_dict, "[[" ("x"), ...]`) are STRUCTURALLY CORRECT and STRUCTURALLY REVERSIBLE (round-trip passes). BDD scenarios

Each `Then` is the contract. Tests asserting a contradicting predicate are a SCOPE VIOLATION (auditor must FAIL).

Feature: Canonical pipeline lift collapses |>, %>%, and nested-call surface forms to one structurally-identical tree.

Scenario: Pure desugar — %>% becomes nested
  Given the R expression `quote(penguins %>% filter(x > 1) %>% mutate(y = z))`
  When  I call ggpaintr:::desugar_pipes_to_nested(expr)
  Then  identical(result, quote(mutate(filter(penguins, x > 1), y = z))) is TRUE

Scenario: Pure desugar — native-pipe sentinel becomes nested
  Given the R expression `quote(penguins %ptrPipeNative% filter(x > 1) %ptrPipeNative% mutate(y = z))`
        (the sentinel preprocess_native_pipe substitutes in for `|>` in string input)
  When  I call ggpaintr:::desugar_pipes_to_nested(expr)
  Then  identical(result, quote(mutate(filter(penguins, x > 1), y = z))) is TRUE

Scenario: Pure desugar — mixed surface form
  Given the R expression `quote(mutate(filter(penguins, x > 1), y = z) %>% summarise(n = n()))`
  When  I call ggpaintr:::desugar_pipes_to_nested(expr)
  Then  identical(result, quote(summarise(mutate(filter(penguins, x > 1), y = z), n = n()))) is TRUE

Scenario: Pure desugar — no pipes is a no-op
  Given the R expression `quote(mutate(filter(penguins, x > 1), y = z))`
  When  I call ggpaintr:::desugar_pipes_to_nested(expr)
  Then  identical(result, quote(mutate(filter(penguins, x > 1), y = z))) is TRUE

Scenario: Split + rebuild round-trips on a chain with named first arg
  Given the R expression `quote(ggplot(data = penguins, mapping = aes(x = bill_length)))`
  When  I call parts <- ggpaintr:::resugar_pipeline_stages(expr); rebuilt <- ggpaintr:::rebuild_nested_from_stages(parts)
  Then  identical(rebuilt, expr) is TRUE
  And   parts$stages[[1]]$first_arg_name equals "data"

Scenario: Split + rebuild round-trips on a 3-level chain
  Given the R expression `quote(summarise(mutate(filter(penguins, x > 1), y = z), n = n()))`
  When  I call parts <- ggpaintr:::resugar_pipeline_stages(expr); rebuilt <- ggpaintr:::rebuild_nested_from_stages(parts)
  Then  identical(rebuilt, expr) is TRUE
  And   identical(parts$source, quote(penguins)) is TRUE
  And   length(parts$stages) equals 3

Scenario: try_lift fires for plain bare-symbol source (any surface form, all give the same canonical tree)
  Given the three R expressions:
        e_native   = quote(penguins %ptrPipeNative% filter(ppVar > 1) %ptrPipeNative% mutate(y = ppNum))
        e_magrittr = quote(penguins %>% filter(ppVar > 1) %>% mutate(y = ppNum))
        e_nested   = quote(mutate(filter(penguins, ppVar > 1), y = ppNum))
        (each contains placeholder ppVar / ppNum)
  When  I call try_lift_to_pipeline(e_native), try_lift_to_pipeline(e_magrittr), try_lift_to_pipeline(e_nested)
  Then  every result has $success equal to TRUE
  And   identical(result_native$parts$source, result_magrittr$parts$source) is TRUE
  And   identical(result_magrittr$parts$source, result_nested$parts$source) is TRUE
  And   length(result_native$parts$stages) equals length(result_magrittr$parts$stages)
  And   length(result_magrittr$parts$stages) equals length(result_nested$parts$stages)

Scenario: try_lift refuses single-stage list (no chain to lift)
  Given the R expression `quote(penguins)` (a bare symbol — no chain)
  When  I call try_lift_to_pipeline(expr)
  Then  result$success is FALSE
  And   result$reason equals "single-stage"

Scenario: try_lift refuses an opaque-call source (must have chain depth >= 2 to reach the grounding gate)
  Given the R expression `quote(mutate(filter(make_data(), ppVar > 1), y = z))`
        — after Step 1 desugar this is unchanged; after Step 2 split, parts$source = make_data() (a 0-arg CALL)
        and parts$stages has length 2, so the single-stage GATE 0 does not fire and execution reaches the grounding gate
  When  I call try_lift_to_pipeline(expr)
  Then  result$success is FALSE
  And   result$reason equals "opaque-call-source"

Scenario: try_lift refuses a single-call data arg as single-stage (regression check on the gate ordering)
  Given the R expression `quote(filter(make_data(), ppVar > 1))` whose split gives stages of length 1

Definition of Done

Authoritative gate (copied verbatim from `CLAUDE.md`'s "Authoritative gate (Definition of Done)" block):

NOT_CRAN=true Rscript -e 'suppressMessages(devtools::load_all(".")); testthat::test_dir("tests/testthat", reporter="progress", stop_on_failure=FALSE)'

Expected: `FAIL 0 / WARN 0 / SKIP 0 / PASS (post-revert-baseline 1987 + N_new)` where N_new is approximately 18–24 across the three new test files. All previously-passing tests still pass.

Proxy traps — do not be fooled (copied verbatim from CLAUDE.md):

  • A raw `Rscript -e 'testthat::test_dir()/test_file()'` with no `NOT_CRAN` → `skip_on_cran()` fires → every shinytest2 browser test silently SKIPs. "green, SKIP n" off that is NOT the gate.
  • `devtools::check(... --as-cran)` does NOT make `skip_on_cran()` fire (devtools runs the check subprocess with `NOT_CRAN=true`). The browser fixtures instead cleanly SKIP via `boot_vignette_app()`'s source-root guard. If a check run shows the e2e file ERRORing, that guard is missing/broken — a harness defect, not a product failure, and never the gate.
  • Only `NOT_CRAN=true` / `devtools::test()` runs the browser tests (source tree present → 0 SKIP). That alone is the gate.

Out-of-scope handling

Findings outside this plan's owned files are surfaced in the audit report, never absorbed. The realistic risks worth flagging:

  • A test that snapshots the internal tree shape of a `%>%`-source data_arg (e.g. inspects `length(data_arg$stages)` directly on a `%>%` formula) may see a different stage count post-lift if the formula's source is itself a call (e.g. `make_data(args) %>% filter(ppVar > 1)`, where today's `%>% path produces 2 stages and the new always-aggressive lift produces 3). Record as a follow-up; do NOT narrow the lift to preserve the old count.
  • A test that asserts on the `$op` value of a `%>%`-source pipeline (e.g. `expect_equal(data_arg$op, "%>%")`) will fail because all lifted pipelines now carry `$op = "|>"`. Record as a follow-up; update the assertion, do NOT preserve `$op = "%>%"`.
  • Preserve-mode render output text is NOT a risk here: PLAN-04 ships the prefix-collapse render rule which renders placeholder-free prefix stages as a nested-call source, so user-facing preserve-mode text remains stable across the surface form change (verified by the dry-run discussion in this plan's grill session).
