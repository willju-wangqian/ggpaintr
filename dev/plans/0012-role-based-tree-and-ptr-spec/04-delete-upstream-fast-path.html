PLAN-04 — Delete runtime_upstream_data per-layer fast-path (closes bug 3a)

Source: ADR 0012 (§2 row "Bug 3a fast-path", §3.7 "Bug 3a fast-path deletion", §4 S4) · Depends on: 02 · Parallel group: G3 · Status: draft

Context

ADR 0012 §3.7 locks the deletion: "The if (!is.null(state$resolved_data[[layer_name]])) { ... } block inside runtime_upstream_data (in R/paintr-server.R) — the per-layer fast-path — goes away. With the role-based tree, every consumer's c$upstream already encodes its true resolution point — for in-stage consumers, that subtree is the prior stage, not the full pipeline. The existing ptr_resolve_upstream(c$upstream, snapshot, ...) fallback handles every case correctly."

Plan 02's structural lift makes this deletion safe. Before plan 02, deletion would have broken in-aes pickers under |> input. After plan 02, every consumer has correct structural c$upstream.

Success Criteria

  • The per-layer fast-path block in runtime_upstream_data — the if (!is.null(layer_name) && !is.null(state$resolved_data[[layer_name]])) { ... } block whose body assigns from state$resolved_data[[layer_name]]() to out[[c$id]] with a next — locate by symbol name in R/paintr-server.R is removed. The shared branch and the c$upstream fallback branch remain untouched.
  • An e2e fixture tests/testthat/fixtures/vignette-apps/adr12-bug-3a/ exists with an app.R file using the app-dir + pkgload pattern (per the shinytest2 conventions in .claude/rules/testing.md). The fixture's formula is ppUpload |> filter(ppVar > ppNum) |> ggplot(aes(ppVar, ppVar)) + geom_point().
  • A corresponding test file tests/testthat/test-adr12-bug-3a.R exists, gated with skip_on_cran() + skip_if_not_installed("shinytest2") + skip_if_not_installed("chromote"). The test:
    1. Boots the fixture app via AppDriver$new(test_path("fixtures","vignette-apps","adr12-bug-3a")) with the standard GGP_PKG env-var dance.
    2. Uploads penguins.csv (or whatever test data the fixture provides) into the ppUpload widget.
    3. Asserts the in-filter ppVar picker (id ggplot_2_1_1_ppVar_NA — confirm the exact id by inspecting the translated tree's consumer node ids in the fixture's app.R header comment) shows the uploaded data's columns as options.
    4. Asserts the aes-side ppVar pickers (ids matching pattern ggplot_1_*_ppVar_NA) also show the columns.
  • Authoritative gate (Definition of Done): FAIL 0 / ERROR 0 / SKIP 0 / PASS = baseline_after_plan_02 + N_new_test_assertions. No previously-passing test regresses (especially the existing tests/testthat/test-runtime-*.R coverage of runtime_upstream_data).
  • The deleted code's previous comment about "per-layer resolved-data fast-path" is removed cleanly — no orphan comments referencing a vanished block.

Definition of Done

Authoritative gate (copied verbatim from CLAUDE.md):

NOT_CRAN=true Rscript -e 'suppressMessages(devtools::load_all(".")); testthat::test_dir("tests/testthat", reporter="progress", stop_on_failure=FALSE)'

Expected: FAIL 0 / ERROR 0 / SKIP 0 / PASS (baseline_after_plan_02 + N_new). The browser e2e test must run (not silently SKIP) under this gate — that's the point of the NOT_CRAN=true envvar.

Proxy traps — do not be fooled (copied verbatim from CLAUDE.md):

  • A raw Rscript -e 'testthat::test_dir()/test_file()' with no NOT_CRAN → skip_on_cran() fires → every shinytest2 browser test silently SKIPs. "green, SKIP n" off that is NOT the gate.
  • devtools::check(... --as-cran) does NOT make skip_on_cran() fire (devtools runs the check subprocess with NOT_CRAN=true). The browser fixtures instead cleanly SKIP via boot_vignette_app()'s source-root guard (no DESCRIPTION in the .Rcheck sandbox). If a check run shows the e2e file ERRORing, that guard is missing/broken — a harness defect, not a product failure, and never the gate.
  • Only NOT_CRAN=true / devtools::test() runs the browser tests (source tree present → 0 SKIP). That alone is the gate.

This plan adds 1 new test file with ~3 browser-driven assertions plus 1 deletion in R/paintr-server.R. The gate must show PASS rise by ≥ 3, FAIL/ERROR still 0, SKIP unchanged.
