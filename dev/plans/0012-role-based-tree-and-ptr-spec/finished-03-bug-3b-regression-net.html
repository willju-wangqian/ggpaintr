PLAN-03 — Bug-3b regression net (empty filter elides under |>)

Source: ADR 0012 (§1.3 row "ADR-0011 Bug 3b", §4 S3) · Depends on: 02, 04 (the atomic G2 pair — see index.md) · Parallel group: G3 · Status: draft

Context

ADR 0012 §1.3 records: "empty filter() survives prune under |> input … prune_walk.ptr_pipeline elides stages whose args all prune to missing. With no ptr_pipeline node, that elision never runs." Plan 02 lifts nested-call data args into a canonical ptr_pipeline, which restores the elision path. This plan adds the regression net that asserts the bug stays fixed.

Pure unit test against the public translate / substitute / prune / render pipeline. No browser, no Shiny — the bug is provable at the R level (ADR 0011 §2.2 confirmed the R-level reproduction).

Success Criteria

  • tests/testthat/test-prune-bare-pipe-stage.R exists with at least four assertions:
    1. ptr_render(ptr_prune(substitute_walk(ptr_translate("ppUpload |> filter(ppVar > ppNum) |> ggplot(aes(ppVar, ppVar)) + geom_point()"), list(snapshot = list(), upstream_cols = list())))) does not contain the substring "filter". (Empty filter must be elided.)
    2. The same assertion with %>% in place of |> (regression net for the previously-working path).
    3. The same assertion against the nested-call form "ggplot(filter(ppUpload, ppVar > ppNum), aes(ppVar, ppVar)) + geom_point()" — written directly as nested calls, not as a pipe at all. This proves the lift catches the structural pattern regardless of surface syntax.
    4. Non-empty filter is preserved: ptr_render(ptr_prune(substitute_walk(ptr_translate("penguins |> filter(bill_length_mm > 40) |> ggplot(aes(bill_length_mm)) + geom_point()"), list(snapshot = list(), upstream_cols = list())))) does contain "filter" and "bill_length_mm > 40". (Don't over-elide.)
  • An additional assertion across all elidable verbs in default_drop_when_empty(): for at least filter, mutate, arrange, group_by — empty form under |> elides; non-empty form survives. Use a loop / purrr::walk over a small named vector of verb name → empty-args fixture pair.
  • Authoritative gate (see Definition of Done) shows FAIL 0 / ERROR 0 / SKIP 0 with PASS rising by exactly the number of test assertions added.

Constraints

  • Files this plan owns: tests/testthat/test-prune-bare-pipe-stage.R (new file).
  • Files this plan must NOT touch: any R/paintr-*.R source file, any vignette, any fixture under tests/testthat/fixtures/, NAMESPACE, man/. This is a pure test-additions plan — no production source changes.
  • Use only public + already-existing internal API: ptr_translate, ggpaintr:::substitute_walk, ptr_prune, ptr_render, ggpaintr:::default_drop_when_empty. Do not introduce new internals.
  • No library(dplyr) at test file top level — the test runs against the package's own substitute+prune+render machinery, not against dplyr's filter behavior. Where the test needs filter to resolve to dplyr::filter for qualified_call_name, attach "dplyr" in the test setup via withr::local_attached_package("dplyr") or equivalent, scoped to the test.
  • testthat edition 3 conventions: test_that("…", { … }) blocks; group by behavior, not by surface syntax.

BDD scenarios

Feature: Empty pipe-stage calls elide regardless of surface syntax

Scenario: Empty filter elides under |>
  Given the formula "ppUpload |> filter(ppVar > ppNum) |> ggplot(aes(ppVar, ppVar)) + geom_point()"
  When  the tree is translated, substituted with an empty input snapshot, pruned, and rendered
  Then  the rendered code text does NOT contain the substring "filter"

Scenario: Empty filter elides under %>%
  Given the same formula with %>% instead of |>
  When  translated, substituted, pruned, rendered
  Then  the rendered code text does NOT contain the substring "filter"

Scenario: Empty filter elides when written as nested calls
  Given the formula "ggplot(filter(ppUpload, ppVar > ppNum), aes(ppVar, ppVar)) + geom_point()"
  When  translated, substituted, pruned, rendered
  Then  the rendered code text does NOT contain the substring "filter"

Scenario: Non-empty filter is preserved
  Given the formula "penguins |> filter(bill_length_mm > 40) |> ggplot(aes(bill_length_mm)) + geom_point()"
  When  translated, substituted (with empty snapshot — no placeholders to fill), pruned, rendered
  Then  the rendered code text contains "filter" AND contains "bill_length_mm > 40"

Scenario: Across-verb elision sweep
  Given a verb name in c("filter", "mutate", "arrange", "group_by")
  When  the empty-args form of the verb is placed in a |>-piped formula and pruned
  Then  the verb name does NOT appear in the rendered code
  And   when the verb has non-empty args, it does appear

Proxy traps — do not be fooled (copied verbatim from CLAUDE.md):

  • A raw Rscript -e 'testthat::test_dir()/test_file()' with no NOT_CRAN → skip_on_cran() fires → every shinytest2 browser test silently SKIPs. "green, SKIP n" off that is NOT the gate.
  • devtools::check(... --as-cran) does NOT make skip_on_cran() fire (devtools runs the check subprocess with NOT_CRAN=true). The browser fixtures instead cleanly SKIP via boot_vignette_app()'s source-root guard. If a check run shows the e2e file ERRORing, that guard is missing/broken — a harness defect, not a product failure, and never the gate.
  • Only NOT_CRAN=true / devtools::test() runs the browser tests. That alone is the gate.
