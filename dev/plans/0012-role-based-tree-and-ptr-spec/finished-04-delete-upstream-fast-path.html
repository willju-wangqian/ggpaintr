PLAN-04 — Downstream uniformity (preserve-mode collapse + runtime fast-path delete + consumer-uniformity test net + bug-3a e2e)

Source: ADR 0012 (§1, §2 row "Bug 3a fast-path", §3.4 "+ fence", §3.7 "Bug 3a fast-path deletion", §4 S4) · Depends on: 01 · Parallel group: G2 (atomic pair with PLAN-02) · Status: draft (supersedes the cd9b74a-era PLAN-04 which scoped only the runtime fast-path deletion; this revision pairs with the rewritten PLAN-02 — see Governing principle and Atomic-pair section)

Governing principle (overrides scope tactics)

The tree is semantic, not syntactic. ADR 0012 §1 / §2 row 1: pipe operators are pure first-arg-insertion sugar that R desugars away (`|>` at parse time; `%>%` survives only because magrittr is a function). After PLAN-02's lift, every layer's `data_arg` arrives at downstream consumers in one canonical `ptr_pipeline` shape, regardless of which surface form the user wrote. This plan locks that uniformity in two ways: (a) it verifies it directly via a consumer-uniformity test net that asserts every downstream consumer of `data_arg` (eval, prune, safety, runtime upstream resolution) produces structurally equivalent output across the three surface forms (`|>`, `%>%`, nested-call); (b) it makes the always-aggressive lift invisible to existing `%>%` / `|>` users at the preserve-mode render layer via a prefix-collapse rule (placeholder-free prefix stages render as a single nested-call source). It also lands the original ADR §3.7 runtime fast-path deletion, which is safe only after PLAN-02's lift makes every consumer's `c$upstream` structurally correct. The implementer's job is to land these three changes uniformly; narrowing any of them to preserve a surface-syntax divergence is a SCOPE VIOLATION the auditor must mechanically FAIL.

This plan does NOT modify the `data_arg`-consuming walkers in `R/paintr-prune.R`, `R/paintr-safety.R`, `R/paintr-eval.R`, or `R/paintr-build-ui.R` themselves. Those consumers were already ADR-correct against `ptr_pipeline` input; the consumer-uniformity test net VERIFIES this rather than changing it. Should the test net surface a real consumer-side gap, the response is to fix the consumer (in this plan, since plan 02's atomic-pair commitment makes the half-state intolerable), not to narrow plan 02's lift.

Context

PLAN-02 introduces the canonical-pipeline lift: every layer's `data_arg` becomes a `ptr_pipeline` whenever its three gates (single-stage, round-trip, grounding) pass. Two consequences need explicit downstream treatment in this plan:

  1. Always-aggressive descent produces extra structural stages for nested-call and `%>%`-source inputs whose source is itself a call-shape (e.g. `my_dict[["x"]]` source). These extra stages are STRUCTURALLY CORRECT and STRUCTURALLY REVERSIBLE. They are invisible to the stage-checkbox UI today (verified: `R/paintr-build-ui.R:229`, `if (length(fields) > 0L)` gates each stage's `controllable_region` emission on placeholder presence — placeholder-free extra stages emit no UI). They are NOT invisible to the preserve-mode panel today, which would render every stage as a `|> stage` and thus expose the new structural stages to users. The prefix-collapse render rule below fixes that.
  2. Every lifted `ptr_pipeline` now carries `$op = "|>"` (PLAN-02 SC, per ADR §5 OQ2 default). For `%>%`-source users in preserve mode, this would otherwise change the operator displayed in their copyable code. The prefix-collapse render rule below subsumes this concern: for placeholder-free prefixes (the common `%>%`-source case), the collapsed prefix renders as a nested-call source, not as `|>` stages.

ADR §3.7 (runtime fast-path) deletion. The per-layer fast-path appears as TWO sibling blocks in `R/paintr-server.R` — both with identical pattern (`if (!is.null(layer_name) && !is.null(state$resolved_data[[layer_name]])) { ... }`) and identical fall-through to `ptr_resolve_upstream(...)`:

  • The block inside `runtime_upstream_data` (function near line 867, fast-path block near line 903 — the ADR §3.7 named block).
  • The block inside `runtime_consumer_entry` (function near line 1301, fast-path block near line 1304 — a sibling per-consumer function whose own comment explicitly says "mirroring the per-consumer slot of `runtime_upstream_data`"; the same architectural concern applies).

Both blocks go away. With PLAN-02's lift, every consumer's `c$upstream` already encodes its true resolution point (for in-stage consumers, the prior stage; for in-aes consumers, the layer's data_arg pipeline). The existing `ptr_resolve_upstream(c$upstream, snapshot, ...)` fallback handles every case correctly. Before PLAN-02 this deletion was unsafe under `|>` (the in-aes pickers' `c$upstream` was a `ptr_call` whose subtree the resolver could not walk uniformly); after PLAN-02 it is safe across all three surface forms. Deleting only one of the two blocks while leaving the other reproduces the exact half-state the atomic-pair is designed to prevent: callers routing through `runtime_consumer_entry` (the single-consumer resolution path) would short-circuit incorrectly under canonical-shape input.

e2e fixture for bug 3a. The original cd9b74a-era PLAN-04 included an e2e fixture `adr12-bug-3a` exercising the in-filter picker populating after upload under `|>` input. That scope is kept verbatim; it doubles as the user-facing regression test for both the fast-path deletion and PLAN-02's lift.

History note (do not repeat). The prior PLAN-02 revision (merged as cd9b74a, reverted at f2954e5) shipped a narrowed lift predicate to avoid breaking downstream consumers (`paintr-prune`, `paintr-disable`). The narrowing preserved the very `|>` / `%>%` / nested-call divergence the ADR was supposed to eliminate. The structural fix is the PLAN-02 ⇄ PLAN-04 atomic pair: PLAN-02 lifts uniformly, PLAN-04 verifies downstream consumers handle the lifted shape uniformly, and the render-side concern (extra stages being visible to users) is addressed by the prefix-collapse render rule below — not by narrowing the lift.

2. Prefix-collapse preserve-mode render rule in R/paintr-render.R

In `R/paintr-render.R`, the three symbols responsible for rendering a `ptr_pipeline` in preserve mode are:

  • `render_walk.ptr_pipeline(node, indent, preserve_placeholders)` (S3 dispatch entry; located near line 89) — delegates to the body helper below.
  • `render_pipeline_body(node, indent, preserve_placeholders)` (located near line 93) — today emits every stage as a chain link via the helper below.
  • `render_pipe_chain(seg_thunks, op, indent)` (located near line 106) — assembles the `|>` / `%>%` chain text from per-stage thunks.

Replace the body of `render_pipeline_body` (and any adjacent helpers as needed). The new body branches on `preserve_placeholders`:

  • When `preserve_placeholders = FALSE` (eval-mode rendering — placeholders have been substituted with concrete values upstream; this codepath supplies the code-panel text users see after pressing Update): the existing chain rendering is preserved verbatim — every stage emits as a `|>` (or `%>%`) chain link via `render_pipe_chain`. PLAN-04 does NOT change non-preserve-mode rendering for any input.
  • When `preserve_placeholders = TRUE` (preserve-mode rendering — placeholders are shown as their token names; this codepath supplies the preserve-mode panel users see while editing): the NEW three-branch prefix-collapse rule fires (pseudo-code below; final form may vary slightly for R idiom but must be semantically equivalent). `render_pipe_chain` is reused for the all-`|>` and tail-`|>` branches; a new helper renders the collapsed nested-call source as a single atom.
# stages[[1]] is the SOURCE node (per PLAN-02 ptr_classify_calls SC: post-loop
# split places `cur` — the deepest non-call — as the source, and the typed
# source node is stages[[1]] in the constructed ptr_pipeline).
# stages[[2..N]] are the verb stages.

render_pipeline_body(node, indent = 0L, preserve_placeholders = FALSE) {
  if (length(node$stages) == 0L) return("")

  if (!preserve_placeholders) {
    # Eval-mode rendering: existing chain behavior verbatim. PLAN-04 does
    # not change this codepath. (Matches today's render_pipeline_body body.)
    thunks <- lapply(node$stages, function(s) {
      force(s)
      function(ci) render_walk(s, ci, preserve_placeholders = FALSE)
    })
    return(render_pipe_chain(thunks, node$op, indent))
  }

  # preserve_placeholders = TRUE: the new three-branch prefix-collapse rule.
  stages <- node$stages
  k <- first_index_where_subtree_contains_any_placeholder(stages)  # NA if no stage contains a placeholder

  if (is.na(k)) {
    # No placeholders anywhere in the pipeline. Collapse the entire structure
    # back to a single nested-call source. Render that nested-call source as
    # a single rendered atom, not as a chain.
    nested_expr <- rebuild_nested_from_stages_typed(stages)   # uses PLAN-02's rebuild helper, adapted for typed nodes
    return(render_nested_atom(nested_expr, indent = indent))
  }

  if (k <= 2) {
    # The source itself (k == 1) or the first verb stage (k == 2) already
    # contains a placeholder — there is no all-placeholder-free prefix to
    # collapse. Render the entire pipeline as a `|>` chain (today's
    # behavior under preserve mode).
    thunks <- lapply(stages, function(s) {
      force(s)
      function(ci) render_walk(s, ci, preserve_placeholders = TRUE)
    })
    return(render_pipe_chain(thunks, node$op, indent))
  }

  # Mixed: stages[1..k-1] are all placeholder-free; stages[k..N] contain at
  # least one placeholder somewhere. Collapse the prefix to a nested-call
  # source, then render the tail as a `|>` chain whose first element is the
  # collapsed nested source.
  prefix_nested_expr <- rebuild_nested_from_stages_typed(stages[seq_len(k - 1L)])
  tail_stages        <- stages[k:length(stages)]
  prefix_thunk       <- function(ci) render_nested_atom(prefix_nested_expr, indent = ci)
  tail_thunks        <- lapply(tail_stages, function(s) {
    force(s)
    function(ci) render_walk(s, ci, preserve_placeholders = TRUE)
  })
  return(render_pipe_chain(c(list(prefix_thunk), tail_thunks), node$op, indent))
}

The helper `rebuild_nested_from_stages_typed` is a typed-node analogue of PLAN-02's `rebuild_nested_from_stages` — it folds the prefix using each stage's `$first_arg_name` to restore named-arg shape, but operates on the typed node shapes used by render-time rather than raw AST. Either share PLAN-02's helper with a typed-vs-raw flag, or duplicate the small folding loop in `R/paintr-render.R` — implementer's choice as long as the unit tests for both pass.

The helper `first_index_where_subtree_contains_any_placeholder` walks a single stage's subtree (including its `$args` / `$children` / `$stages` fields) and returns TRUE iff any node satisfies `is_ptr_placeholder(node)` — the existing union helper at `R/paintr-nodes.R:103-105` that checks for any of the three concrete placeholder classes (`ptr_ph_value`, `ptr_ph_data_consumer`, `ptr_ph_data_source`; there is no `ptr_ph` base-class string — placeholders are flat S3, not a class hierarchy). The pipeline-level function maps this over `stages` left-to-right and returns the first TRUE index, or `NA_integer_` if none.

3. Consumer-uniformity test net (tests/testthat/test-adr12-consumer-uniformity.R)

A new test file that, for each downstream consumer of `data_arg`, asserts the consumer produces structurally equivalent output across the three surface forms (`|>` string-input, `%>%`, nested-call) for placeholder-containing chains. The four consumers covered:

  • eval (`R/paintr-eval.R :: layer_to_eval_expr / pipeline_to_eval_expr`) — assert `identical(eval_expr_native, eval_expr_magrittr) && identical(eval_expr_magrittr, eval_expr_nested)` for a fixed placeholder snapshot.
  • prune (`R/paintr-prune.R :: prune_walk.ptr_pipeline / prune_walk.ptr_call`) — assert the pruned tree is `ptr_tree_structural_equal` across the three surface forms.
  • safety (`R/paintr-safety.R` deny-list AST walker) — assert the same allow/deny verdict across the three surface forms for both a safe formula and a deny-listed formula.
  • picker upstream resolution (`R/paintr-server.R :: runtime_upstream_data AFTER deletion of the fast-path) — assert each consumer id's resolved upstream data is `identical()` across the three surface forms for a fixed uploaded data snapshot.

Concrete formulas reused from PLAN-02's BDD scenarios — specifically the three-way set:

f_native   = "ppUpload |> filter(ppVar > ppNum) |> mutate(new_var = ppVar + 1) |> ggplot(aes(ppVar, ppVar)) + geom_point()"
f_magrittr = "ppUpload %>% filter(ppVar > ppNum) %>% mutate(new_var = ppVar + 1) %>% ggplot(aes(ppVar, ppVar)) + geom_point()"
f_nested   = "ggplot(mutate(filter(ppUpload, ppVar > ppNum), new_var = ppVar + 1), aes(ppVar, ppVar)) + geom_point()"

The test net runs in-process (no shinytest2 required) and is gated only on `NOT_CRAN` if it touches eval; otherwise no gates. Each consumer's assertion is one `test_that` block; the file as a whole adds approximately four tests.
