PLAN-01 — ptr_classify_calls no-op pass

Source: ADR 0012 (§2 "Classification slot", §4 S1) · Depends on: none · Parallel group: G1 · Status: draft

Context

ADR 0012 §2 locks the classification slot: "New pass ptr_classify_calls inserted between ptr_classify_data and ptr_shared_bind in ptr_translate (R/paintr-translate.R:54-64). Tree-rewriting, not pure annotation."

This plan only carves out the slot. The pass exists as a true no-op (identity function on the tree) — no lift, no role tagging. The next plan fills in the body. Splitting these reduces the blast radius of S2's logic change: if S2 regresses, S1 stays merged and S2 can be reverted in isolation.

Success Criteria

  • A new exported-internal function ptr_classify_calls(root) in R/paintr-translate.R that returns its root argument unchanged, asserting is_ptr_root(root).
  • ptr_translate's annotation block (the if (annotate) { ... } block near the end of ptr_translate in R/paintr-translate.R — locate by the ptr_assign_ids / ptr_classify_data / ptr_shared_bind sequence) calls ptr_classify_calls(root) immediately after ptr_classify_data(root) and before ptr_shared_bind(root).
  • Authoritative gate (see Definition of Done) stays at the pre-plan baseline counts plus exactly the new tests this plan adds. Specifically: FAIL 0 / ERROR 0 / SKIP 0 / PASS (baseline + N_new).
  • tests/testthat/test-classify-calls-noop.R exists, asserts: (a) ptr_classify_calls(root) is identity on a translated tree; (b) ptr_translate(formula) output for a representative %>% formula and a representative |> formula is bit-identical to ptr_translate(formula) with the call commented out (capture pre/post-baseline trees from this commit via identical()).

Constraints

  • Files this plan owns: R/paintr-translate.R (add the new function + one call site only), tests/testthat/test-classify-calls-noop.R (new file).
  • Files this plan must NOT touch: R/paintr-prune.R, R/paintr-server.R, R/paintr-app.R, R/paintr-safe.R, R/paintr-resolve.R, any vignette, any fixture under tests/testthat/fixtures/vignette-apps/, NAMESPACE, man/.
  • The new function must be unexported (internal). No @export tag. No roxygen entry beyond a one-line internal comment.
  • No scope creep into the lift transform, terminal-grounding, or any tree rewriting. Identity only.
  • Honor project conventions: ptr_ prefix only on public API; this is an internal helper named ptr_classify_calls for consistency with the existing classify-pass naming (ptr_classify_data, ptr_assign_ids, etc.). snake_case, 2-space indent, no stop()/warning(); use rlang::abort() and assertthat::assert_that().

BDD scenarios

Feature: Reserve a classification-pass slot in ptr_translate without changing tree shape

Scenario: Identity on a translated tree
  Given a tree `root` produced by ptr_translate("ggplot(penguins, aes(bill_length_mm)) + geom_point()")
  When  I call ggpaintr:::ptr_classify_calls(root)
  Then  the returned value is identical (via base::identical()) to `root`

Scenario: Translate output unchanged for %>% input
  Given the formula string "penguins %>% filter(bill_length_mm > 40) %>% ggplot(aes(bill_length_mm)) + geom_point()"
  When  I call ptr_translate(formula)
  Then  the returned ptr_root tree's $layers slot matches a saved baseline tree captured before the slot was added
  And   no new classes are introduced on any node

Scenario: Translate output unchanged for |> input
  Given the formula string "penguins |> filter(bill_length_mm > 40) |> ggplot(aes(bill_length_mm)) + geom_point()"
  When  I call ptr_translate(formula)
  Then  the returned ptr_root tree's $layers slot matches a saved baseline tree captured before the slot was added

Scenario: Rejects non-ptr_root input
  Given a value that is not a ptr_root (e.g., list())
  When  I call ggpaintr:::ptr_classify_calls(value)
  Then  the call errors with class "rlang_error"

Proxy traps — do not be fooled (copied verbatim from CLAUDE.md):

  • A raw Rscript -e 'testthat::test_dir()/test_file()' with no NOT_CRAN → skip_on_cran() fires → every shinytest2 browser test silently SKIPs. "green, SKIP n" off that is NOT the gate.
  • devtools::check(... --as-cran) does NOT make skip_on_cran() fire (devtools runs the check subprocess with NOT_CRAN=true). The browser fixtures instead cleanly SKIP via boot_vignette_app()'s source-root guard (no DESCRIPTION in the .Rcheck sandbox). If a check run shows the e2e file ERRORing (R CMD check found ERRORs, pkgload "DESCRIPTION not found"), that guard is missing/broken — a harness defect, not a product failure, and never the gate.
  • Only NOT_CRAN=true / devtools::test() runs the browser tests (source tree present → 0 SKIP). That alone is the gate. Never claim the suite green, merge a branch, mark a step done, or clear context off the skipping/erroring harness.

This plan adds approximately 4 new tests in tests/testthat/test-classify-calls-noop.R. The gate must show PASS rise by ≥ 4 with FAIL/ERROR still 0 and SKIP unchanged from the pre-plan baseline.
