PLAN-07 — CONTEXT.md amendment + vignette updates

Source: ADR 0025 Scope cuts (vignettes + CONTEXT.md owed) · Depends on: 01–06 · Parallel group: G4 · Status: draft

Context

ADR 0025's Scope cuts section explicitly defers three documentation updates to the implementation PR:

  • Vignette updates — vignettes/ggpaintr-customization.Rmd:297 (the table row for node$companion_id) and vignettes/ggpaintr-customization.Rmd:502 (the companion_id_fn = function(id) ... prose) reference the retired surface. Rewrite to describe shortcut = TRUE/FALSE + node$shortcut_id.
  • NEWS.md entry — a single bullet (or a tight set under a version heading) describing the hard rename + the new auto-name / mutex / prologue / vacate / debounce / spec-roundtrip behaviours.
  • CONTEXT.md amendment — lines 48 and 132 currently say "exactly one [shared panel] per page". The source at R/paintr-shared-coordinator.R:220–234, 292 already accepts id and namespaces every shared id; the new auto-name design (Plan 02) depends on multi-panel support being part of the documented surface.

This plan is documentation-only. No R/ source code is touched. Plans 01–06 must be merged first (the vignette/NEWS text describes their final behaviour).

Governing principle

Documentation that contradicts the source code is a liability — it sends consumers down dead paths and undermines the audit chain. The author-facing description of the registry knob (shortcut vs. retired companion_id_fn), the per-page count of shared panels, and the upload binding-name behaviour must each match exactly one place: the code. ADR 0025's scope cuts deferred these doc updates so the implementation PR could land first; this plan closes the loop. Naming the WHY: every future /grill-with-docs session and every external user reading the vignette assumes the words on the page reflect the package's actual surface — when they do not, the project's documentation contract collapses silently.

Success Criteria

  • vignettes/ggpaintr-customization.Rmd contains no occurrence of the literal strings companion_id or companion_id_fn. The table row at the previous line ~297 reads node$shortcut_id (with prose describing the new opt-in surface). The prose section at the previous line ~502 describes shortcut = TRUE + the author's responsibility to render shiny::textInput(node$shortcut_id, ...) in build_ui, with a one-paragraph note on the default-FALSE-author-emit policy (Example 6's silent-unreachability trap). Verified by grep -nE 'companion_id' vignettes/ returning zero matches.
  • vignettes/ggpaintr-customization.Rmd documents the per-source auto-name + mutex + prologue behaviour in one new subsection (≤ 150 words) covering: when the textbox is empty an upload binds under node$auto_name; the textbox auto-clears on file pick (and vice versa); the code panel emits a one-line <auto-name> <- read.<ext>(...) prologue. Verified by reviewer reading the rendered devtools::build_vignettes() output (manual; no automated test for vignette prose content).
  • CONTEXT.md no longer says "exactly one [shared panel] per page" at line 48 or any other line carrying the same claim (re-grep to locate; the ADR cites lines 48 and 132 but current source has the literal only at 48 — locate the second site by content, not by line number). Replacement wording describes that ptr_shared(..., id = ...) namespaces every shared id; multiple coordinators on one page are supported. Verified by grep -nE 'exactly one.*per page' CONTEXT.md returning zero matches.
  • NEWS.md has a new entry (top of the Unreleased section, or new version heading per project convention) summarising ADR 0025's behaviours: hard rename companion_id_fn → shortcut + node$companion_id → node$shortcut_id; auto-name on upload; UI mutex; prologue; vacate-on-empty; 400ms debounce; spec round-trip via bound names. Single tight bullet group or paragraph (≤ 100 words). Verified by grep -nE 'shortcut|auto-name|mutex|prologue' NEWS.md matching the new section.
  • The vignette still builds cleanly: devtools::build_vignettes() finishes without error and the produced HTML renders the new subsection. Verified by running the build locally.
  • Full authoritative gate (Definition of Done) shows FAIL 0 / WARN 0 / SKIP 0 — no regressions from doc edits.

Constraints

(Line numbers may shift after Plans 01–06 merge; re-grep companion_id in the vignette at the start of this plan to find the actual touchsites.)
    • CONTEXT.md — lines 48 and 132 ("exactly one per page" claim); any other line found by grep -nE 'exactly one.*per page|one shared panel' CONTEXT.md.
    • NEWS.md — one new entry. If Plan 01 added a partial NEWS entry (the rename note), this plan replaces or extends it with the full ADR-0025 summary.
  • Files this plan must NOT touch:
    • Any file under R/, tests/, man/, or dev/ (audits and notes excluded). This plan is documentation-only.
    • The ADR dev/adr/0025-…html itself. It is audit-stamped (hash 5f437d0f279f) and any content edit invalidates the stamp.
    • Other vignettes (ggpaintr-introduction.Rmd, ggpaintr-use-cases.Rmd, etc.) — surface drift in the post-impl audit; do not rewrite preemptively here unless the audit explicitly flags a contradiction with the new surface.
    • Generated docs (pkgdown docs/ output) — those regenerate via source("dev/build-pkgdown.R"); build_pkgdown_clean() as an integration step.
  • Prose follows the project's .Rmd rule (no hard-wrap; one paragraph = one line; per global CLAUDE.md). Code blocks use the project's existing R-fenced style.
  • No new examples that exercise unimplemented features. The vignette text describes only what Plans 01–06 actually ship.
  • SCOPE NARROWING FORBIDDEN. Do not selectively rewrite some companion_id references and leave others "for a follow-up" — every occurrence must move in this plan.
  • Conventions: tidyverse style; do not introduce new headings unless mandated by content; keep diffs minimal.
