PLAN-06 — Spec round-trip fallback to bound_names

Source: ADR 0025 §6 + Worked example #3 · Depends on: 01, 02 · Parallel group: G3 · Status: draft

Context

ADR 0025 §6 specifies that ptr_runtime_input_spec()'s source-placeholder dump path (the snapshot writer that produces spec[[shortcut_id]]) falls back to state$bound_names[[key]]() when the textbox is empty AND a bound name is available:

# R/paintr-input-spec.R, source-placeholder branch with `shortcut = TRUE`
val <- input[[shortcut_id]] %||% ""
if (!nzchar(val)) {
  bound <- state$bound_names[[key]]()
  if (!is.null(bound)) val <- bound
}
spec[[shortcut_id]] <- val

Reproducing a session = the consumer runs <auto-name> <- read.csv("...") (the prologue from Plan 04) in their R session, then ptr_app(formula, spec = list(...)). The spec seeds the textbox at boot with the auto-name; resolution finds <auto-name> via env shortcut. Equivalent to running ppUpload(<auto-name>) as the formula — the spec is the bridge between live upload state and reproducible env-resident state. The actual snapshot-write site in this codebase is R/paintr-server.R:1521–1525 (the per-row snapshot loop) plus the panel-shared snap update at R/paintr-server.R:2371–2382.

Governing principle

The spec is the round-trip contract — a value dumped by ptr_runtime_input_spec() must, when supplied as spec = … to a fresh ptr_app() with the same formula and caller-env bindings, reproduce the rendered plot. Group 3 (the deployed-app consumer who uploaded a file) and Group 1 (the formula author who reproduces it from disk) live on opposite ends of this bridge; the dumped spec[[shortcut_id]] value is the only handle between them. If the dump records "" when the user has actually uploaded a file, the bridge is broken. Naming the WHY: ADR 0010's ppUpload(name) form already establishes that "shortcut textbox seed = env name to look up"; ADR 0025 §6 extends that contract to the spec round-trip so the same name flows through dump→seed→resolve.

Success Criteria

  • After a user uploads a file (no text in shortcut textbox), state$spec() returns a list containing the entry <ns(shortcut_id)> = <node$auto_name> (a single character string equal to the auto-name) — NOT "", NOT absent. Verified by test-spec-roundtrip-bound-names-fallback.R using a shinytest2 case that uploads, clicks Update, and inspects app$get_value(output = "ptr_code") in "spec" mode (the rendered spec text contains the auto-name string).
  • When the textbox has a user-typed value (e.g. "mtcars") and state$bound_names[[key]]() is also non-NULL, the dumped spec value is "mtcars" (textbox wins over bound-name fallback). Verified by the same test.
  • When the textbox is empty AND state$bound_names[[key]]() is NULL (no upload, no bind), the dumped spec value is "" or the entry is dropped via the sparse-diff (matches today's behaviour). Verified by test-spec-roundtrip-empty.R.
  • ADR Example 3 round-trip: a session's spec dump list(<ns(shortcut_id)> = "ggplot_1_0_ppUpload_NA"), when supplied to a fresh ptr_app(formula, spec = ...) with ggplot_1_0_ppUpload_NA <- read.csv("...") in the caller env, boots with the shortcut textbox seeded to "ggplot_1_0_ppUpload_NA" AND the env-shortcut resolves the caller-env frame AND the plot renders. Verified by test-spec-roundtrip-end-to-end.R using two shinytest2 boots (boot-1 uploads a file and exports the spec via app$get_value(output = "ptr_code"); boot-2 supplies that spec + the env binding and asserts the plot renders).
  • For panel-shared sources, the panel-side snap update at R/paintr-server.R:2371–2382 also applies the bound-name fallback (so multi-coordinator apps round-trip too). Verified by test-spec-roundtrip-shared.R.
  • Full authoritative gate (Definition of Done) shows FAIL 0 / WARN 0 / SKIP 0.

Constraints

  • Files this plan owns:
    • R/paintr-server.R — modify the snapshot-write loop at lines ~1518–1525 (inside ptr_setup_runtime()'s observe block). After the raw read snapshot[[raw_id]] <- input[[ns(raw_id)]], for rows whose role is "source_companion": if the read value is NULL or empty-string AND the corresponding state$bound_names[[key]]() is non-NULL, overwrite snapshot[[raw_id]] with the bound name. The mapping from raw_id (the companion id) to key (the source's node$id) goes through the existing spec$source_id linkage in state$input_spec.
    • R/paintr-server.R — apply the same fallback to the panel-shared snap update at lines ~2366–2387 (the snap[[psn$shortcut_id]] <- bname path — post-Plan-01 rename; the current source still reads psn$companion_id) so coordinator-eval-env-bound auto-names also flow into the spec.
    • New tests:
      • tests/testthat/test-spec-roundtrip-bound-names-fallback.R — upload → empty textbox → spec dump contains the auto-name.
      • tests/testthat/test-spec-roundtrip-empty.R — no upload + empty textbox → "" or dropped (matches today).
      • tests/testthat/test-spec-roundtrip-end-to-end.R — ADR Example 3 two-boot round-trip.
      • tests/testthat/test-spec-roundtrip-shared.R — coordinator-bound auto-name flows through the panel-shared snap path.
  • Files this plan must NOT touch:
    • Auto-name derivation (Plan 02), UI mutex (Plan 03), prologue (Plan 04), vacate-on-empty + debounce (Plan 05), docs (Plan 07).
    • R/paintr-input-spec.R's ptr_runtime_input_spec() tree-walker — that function returns the input schema (a data.frame of which inputs exist); the value-dump fallback belongs in the snapshot loop, NOT in the schema walker. ADR §6's pseudocode is illustrative; the actual write site is R/paintr-server.R:1521–1525 / 2371–2382. (The drift audit will flag this.)
    • The sparse-diff helper (ptr_spec_from_snapshot at R/paintr-server.R:391) — its contract is unchanged; this plan only changes what value lands in snapshot for the companion rows before the diff runs.
    • Generated man/*.Rd — integration step.
  • The fallback reads state$bound_names[[key]]() reactively in production (inside the observe's isolate block that already wraps the snapshot loop). Plan 05's A1 makes bound_names[[key]]() return NULL after a vacate — the fallback then leaves snapshot[[raw_id]] as "", which the sparse-diff drops; matches the "empty after vacate" criterion above.
  • The key for a source row is its node$id (the source's own id), distinct from the raw_id (the companion id == node$shortcut_id). Use the existing spec$source_id column to map companion-row → source-id.
  • SCOPE NARROWING FORBIDDEN. Do not gate the fallback on "only when the value didn't come from a spec= seed" — both paths must fall back, by design.
  • Conventions: snake_case, 2-space indent, %||% guard for the NULL companion row case.

BDD scenarios

Feature: Spec round-trip falls back to bound_names (ADR 0025 §6)

Scenario: upload with empty textbox dumps the auto-name into spec
  Given a `ppUpload() |> ggplot(aes(x = ppVar('mpg'))) + geom_point()` app
   When the user uploads `mtcars.csv` and clicks Update (textbox stays "")
   Then `state$spec()[[ns(node$shortcut_id)]]` equals node$auto_name (a non-empty character string)

Scenario: typed textbox wins over bound-name fallback
  Given the user typed "my_data" into the shortcut textbox
    And state$bound_names[[node$id]]() is non-NULL (a prior upload's auto-name)
   When state$spec() reactive fires
   Then `state$spec()[[ns(node$shortcut_id)]]` equals "my_data" (NOT the prior bound-name)

Scenario: no upload + empty textbox produces no spec entry
  Given the user has not interacted with the source surface
   When state$spec() reactive fires
   Then the spec list either has no entry for node$shortcut_id OR the entry is "" (and is dropped by the sparse diff)

Scenario: ADR worked example #3 — full round-trip via auto-name
  Given a boot-1 session ends with state$bound_names[[node$id]] = "ggplot_1_0_ppUpload_NA"
    And ptr_runtime_input_spec() (via state$spec()) was dumped to obtain `spec_round_trip`
   When a boot-2 session calls `ptr_app(formula, spec = spec_round_trip)` with `ggplot_1_0_ppUpload_NA <- read.csv("penguins.csv")` in caller env
   Then the shortcut textbox at boot-2 reads "ggplot_1_0_ppUpload_NA"
    And the env-shortcut path resolves the caller-env frame
    And the rendered plot at boot-2 is identical to boot-1's rendered plot (same data, same aesthetics)

Scenario: panel-shared source round-trip
  Given a `ptr_shared(list(f1, f2), id = "main")` coordinator with `ppUpload(shared = 'ds')`
    And the user uploads to the panel and the textbox stays ""
   When state$spec() fires
   Then the dumped value at the namespaced shortcut id equals "main_ds" (NOT "")
