PLAN-02 ‚Äî Auto-name derivation for upload binding

Source: ADR 0025 ¬ß3 + ¬ß5 + ¬ß8 (auto-name table, eval_env contract, ADR 0010 reinterpretation) ¬∑ Depends on: 01 ¬∑ Parallel group: G2 ¬∑ Status: draft

Context

ADR 0025 ¬ß3 specifies a deterministic auto-name for upload-bound sources when the shortcut textbox is empty:

  • ppUpload(shared='ds_a') with obj$id = NULL ‚Üí symbol ds_a (coordinator eval_env).
  • ppUpload(shared='ds_a') with obj$id = "main" ‚Üí symbol main_ds_a.
  • ppUpload() non-shared, no explicit name ‚Üí symbol node$id (e.g. ggplot_1_0_ppUpload_NA).
  • ppUpload(penguins) (ADR 0010 form) ‚Üí symbol penguins (the node$default verbatim).

ADR ¬ß5 says the substitute walker reads ctx$snapshot[[node$shortcut_id]] first; on empty, falls back to node$auto_name (a new field stamped at translate time alongside node$id). Per-instance eval_envs and per-coordinator eval_envs (already isolated, ADR 0023) make every auto-name unique within its scope. No counter, no reserved-namespace beyond Plan 01's shared-key validator.

Governing principle

The auto-name is the slot's binding-name contract ‚Äî one deterministic symbol per source node, owned by ggpaintr, derived from existing id infrastructure. Every reader of the slot's binding (substitute walker, code emission, eval_env writer, spec dumper) must consult the same single source of truth (node$auto_name), not re-derive the rule locally. Naming the WHY: F3 (eval-env pollution) is what you get when uploads write under whatever name happens to be in scope at the call site; one shared deterministic name eliminates the class.

Success Criteria

  • After ptr_translate_formula() on ppUpload() |> ggplot(aes(x = ppVar('mpg'))) + geom_point(), the resulting source node satisfies identical(node$auto_name, node$id) (a non-shared upload's auto-name is its node$id). Verified by test-auto-name-derivation.R.
  • After translation of ppUpload(penguins) |> ggplot(...), node$auto_name == "penguins" (the node$default verbatim). Verified by the same file.
  • In a ptr_shared(list(f1, f2), id = "main") coordinator where both formulas contain ppUpload(shared = 'ds'), after ptr_setup_panel_sources() runs, the panel's first-occurrence source node has node$auto_name == "main_ds" (the canonical with obj$id prefix). Verified by test-auto-name-shared-canonical.R.
  • In the same coordinator with id = NULL (the default), the auto-name is "ds" (no prefix). Verified by the same file.
  • When resolve_upload_source runs with file_info non-NULL and companion_slot$value empty/NULL, bind_source_value is called with name = node$auto_name (NOT NULL, NOT a re-derived value). Verified by test-upload-binds-under-auto-name.R using testthat::local_mocked_bindings(.package = "ggpaintr", bind_source_value = function(state, key, name, df, slot) { ... }) to capture the name argument.
  • When resolve_upload_source runs with file_info non-NULL and companion_slot$value = "my_name" (a valid R name), bind_source_value is called with name = "my_name" (textbox wins over auto-name when both available). Verified by the same file.
  • The substitute walker, given a source node with node$shortcut_id set and ctx$snapshot[[node$shortcut_id]] == "", returns an expression whose symbol equals as.name(node$auto_name). Verified by test-substitute-auto-name-fallback.R.
  • Full authoritative gate (Definition of Done) shows FAIL 0 / WARN 0 / SKIP 0 with PASS rising by at least the new-case count.

Constraints

  • Files this plan owns:
    • R/paintr-ids.R ‚Äî inside assign_id_to_placeholder() (or the equivalent translate-time stamp helper for source nodes), set node$auto_name <- node$default %||% node$id for non-shared sources, and leave node$auto_name = NULL for shared sources (the runtime stamp in ptr_setup_panel_sources overwrites this for shared sources). The node$default path handles the ppUpload(penguins) case verbatim.
    • R/paintr-shared-ui.R ‚Äî inside ptr_setup_panel_sources() (the stamp block currently at lines ~511-516), after the existing node$id <- canonical assignment, add node$auto_name <- if (!is.null(obj$id)) paste0(obj$id, "_", key) else key. The key here is the shared key (e.g. "ds"); obj$id is the coordinator object's id slot. Cross-site check: R/paintr-shared-ui.R:152 also calls n$id <- canonical_shared_id(k) in a non-ptr_setup_panel_sources helper; verify whether that site is on the panel-first-occurrence binding path and either stamp n$auto_name there too or record a one-line justification for why the later ptr_setup_panel_sources overwrite supersedes it.
    • R/paintr-substitute.R ‚Äî extend the source-node substitute walk (lines ~179‚Äì213) so that when ctx$snapshot[[node$shortcut_id]] is empty (NULL or empty string), the walker falls through to as.name(node$auto_name) when node$auto_name is non-NULL. BDD scenarios

Feature: Auto-name derivation for upload binding (ADR 0025 ¬ß3, ¬ß5, ¬ß8)

Scenario: non-shared upload ‚Äî translate-time auto-name equals node$id
  Given a formula `ppUpload() |> ggplot(aes(x = ppVar('mpg'))) + geom_point()`
   When ptr_translate_formula() runs
   Then the source node has node$auto_name identical to node$id
    And node$auto_name has the literal suffix "_ppUpload_NA"

Scenario: ppUpload(penguins) ‚Äî translate-time auto-name equals node$default verbatim

   When ptr_translate_formula() runs
   Then the source node's node$default is "penguins"
    And node$auto_name is identical to "penguins"

Scenario: ADR worked example #2 (part A) ‚Äî shared upload with obj$id prefix
  Given a coordinator `ptr_shared(list(f1, f2), id = "main")` where both formulas reference `ppUpload(shared = 'ds')`
   When ptr_setup_panel_sources() stamps the panel-first-occurrence node
   Then the stamped node has node$auto_name identical to "main_ds"

Scenario: ADR worked example #2 (part B) ‚Äî shared upload without obj$id
  Given a coordinator `ptr_shared(list(f1, f2))` (id defaulted to NULL) where both formulas reference `ppUpload(shared = 'ds')`
   When ptr_setup_panel_sources() stamps the panel-first-occurrence node
   Then the stamped node has node$auto_name identical to "ds"

Scenario: upload binds under auto-name when shortcut textbox is empty
  Given a mocked bind_source_value (via testthat::local_mocked_bindings)
    And a source node whose node$auto_name is "ggplot_1_0_ppUpload_NA"
    And a file_info argument that is non-NULL
    And companion_slot$value is NULL (or "")
   When resolve_upload_source() runs
   Then bind_source_value is invoked exactly once
    And the captured `name` argument is identical to "ggplot_1_0_ppUpload_NA"

Scenario: shortcut textbox wins over auto-name when both available
  Given a mocked bind_source_value
    And a source node whose node$auto_name is "ggplot_1_0_ppUpload_NA"
    And companion_slot$value is "my_name" (a valid R name)
   When resolve_upload_source() runs
   Then the captured `name` argument is identical to "my_name"

Scenario: substitute walker falls back to node$auto_name on empty snapshot
  Given a source node with node$shortcut_id = "x_shortcut" and node$auto_name = "auto_x"
    And ctx$snapshot[["x_shortcut"]] is "" (empty string)
   When substitute_walk dispatches on the source node
   Then the resulting expression's substituted symbol is identical to as.name("auto_x")
