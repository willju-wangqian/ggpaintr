PLAN-05 — Vacate-on-empty (A1) + shortcut-textbox debounce (A2)

Source: ADR 0025 §7 · Depends on: 02, 03, 04 · Parallel group: G3 · Status: draft

Context

ADR 0025 §7 carries two coupled fixes for residual staleness/flicker after the mutex is in place:

  • A1 — Vacate-on-empty. When resolve_upload_source falls through to slot(NULL) (file_info NULL AND try_bind_source_default_resolved returned FALSE), actively vacate the prior eval_env binding: 
    prior_name <- shiny::isolate(state$bound_names[[key]]())
if (!is.null(prior_name) &&
    exists(prior_name, envir = state$eval_env, inherits = FALSE)) {
  rm(list = prior_name, envir = state$eval_env)
}
state$bound_names[[key]](NULL)
slot(NULL)
    Without A1 the textbox-deletion sequence (upload → type → delete text) leaves state$bound_names[[key]] stale, consumer pickers stay populated with stale columns, and ptr_runtime_input_spec() dumps the stale name.
  • A2 — Debounce. Wrap the shortcut-textbox input read in a 400ms shiny::debounce so intermediate keystrokes don't trigger env manipulation: 
    shortcut_val <- shiny::reactive(input[[shortcut_id]]) %>% shiny::debounce(400)
    Eliminates the "Object 'm' not found / Object 'mt' not found / ..." error-pane flicker as the user types a name; reduces rm()/assign() churn from one-per-keystroke to one-per-decision.

Governing principle

System-state surfaces (bound_names, eval_env, active_uploads) must reflect current source-of-truth, not historic. Two failure modes share this root: (a) reactive observers that re-read input mid-keystroke act on transient states the user never intended; (b) bindings written into eval_env that survive the user clearing their source surface produce stale downstream reads (consumer pickers, code panel, spec dump). The fix on both sides is the same — collapse the observer's "what is the current state" sample to exactly one decision per user action. Naming the WHY: the plot widget's last-good preservation behaviour is intentional (it covers the resolve-error case); but the slot's system-state surfaces are not part of that contract and must reflect now, not last successful resolve.

Success Criteria

  • After uploading a file then clearing the shortcut textbox to empty, state$bound_names[[key]]() returns NULL, and the prior eval_env binding (the auto-name symbol) is no longer present in state$eval_env. Verified by test-vacate-on-empty.R using a shinytest2 case + app$get_value(input = ...) on a debug output that exposes shiny::isolate(state$bound_names[[key]]()) and exists(node$auto_name, envir = state$eval_env).
  • After the same vacate, ptr_runtime_input_spec() dumps an empty string ("") for the shortcut id (NOT the stale auto-name from before the clear). Verified by the same test.
  • After the vacate, state$active_uploads[[key]] (the slot Plan 04 introduced) is also vacated (NULL or absent), so the code panel emits no prologue line for that key. Verified by reading the code-panel text and asserting no prologue line.
  • Typing six characters into the shortcut textbox within a 400ms window triggers exactly one resolve cycle (not six). Verified by test-debounce-keystroke-burst.R using a shinytest2 case that instruments resolve_upload_source via assignInNamespace (per project memory mock-internal-calls + shinytest2-appdir-pkgload), counts invocations into a child-process counter logged via message(), and asserts the count equals 1 after a burst with wait_ = FALSE on each set_inputs call.
  • During the keystroke burst, the error pane shows zero "Object '<partial-name>' not found" messages (no flicker). Verified by the same test asserting app$get_html("#<error-pane-id>") never contains the substring "not found" mid-burst.
  • A1's rm() is scoped: inherits = FALSE in the exists() probe and envir = state$eval_env in the rm(), so caller-env bindings reached via parent chain are untouched. Verified by test-vacate-on-empty-scope.R setting up a parent.frame() binding for mtcars, vacating, and asserting the caller-env mtcars is intact.
  • Full authoritative gate (Definition of Done) shows FAIL 0 / WARN 0 / SKIP 0.

Constraints

  • Files this plan owns:
    • R/paintr-server.R — the A1 block inserted into resolve_upload_source()'s no-file branch (currently lines ~1046–1059 of the post-rename file) AND inside try_bind_source_default_resolved()'s no-bind fall-through. The vacate logic also clears state$active_uploads[[key]] (the slot Plan 04 introduced).
    • R/paintr-server.R — the A2 debounce: wrap the shortcut-input read site (inside the per-source observer setup closure that Plan 03's mutex sits in) with shiny::debounce(400). The debounce window is a literal 400L in this plan; a configurable option (options(ggpaintr.shortcut_debounce = …)) is deferred per ADR risk-mitigation note (out of scope for this plan).
    • New tests:
      • tests/testthat/test-vacate-on-empty.R — full clear sequence + spec dump assertion.
      • tests/testthat/test-vacate-on-empty-scope.R — caller-env shadow is preserved.
      • tests/testthat/test-debounce-keystroke-burst.R — single resolve invocation per burst.
  • Files this plan must NOT touch:
    • Auto-name derivation (Plan 02), UI mutex observers (Plan 03 — though both plans touch the same observer-setup closure, the regions are disjoint: Plan 03 owns the typed/file event observers, Plan 05 owns the input-read debounce wrap and the no-file-branch vacate body), prologue (Plan 04 — Plan 05 only clears the slot Plan 04 reads), spec round-trip (Plan 06), docs (Plan 07).
    • The plot widget's last-good preservation behaviour in state$last_ok_runtime() — A1 fixes system-state surfaces, not the plot's carry-over.
    • Custom-source-author paths — A1 acts on state$bound_names which is keyed by ggpaintr-managed keys; third-party state slots are not vacated.
    • state$eval_env's parent chain — A1 uses inherits = FALSE deliberately.
    • Generated man/*.Rd — integration step.
  • exists(prior_name, envir = state$eval_env, inherits = FALSE) MUST be guarded by is.character(prior_name) && length(prior_name) == 1L && nzchar(prior_name) && make.names(prior_name) == prior_name (or the equivalent — exists() errors on non-syntactic names with inherits = FALSE on some R versions). Use the same name-validity predicate Plan 02 uses inside resolve_upload_source's binding_name derivation; do not duplicate the regex.
  • The 400ms debounce window is a single literal in code, not three (one per use site). Centralize the value at the wrap site so a future configurable-option change touches one location.
  • SCOPE NARROWING FORBIDDEN. Do not condition the vacate on "only when the textbox change came from the user (not from Plan 03's mutex auto-clear)" — both paths must vacate, by design. Plan 03's auto-clear writes "" into the textbox; that change must trigger the vacate exactly as a user-deletion would.
  • Conventions: snake_case, 2-space indent, shiny::debounce over hand-rolled timers, shiny::isolate() around the bound_names read inside the no-file branch (the read happens at observer-fire time and must not subscribe).
