PLAN-01 — Rename companion_id_fn → shortcut + node$companion_id → node$shortcut_id

Source: ADR 0025 §1 + §8 (rename) and Example 4 / Example 6 · Depends on: none · Parallel group: G1 · Status: draft

Context

ADR 0025 §1 retires the dual-role companion surface and renames it to the single-role shortcut surface. The registry parameter on ptr_define_placeholder_source() changes from companion_id_fn = function(id) -> string (NULL default) to shortcut = FALSE (boolean default). The node field node$companion_id becomes node$shortcut_id; the id suffix "_name" becomes "_shortcut". The built-in ppUpload registers with shortcut = TRUE to keep its textbox. A translate-time validator rejects shared = "shortcut" with a clear error (ADR Example 4). Hard rename, no lifecycle::deprecate_warn().

Quoting the ADR §1 binding sentence: "The '_shortcut' suffix replaces today's '_name'. Naming is consistent across the field, the suffix, and the public arg."

Governing principle

One surface, one role. The companion textbox carries only the env-shortcut role (a typed name resolves a caller-env frame); the upload-name-override role is retired. Every artefact named after the old dual role — argument name, node slot, id suffix, hook variable, find-helper — must move to the single-role name in lock-step, so no reader of any single file is led to believe the old dual role still exists. Naming the WHY: divergent naming is what let F1/F2/F3 (sticky-after-clear, code/plot mismatch, eval-env pollution) hide in ADR 0024's review; uniform naming makes the new contract auditable on sight.

Success Criteria

  • ptr_define_placeholder_source(keyword = "ppX", role = "source", shortcut = TRUE, build_ui = ..., resolve_data = ..., resolve_expr = ...) returns without error and the resulting registry entry carries shortcut = TRUE (a boolean), not companion_id_fn. Verified by devtools::test(filter = "registry") passing the new test_that case "ptr_define_placeholder_source accepts shortcut = TRUE".
  • Calling ptr_define_placeholder_source(..., companion_id_fn = function(id) paste0(id, "_x")) errors with a message naming the removed argument. Verified by an expect_error in test-registry-shortcut.R.
  • For a formula containing ppUpload(), after translation the source node$shortcut_id equals paste0(node$id, "_shortcut") (e.g. "ggplot_1_0_ppUpload_NA_shortcut"), node$companion_id is absent (is.null(node$companion_id) is TRUE), and no node field, helper, registry entry, or local variable in R/paintr-*.R contains the literal text companion_id_fn or companion_id. Verified by devtools::test(filter = "id-table") + a fresh grep -r 'companion_id' R/ returning zero matches.
  • For a formula ppUpload(shared = 'shortcut') |> ggplot(...), ptr_app() translation aborts with an error whose message contains the substring "Shared key 'shortcut' is reserved". Verified by test-shortcut-reserved-key.R.
  • A custom source registered with shortcut = TRUE whose build_ui omits a textInput(node$shortcut_id, ...) boots without error and renders only the widgets the author named (no auto-injected textbox). Verified by test-shortcut-author-trap.R asserting expect_no_error on translation + boot and the rendered tag list contains no input bound to node$shortcut_id.
  • Full authoritative gate (Definition of Done below) shows FAIL 0 / WARN 0 / SKIP 0 with PASS rising by at least the number of new test cases added in this plan, and zero new failures in pre-existing tests.
