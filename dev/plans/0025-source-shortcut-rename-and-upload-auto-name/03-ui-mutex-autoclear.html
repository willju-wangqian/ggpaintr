PLAN-03 — UI mutex with auto-clear (Q3-B)

Source: ADR 0025 §2 + Worked example #5 · Depends on: 01 · Parallel group: G2 · Status: draft

Context

ADR 0025 §2 establishes that the shortcut textbox and the fileInput are mutually exclusive: filling one auto-clears the other. There is no hard-greyed state. Boot-time seeding of the textbox via ppUpload(penguins) (ADR 0010 form) does NOT count as a first interaction — picking a file silently wipes the seed.

This replaces the existing ptr_upload_autoname() mechanism (R/paintr-upload.R:45; R/paintr-server.R:1298–1331), which derives a friendly textbox value from the picked filename. Under ADR 0025 the textbox is reserved for env-shortcut typing; the upload's binding-name path is the auto-name (Plan 02), not the filename slug. The old autoname machinery is removed wholesale.

Governing principle

The two affordances feed the same source slot, and the only way to make F2 (code/plot mismatch from concurrently active file + textbox) impossible by construction is to forbid the concurrently-active state. "Whichever was last touched is the live one" collapses the resolution-precedence matrix to a single rule with no arbitration. Naming the WHY: every alternative that preserves the both-active state requires runtime arbitration code that has to be re-verified on every change and is testable only by combinatoric scenario; mutex removes the class of bugs rather than guarding against it.

Success Criteria

  • Typing a value into the shortcut textbox triggers shiny::reset(srcId) on the sibling fileInput (the file pill disappears, input[[srcId]] returns to NULL). Verified by a shinytest2 case test-mutex-typed-textbox-resets-file.R that uploads a file, types a name, and asserts the file pill text is no longer present in app$get_html() (or that app$get_value(input = srcId) is NULL).
  • Picking a file triggers updateTextInput(session, compId, value = "") on the sibling shortcut textbox. Verified by a shinytest2 case test-mutex-file-pick-clears-textbox.R that types a name first, picks a file, and asserts app$get_value(input = compId) == "".
  • For a formula ppUpload(penguins): at boot, the shortcut textbox shows "penguins" (the seed) AND the fileInput is empty/live. Picking a file after boot silently clears the seeded textbox (the boot seed is not "user interaction"). Verified by test-mutex-boot-seed-not-first-interaction.R.
  • ptr_upload_autoname() and its supporting reactive (last_auto_set) are deleted from R/. grep -nE 'ptr_upload_autoname|last_auto_set|ptr_bind_source_autoname' R/ returns zero matches. The corresponding test cases in tests/testthat/test-upload.R are deleted (the test_that("ptr_upload_autoname derives a default only when the companion is blank", …) block).
  • Full authoritative gate (Definition of Done) shows FAIL 0 / WARN 0 / SKIP 0 with PASS rising by ≥ 3 shinytest2 cases minus the 6 expectations removed from test-upload.R (net PASS delta sign: depends on environment; the gate's primary signal is FAIL 0 / SKIP 0).

Constraints

  • Files this plan owns:
    • R/paintr-server.R — REPLACE the shiny::observeEvent(input[[src_id]], { ... ptr_upload_autoname ... }) block (currently lines ~1298–1331) with two mutex observers:
      1. observeEvent(input[[src_id]], { … updateTextInput(session, comp_id, value = "") …}, ignoreInit = TRUE) — clears the textbox on file pick.
      2. observeEvent(input[[comp_id]], { … if (nzchar(input[[comp_id]])) shiny::reset(src_id) … }, ignoreInit = TRUE) — resets fileInput on typing.
      Both wrapped in the existing per-source setup closure; both gated by ignoreInit = TRUE so the boot-seed write does not fire either observer; both inside the per-instance session scope so cross-source aliasing is impossible.
    • R/paintr-upload.R — DELETE ptr_upload_autoname() (lines 45–onwards). If the helper is exported in NAMESPACE/roxygen, remove the export tag too (integration step regenerates NAMESPACE).
    • R/paintr-server.R — DELETE the wrapper ptr_bind_source_autoname() (lines ~1297-1332 — the closure that contained the last_auto_set reactiveVal + the old observeEvent(input[[src_id]], { ... ptr_upload_autoname ... }) body). The two new mutex observers above replace its job; the wrapper's callers in ptr_setup_pipelines() (line ~1290) and bare-layer wiring update to call the new observer-pair directly (or a tightly-scoped replacement wrapper that returns nothing and only owns the mutex bodies).
    • tests/testthat/test-upload.R — DELETE the test_that("ptr_upload_autoname derives a default only when the companion is blank", { ... }) block (currently lines 374–385) and any helper-of-helper-only state still referencing the deleted function.
    • New tests (shinytest2):
      • tests/testthat/test-mutex-typed-textbox-resets-file.R
      • tests/testthat/test-mutex-file-pick-clears-textbox.R
      • tests/testthat/test-mutex-boot-seed-not-first-interaction.R
      • tests/testthat/test-shortcut-custom-source-env-bind.R — ADR worked example #1 end-to-end: register a custom ppDataset source with shortcut = TRUE (fixture app.R does the registration via ptr_define_placeholder_source), type a caller-env frame name into the shortcut textbox, assert the downstream ppVar picker populates and the plot renders.
      Each follows the project's app-dir + pkgload::load_all() pattern (see tests/testthat/fixtures/vignette-apps/<slug>/app.R; project memory shinytest2-appdir-pkgload). Each starts with skip_on_cran(); skip_if_not_installed("shinytest2"); skip_if_not_installed("chromote"). app$set_inputs uses wait_ = FALSE for placeholder writes per the project's testing rules.
  • Files this plan must NOT touch:
    • Auto-name derivation (Plan 02), prologue emission (Plan 04), resolve_upload_source no-file branch + debounce (Plan 05), spec round-trip (Plan 06), docs (Plan 07).
    • The seed_initial/default_name logic that determines the boot-seed value — that's part of the existing ppUpload(name) seeding path (ADR 0010) and is unchanged here. Only the observer's body changes; the seed write still goes through the existing path.
    • R/paintr-builtins.R ppUpload build_ui — Plan 01 already updates it to bind shiny::textInput(inputId = node$shortcut_id, ...).
    • Generated man/*.Rd — integration step.
  • ignoreInit = TRUE on both observers is load-bearing — it is what makes ADR §2's "boot-time exception" work without a separate "is this the boot" predicate.
  • The mutex observers fire on the per-instance session scope (the one inside the source's setup closure), NOT the top-level session. This matches the existing autoname observer's scope and keeps multi-instance / multi-coordinator apps isolated.
  • SCOPE NARROWING FORBIDDEN. Do not silently keep ptr_upload_autoname() "just in case"; ADR §2 retires the filename-derived-textbox-value behaviour outright.
  • Conventions: snake_case; shiny::reset() over hand-rolled DOM reset.
