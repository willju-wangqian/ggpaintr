Step 05 — ptr_module_ui rebuilt on the L3 API (pass #M)

Phase 5. Depends on: Steps 03 (combinators) + 04 (folded controls). Authority: ADR 0005 §5. Also removes ptr_controls_ui/ptr_outputs_ui.

Goal

Reimplement ptr_module_ui (R/paintr-app.R:570–608) as an internal composition of bare pieces + its own self-contained shell + combinators + (single-formula) the folded inline shared section. Delete ptr_controls_ui (:691–709) and ptr_outputs_ui (:736–745); rewire all callers.

Internal target shape

ptr_module_ui(id, formula, ui_text=NULL, checkbox_defaults=NULL,
              expr_check=TRUE, css=NULL) =
  own_shell(css,                                  # fluidPage > div.ptr-app + assets
    sidebarLayout(
      sidebarPanel( ptr_ui_controls(id, formula, ui_text, checkbox_defaults,
                                    expr_check, shared = NULL) ),
      mainPanel( ptr_ui_toggle_code(
                   ptr_ui_inline_error(ptr_ui_plot(id), ptr_ui_error(id)),
                   ptr_ui_code(id)) )))

Success criteria

  • External contract preserved: rendered DOM ids (<id>-ptr_plot/-ptr_error/-ptr_code/-ptr_layer_select/-ptr_update_plot) and the inputId set are byte-identical to the pre-redesign ptr_module_ui. ptr_module_server unchanged still drives it.
  • ptr_controls_ui, ptr_outputs_ui removed from NAMESPACE + source.
  • Internal callers rewired: ptr_module_ui itself (was app.R:595), ptr_app_bslib (R/paintr-app-bslib.R:75).
  • Roxygen [ptr_controls_ui()]/[ptr_outputs_ui()] links removed/redirected. Known sites (verified 2026-05-16, treat as non-exhaustive — re-grep before editing): R/paintr-css-doc.R:4; R/paintr-app-bslib.R:4; R/paintr-app.R lines 479, 669, 677, 690, 715, 725, 734. Authoritative check is grep -rn '\[ptr_controls_ui()\]\|\[ptr_outputs_ui()\]' R/ returning zero, and R CMD check clean (covered with Step 07 for prose, but man-page xrefs must resolve at this step).
  • devtools::check() 0/0; test-rewrite-app.R + test-ui-pieces.R green.

Constraints

DOM/id parity is the hard gate. If composition changes any server-targeted id, ptr_module_server breaks silently. Snapshot the old ptr_module_ui("m", f) HTML before this step and diff after.
  • The shell here is ptr_module_ui's own (self-contained, L2) — not ptr_ui_page (ADR 0005 rejected routing L2 through the shell).
  • Bare .ptr-app only (no --page canvas — that stays opt-in for ptr_app/ptr_app_grid).
  • No deprecation shims for the removed split functions.
  • Single-formula module ⇒ shared = NULL (inline section shows all its shared keys). Multi-instance shared wiring is the user's job at L2 via ptr_shared; ptr_module_ui takes no shared_panel= boolean (removed, ADR 0005).
