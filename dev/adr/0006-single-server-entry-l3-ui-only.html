ADR 0006 — Single public server entry; L3 is UI-side only; ptr_module_* rename

Status: accepted · Date: 2026-05-18 · Supersedes: ADR 0005 in part — its §2 L3 definition ("own the layout / render path"; "Custom render is L3 … using it to replace a pane is owning the render path"), its §2/§5 sealed-bundle name exception (ptr_module_ui/ptr_module_server "keep their established names"), and the justification of its Rejected-alternatives bullet on state$server_ns_fn/ui_ns_fn (the canonical "ptr_server inside the user's own moduleServer(id)" pattern). Also voids ADR 0004's surviving name protection for the ptr_module_* pair. · Keeps from ADR 0005: §1 (ptr_<x> = self-contained / ptr_ui_<x> = bare convention), §3 (shared partition model), §4 (orthogonal outputs + combinators), and the conclusion that state$server_ns_fn/ui_ns_fn stay internal plumbing. · Related: ADR 0002 (public-API tiering), ADR 0005 (L2/L3 redesign) · Domain language: CONTEXT.md (lock re-pointed from ADR 0005 to this ADR) · Implementation: order-sensitive rename + doc/roxygen/vignette fallout (this date)

Context

A grill-with-docs session (2026-05-18) tested ADR 0005's L3 definition against the code and found it internally inconsistent. ADR 0005 §2 defines L3 as "own the layout / render path" and CONTEXT.md (line 23) documents the canonical custom-render server pattern as bare ptr_server() inside the user's own shiny::moduleServer(id, …). But the exported low-level engine ptr_server() cannot bind shared placeholders:

  • ptr_setup_consumer_uis() (called by ptr_server(), R/paintr-server.R) skips every shared consumer: if (!is.null(c$shared)) next — comment: "Shared consumers are owned by the host … skip here so we don't double-write."
  • The engine performs no formula-local shared self-binding. That logic exists only in ptr_module_server(): the pre-flight contract abort, the missing_keys → local_shared reactive built off input$shared_<key>, and the post-call ptr_bind_local_shared_consumers() (an unexported helper an L3 user cannot call).

Consequence: a legitimate single-instance self-shared formula such as ggplot(mtcars, aes(x = var(shared = 'v'), y = var(shared = 'v'))) + geom_point() works under ptr_module_server() but is silently dropped under the documented bare-ptr_server L3 pattern. This directly contradicts CONTEXT.md line 99 ("a formula-local shared key is owned by that one instance's own server") — the documented L3 instance server (bare ptr_server) does not in fact bind it. The defect is not in ptr_server()'s behaviour (its skip is a correct layering choice — shared binding is a host responsibility); the defect is in treating the low-level engine as a public L3 server pattern.

Decision

1. One public server entry

ggpaintr exposes exactly one public server-side entry point. It owns shared-placeholder binding (formula-local self-bind + consumer pickers) for every supported configuration. The low-level engine that skips shared binding is not public API.

2. Rename swap (mechanical, order-sensitive)

OldNewRole
ptr_server (4-arg engine)ptr_server_internalUnexported plumbing. @keywords internal, no @export.
ptr_module_serverptr_serverThe sole public server entry. Returns state.
ptr_module_uiptr_uiThe L2 self-contained UI bundle.

The name ptr_server is reused for a different function with a different signature ((input,output,session,formula,…) → (formula, id = NULL, …)). The rename is therefore a swap, valid only in this order: (1) engine → ptr_server_internal everywhere (including the do.call(ptr_server, …) call site inside the former ptr_module_server, and any test/vignette/fixture calling the bare 4-arg form); (2) ptr_module_server → ptr_server; (3) ptr_module_ui → ptr_ui. Reversing (1)↔(2) aliases the two functions.

3. L3 is UI-side only

L3 means a UI-side split: bare ptr_ui_* pieces composed into the user's own layout (optionally ptr_ui_page). It is not a server-side concept. There is no user-authored moduleServer wrapping a ggpaintr engine. The server is always the single ptr_server(), at L2 and L3.

Custom render is done by extracting components from the state returned by ptr_server() — e.g. state <- ptr_server(formula, id); output$my_plot <- plotly::renderPlotly(state$runtime()$plot) with the user's own widget at shiny::NS(id)("my_plot"). Custom render is not a separate level and not a server pattern; it is L3-UI fed by the one server entry's return value. ADR 0005 §2's "using state to replace a pane is owning the render path" framing is withdrawn.

Kept intact from ADR 0005 (do not re-litigate): the ptr_<x> = self-contained / ptr_ui_<x> = bare naming convention (§1); the shared partition model and coordinator (§3); orthogonal output pieces + nestable combinators (§4); and the conclusion that state$server_ns_fn/ui_ns_fn remain non-public. Only their justification changes: those stay internal because there is one server entry and custom render extracts from its returned state — not because of a "ptr_server-inside-your-own-moduleServer" pattern, which no longer exists.
Superseded — do NOT carry forward (a future reader will find these in old docs):
  • ADR 0005 §2: "L3 — own the layout / render path" and "Custom render is L3 (… using it to replace a pane is owning the render path)." L3 is UI-side only; the server is unchanged and singular.
  • ADR 0005 §2/§5 + ADR 0004: the sealed-bundle exception keeping ptr_module_ui/ptr_module_server names. Both renamed (ptr_ui/ptr_server); the "module" qualifier is now misleading and void.
  • CONTEXT.md line 23's "Canonical custom-render server pattern: moduleServer(id, \(i,o,s){ state <- ptr_server(i,o,s,formula); … })" and the "ptr_module_server is the scope-hidden path / replacing panes uses ptr_server inside your own moduleServer" sentences. There is no public bare engine to wrap.
  • Any exported ptr_server with the old (input, output, session, formula, …) signature. That symbol is now ptr_server_internal and unexported; the name ptr_server is the formula-first public entry.

Rejected alternatives

  • Make the engine absorb formula-local self-binding so the bare-ptr_server-in-moduleServer pattern works with shared keys: rejected — duplicates host-binding logic into the engine, leaves two public server entries with overlapping responsibility, and enlarges the surface. This is exactly the dual-path maintenance cost ADR 0005 set out to remove.
  • Export ptr_bind_local_shared_consumers as an L3 escape hatch: rejected — re-creates the "exported portal whose runtime companions are internal" smell ADR 0002/0005 explicitly fought; pushes namespace plumbing onto users.
  • Keep ptr_module_* names (ADR 0005's sealed-bundle exception): rejected — with a single server entry the "module" qualifier no longer distinguishes anything and misleads (it implies a sibling non-module entry that no longer exists). ptr_server/ptr_ui are the honest names.
  • Keep L3 as a server-side "own the render path" concept (ADR 0005 status quo): rejected — it is the source of the contradiction; no public engine should be wrapped by user moduleServer code, because the engine deliberately does not bind shared placeholders.

Consequences

  • Breaking, no deprecation (single user, pre-1.0). The hardest-to-debug part is the name reused under a changed signature: external callers of old ptr_server(input,output,session,formula) silently bind to new ptr_server(formula, id=NULL,…). In-repo instances are eliminated by the order-sensitive rename; external breakage is accepted.
  • CONTEXT.md: the L3 section (lines 19–27), line 5 lock pointer (ADR 0005 → ADR 0006), line 23 canonical pattern, line 82 L3 surface, line 98 relationships, line 113 example dialogue — all rewritten to "L3 = UI-side split; one server entry ptr_server; custom render = extract from returned state."
  • vignettes/ggpaintr-use-cases.Rmd L3 section (lines ≈340, 423, 464–469, 484, 569, 614) and any shinytest2 fixtures wired on bare ptr_server-in-moduleServer must be rewritten and re-verified against the authoritative test gate.
  • roxygen/NAMESPACE: ptr_server_internal loses @export (gains @keywords internal); the new ptr_server roxygen documents the extract-from-state custom-render pattern; devtools::document() regenerates NAMESPACE (export(ptr_server) retained for the new function; ptr_server_internal absent).
  • ADR 0002's "exported but companions internal" portal-smell guard is still honored — state$server_ns_fn/ui_ns_fn stay non-public; only the reason recorded for it changes (one entry + extract-from-state).
