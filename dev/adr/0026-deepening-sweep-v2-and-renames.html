Context

The user's stated #1 friction: logic copy-pasted across 2+ spots that should be a single shared helper (must maintain in multiple places). 41 raw candidates deduped to 12 ranked deepenings + a deep-leave/dropped list.

Baseline before any change (2026-05-30): serial full suite FAIL 0 / ERROR 0 / SKIP 0 / PASS ≈3433 (6 parallel-only failures in the consumer-seeding browser cluster re-confirmed as boot-tail flakes — pass serially). The authoritative gate per CLAUDE.md is the serial devtools::test() at 0/0/0; parallel reproducibly flakes that cluster, so each candidate's landing gate is run serially.

Constraints (locked by the user)

  • Functionality of the frequently-used public API is frozen: ptr_app, ptr_server, ptr_ui, ptr_shared/_panel/_server, the pp* keywords, ptr_define_placeholder_*. Renaming arguments is allowed; behaviour is not changed.
  • Tests for exported functions are not edited except where an authorised rename literally changed a name (flagged in that commit). Tests for internal functions may change with the interface contract.
  • Low-value / safe-to-restructure exports: ptr_app_grid (user plans to remove), ptr_app_bslib (failed attempt), ptr_extract_* (only tests use them). Their deprecation is a product decision, deferred to a separate pass — not actioned as a deepening here.
  • Hard-rename (no deprecated aliases) for the ptr_default_* factories and the confusing ptr_define_placeholder_* argument names.

Locked rename decisions (ranks 3 & 4)

Why the ptr_default_* names are wrong: they read like "produce a default value of type X" or a coercer, but each is a factory returning a validator closure function(arg_expr) → canonical_value | abort() that checks the unevaluated AST of a placeholder keyword's formula-time argument at registration time. The role is "validate an argument," not "make a default."

Internal helpers harmonise: validate_default_arg → validate_positional_arg; the roxygen @name ptr_default_args group → ptr_arg_validators. Key synthesis correction: the cross-file source/consumer duplication (#10) is narrower than first claimed — resolve_upload_source is already shared (the panel-host path calls the master impl). The genuine divergent copies are (a) the renderUI bodies, (b) the seeding-decision closures + latches, (c) the spec-value normalization (apply_spec_at_boot vs its host variant). The two paths' reactive dependencies genuinely differ (state-backed vs input-id-only, per ADR 0006's state=NULL-at-host), so a forced full collapse would couple them artificially — hence #10 is phased and its full seam-collapse is optional, landed only if it goes green cleanly.

ADR 0017 reconciliation

  • C1 partition-view → promoted (rank 11). Deferral condition ("worth it if test leverage on partition logic is wanted") met given the recurring 2026-05-22 binder bug.
  • C2a pure cores → promoted (rank 9). Deferral condition met per the seeding-bug memories.
  • C2b named phases → promoted (rank 6). ADR 0017 already said "worth doing regardless."
  • C3a pass-driver (ADR 0018) → dropped from this sweep. High risk, widest blast radius (all four typed-AST passes, ADR 0012 territory). The ~80% boilerplate is real but the per-node payloads genuinely differ and a pass-driver is a large speculative abstraction; not worth the regression risk before the cheaper, higher-locality duplications are banked. ADR 0018 stays a parked proposal.
  • C3c public staged pipeline (ADR 0019) → dropped. The pass functions are deep infrastructure; the 186 ::: test reach-ins are a test-coupling concern, not a duplication. Better served incrementally by the rank-9 pure cores than by a new public API. ADR 0019 stays parked.
  • C4 (app split) / C5 (safe/safety merge) → folded into rank 12 as low-value organisational moves (optional).
  • C6 (placeholder spec / single seam) → partially realised by rank 3 (the lighter consolidation+rename, not the full ptr_register(spec) collapse).

Deep-leave / dropped (do not re-suggest)

  • validate_keyword/_hook/_default_arg/_named_args/_copy_defaults — deep-leave: each encodes a distinct single-use registration contract; not duplication. They become parameters of rank 3's core, not a target.
  • Per-role runtime_fn default (identity for value/consumer, abort-guard for source) — deep-leave: semantically required, a one-line role branch in rank 3's core.
  • classify_walk/disable_walk shared S3 structure — deep-leave: a design pattern, independent behaviour; folding re-introduces the C3a boilerplate-driver risk for no locality gain (ADR 0012 territory).
  • invoke_build_ui label+widget coupling — deep-leave: intentional synchronisation on the final label.
  • layer_panel_default_shell_copy's 5 ptr_resolve_ui_text calls — deep-leave: a named semantic unit, not shallow batching.
  • paintr-utils.R 21-function split — dropped: 0-net-LOC churn, no bug payoff, costs git-blame continuity. Revisit only if it becomes an editing hotspot.
  • ptr_app_bslib / ptr_extract_* / ptr_options export-surface — dropped from this sweep: deprecation/unexport is a product/API decision, not a depth fix. Handle in a separate deprecation/export-surface pass.
  • shared_placeholders vapply(..., [[, 'key') repeated 3× — dropped: a stable idiom, not divergent logic; a collect_shared_keys() helper is mild sugar not worth the arg-rename risk. Touch only if in the area for C1.
