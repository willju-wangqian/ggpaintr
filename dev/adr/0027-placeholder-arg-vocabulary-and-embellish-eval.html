ADR 0027 — Placeholder argument vocabulary + explicit plain-R embellish_eval

Status: Locked decision (pending implementation). Routes to /decision-to-plan for the implementable plan; this doc records the decision + its verified baseline, it is not itself the plan.

Date: 2026-05-31. Supersedes naming for: the ptr_define_placeholder_* argument surface introduced with ADR 0009 (pp* rename) and the ptr_arg_* validator factories.

The story, in plain language

A ggpaintr placeholder is a token (ppVar, ppNum, a custom colvars/ppVars2) you write inside an otherwise-ordinary ggplot2 expression. That expression is meant to work two ways:

  1. Through ptr_app()/ptr_server() — the token becomes a Shiny widget. Its parenthesized arguments (e.g. ppNum(50), colvars(c(mpg, hp))) are the default seed for that widget, parsed at translate time.
  2. As plain R, with no app — the same expression still renders a static ggplot, because each placeholder keyword is bound to a function that gives the token a live-R meaning. This is a real, documented feature (README + tutorial lead with the unquoted-expression form).

Path 2 is under-built and under-documented, and it broke on a real example. The constructor argument that governs path 2 is currently named runtime, defaults to function(x, ...) x (identity), and for any column-selecting placeholder used inside a tidyselect verb the identity default is wrong: it eagerly forces the bare column symbols in ordinary scope.

Verified baseline (the premise, grounded against HEAD)

The customization vignette's own colvars example —

mtcars |>
  dplyr::select(colvars(mpg, hp)) |>
  ggplot(aes(x = ppVar(mpg), y = ppVar(hp))) + geom_point()

— fails two independent ways (reproduced 2026-05-31 against devtools::load_all(".")):

  1. Rejected inside ptr_app. colvars(mpg, hp) is two positional args; extract_placeholder_args caps positional args at one → "Placeholder colvars(...) accepts at most one positional argument (the default); got 2." Source: R/paintr-translate.R:925-930.
  2. Naked expression fails. With the identity runtime, colvars(mpg, hp) returns the whole tibble and tidyselect rejects it: "must be numeric or character, not a <tbl_df> object." The same mechanism makes ppVars2(c(species, tars1, …)) throw "object 'species' not found" — tidyselect evaluates the un-substituted wrapper call in non-masked scope. Source: identity default at R/paintr-registry.R:393-404; tidyselect behavior confirmed empirically (/tmp/runtime_test*.R).

Why the ptr_app path does not hit defect 2: ptr_resolve_upstream substitutes the placeholder before evaluating, splicing c(species, tars1, …) inline with no wrapper, so tidyselect masks correctly (verified: a 518-row pcp data frame). Source: R/paintr-resolve.R:24-47, R/paintr-substitute.R:132-176. The defect is exclusively on the plain-R path, which is exactly the path runtime exists to serve and which no test/vignette exercises.

Current argument surface (verified): ptr_define_placeholder_* take positional_arg, named_args, validate_input, runtime. Exported validator factories: ptr_arg_numeric, ptr_arg_numeric_vector, ptr_arg_string, ptr_arg_symbol_or_string, ptr_arg_expression (no standalone ptr_arg_symbol). Source: R/paintr-default-args.R:201-289, R/paintr-registry.R:592-606,713-719.

The structural problem

For a column placeholder, the seed-parse slot (positional_arg) and the plain-R slot (runtime) perform the same transformation — "bare column names → character vector" — declared twice. That duplication, plus arbitrary/disconnected names (positional_arg, named_args, runtime all govern the same parenthesized arguments but share no vocabulary), is the root of the confusion.

Decisions (locked)

D1 — Keep the plain-R evaluation slot explicit; do NOT derive it (Direction A)

The plain-R behavior is not auto-derived from the seed-parse spec. Rationale: derivation is unreliable and would silently guess wrong — e.g. a numeric placeholder may legitimately need string treatment in plain R, which no type-derivation can know. Forcing authors to see and own the plain-R slot is the feature, not overhead.

D2 — Rename the argument vocabulary (one coherent family)

nowfinalrole
positional_argparse_positional_argtranslate-time: parse+validate the positional default into the widget seed
named_argsparse_named_argstranslate-time: whitelist + per-arg parse/validate of named defaults
validate_inputvalidate_session_inputsession-time: validate the live widget value before resolve
runtimeembellish_evalplain-R: how a placeholder-embellished ggplot expression evaluates with no app

parse_* (not validate_*) because these factories return the parsed seed value, not merely a pass/fail (default <- entry$default_arg(positional[[1L]]), R/paintr-translate.R:931). The internal registry-record keys (default_arg, copy_defaults) stay as-is — only the public argument names change.

D3 — Collapse the vector variants; add ptr_arg_symbol

  • ptr_arg_numeric(vector = FALSE, length = NULL) absorbs ptr_arg_numeric_vector (vector = TRUE; length applies only when vector).
  • ptr_arg_string(vector = FALSE), ptr_arg_symbol_or_string(vector = FALSE) gain the flag.
  • New ptr_arg_symbol(vector = FALSE) (bare-name → string), needed so colvars(c(mpg, hp)) has a vector-capable positional parser.
  • vector = TRUE accepts a single c(...) literal; it does not enable multiple positional args (see D6).

D4 — Built-in embellish_eval helpers

  • embellish_identity() — the default (function(x, ...) x); correct for value placeholders used in data-masked contexts (aes()).
  • embellish_symbol_to_string() — NSE-captures the call's argument (rlang::enexpr) and returns column names as a character vector, which tidyselect accepts; correct for column-selecting consumers used in tidyselect verbs (verified working, /tmp/runtime_test2.R).

Helper prefix is embellish_* to match the embellish_eval slot (no eval_arg_*/evaluate_*_R split). Authors may always supply their own function.

D5 — Accept the breaking change; bump; sweep all call sites

Renaming exported constructor arguments and removing ptr_arg_numeric_vector is a hard break (argument names cannot be soft-deprecated). At 0.10.0 this is acceptable. The change is atomic: rename, regenerate NAMESPACE/man, and update every in-repo call site (tutorial vignette, tests, examples, any fixture app.R) in the same change. Version bump accompanies it.

D6 — colvars(c(mpg, hp)) is canonical; bare-multi is out of scope

The one-positional cap in extract_placeholder_args stays (it is shared by every placeholder; loosening it is a separate, higher-risk decision). The supported multi-column form is a single c(...) positional. The vignette example must change from colvars(mpg, hp) to colvars(c(mpg, hp)).

Worked examples — what becomes possible

Example 1 demonstrates the core win: one column-selecting consumer that works through ptr_app and as a naked static expression, because embellish_eval is set to the new NSE helper.

1. A consumer that survives both paths

colvars <- ptr_define_placeholder_consumer(
  keyword = "colvars",
  build_ui = function(node, cols, data, label = NULL, selected = NULL, ...)
    shiny::selectInput(node$id, label %||% "Columns", choices = cols,
                       selected = intersect(selected %||% character(0), cols),
                       multiple = TRUE),
  resolve_expr   = function(value, node, ...)            # ptr_app path: inline c()
    if (length(value) == 0L) NULL else rlang::call2("c", !!!rlang::syms(value)),
  embellish_eval = embellish_symbol_to_string(),         # plain-R path: NSE -> character vector
  parse_positional_arg = ptr_arg_symbol(vector = TRUE)   # seed: colvars(c(mpg, hp))
)

# naked, no app -- now renders instead of erroring:
mtcars |> dplyr::select(colvars(c(mpg, hp)))             # data frame with mpg, hp

Example 2 demonstrates D1's rationale: a numeric placeholder whose plain-R meaning is deliberately a string, which no derivation could infer — so the author sets embellish_eval by hand.

2. A value placeholder with a non-identity plain-R meaning

ppCode <- ptr_define_placeholder_value(
  keyword = "ppCode",
  build_ui = function(node, label = NULL, selected = NULL, ...)
    shiny::numericInput(node$id, label %||% "Code", value = selected %||% 1),
  resolve_expr   = function(value, node, ...) as.character(value),
  embellish_eval = function(x, ...) as.character(x),      # plain-R: number -> string
  parse_positional_arg = ptr_arg_numeric()
)

Worked examples — what is rejected by design

Example 1 stays rejected after this ADR: the bare comma-separated form is multiple positional args, and the one-positional cap is retained (D6).

1. Bare multi-positional selection (use c(...) instead)

# still aborts at translate: "accepts at most one positional argument; got 2"
ptr_app( mtcars |> dplyr::select(colvars(mpg, hp)) |> ggplot(aes(ppVar(mpg), ppVar(hp))) + geom_point() )
# canonical form:
ptr_app( mtcars |> dplyr::select(colvars(c(mpg, hp))) |> ggplot(aes(ppVar(mpg), ppVar(hp))) + geom_point() )

Example 2 is the anti-pattern this ADR exists to prevent: leaving a tidyselect consumer on the default embellish_identity(), which forces the bare symbols and breaks the naked path.

2. Relying on the identity default for a tidyselect consumer

# embellish_eval defaults to embellish_identity() -> forces c(mpg, hp) in ordinary scope
colvars_bad <- ptr_define_placeholder_consumer(
  keyword = "colvars_bad",
  build_ui = function(node, cols, data, ...) shiny::selectInput(node$id, "Columns", cols, multiple = TRUE),
  resolve_expr = function(value, node, ...) rlang::call2("c", !!!rlang::syms(value))
  # no embellish_eval -> naked `dplyr::select(colvars_bad(c(mpg, hp)))` errors "object 'mpg' not found"
)

Scope

In: the rename map (D2), the vector= collapse + ptr_arg_symbol (D3), the two embellish_* helpers (D4), NAMESPACE/man regen, in-repo call-site sweep, version bump, focused tests for the new plain-R path.

Out (follow-ups): the vignette's missing plain-R-use-case coverage is a known, larger gap tracked separately — this ADR fixes the example's correctness (c(...) form + a working embellish_eval) but does not rewrite the vignette's narrative. Bare-multi positional support (D6). Any Direction-B unification.

Success criteria

  • A column-selecting consumer with embellish_eval = embellish_symbol_to_string() renders a static ggplot from the naked expression (dplyr::select(colvars(c(mpg, hp))) path) — no "object not found", no "must be numeric or character".
  • The same placeholder still works through ptr_app (substituted-inline path unchanged).
  • All ptr_define_placeholder_* calls in the repo use the new argument names; devtools::document() shows no NAMESPACE drift; the authoritative test gate is FAIL 0 / ERROR 0 / SKIP 0.
  • No remaining reference to runtime/positional_arg/named_args/validate_input/ptr_arg_numeric_vector as public arg/function names.
