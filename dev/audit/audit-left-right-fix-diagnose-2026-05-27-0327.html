Diagnose — DOM-id collision in ptr_setup_panel_sources()

Branch left-right-fix · base 9aaaa10 · 2026-05-27 · /diagnose, no source edits

TL;DR

  • Bug reproduced. Two ptr_shared(formulas, id = "left"|"right") coordinators on one page each render an inner fileInput + companion textInput at the bare DOM ids shared_ds / shared_ds_shortcut. Both modules emit identical ids → Shiny binds one element per id; the other is dead. 5 collisions in the probe (label, fileInput, progress bar, shortcut label, shortcut input).
  • Recommendation: Option A, minimal form. Mirror the already-shipped pattern at R/paintr-build-ui.R:809-810 (invoke_build_ui) and R/paintr-server.R:2229-2233 (ptr_setup_source_uis): inside the render closure of ptr_setup_panel_sources(), stamp rendered_node$id <- ns(node$id) (and rendered_node$shortcut_id) on the local copy, leave node bare. No public API surface widens. ~3-line edit; the existing rendered_node <- node line at R/paintr-shared-ui.R:533 already exists — it just forgot to apply the stamp.
  • Counterfactual confirmed. Applying the stamp in a side-by-side probe yields fully namespaced ids (left-shared_ds / right-shared_ds, etc.) — zero collisions.
  • No new node$dom_id field needed (Option B). grep -rn '$dom_id' R/ returns nothing — Option B would introduce a brand-new field; Option A is purely a behavioural fix at one site.

Proposed fix (Option A, minimal)

Single edit at R/paintr-shared-ui.R:533-568. Lines added inside the existing renderUI closure, immediately after rendered_node <- node:

output[[output_id]] <- shiny::renderUI({
  rendered_node <- node
+ # Mirror invoke_build_ui() (R/paintr-build-ui.R:810) and
+ # ptr_setup_source_uis() (R/paintr-server.R:2230): the rendered widget
+ # must bind at the coordinator-namespaced DOM id so two ptr_shared(id=)
+ # coordinators on one page don't collide on bare shared_<key>.
+ rendered_node$id <- input_id
+ if (!is.null(node$shortcut_id)) {
+   rendered_node$shortcut_id <- shortcut_input_id
+ }
  copy <- ptr_resolve_ui_text(...)
  ...
  do.call(entry$build_ui, c(list(rendered_node, label = copy$label), extra_named))
})

Where input_id and shortcut_input_id are already computed at lines 526-527 (ns(canonical) / ns(node$shortcut_id)). No new helpers, no new fields, no API change.

Blast radius: one closure body. No exported function signature changes. No third-party build_ui hook needs an update because every other render path already passes pre-namespaced node$id — third-party authors that wrote a custom ptr_define_placeholder_source(build_ui = …) already expect node$id to be the DOM id to bind to.
