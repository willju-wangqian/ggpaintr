Headline verdict

The checklist is broadly accurate on the placeholder grammar, registry shape, and code-mode wiring, but contains three concrete factual errors in the public-API surface and one outdated header. The 57-commit gap since the doc was written matters most for preserve-mode rendering (semantics rewritten by ADR-0012) and ppUpload (companion name= added by ADR-0010) — both of which are nominally inside sections B / H but the doc still describes the pre-ADR-0010/0012 shape.

Recommended doc edits

The minimum patch to bring the checklist back into alignment without rewriting it:

  1. A7 / H4 / H5: replace "csv / tsv / xlsx / rds / fst" with "csv / tsv / rds / xlsx / xls / json" (the actual ptr_upload_accept_formats() set).
  2. B5: replace ptr_ui_code_block(style = "panel") with ptr_ui_code(style = "panel").
  3. F2 + L2: either (a) drop ptr_app_components() from the "public L3 surface" framing, replacing with ptr_app_grid() + ptr_ui()/ptr_server(); or (b) decide to export ptr_app_components and add the @export tag. Today's NAMESPACE makes option (a) the truthful patch.
  4. F9: same fork — either drop the ptr_render() example, or export ptr_render (currently @keywords internal). The ptr_extract_code(state, preserve = TRUE) alternative the doc itself flags as UNTESTED in B8 is the cleanest forward fix.
  5. Header HEAD ref + the "Known-broken" table: refresh to 984b689 and flag ADR-0010 (ppUpload name=) + ADR-0012 (pipe-op preserve) as landed.
  6. A7 / H4: surface the ppUpload(name = "object_name") companion-input shape (ADR-0010 §C1–C6).
