Findings

BUG — R/paintr-server.R:308 · ptr_server() @return claims "invisibly" but the function returns visibly

Doc text (line 308): #' @return The `ptr_state` list, invisibly. The @description (lines 283–285) also says "Returns the state list so embedders can attach extras".

Source: the function body's final expression is a bare state (line 372), immediately followed by the closing } (line 373). There is no invisible() wrapper anywhere on the return path. Verified by direct read of lines 358–373; the only invisible() calls in the function family belong to the unrelated internal ptr_register_code() helper (line 1398) and ptr_gg_extra() (lines 1474/1486). So ptr_server(input, output, session, f) returns its ptr_state visibly — at a REPL / in shiny::testServer() it auto-prints, contrary to the documented contract.

Why it matters: a user relying on the doc would expect no console output (e.g. not bother assigning the result, or expect ptr_server(...) at the end of a server function to print nothing). The sibling ptr_extract_* / ptr_gg_extra blocks correctly document visibility, so this is an isolated inaccuracy, not a house style.

Suggested correction (text only — not applied): either change the doc to #' @return The `ptr_state` list. (drop "invisibly"), or, if the invisible contract is intended, change the source last line from state to invisible(state). Pick one so doc and code agree; the doc-only edit is the lower-risk choice since visible return is the long-standing behaviour and tests/embedders may already rely on inspecting it. (Source must not be edited by this audit.)
