Bugs found

BUG 1 — UC5: stale #> comment in ptr_normalize_column_names() example

Vignette claimed:

names(clean)
#> "first.column" "if."

Actual output (verified):

[1] "first_column" "if_"

The function replaces non-syntactic characters with _, not .. Patched vignettes/ggpaintr-use-cases.Rmd.

BUG 2 — CU4: dplyr::all_of() claim in colvars walk-through

Vignette claimed the rendered code would be mtcars |> dplyr::select(dplyr::all_of(c('mpg', 'hp', 'wt'))). Browser-verified actual output is mtcars |> dplyr::select(c("mpg", "hp", "wt")) — there is no all_of() wrap because the example's resolve_expr returns rlang::call2("c", …). Patched vignettes/ggpaintr-customization.Rmd.

BUG 3 — Gallery: range / colvars placeholders referenced but never registered

Gallery §3, §4.1–4.6, §5.1, §6.1 use custom range / colvars keywords. §2 told the reader to "Run the placeholder-registration chunk from ggpaintr-customization", but that vignette has the registrations scattered across multiple worked-example chunks and was easy to miss. Added a single concrete custom-placeholders-setup chunk in §2 of the gallery that registers both keywords. Examples now copy-paste-runnable after one setup chunk.

BUG 4 — Gallery §6.2 (ridges): lincoln_weather non-syntactic columns

The ggridges::lincoln_weather dataset ships with names like "Mean Temperature [F]". The var placeholder requires syntactic identifiers, so the picker rejects the column with Input error: Invalid value for placeholder `var`: Column `Mean.Temperature..F.` is not in the data. Patched vignettes/ggpaintr-gallery.Rmd §6.2 to run ptr_normalize_column_names(lincoln_weather) first, with a cross-link to the use-cases vignette §"Local data with non-syntactic columns".

BUG 5 (runtime, FULLY FIXED) — anonymous function(.x) … literals inside formulas

Gallery §4.3 (k-means) and §4.6 (group-wise regression) failed on function(.x) … lambdas inside map(). Four distinct defects, all now fixed and regression-tested (tests/testthat/test-rewrite-closure.R, 9 cases):

  1. Parser crash. AST walkers (visit_collect in R/paintr-shared.R, visit in find_layer_placeholders_with_stage in R/paintr-build-ui.R, rec in ptr_walk in R/paintr-walk.R) iterated for (el in x) visit(el) over any is.list(x). is.list() is also TRUE for pairlists; function formals are a pairlist whose no-default elements are the missing-argument marker, so forcing the loop variable raised argument "el" is missing. Fixed: skip pairlists (is.list(x) && !is.pairlist(x)).
  2. Mis-translation. translate_call() treated function(.k) body (a length-4 call: function, formals-pairlist, body, srcref) as an ordinary head(arg, arg, arg) call, so the formals round-tripped through a literal node and the srcref became a bogus 3rd argument — re-rendering as function(as.pairlist(alist(.k = )), body, NULL). Fixed: a dedicated ptr_closure node (constructor + predicate in R/paintr-nodes.R) that keeps the formals pairlist verbatim and translates only the body, with matching node_to_lang / render_walk / substitute_walk / prune_walk methods and a "body" cursor slot in ptr_rewrite_pre so closure-body placeholders get ids.
  3. Server input wiring. collect_layer_placeholders() in R/paintr-input-spec.R (which builds the runtime input spec / snapshot) handled ptr_call/ptr_pipeline but not closures, so a placeholder inside a lambda body got a widget yet its typed value never reached substitution (the code pane showed the arg pruned away). Fixed: added a ptr_closure branch that recurses into $body.
  4. Pre-existing, closure-independent. node_list_to_lang() built args with out[[i]] <- node_to_lang(...); when a node's lang is NULL (e.g. a literal labs(y = NULL) arg) [[<- deletes the slot, desyncing the list from names(args) → 'names' attribute [n] must be the same length as the vector [n-1]. This bit any formula with a literal NULL argument, independent of closures. Fixed: out[i] <- list(node_to_lang(...)) preserves NULL slots.

Both §4.3 and §4.6 now run end-to-end in the browser (verified: plot renders, no error, code pane shows correct function(.k) … / function(.d) …). §4.6 reverted to a normal runnable chunk. §4.3 additionally had a design error — it fed colvars (a consumer placeholder that resolves against pipeline upstream, which here is tibble(k = …), not iris) — corrected to the fixed four-column iris[, c(...)] set matching the §4.3 original recipe.

Minor cosmetic, not fixed (out of scope): the code pane renders broom::glance as ::(broom, glance) (prefix instead of infix ::). It evaluates correctly; only the displayed text is ugly.
