Caveats banked (not refiled as bugs)

testServer + unset column-picker → object '' not found

When shiny::testServer() runs a ptr_server() session without pre-seeding the column-picker inputs, session$setInputs retains the empty-string default and the reactive flush surfaces object '' not found during the data-source resolver pass. Pre-seeding the picker (e.g. session$setInputs(ggplot_1_1_var_NA = "Sepal_Length")) clears the flush and the runtime resolves normally.

This is a testServer driver artefact, not a runtime bug — in a real Shiny session the pickerInput initialises with the first column rather than "", so the resolver never sees an empty symbol. Documented here so future headless walks pre-seed pickers before reading state$runtime().
