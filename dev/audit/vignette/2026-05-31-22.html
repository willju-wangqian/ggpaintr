ggpaintr-tutorial.Rmd

STALE-WRONG — tutorial:191 (and cross-ref tutorial:104) — the plain-R-meaning slot is documented under its old argument name runtime

What's there. Section 2.1 carries a subsection heading ### `runtime` (line 191) describing "the plain-R meaning of the keyword outside ptr_app() … value/consumer keywords default to the identity function(x, ...) x". Line 104 forward-references it: "returns a plain-R function, see runtime below". The accompanying chunk label is {r runtime-bind} (line 195).

What it should be. The public constructor argument was renamed by ADR 0027 from runtime to embellish_eval. The current signatures take embellish_eval (R/paintr-registry.R:600 ptr_define_placeholder_value(... embellish_eval = NULL ...); same on _consumer L719, _source L877, ptr_define_core L354). The old name runtime now raises an R "unused argument" error if written in a registration call.

Why this is the load-bearing finding. The vignette is internally contradictory: the same vignette's code chunks already use the new name — embellish_eval = function(x, ...) as.numeric(x) / 100 (line 141) — and the prose at lines 208, 249, 253 already calls it embellish_eval. Only the 2.1 subsection heading (191), its cross-reference (104), and the chunk label (195) were missed in the rename. A reader who reads the 2.1 argument walkthrough learns the wrong argument name, then sees a chunk that uses a different one.

Suggested correction (text only, not applied). Rename the subsection heading ### `runtime` → ### `embellish_eval`; change line 104 "see runtime below" → "see embellish_eval below"; rename the chunk label runtime-bind → embellish-bind. Within the section body, keep the explanation but state the default precisely: for value/consumer keywords embellish_eval defaults to embellish_identity() (the identity function(x, ...) x) — confirmed at ptr_define_core R/paintr-registry.R (runtime_default <- … embellish_identity()), and for a source it defaults to an abort guard (already stated correctly at line 253).
