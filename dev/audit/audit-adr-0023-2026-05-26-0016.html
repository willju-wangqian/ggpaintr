Verdict

FAIL — 2 findings. Both are the same class of error: positional-arg slips in the worked-example code blocks (and the Context recipe at line 30). The Decision, Consequences, and Alternatives sections are baseline-clean — every named symbol, file path, line citation, and bundle-field claim verifies against source. The slip is isolated to the example R code: ptr_ui("p1") binds "p1" to the required formula argument (not id), and ptr_server("p1", formula = panel$formulas[[1]], ...) binds "p1" to formula positionally and then also passes a named formula = — a duplicate-arg R error. As written, the worked-example apps will not parse or run.

Per the audit-adr contract, no stamp / commit / worktree on FAIL. Fix the two slips (or, in the very unlikely alternative, propose a ptr_ui / ptr_server signature change in the ADR), then re-run /audit-adr.

FINDING 2: ptr_server("p1", formula = panel$formulas[[1]], shared_state = s) duplicates formula

Category: 1 (code-shaped literal — function signature)

ADR quotes (Example 1, lines 156-157):

s <- ptr_shared_server(panel)
ptr_server("p1", formula = panel$formulas[[1]], shared_state = s)
ptr_server("p2", formula = panel$formulas[[2]], shared_state = s)

Source evidence:

R/paintr-app.R:1075  ptr_server <- function(formula, id = NULL,
                                            envir = parent.frame(), ...,
                                            shared_state = NULL, spec = NULL)

First positional arg is formula. The ADR call binds "p1" to formula positionally AND passes a named formula = panel$formulas[[1]] — R raises a "formal argument matched by multiple actual arguments" error at the call site. The example will not run.

Two interpretations:

  1. If the ADR is wrong — proposed edit (NOT applied): 
    - ptr_server("p1", formula = panel$formulas[[1]], shared_state = s)
- ptr_server("p2", formula = panel$formulas[[2]], shared_state = s)
+ ptr_server(panel$formulas[[1]], "p1", shared_state = s)
+ ptr_server(panel$formulas[[2]], "p2", shared_state = s)

    Or, fully named (more readable):

    + ptr_server(formula = panel$formulas[[1]], id = "p1", shared_state = s)
+ ptr_server(formula = panel$formulas[[2]], id = "p2", shared_state = s)
  2. If the source is wrong — verification path:

    Same as Finding 1: would require a ptr_server signature change explicitly proposed in the ADR. None is. Confirm with grep -n "ptr_server" dev/adr/0023-panel-owned-shared-sources-end-to-end.html — every reference treats ptr_server as already existing.

Heuristic hint: Interpretation 1. Same authoring lapse as Finding 1; both call sites have "p1"/"p2" in the first positional slot expecting it to be id. Fix both findings together — they're a pair.
