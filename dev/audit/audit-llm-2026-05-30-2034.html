ggpaintr LLM-assist surface — audit

Date: 2026-05-30 20:34 · Branch: llm-update · Method: static introspection of the source package (pkgload::load_all) vs. the shipped inst/llm/ corpus, via the new linter dev/build-llm.R lint. Read-only audit; no content edited yet.

Verdict: the LLM-assist content is the four file-reader functions in R/paintr-llm.R (sound) plus a 13-file markdown corpus under inst/llm/ that has systemically drifted behind several shipped API changes. 101 HARD + 21 WARN machine-verified breakages across all 13 files: an LLM following the primer/topics today emits R that fails on nonexistent function names, swapped argument order, removed arguments, renamed arguments, and non-parsing placeholder tokens. The four R functions need no behavioral change — only the bundled content (and a generation/gate mechanism so it cannot drift silently again).

Also ptr_translate() (level3_layout) — real but not exported. Rewrite every L2/L3 example to ptr_ui(formula, id = "…") / ptr_server(formula, id = "…"); inject signatures from formals().

Replace hand-typed signature lines with formals()-injected managed blocks; rewrite the naming-convention prose.

Root cause: the content is a second, hand-synced copy of the API contract written before ADR-0009 (pp* rename), the module→ptr_ui/ptr_server rename, the shared_ui removal, the checkbox_defaults removal, and the placeholder-constructor signature change. Nothing checked the copy against the source, so it rotted silently.

3. Mechanism built during the audit

dev/build-llm.R lint — loads the source package and asserts, against live introspection: every ptr_*() call names a real export (drift class 1); no removed/renamed argument appears in arg = position (class 2); no pre-pp* bare token appears in call position (class 3, WARN); no internal (non-exported) S3 generic is documented as a call (class 4); the 5 user tokens are all registered. Exit 1 on any HARD violation. This is the worklist producer for the rewrite and the seed of the sync gate.

4. Proposed rewrite (build-time generation — decisions locked)

Locked: managed-block injection (keep the 13 markdown files as prose source; regenerate only the volatile factual blocks from introspection) + remove the build_ui_for section.

  1. Content correctness — rewrite all 13 files to clear lint to 0 HARD: classes 1–5 above.
  2. Managed blocks — sentinel-delimited regions for signatures (formals()), the token table (ptr_registry_keywords()), and the topic index (files on disk); dev/build-llm.R generate + verify (the document()-style "regenerate ⇒ no diff" gate).
  3. Tests — tests/testthat/test-llm.R: lint-clean, topic-list == files, token table == registry, sync-gate no-diff, plus the four functions' behavioral contracts (currently zero coverage).
  4. Gate wiring — add the lint + verify to the pre-commit hook alongside document().

Definition of Done: Rscript dev/build-llm.R lint → 0 HARD; verify → no diff; devtools::test(filter="llm") → FAIL 0 / ERROR 0; full authoritative gate unchanged.
