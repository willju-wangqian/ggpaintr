1. The lens (one-line restatement)

For each absorbed L2: if the orchestrator/runtime regressed only at THIS contract, would the browser DOM differ? If no → RETAIN as unit pin, do not migrate. If yes → MERGE. If yes-but-needs-extra-fixture → MERGE-deferred.

Two sub-questions sharpen the lens (from v6/v7/v8 §1):

  1. Does the assertion observe an internal-shape value (S3 class, list field name, NULL vs empty-list, named-list shape, object identity) that the rendered DOM does not reflect?
  2. Does the assertion's trigger come from an R-side mutation (e.g. local_mocked_bindings(.package=), direct call to an internal helper with fabricated state) that no DOM event can dispatch?

"Yes" to either → not browser-encodable as-is.
