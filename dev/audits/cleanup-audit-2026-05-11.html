1 · The boot-skip triggered() guard the pattern you described

paintr-server.R · ptr_setup_runtime() · lines ~386–452

✓ Verified by reading the function.

This is the exact shape you called out: a plain shiny::observe() wrapping a hand-written predicate that re-derives "has the user actually triggered anything yet" from raw input counters:

triggered <- function() {
  up <- input[[ns("ptr_update_plot")]]
  da <- if (!is.null(state$draw_trigger)) state$draw_trigger() else NULL
  extras <- state$extras()
  (!is.null(up) && is.numeric(up) && up >= 1L) ||
    (!is.null(da) && is.numeric(da) && da >= 1L) ||
    (length(extras) > 0L)
}
shiny::observe({
  if (!triggered()) return(invisible())
  shiny::isolate({ ... })
})

The natural Shiny form is observeEvent / bindEvent over the union of triggers, with ignoreInit = TRUE doing the boot-skip:

cleaner shape
shiny::observeEvent(
  {
    input[[ns("ptr_update_plot")]]
    if (!is.null(state$draw_trigger)) state$draw_trigger()
    state$extras()
  },
  {
    shiny::isolate({ ... })   # body unchanged
  },
  ignoreInit = TRUE,
  ignoreNULL = FALSE
)
why it's cleaner
  • No is.numeric() && >= 1L hand-checks — observeEvent already de-dups on value change and skips the initial flush.
  • The "first eval held back until a real click" intent becomes declarative (ignoreInit) instead of an imperative early-return.
  • Removes the dependency-take/early-return split that currently makes the observer fire-then-bail on every unrelated invalidation.
⚠ Constraint to verify before doing this. The current comment (server.R ~393–396) claims shiny::testServer() "does not auto-flush at session start, so we cannot rely on observeEvent(..., ignoreInit = TRUE)". If true, the test suite is the reason for the hand-rolled guard. Action item: write a throwaway testServer case with observeEvent(..., ignoreInit = TRUE) and confirm whether it fires on session$flushReact() before recommending the swap. If the claim holds, a smaller win is still available: collapse (!is.null(x) && is.numeric(x) && x >= 1L) into a one-line helper clicked <- function(x) isTRUE(is.numeric(x) && length(x) == 1L && x >= 1L).

2 · Seven hand-rolled recursive tree walkers with one skeleton

✓ Verified — read all seven.

The codebase has a recurring "descend the typed AST, skip the upstream back-pointer" recursion, copy-pasted into:

FunctionFileWhat it collects
find_nodes(node, pred)build-ui.Rnodes matching a predicate
find_layer_placeholders(x)build-ui.Rnon-shared placeholders, stops at placeholder
find_layer_placeholders_with_stage(x)build-ui.Rplaceholders + enclosing stage_id
collect_shared_placeholders(tree)build-ui.Rfirst occurrence per shared key
collect_stage_ids(node)disable.Revery stage_id in formula order
find_consumer_ids_in_upstream(upstream)server.Rconsumer ids inside an upstream subtree
find_producer_ids_in_upstream(upstream)server.Rproducer ids inside an upstream subtree
walk_has_placeholder(node)ids.R (referenced from disable.R)boolean: does subtree contain a placeholder

Every one of them is the same loop: if is_ptr_node(x) → iterate names(x) skipping "upstream"; else if is.list(x) → iterate elements. A single combinator subsumes all of them:

# R/paintr-walk.R  (new file)
# Pre-order visit of every typed-tree node. `descend_upstream = FALSE`
# matches the existing convention (the upstream field is a back-pointer).
ptr_walk <- function(node, visit, descend_upstream = FALSE, prune = NULL) {
  rec <- function(x) {
    if (is_ptr_node(x)) {
      visit(x)
      if (!is.null(prune) && isTRUE(prune(x))) return(invisible())
      for (nm in names(x)) {
        if (!descend_upstream && identical(nm, "upstream")) next
        rec(x[[nm]])
      }
    } else if (is.list(x)) {
      for (el in x) rec(el)
    }
  }
  rec(node)
  invisible()
}
# collector variant
ptr_collect <- function(node, pred, ...) {
  out <- list(); ptr_walk(node, function(x) if (pred(x)) out[[length(out)+1L]] <<- x, ...); out
}

Then e.g. find_nodes(tree, p) → ptr_collect(tree, p); collect_stage_ids → a 3-line wrapper; find_consumer_ids_in_upstream/find_producer_ids_in_upstream collapse into collect_upstream_ids(u, pred) with descend_upstream = FALSE and a prune that stops at the matched node. Estimated net deletion: ~120 lines across four files, and the "skip upstream" invariant lives in exactly one place instead of seven (it's currently easy to forget — and a forgotten skip would double-count or loop).

3 · find_consumer_ids_in_upstream ≡ find_producer_ids_in_upstream

paintr-server.R · lines ~771–819

✓ Verified — the two function bodies are identical except is_ptr_ph_data_consumer vs is_ptr_ph_value.

Independent of the larger #2 refactor, these two are a same-day, zero-risk merge:

collect_upstream_ids <- function(upstream, pred) {
  if (is.null(upstream)) return(character())
  ids <- character()
  visit <- function(n) {
    if (is.null(n)) return()
    if (pred(n) && !is.null(n$id)) { ids <<- c(ids, n$id); return() }
    if (is_ptr_node(n)) for (nm in names(n)) { if (identical(nm, "upstream")) next; visit(n[[nm]]) }
    else if (is.list(n)) for (el in n) visit(el)
  }
  visit(upstream)
  unique(ids)
}

# find_consumer_ids_in_upstream(u)  ->  collect_upstream_ids(u, is_ptr_ph_data_consumer)
# find_producer_ids_in_upstream(u)  ->  collect_upstream_ids(u, is_ptr_ph_value)

5 · validate_expr_safety repeats one abort 8×

paintr-utils.R · validate_expr_safety() · lines ~597–727

✓ Verified — counted 8 rlang::abort(paste0("expr placeholder: ` … ` is not allowed …")) / "is not in the allowlist" sites, one per (string-literal | char-vector-element | symbol | call-head) × (denylist | allowlist).

All differ only in the trailing parenthetical ("found as string literal", "found in character vector", "found as symbol reference", "found as call head"). Collapse to:

deny_abort <- function(name, where) {
  rlang::abort(paste0(
    "expr placeholder: `", name, "` is not allowed (", where, "). ",
    "Set expr_check = FALSE to allow arbitrary expressions."
  ))
}
# and a single check used in every branch:
deny_check <- function(name, where) {
  hit <- if (resolved$mode == "denylist") name %in% resolved$fns else name %in% unsafe_expr_denylist
  if (isTRUE(hit)) deny_abort(name, where)
}

The walker itself is fine — the recursive descent into pairlists / string literals / symbols / call heads is the security mechanism (see .claude/rules/reviewing.md) and should not be touched. Only the message construction is duplicated.

6 · Five *_walk S3 generics, each ~13 near-identical methods architectural — discuss first

✓ Verified — classify_walk, substitute_walk, prune_walk, disable_walk, render_walk (plus build_ui_for and walk_ptr_safety) each implement .ptr_root (loop layers), .ptr_layer (recurse data_arg + children), .ptr_pipeline (loop stages), .ptr_call (loop args), plus trivial .default / .ptr_literal / .ptr_missing passthroughs.

That's roughly 5 × the same structural-recursion boilerplate. A "transform" combinator that knows how to descend the container nodes and takes per-leaf callbacks (what to do at a placeholder, what to do at a call, whether to drop a stage) would let disable, prune, and substitute each shrink to "the interesting cases only". render is genuinely different (it produces text, not a tree) and build_ui_for is dispatch sugar — leave those.

This is the highest-leverage but also the riskiest item: it touches the transform pipeline that every test exercises. Recommend treating it as its own change with the full devtools::test() + manual-app pass, after we've agreed the shape. I have not prototyped this one, so I'm flagging it as a direction, not a verified drop-in.

7 · for (x) local({ x_local <- x; … }) capture-shuffle

paintr-server.R lines 230, 287, 369, 618, 721 · paintr-app.R line 410

✓ Verified — 6 occurrences.

Every observer-setup loop does for (item in items) local({ item_local <- item; ...use item_local... }) to force per-iteration capture. lapply(items, function(item) { ...use item... }) gives the same fresh binding without the shadow-variable rename and without the reader having to know why local() is there. (Functionally equivalent for side-effecting observer registration; lapply's return value is discarded.) Pure readability — do it opportunistically when touching those functions, not as a standalone churn PR.
