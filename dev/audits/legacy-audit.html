0. Executive summary

The rewrite is a deliberate re-architecture: a typed AST (ptr_root/ptr_layer/ptr_ph_value/ptr_ph_data_source/ptr_ph_data_consumer/ptr_user_expr/ptr_pipeline/…) with S3 walkers replaces the legacy index-path ptr_obj; the placeholder registry is now process-global (no placeholders= arg) with three role-specific constructors; the data-pipeline "Update data" click-gated cache model is dropped in favor of lazy per-stage resolution with enable checkboxes. Safety (AST denylist walker) is byte-identical and net-stronger (re-applied to resolver output + resolved upstream + depth guards).

Biggest user/integrator impacts:

  1. HIGH "Update data" button + stale-input notice removed. Data-pipeline placeholders now feed consumers live (debounced); per-stage enable checkboxes replace the button. Whole data-manipulation UX changed.
  2. HIGH Control panel layout changed from a server-rendered tabbed panel to a static layer pickerInput + hidden per-layer tabset. Output ids renamed (outputPlot/outputError/outputCode → ptr_plot/ptr_error/ptr_code), draw button renamed (draw → ptr_update_plot, label "Update plot").
  3. HIGH Custom-placeholder API redesigned. ptr_define_placeholder() / ptr_merge_placeholders() / ptr_missing_expr() removed; replaced by ptr_define_placeholder_value/_source/_consumer() with new hook signatures (build_ui(node, label, …), resolve_expr(value, node, …), return NULL not ptr_missing_expr()). resolve_input, bind_ui, prepare_eval_env hooks dropped.
  4. HIGH L2/L3 integration API removed: ptr_parse_formula, ptr_exec, ptr_assemble_plot, ptr_build_ids, ptr_input_ui, ptr_output_ui, ptr_setup_controls, ptr_register_draw, ptr_resolve_layer_data all gone. ptr_extract_plot/error/code kept the names but changed the argument from a runtime-result list to the server state object (silent compat trap). ptr_module_ui now requires formula.
  5. MED Error panel formatting changed: stage-prefixes ("Input error:" / "Plot error:") dropped, ANSI stripped, ptr_extract_error returns a raw string not a UI tag, shared-resolution errors prepended. "No layers" text changed.
  6. MED On render failure the plot canvas is no longer cleared (req() instead of plot.new()) — stale plot lingers.
  7. MED Debounce default flipped from always-300 ms to 0 ms (auto, flips to 300 ms only after 3 consecutive slow resolves); new producer_debounce_ms knob.
  8. MED Several ui_text keys removed (shell$update_data_button, shell$update_data_stale_class) — existing overrides now hard-error. help/placeholder/empty_text copy may no longer reach builtin widgets (VERIFY). Pipeline-control labels lost the "in mutate()" suffix.
  9. MED ptr_gg_extra() return changed (returns state invisibly, auto-folds into plot) — legacy idiom plot_obj + ptr_gg_extra(...) no longer works.
  10. LOW Package version still 0.9.1 despite breaking changes; purrr dropped from Imports; %||% localized.

No regression found in: formula syntax accepted (single expr, + layers, bare-symbol layers, pkg::fn/pkg:::fn, %>%/|>, kw(shared="..."), .data[["x"]] escape), the safety denylist/walker (identical, net-stronger), ptr_options, ptr_normalize_column_names, ptr_ns_id, the LLM helpers, checkbox-defaults, paintr-data.R, paintr-upload.R core.

2. Changed user / UI behavior

  1. "Update data" button + stale-input notice gone. HIGH Legacy: edit a data-pipeline placeholder → a yellow "Unsaved data inputs — click 'Update data'" card appears + the button gets a ptr-update-data-stale class → click recomputes the cached layer frame. Rewrite: each data-manipulation pipeline stage gets a leading checkboxInput (value=TRUE) in a div.ptr-stage-row; unchecking drops that stage. Edits flow live (debounced) into consumer pickers and into the next "Update plot" render. (legacy paintr-app.R::ptr_setup_data_pipeline_observers / paintr-runtime.R::ptr_stale_notice_ui → rewrite paintr-build-ui.R::build_ui_for.ptr_layer + paintr-server.R::ptr_setup_consumer_uis.)
  2. Control-panel layout: tabbed server-rendered panel → static layer pickerInput + hidden per-layer tabset. HIGH The end user now picks a layer from a dropdown instead of clicking layer tabs. Per-layer inner structure (Data / Controls sub-tabs in a tabsetPanel) is preserved, but the inner tabset now carries id = ns_fn("<layer>_subtab") (legacy had none). Applies to both ptr_app and ptr_app_bslib (bslib sidebar contents changed identically).
  3. Element ids renamed. MED Outputs: outputPlot/outputError/outputCode → ptr_plot/ptr_error/ptr_code. Draw button: draw → ptr_update_plot; label copy key draw_button → shell$update_plot_label, default text "Draw plot" → "Update plot". Layer select: ptr_layer_select; layer tabset: ptr_layer_tabset. Anyone CSS-targeting or testing the old ids breaks.
  4. Several ui_text keys removed → existing overrides hard-error. MED shell$update_data_button, shell$update_data_stale_class dropped from ptr_ui_text_component_paths(), ptr_default_ui_text(), and allowed_shell — passing them now aborts "unsupported entries". Also component = "update_data_button" to ptr_resolve_ui_text aborts.
  5. help / placeholder / empty_text copy may no longer render on builtin widgets. MED VERIFY Legacy builtins wrapped controls in ptr_attach_help(ui, copy$help) and passed placeholder = copy$placeholder to textInput/textAreaInput, and noneSelectedText = copy$empty_text for var. Rewrite builtins emit bare widgets; build_ui_for forwards only label = copy$label to hooks. So ui_text help/placeholder entries and the distinct "Choose one column" empty-text for var appear not to surface (unless the registry hook re-resolves copy — needs confirmation). Also default label wording changed for several builtins (text: "Enter text for {param}" → "Enter a value for {param}"; var: "Choose a column for {param}" → "Pick a column for {param}"; etc.).
  6. Plot canvas not cleared on render failure. MED Legacy ptr_register_plot did graphics::plot.new() on a NULL plot; rewrite uses shiny::req(), so after an error the previous plot lingers instead of going blank.
  7. Error-panel content reformatted. MED Stage prefixes "Input error:" / "Plot error:" dropped; ANSI escapes stripped (cli::ansi_strip); shared_resolution_errors concatenated ahead of the runtime error; ptr_extract_error returns a raw string (was a styled Shiny tag). "No layers" message text: legacy "No plot layers remain after processing the selected inputs." → rewrite "No layers to render (after pruning)." / "No layers to evaluate (after pruning)."
  8. Debounce default flipped. MED Legacy: a single always-on 300 ms debounce on the shared env feeding all var pickers. Rewrite: producer (text/num/expr) inputs debounced with a window defaulting to 0 ms, auto-flipping to 300 ms only after 3 consecutive >150 ms upstream resolves; new producer_debounce_ms arg (NULL=auto, 0=off, n=pinned); new verbose cli::cli_inform on auto-flip. var picker choices now reflect live placeholder edits before clicking "Update plot".
  9. Runtime re-renders on ptr_gg_extra() change. MED Rewrite's runtime trigger includes length(state$extras()) > 0 and folds extras in via Reduce(`+`, extras, res$plot); legacy required the embedder to do plot_obj + ptr_gg_extra(...) in their own renderPlot and ptr_gg_extra didn't itself trigger a draw.
  10. Layer panels no longer get a ptr-layer-disabled CSS class when the layer's include-checkbox is unticked. LOW VERIFY Legacy sent a ptr_set_class custom message; the rewrite's ptr_setup_layer_picker only does updateTabsetPanel. (The layer is still functionally dropped via disable_walk; only the greyed-out visual cue is gone — unless re-wired elsewhere in paintr-server.R.)
  11. Pipeline-control labels lose the "in verb()" suffix. MED Legacy appended " in mutate()" (etc.) to a data-pipeline placeholder's label and used verb() as the param for unnamed args; rewrite uses the plain resolved label.
  12. ptr_app_grid now auto-renders default widgets for any shared= key not covered by shared_ui (via build_ui_for(node, ns_fn=identity)), and now errors if shared_ui references a key not used in any formula (legacy didn't validate this). MED
  13. expr placeholder widget changed from single-line textInput to multi-line textAreaInput. MED Plus parse-error message text changed ("could not parse input as R expression: …" → "expr placeholder requires exactly one expression as a string."). upload fileInput lost its accept filter (was c(".csv",".tsv",".rds",".xlsx",".xls",".json")) and now has a hard-coded "Upload data" label + "Dataset name" companion (was driven by upload_file/upload_name ui_text keys). MED VERIFY
  14. Layer-include checkbox default label fallback changed — still defaults to "Include this layer in the plot" via ui_text, but a custom shell_copy arg can now override per-panel and the ultimate fallback is the bare layer name (legacy had no such fallback). LOW
  15. Otherwise unchanged for end users: formula syntax accepted, duplicate-layer-name -2/-3 suffixing, non-ggplot layer <name>_checkbox ids, num blank/NA → arg dropped (now also handles logical-NA / empty-string from numericInput), text auto-quote-stripping, .data[["x"]] placeholder-conflict escape, title from ui_text title key.
