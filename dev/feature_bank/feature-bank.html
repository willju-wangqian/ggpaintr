How coverage is detected

Each row carries a set of search terms (function names, argument names, behavior keywords). Each source file is parsed with base::parse() and the resulting expressions are deparsed back to text -- this drops every comment, so a feature mentioned only in a #-comment does not count. For tests/testthat/test-*.R files the search is further restricted to text inside test_that(...) bodies, so a name dropped in a top-level helper or setup block does not by itself satisfy a row. A source counts as evidence when at least one term appears literally in that restricted text. Sources searched: tests/testthat/test-*.R (automated; in-test bodies only), dev/scripts/feature-coverage-examples.R (canonical demo + headless source-eval; full code), and dev/audit/2026-05-15-headless/driver-headless.R (supplemental headless audit; full code). Status badges are three-level: tested when at least one tests/testthat/test-*.R file hits, demo only when only the canonical example or the headless driver hits (the row is exercised but has no dedicated unit test), none when no source contains any search term -- either the feature genuinely lacks coverage, or its search terms need refinement in dev/feature_bank/build-feature-bank.R.
