# Level 2 — custom top-level ids with `ptr_build_ids()`

Use when: the host app already uses one of the default ggpaintr ids, or
when ggpaintr outputs must land in specific containers.

## Signature and defaults

```r
ptr_build_ids(
  control_panel = "controlPanel",  # uiOutput with the tabbed controls
  draw_button   = "draw",          # actionButton triggering a redraw
  plot_output   = "outputPlot",    # plotOutput
  error_output  = "outputError",   # uiOutput for inline errors
  code_output   = "outputCode"     # verbatimTextOutput for generated code
)
```

Each field is a single non-empty string; duplicates are rejected.
Returns an object of class `ptr_build_ids` — the only accepted type for
`ids =` on every Level-2 helper.

## Example — custom ids threaded through UI and server

```r
library(shiny); library(ggpaintr)

ids <- ptr_build_ids(
  control_panel = "builder_controls",
  draw_button   = "render_plot",
  plot_output   = "main_plot",
  error_output  = "main_error",
  code_output   = "main_code"
)

ui <- fluidPage(
  sidebarLayout(
    sidebarPanel(ptr_input_ui(ids = ids)),
    mainPanel(ptr_output_ui(ids = ids))
  )
)

server <- function(input, output, session) {
  ptr_state <- ptr_server_state(
    "ggplot(data = mtcars, aes(x = var, y = var)) + geom_point()",
    ids = ids
  )
  ptr_setup_controls(input, output, ptr_state, ids = ids)
  ptr_register_draw(input, ptr_state, ids = ids)
  ptr_register_plot(output, ptr_state, ids = ids)
  ptr_register_error(output, ptr_state, ids = ids)
  ptr_register_code(output, ptr_state, ids = ids)
}

shinyApp(ui, server)
```

## Notes

- `ptr_server_state()` stores `ids` on `ptr_state$ids`, so every other
  helper defaults `ids =` to that value. Passing `ids =` explicitly is
  only needed when one state object drives multiple output locations.
- **Only these five top-level ids are configurable.** Per-placeholder
  input ids (generated by `ptr_parse_formula()`) and internal `var-*`
  helper ids are deterministic and package-owned. Discover them with
  `ptr_runtime_input_spec()`.
