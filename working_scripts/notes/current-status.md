## Current placeholder support

The supported placeholders in the `paintr2` formula workflow are:

- `var`
- `text`
- `num`
- `expr`
- `upload`

Current behavior:

- `var` selects or injects variable-like expressions into the plot formula  
  Reference: `R/ui_function.R:22-27`, `R/ui_function.R:65-85`, `R/paintr2_func.R:254-265`
- `text` supplies character arguments  
  Reference: `R/ui_function.R:29-39`, `R/paintr2_func.R:277-286`
- `num` supplies numeric arguments  
  Reference: `R/ui_function.R:41-51`, `R/paintr2_func.R:267-275`
- `expr` supplies parsed R expressions  
  Reference: `R/ui_function.R:53-63`, `R/paintr2_func.R:288-295`

  Reference: `R/ui_function.R:1-20`, `R/paintr2_func.R:152-251`, `R/paintr2_func.R:297-304`

## Placeholder behavior details

### `var`

Current behavior:

- UI side: `var` first creates a placeholder `uiOutput()`
- once the data source is known, `output_embed_var()` replaces that placeholder with a picker input of column names
- `var` depends on a data source being available either from:
  - global `ggplot(data = ...)`
  - layer-specific `data = ...`
- on substitution, `var` input is parsed with `parse_expr()`
- this means `var` can behave like a column selector or a lightweight expression input
- examples that may work:
  - `mpg`
  - `log(mpg)`
  - `disp + 1`
- if no data is available for a `var` placeholder, the current path can fail with `data is not provided!`

Reference:

- `R/ui_function.R:22-27`
- `R/ui_function.R:65-85`
- `R/ui_function.R:113-260`
- `R/paintr2_func.R:254-265`

### `text`

Current behavior:

- UI side: `textInput()`
- intended for character arguments such as:
  - `labs(title = text)`
  - `theme(legend.position = text)`
- on substitution, non-empty input is inserted as a quoted string
- blank input is converted to `_NULL_PLACEHOLDER` and later removed from the expression

Reference:

- `R/ui_function.R:29-39`
- `R/paintr2_func.R:277-286`
- `R/paintr2_func.R:395-408`

### `num`

Current behavior:

- UI side: `numericInput()`
- intended for numeric arguments such as:
  - `size = num`
  - `alpha = num`
  - `hjust = num`
- on substitution, numeric input is inserted directly into the expression
- missing or `NA` input is converted to `_NULL_PLACEHOLDER` and later removed

Reference:

- `R/ui_function.R:41-51`
- `R/paintr2_func.R:267-275`
- `R/paintr2_func.R:395-408`

### `expr`

Current behavior:

- UI side: `textInput()`
- intended for free-form R expressions entered as text
- examples:
  - `facet_wrap(expr)`
  - `facet_grid(var ~ var, labeller = expr)`
- on substitution, the text is parsed with `parse_expr()`
- blank input is converted to `_NULL_PLACEHOLDER` and later removed
- invalid R syntax in an `expr` field will currently fail during parsing/substitution

Reference:

- `R/ui_function.R:53-63`
- `R/paintr2_func.R:288-295`
- `R/paintr2_func.R:395-408`

### `upload`

Current behavior:

- UI side: `fileInput()` plus a companion dataset-name `textInput()`
- supported upload formats are:
  - `.csv`
  - `.rds`
- users can optionally provide a dataset object name
- if no name is provided, the uploaded file name is normalized into a default object name
- uploaded data is read on the server side and assigned into the evaluation environment under the resolved object name
- uploaded data can feed `var` choices
- both global `ggplot(data = upload, ...)` and layer-level `data = upload` are supported
- exported Shiny apps preserve the same upload-aware runtime path

Reference:

## Upload support status

`upload` is currently implemented for the `paintr2` path.

Current behavior:

- `upload` creates a Shiny file upload control
- users can upload `.csv` or `.rds`
- users can optionally provide a dataset object name
- if no name is provided, the uploaded file name is converted into a default object name
- uploaded data becomes available to the plotting evaluation environment
- uploaded data can drive `var` choices
- both global `ggplot(data = upload, ...)` and layer-level `data = upload` are supported
- exported Shiny apps preserve the same upload-aware runtime path

## Test commands

Run all automated tests:

```r
testthat::test_dir("tests/testthat", reporter = "summary")
```

Run package tests:

```r
devtools::test()
```

Run one test file:

```r
testthat::test_file("tests/testthat/test-upload.R")
```

Run package check:

```r
devtools::check()
```

Run from shell:

```bash
Rscript -e "testthat::test_dir('tests/testthat', reporter = 'summary')"
Rscript -e "devtools::test()"
Rscript -e "devtools::check()"
```
