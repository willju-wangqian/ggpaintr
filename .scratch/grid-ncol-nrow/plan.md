# Plan: Add `ncol`/`nrow` layout parameters to `ptr_app_grid`

## Context

`ptr_app_grid` currently places all N plots in a single `fluidRow` with equal Bootstrap column widths (`col_width = 12 / N`). There is no way to control the number of rows. The feature adds `ncol` and `nrow` parameters following the ggplot2 `facet_wrap(ncol=, nrow=)` convention already present in the copy rules.

## Success Criteria

1. `ptr_app_grid(plots, ncol = 2)` with 6 plots renders 3 Bootstrap rows, each with 2 columns of width 6.
2. `ptr_app_grid(plots, nrow = 2)` with 6 plots renders 2 Bootstrap rows with 3 columns of width 4.
3. `ptr_app_grid(plots, ncol = 2, nrow = 3)` with 6 plots renders 3 rows × 2 columns.
4. `ptr_app_grid(plots, ncol = 2, nrow = 2)` with 5 plots calls `rlang::abort()` with a message containing `ncol * nrow` and `length(plots)`.
5. `ptr_app_grid(plots, ncol = 3, nrow = 2)` with 5 plots renders 2 rows (first row 3 plots, second row 2 plots) — no error for `ncol * nrow > n_plots`.
6. Neither parameter specified → current one-row behavior unchanged (regression safety).
7. `ptr_app_grid_components` mirrors the same signature and behavior (it is the implementation; `ptr_app_grid` delegates to it).

## Constraints

- Parameter names: `ncol` and `nrow` (no underscore), matching ggplot2/`facet_wrap` convention.
- Both default to `NULL`.
- Validation uses `assertthat::assert_that()` for type/range checks and `rlang::abort()` for the underflow semantic error (consistent with existing codebase patterns).
- No new dependencies. Layout stays on Bootstrap `fluidRow` / `column`.
- `ptr_app_grid_components` is the single implementation site; `ptr_app_grid` passes `ncol`/`nrow` through unchanged.
- Roxygen docs updated for both exported functions. Run `devtools::document()` after editing `R/paintr-app.R` to regenerate `man/` files.
- No changes to `CONTEXT.md` or `dev/adr/` — this is additive behavior, not a domain/architecture decision.
- Do not break existing callers — both params default to `NULL` (backward-compatible).

With:

```r
n_plots <- length(plots)

# Resolve ncol/nrow (NULL = auto, matching facet_wrap convention)
if (is.null(ncol) && is.null(nrow)) {
  ncol <- n_plots
  nrow <- 1L
} else if (is.null(ncol)) {
  assertthat::assert_that(
    rlang::is_integerish(nrow, n = 1L) && nrow >= 1L,
    msg = "`nrow` must be a positive integer."
  )
  nrow <- as.integer(nrow)
  ncol <- ceiling(n_plots / nrow)
} else if (is.null(nrow)) {
  assertthat::assert_that(
    rlang::is_integerish(ncol, n = 1L) && ncol >= 1L,
    msg = "`ncol` must be a positive integer."
  )
  ncol <- as.integer(ncol)
  nrow <- ceiling(n_plots / ncol)
} else {
  assertthat::assert_that(
    rlang::is_integerish(ncol, n = 1L) && ncol >= 1L,
    msg = "`ncol` must be a positive integer."
  )
  assertthat::assert_that(
    rlang::is_integerish(nrow, n = 1L) && nrow >= 1L,
    msg = "`nrow` must be a positive integer."
  )
  ncol <- as.integer(ncol)
  nrow <- as.integer(nrow)
  if (ncol * nrow < n_plots) {
    rlang::abort(paste0(
      "`ncol * nrow` (", ncol * nrow, ") < `length(plots)` (", n_plots, "). ",
      "Increase `ncol`, `nrow`, or both."
    ))
  }
}

col_width <- max(1L, 12L %/% ncol)

plot_area <- do.call(shiny::tagList, lapply(seq_len(nrow), function(r) {
  row_start <- (r - 1L) * ncol + 1L
  row_end   <- min(r * ncol, n_plots)
  if (row_start > n_plots) return(NULL)
  do.call(
    shiny::fluidRow,
    lapply(row_start:row_end, function(i) {
      shiny::column(
        width = col_width,
        ptr_module_ui(plot_module_ids[[i]], plots[[i]],
                      expr_check = expr_check, shared = obj)
      )
    })
  )
}))
```
